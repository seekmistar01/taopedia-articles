# Alpha Reserve

An alpha reserve is the pool-held side of a subnet's alpha token in Dynamic TAO tokenomics.
Bittensor subnets use paired reserve contexts: a TAO reserve and a subnet-specific alpha reserve
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The term names location. Alpha reserve means alpha inside the subnet pool, not alpha outside the
pool in staking, rewards, or other subnet-token contexts.

## Pool Role

Each subnet pool is described as an automated market maker with two reserve sides. TAO reserve names
the TAO side, while alpha reserve names the subnet-alpha side
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

That makes alpha reserve a location-specific term. It does not mean every alpha token associated
with a subnet; it means the portion held in the pool that pairs with TAO.

## Ratio Context

The reserve pair gives readers a way to understand the pool relationship between TAO and alpha. The
[Emission](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection) documentation
uses the TAO reserve and alpha reserve as the two sides of that relationship.

For a reference article, the important boundary is simple: alpha reserve is one side of the pool
ratio. The article does not need short-term pool math or changing reserve amounts.

## Emission Context

Subnet emission can add alpha to the reserve side. Alpha reserve injection adds alpha in proportion
to TAO injection
([Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)).

This is separate from alpha allocated outside the pool. Alpha reserve injection supports the pool
side of subnet tokenomics, while other alpha flows describe outside-pool contexts.

## Alpha Outstanding Contrast

Alpha reserve and alpha outstanding describe subnet alpha by location. Alpha reserve is pool-held
alpha; alpha outstanding is alpha outside that reserve context
([Emission](https://docs.learnbittensor.org/learn/emissions)).

The distinction prevents a common reading mistake. A pool-reserve statement and an outside-pool
alpha statement can both be about the same subnet asset, but they do not describe the same location.

## ADR Context

Alpha reserve also appears in Alpha Distribution Ratio discussions. ADR compares alpha emitted
outside the pool with alpha injected into the automated market maker pool
([Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)).

In that comparison, alpha reserve belongs to the pool-injection side. It is the reserve-side context
opposite alpha distributed outside the pool.

## TAO Reserve Boundary

Alpha reserve should be read together with TAO reserve, but the two sides are not interchangeable.
The subnet pool uses both sides to describe the relationship between network TAO and subnet alpha
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)).

That pair is part of Dynamic TAO tokenomics. Alpha reserve names the subnet-alpha side of the pool;
TAO reserve names the TAO side.

## Protocol Alpha Context

Protocol alpha is related to reserve injection but is not the same term. Protocol alpha names alpha
gathered by the protocol during reserve-injection activity, while alpha reserve names the pool-held
alpha side itself
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha)).

Keeping those terms separate makes the page easier to read. Alpha reserve identifies the pool side;
protocol alpha identifies alpha retained by the protocol as a result of activity connected to that
side.

## Reader Boundary

Alpha reserve is a tokenomics term, not a separate token, staking guide, or pool strategy. It should
be used when the focus is subnet alpha inside the pool
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

When the focus is the paired TAO side, TAO reserve is more precise. When the focus is alpha outside
the pool, alpha outstanding is more precise. When the focus is the broader system, Dynamic TAO is
the broader framing.
