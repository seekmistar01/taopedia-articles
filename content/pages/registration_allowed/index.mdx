# Registration Allowed

Registration allowed is a per-subnet hyperparameter, listed in the documentation as
`NetworkRegistrationAllowed`, that controls whether neuron registration is enabled on a subnet. When
it is on, new neurons can register on that subnet; when it is off, the subnet does not accept new
registrations.

## What It Controls

The hyperparameter acts as a switch over registration for a subnet. It does not change how
registration works or what it costs; it only sets whether registration is currently open. With the
switch off, attempts to register on that subnet are not accepted until it is turned back on.

## Documented Type, Default, and Setter

The [Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)
reference lists `NetworkRegistrationAllowed` as a Boolean with a default of `True` and describes it
as determining whether neuron registration is enabled on the subnet; if disabled, the subnet will
not receive new neurons. It is set through the `sudo_set_network_registration_allowed` extrinsic and
requires root permissions, which is why this page treats the switch as root-governed rather than
freely owner-tuned
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

Whether registration is open on netuid 1 can differ from another subnet's toggle. The live setting
is per-subnet chain state and should be read for that subnet rather than assumed from the documented
default ([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

## Open Registration Still Requires a Registration Burn

When registration is allowed on a subnet such as netuid 1, the
[Glossary: Register](https://docs.learnbittensor.org/resources/glossary#register) still describes
subnet entry as purchasing a UID slot through a dynamic registration burn in TAO. The switch opens
or closes registration; it does not remove the burn cost when entry is permitted
([Mining: Miner registration](https://docs.learnbittensor.org/miners#miner-registration)).

## A Full Subnet Still Uses Displacement Rules

Registration allowed does not remove UID capacity limits. The
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters) reference
includes `max_allowed_uids` as the ceiling for a subnet, and mining documentation describes
displacement when a full subnet admits a new registrant
([Mining: Miner deregistration](https://docs.learnbittensor.org/miners#miner-deregistration)).

The registration switch and the slot ceiling answer different questions. Registration allowed names
whether new entry is open; capacity and displacement rules name what happens when a full subnet
still receives a registration attempt
([Glossary: UID Slot](https://docs.learnbittensor.org/resources/glossary#uid-slot)).

## PoW Registration Is Not the Burn Switch

The hyperparameters reference also lists `NetworkPowRegistrationAllowed`, defaulting to `False`, and
notes that it is not used for neuron registration because neuron registration is burn-based and
always open in that sense. Readers should not confuse the proof-of-work toggle with
`NetworkRegistrationAllowed`, which is the switch that actually gates burn-based neuron entry on a
subnet ([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

## Distinction from Min Burn

Registration allowed is the Boolean that decides whether a subnet accepts new registrations at all,
while min burn sets the lower bound on the dynamic TAO burn required to register a neuron when
registration is open
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

- **Registration allowed** — whether new registration is currently open.
- **Min burn** — the floor on the registration burn amount when it is.

## Distinction from Subnet Is Active

Registration allowed governs whether new neurons may join a subnet, while subnet is active reports
whether the subnet's emissions have started
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

- **Registration allowed** — whether new neurons may register.
- **Subnet is active** — whether the subnet's emissions have started.

## Distinction from Register

Registration allowed is the hyperparameter Boolean that decides whether the entry path is open on a
subnet right now, while register is the action that claims a place through the dynamic burn-priced
UID assignment process
([Glossary: Register](https://docs.learnbittensor.org/resources/glossary#register),
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

- **Register** — the entry action and burn-priced UID assignment.
- **Registration allowed** — whether the subnet currently accepts new registrations.

## Distinction from Recycling

Registration allowed controls whether new neurons may register at all, while recycling is the
mechanism by which a registering neuron's TAO is burned to claim a UID; when the toggle is off, no
recycling-based registration can occur on that subnet
([Glossary: Recycling](https://docs.learnbittensor.org/resources/glossary#recycling),
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Recycling and Burning](https://docs.learnbittensor.org/learn/emissions)).

- **Registration allowed** — the toggle for whether a subnet accepts new registrations.
- **Recycling** — the TAO-burn mechanism used to register, available only while the toggle is on.

## Chain Reads for netuid 1

Readers can verify live hyperparameter values for the documented example netuid with
`btcli subnet hyperparameters --netuid 1 --network finney`
([Subnet Hyperparameters: View hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters#view-hyperparameters)).

That read path keeps live hyperparameter claims tied to a parseable netuid.

## Per-Subnet Live Value Boundary

Registration allowed is per-subnet chain state. The toggle configured on one netuid controls whether
new neurons may register on that subnet, but the live setting can differ from reference defaults and
from the toggle on another netuid
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

This article's infobox uses netuid 1 as an example label when reading one subnet's hyperparameter
table on Finney mainnet. That example helps readers verify one row; it is not proof that every
subnet exposes the same live admission toggle
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

Registration allowed gates roster growth; Yuma Consensus still aggregates validator weights from the
active roster each epoch on the active subnet
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Registration allowed** — toggle for accepting new subnet registrations.
- **Per-subnet boundary** — live value is chain state on one netuid.

## Distinction from Yuma Consensus

Registration allowed is a per-subnet hyperparameter toggle that controls whether new neuron
registrations are accepted. Yuma Consensus is the separate tempo process that converts validator
weight submissions into emission shares each epoch
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

- **Registration allowed** — toggle for accepting new subnet registrations.
- **Yuma Consensus** — on-chain settlement that turns validator weights into emission shares.
  Registration allowed gates roster growth; Yuma processes validator weights into rewards
  ([Yuma Consensus: Validator emissions](https://docs.learnbittensor.org/learn/yuma-consensus#validator-emissions)).

## Reader Boundary

This page defines the concept at a high level. It does not report whether registration is currently
open on any particular subnet. That is live chain state and should be checked for the relevant
netuid. The documented default is enabled, and the parameter is governed at the root level
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).
