ReadyAI is Bittensor Subnet 33, an open-source data-structuring pipeline that grew out of the
Conversation Genome Project. Its task is semantic tagging: miners take raw conversational data and
return structured, tagged output that downstream systems can load into vector databases and use for
retrieval-augmented generation, fine-tuning, and other AI applications. The premise is that modern
language models are now cheaper and more consistent than human annotators, so the annotation work
itself can be turned into a rewarded, distributed service.

## What the Subnet Produces

The subnet's output is structured data rather than a model or a prediction. Raw data goes in, and
tagged, machine-readable data comes out, ready to be stored in a vector database or published as a
dataset. Because the useful product is the structured data, the incentive mechanism is built around
measuring how closely a miner's tagging matches a trusted reference rather than around a subjective
quality judgment.

Miners are free to use different language models to do the tagging — the repository's reference
configuration supports providers such as OpenAI and Anthropic as well as a local model — which keeps
the network from depending on a single provider.

## Miner and Validator Roles

Validators define the ground truth. According to the repository, a validator tags a piece of data in
full, then splits it into smaller "data windows" and hands those windows to miners in a pattern the
project calls fractal mining. Miners tag their assigned windows and return the result.

Scoring is concrete: a miner's submission is compared to the validator's ground-truth tags using a
cosine-distance calculation, so a miner is rewarded for how close its tagging is to the reference
rather than for volume. Those scores become the weights that feed into [[Yuma Consensus]], which
converts them into the incentive split across miners and validators. This makes data quality and
integrity, not raw throughput, the thing the subnet pays for.

## On-Chain Identity

Live SN33 data, including metagraph state and alpha token pool information, is available on
[TaoStats](https://taostats.io/subnets/33). Readers can reproduce the subnet's registered identity
fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 33 --network finney`.

The subnet's open-source codebase (afterpartyai/bittensor-conversation-genome-project) is the source
of truth for the tagging task and the scoring process described above.
