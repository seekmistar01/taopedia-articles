# Subnet Task

A subnet task is the work target a Bittensor subnet asks miners to perform. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-task) places the task inside an
incentive mechanism, while
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) describes
subnets as markets where miners produce a specific digital commodity and validators measure that
work.

## Work Target

The task gives a subnet a concrete work target instead of making every subnet produce the same kind
of output. [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)
describes each subnet as having its own goal and standards, and the
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-task) identifies the subnet
task as the work miners are expected to perform.

## Miner Side

Miners are the role that performs the subnet task. A miner's work is meaningful only in relation to
the subnet's target and evaluation standards; a useful answer in one subnet may not be the requested
work in another subnet.
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)
separates the task miners perform from the method validators use to evaluate that work.

## Validator Side

Validators evaluate miner work against the subnet's standards. The task says what work is being
requested; validation decides how well miner responses satisfy that request.
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) describes
validators as measuring miner work, and
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)
connects that evaluation to subnet incentives.

## Incentive Mechanism Context

A subnet task is one part of a larger incentive mechanism. The task names the work target, while the
surrounding incentive design also covers how validators evaluate the work and how those evaluations
fit the subnet's purpose. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-task) connects subnet tasks to
incentive mechanisms, and
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)
places tasks beside miner-validator interaction and evaluation.

## Relationship to Multiple Mechanisms

A subnet task is usually discussed in a single-mechanism context, but subnets can also run multiple
incentive mechanisms in parallel. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-task) notes that when multiple
incentive mechanisms are used within a subnet, each mechanism can define different tasks for miners
to perform.

For readers, a subnet task still names the work miners are asked to perform, but the mechanism
context matters. The work target for one mechanism does not automatically define the task for
another when a subnet hosts parallel evaluation paths.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task)

## Relationship to Subnet Miner

A subnet task and a subnet miner are related but different parts of Bittensor subnet vocabulary. A
subnet task names the work target miners are asked to perform, while a subnet miner names the role
that performs that work inside a subnet. The
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task) defines the
work miners perform, and the
[Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner) describes
the entity that performs tasks given by subnet validators.

For readers, a subnet task answers what work is requested, while a subnet miner answers which role
produces that work inside a subnet market.

References: [Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner)

## Relationship to Subnet Validator

A subnet task and a subnet validator are related but different parts of Bittensor subnet vocabulary.
A subnet task names the work target inside a subnet, while a subnet validator names the role that
evaluates miner output against that target. The
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task) defines the
work miners perform, and the
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)
describes the role that evaluates miner performance and sets weights based on miner output.

For readers, a subnet task answers what work is being measured, while a subnet validator answers
which role judges whether miner responses satisfy that work target.

References: [Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)

## Relationship to Netuid

A subnet task and a netuid are related but different parts of Bittensor subnet vocabulary. A netuid
selects which subnet context is in scope, while a subnet task names the work target inside that
selected subnet. The [Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)
places netuid at the subnet level, and the
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task) describes
the work miners are expected to perform in that subnet.

For readers, netuid answers which subnet market is being discussed, while a subnet task answers what
work that subnet asks miners to produce.

References: [Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid),
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task)

## Relationship to Incentive Mechanism

A subnet task and an incentive mechanism are related but different parts of subnet design
vocabulary. An incentive mechanism covers the task, protocol, and evaluation method together, while
a subnet task names only the work target inside that design. The
[Glossary: Incentive Mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism)
describes the full evaluation system, and the
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task) describes
the work miners perform within it.

For readers, an incentive mechanism answers how a subnet evaluates work, while a subnet task answers
what work miners are asked to produce inside that mechanism.

References:
[Glossary: Incentive Mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism),
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task)

## Relationship to Yuma Consensus

A subnet task and Yuma Consensus address related but different parts of Bittensor subnet incentive
flow. A subnet task names the work miners are expected to perform inside a subnet, while Yuma
Consensus aggregates validator weight signals about that work into miner incentives and validator
dividends after evaluation. The
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task) describes
the work miners perform, and [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)
describes clipping, bonding, and emission calculation.

For readers, a subnet task answers what work miners are asked to produce, while Yuma Consensus names
the aggregation process that turns validator evaluations of that work into subnet outcomes.

References: [Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)

## Relationship to Validator Weights

A subnet task and validator weights address related but different parts of subnet evaluation
structure. A subnet task names the work miners are expected to perform, while validator weights are
the evaluation signals validators submit for miners after judging that work. The
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task) describes
the work target, and the
[Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights)
describes the per-validator evaluation inputs used by Yuma Consensus.

For readers, a subnet task answers what work is being evaluated, while validator weights name the
per-validator submissions produced from that evaluation.

References: [Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights)

## Relationship to Weight Copying

A subnet task and weight copying address related but different parts of subnet evaluation quality. A
subnet task names the work miners are expected to perform, while weight copying describes when a
validator relies on public weight information instead of independently evaluating that work. The
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task) describes
the work target, and the
[The Weight Copying Problem](https://docs.learnbittensor.org/concepts/weight-copying-in-bittensor)
documentation describes copied or derived weights as the problem behavior.

For readers, a subnet task answers what work should be evaluated, while weight copying names when
validator evaluations may fail to reflect independent judgment of that work.

References: [Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[The Weight Copying Problem](https://docs.learnbittensor.org/concepts/weight-copying-in-bittensor)

## Relationship to Commit Reveal

A subnet task and Commit Reveal address related but different parts of validator-weight processing.
A subnet task names the work miners are expected to perform, while Commit Reveal can conceal
validator evaluations of that work until a later reveal time before consensus uses them publicly.
The [Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task)
describes the work target, and the
[Commit Reveal](https://docs.learnbittensor.org/concepts/commit-reveal) documentation describes
concealed weight submission and delayed reveal.

For readers, a subnet task answers what work is being evaluated, while Commit Reveal names a
visibility and timing layer around how fresh evaluations become available before aggregation.

References: [Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Commit Reveal](https://docs.learnbittensor.org/concepts/commit-reveal)

## Interpretation Boundaries

Subnet task should be read as a concept about requested work, not as a claim that every miner result
is useful or that every subnet uses the same evaluation standard. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-task) keeps the term scoped to
work miners perform, and
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) explains why
subnet goals and standards differ.
