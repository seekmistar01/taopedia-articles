Trishool is Bittensor Subnet 23, an AI alignment subnet focused on evaluating behavioral risks in
large language models. Its public site is [trishool.ai](https://trishool.ai), and the GitHub
repository listed in the subnet's Finney identity is
[TrishoolAI/trishool-phase2](https://github.com/TrishoolAI/trishool-phase2).

## What Trishool Provides

The Trishool repository describes a subnet for building an AI guard model. Miners submit adversarial
prompts, and validators evaluate the model behavior those prompts produce.

In Bittensor terms, Trishool turns AI safety probing into a competitive subnet task. Miners search
for inputs that reveal unsafe or jailbreak-prone behavior. Validators score those results, and those
evaluations feed subnet weights through [[Yuma Consensus]].

## Miner and Validator Roles

Miners search for prompts that expose weaknesses in a guard or target model. Strong miner work is
novel enough to be useful and specific enough for the subnet to evaluate.

Validators run the evaluation side of the loop. Their role is to decide how useful the submitted
prompts are for revealing unsafe behavior and turn that evaluation into weights through
[[Yuma Consensus]]. Operators should always treat the repository as the source of truth for current
setup details.

## On-Chain Identity

The live subnet page is available on [TaoStats](https://taostats.io/subnets/23). Readers can also
reproduce the chain identity fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 23 --network finney`.

That Finney identity reports the subnet name as Trishool, the subnet URL as trishool.ai, and the
GitHub repo as TrishoolAI/trishool-phase2.
