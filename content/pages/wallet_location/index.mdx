# Wallet Location

Wallet location is Bittensor wallet-file terminology for where generated Bittensor wallet files are
stored on a user's machine. The official
[Wallet Location glossary](https://docs.learnbittensor.org/resources/glossary#wallet-location)
describes it as the directory path where generated Bittensor wallets are stored locally.

## File-Storage Context

Wallet location is storage-location vocabulary, not a wallet role, address format, or key type. It
describes where wallet files live on a local machine, while separate wallet and key terms describe
the wallet material itself. Because that local directory can hold coldkey and hotkey file material,
the location should be read as part of the workstation security boundary, not as a public account
label ([Wallets, Coldkeys and Hotkeys](https://docs.learnbittensor.org/keys/wallets),
[Coldkey and Hotkey Workstation Security](https://docs.learnbittensor.org/keys/coldkey-hotkey-security)).

## Wallet Context

[wallet documentation](https://docs.learnbittensor.org/keys/wallets) is the surrounding context for
Bittensor wallet concepts. In Taopedia prose, wallet location should be read as a file storage
concept rather than an account-status claim or a security procedure.

## Key Material Storage Context

The [Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location)
identifies wallet location as where generated Bittensor wallets are stored on a local machine.
[wallet documentation](https://docs.learnbittensor.org/keys/wallets) treats that storage as holding
local coldkey and hotkey file material. Wallet location therefore names a filesystem context for
wallet keys, not an on-chain address or subnet participant role.

## Wallet Location Names Local Storage, Not On-Chain Identity

Wallet location should not be read as a wallet address, subnet role, or proof of who controls keys
on chain. It names the local directory path where generated Bittensor wallet files are stored
([Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location),
[Bittensor wallet documentation](https://docs.learnbittensor.org/keys/wallets)).

Coldkey and hotkey material still supplies authority. The path only describes where those files live
on a workstation ([Wallets, Coldkeys and Hotkeys](https://docs.learnbittensor.org/keys/wallets)).

## Path Labels Do Not Replace Key-Management Procedures

Wallet-location vocabulary does not provide recovery steps, backup policy, or signing instructions.
Key-management guidance covers creation, protection, and use of wallet material
([Working with Keys](https://docs.learnbittensor.org/keys/working-with-keys),
[Handle your Seed Phrase/Mnemonic Securely](https://docs.learnbittensor.org/keys/handle-seed-phrase)).

Readers should keep procedural claims tied to those references rather than to directory-path labels
alone.

## Directory Paths Differ From Mnemonic Recovery

A filesystem path helps locate existing wallet files, while mnemonic vocabulary covers regeneration
after loss. Path metadata is not recovery material
([Glossary: Mnemonic](https://docs.learnbittensor.org/resources/glossary#mnemonic),
[Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet)).

## Path and Authority Boundary

Wallet location points to where generated wallet files are stored; it does not itself grant wallet
authority. The authority still comes from the coldkey and hotkey material described in the official
[wallet documentation](https://docs.learnbittensor.org/keys/wallets), while the
[wallet-location glossary entry](https://docs.learnbittensor.org/resources/glossary#wallet-location)
names the local path where those files are kept.

That boundary keeps a storage path from being read as a public address, account label, or separate
wallet role. Moving or naming a directory changes the local file context, not the underlying
cryptographic relationship between wallet keys and on-chain control.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. Wallet-location
examples should be read against that network boundary rather than as proof of live mainnet behavior.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. The local path vocabulary is the same across environments,
but the chain state those files connect to differs by network.

## Workstation Security Context

Wallet location sits beside workstation-security guidance because paths reveal where key files live
on a machine. Knowing a directory does not, by itself, describe who controls the keys inside the
files or whether backups are safe
([Coldkey and Hotkey Workstation Security](https://docs.learnbittensor.org/keys/coldkey-hotkey-security),
[Working with Keys](https://docs.learnbittensor.org/keys/working-with-keys)).

Readers should pair location vocabulary with access-control and backup policy rather than treating a
path label as recovery material or public address data.

## Distinction from Local Wallet

Local wallet names the on-machine wallet file context. Wallet location names the directory path
where those files are stored
([Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet),
[Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location)).

Wallet-type vocabulary and path vocabulary answer different organization questions. A local wallet
can span multiple directories when users customize storage, so the path names one filesystem context
rather than the whole wallet concept.

## Distinction from Bittensor Wallet

Bittensor wallet names the on-chain identity container. Wallet location names where local files for
that identity are stored
([Glossary: Bittensor Wallet](https://docs.learnbittensor.org/resources/glossary#bittensor-wallet),
[Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location)).

Identity vocabulary persists across directories; location vocabulary names one filesystem path.

## Distinction from Coldkeys

Coldkey names the custody key inside a wallet. Wallet location names where coldkey files live on
disk ([Glossary: Coldkey](https://docs.learnbittensor.org/resources/glossary#coldkey),
[Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location)).

Key-role vocabulary and filesystem-path vocabulary sit on different layers. Moving coldkey files to
a new directory changes local storage context, not coldkey ownership on chain.

## Distinction from Mnemonic

Mnemonic names recovery words that can regenerate keys. Wallet location names where existing wallet
files are stored locally
([Glossary: Mnemonic](https://docs.learnbittensor.org/resources/glossary#mnemonic),
[Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location)).

Recovery-root vocabulary must not be confused with directory-path metadata. A mnemonic can rebuild
keys after loss; a path only locates files that still exist locally.

## Distinction from Wallet Address

Wallet address names a public TAO destination. Wallet location names a local filesystem path
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address),
[Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location)).

Public destination vocabulary and local path vocabulary require opposite handling. Paths belong in
local security notes; addresses belong in transfer and delegation context.

## Reader Boundary

Wallet location should not be read as a public address, a backup plan, or a command tutorial. It
names where local wallet files are stored on a machine
([Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location)).
