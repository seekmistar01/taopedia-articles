# Subnet Validator

A subnet validator is the evaluation role inside a Bittensor subnet. Validators measure miner work
and set weights based on miner output
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The term belongs to subnet incentive vocabulary. It names the role that evaluates work, not the
miner role or the weight signal by itself.

In plain language, subnet validator answers: who scores the miners in this subnet? A validator
observes miner outputs against the subnet's standard and submits weights that reflect that
evaluation. Those weights then feed into Yuma Consensus to produce rank and trust outcomes
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)).

For a reader, the distinction from subnet miner matters most when reading weight or rank data. A
subnet miner produces work; a subnet validator evaluates it. Subnet validator also differs from
blockchain validator — the two roles operate in different contexts within Bittensor
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For subnet
validators on a subnet such as netuid 1, that sequence changes how readers should interpret weight
submission examples and evaluation outcomes.

In localnet, validator role mechanics can be demonstrated in isolation. Local validator examples
reflect local chain configuration rather than production subnet performance.

On testnet, validator evaluation can be observed in a shared, non-production network. Testnet weight
outcomes are separate from mainnet subnet state.

On mainnet, subnet validator is the live evaluation role inside production subnet markets on the
connected Bittensor network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. Validator evaluation examples from one environment should
not be read as evidence for production subnet performance in another environment.

## Distinction from Subnet Miner

Subnet validator names the evaluating role inside a subnet. Subnet miner names the work-producing
role whose output validators measure
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner)).

Evaluation and production are complementary sides of the same subnet flow. A miner produces output;
a validator scores it before consensus and emissions processing.

- **Subnet validator** — the evaluating role that scores miner work.
- **Subnet miner** — the work-producing role whose output validators measure.

## Distinction from Subnet Task

Subnet task names the work a mechanism is meant to evaluate. Subnet validator names the role that
measures miner output against that task's standards
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The task defines what work means. The validator measures miner output against that standard.

- **Subnet validator** — the evaluating role that scores miner work.
- **Subnet task** — the work miners are asked to produce.

## Distinction from Subnet Protocol

Subnet protocol names the exchange rules for how validators request work and miners return
responses. Subnet validator names the evaluating role that operates within those rules
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The protocol defines how work is exchanged. The validator evaluates within those rules.

- **Subnet validator** — the evaluating role within exchange rules.
- **Subnet protocol** — rules for requesting and returning work.

## Distinction from Validator Permit

Subnet validator names the evaluation role on a subnet such as netuid 1. A validator permit is the
per-subnet flag that records whether a UID may submit weights that count in aggregation
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit)).

Registration places a validator on a subnet. The permit answers whether that neuron may submit
counting weights.

- **Subnet validator** — the evaluation role on a subnet.
- **Validator permit** — subnet-level right to submit weights that enter aggregation.

## Distinction from Activity Cutoff

Subnet validator names who evaluates miner work. Activity cutoff is the subnet hyperparameter that
sets how early in each epoch a validator must submit weights for them to count
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Subnet Hyperparameters: Activity Cutoff](https://docs.learnbittensor.org/subnets/subnet-hyperparameters#activitycutoff),
[Glossary: Epoch](https://docs.learnbittensor.org/resources/glossary#epoch)).

Validators must act inside the early epoch window for their weights to count in the current round.

- **Subnet validator** — the evaluating role that submits weights.
- **Activity cutoff** — the early-epoch block window for counting weight submissions.

## Distinction from Validator Dividend

Subnet validator names the evaluation role. Validator dividend is the validator-side emission result
that follows from bonds and miner incentives in the same consensus flow
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Yuma Consensus: Validator emissions](https://docs.learnbittensor.org/learn/yuma-consensus#validator-emissions)).

Weights feed rank and miner incentives first. Bonds and dividends translate validator-miner
relationships into validator-side returns.

- **Subnet validator** — the role that evaluates miners and submits weights.
- **Validator dividend** — validator-side emission earned through consensus evaluation.

## Distinction from Root Validator

Subnet validator names the role that evaluates miner work inside an ordinary subnet. Root validator
belongs to the root staking path on subnet zero, which has no miner work to evaluate
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

- **Subnet validator** — evaluates miner work inside a subnet such as netuid 1.
- **Root validator** — validates staking on the root subnet rather than scoring miner work.

## Distinction from Dendrite

Subnet validator names the evaluation role. Dendrite names the client-side component a validator
commonly uses to transmit requests to miner axon endpoints
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Glossary: Dendrite](https://docs.learnbittensor.org/resources/glossary#dendrite),
[Understanding Neurons](https://docs.learnbittensor.org/learn/neurons#neuron-to-neuron-communication)).

Validators initiate requests through dendrite clients. The validator role is who evaluates the
returned work.

- **Subnet validator** — the evaluating role on a subnet.
- **Dendrite** — client-side component used to send requests to miners.

## Distinction from Synapse

Subnet validator names the evaluating role. Synapse names the structured message object that carries
challenge and response fields through the axon-dendrite exchange
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse)).

The validator evaluates work carried in synapse objects. Synapse is the structured message that
holds challenge and response fields.

- **Subnet validator** — the evaluating role on a subnet.
- **Synapse** — structured message object for the exchange.

## Distinction from Pruning Score

A subnet validator and pruning score meet at deregistration eligibility. A subnet validator is a
neuron that evaluates miners and submits weights in a subnet, while pruning score is the per-neuron
value that ranks which UID is removed first when the subnet is full, so a low-scoring validator
outside its immunity window can be displaced like any other low scorer
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Mining: Miner deregistration](https://docs.learnbittensor.org/miners#miner-deregistration)).

## Distinction from Stake Weight

A subnet validator and stake weight describe validation as the role and the influence it carries. A
subnet validator is the participant that measures miner work and submits weights, while stake weight
is the stake-derived value that scales how much each validator's submitted weights count when
consensus is computed, so the role does the evaluating and stake weight decides how heavily that
evaluation lands
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The validator evaluates work and submits weights. Stake weight scales how heavily those weights
count in consensus.

- **Subnet validator** — the role that evaluates miners and submits weights.
- **Stake weight** — stake-derived influence on how weights count in consensus.

## Distinction from Validator Trust

A subnet validator and validator trust describe the role and a measure of its evaluation. A subnet
validator is the neuron that evaluates miner work and submits weight signals, while validator trust
is the measure of how much that validator's weights align with the consensus result, so the role
does the evaluating and validator trust describes how aligned that evaluation turned out to be
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Glossary: Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The validator submits weights. Validator trust measures how aligned those weights were with the
consensus result.

- **Subnet validator** — the role that evaluates miners and submits weights.
- **Validator trust** — measure of how aligned a validator's weights were with consensus.

## Distinction from Mining and Validating

Mining and validating also differs from subnet validator in scope. Subnet validator names the
participant that evaluates miner responses and sets weights, while mining and validating keeps both
role paths in one introductory frame ([Validators](https://docs.learnbittensor.org/validators),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction)).

- **Mining and validating** — combined introduction to both subnet roles.
- **Subnet validator** — the evaluation and weight-setting role on one subnet.

## Distinction from Yuma Consensus

Subnet validator names the participant role that evaluates miner work and sets weights inside a
subnet. Yuma Consensus is the separate mechanism that converts those weight submissions into
emission shares each tempo ([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)).

- **Subnet validator** — evaluation and weight-setting role on a selected netuid.
- **Yuma Consensus** — mechanism that turns validator weights into emission shares.

## Distinction from Off-Chain Validation Systems

Off-chain validation systems name the evaluation logic validators run outside the chain. Subnet
validator names the role that performs that evaluation and submits the resulting weight signals
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Role vocabulary and evaluation-system vocabulary sit at different levels. A subnet validator carries
out off-chain validation; the validation system is the subnet-specific method that role applies.

## Reader Boundary

Subnet validator names the evaluation role inside a subnet. It explains evaluation vocabulary rather
than procedure details, validator lists, ranking reports, or claims about a specific validator's
performance
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

A subnet validator evaluates miner work inside a Bittensor subnet and expresses that evaluation as
weights for the subnet consensus path.
