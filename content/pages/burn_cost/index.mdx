# Burn Cost

Burn cost is the TAO amount required when creating a new Bittensor subnet. The official
[Glossary](https://docs.learnbittensor.org/resources/glossary#burn-cost) defines it as the amount of
TAO recycled when a new subnet is created, and the
[Create a Subnet](https://docs.learnbittensor.org/subnets/create-a-subnet#burn-cost) guide uses the
same term for the current cost to register a new subnet.

## Subnet Creation Cost

For readers, the important point is that burn cost belongs to subnet creation. It is not a general
transaction fee, a staking amount, or a quality score for an existing subnet. It is the cost
attached to registering a new subnet at the time the creation action is made.

The official [Create a Subnet](https://docs.learnbittensor.org/subnets/create-a-subnet#burn-cost)
page describes the cost as dynamic. It can fall gradually over time and increase when new subnets
are created, so a static article should not state a fixed current value.

## Why "Burn" Is a Careful Term

The name can be confusing because Bittensor documentation also distinguishes burning from recycling.
The official
[local subnet creation guide](https://docs.learnbittensor.org/local-build/create-subnet#subnet-creation-cost)
notes that the subnet creation cost is labeled "burn cost" even though the cost is technically
recycled rather than burned.

That accounting detail matters for interpretation. Burn cost is the familiar label used by the
subnet creation flow, while recycling describes how the TAO is treated in the documentation's
token-accounting language.

## Reader Boundary

Burn cost should be read as a dynamic creation-cost concept, not as advice about when to create a
subnet or as a prediction of future cost. The official
[BTCLI reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-burn-cost) documents how
interfaces can show the current value, but Taopedia's role is to explain what the term means rather
than mirror a live number.

## Relationship to Multiple Mechanisms

Burn cost applies when registering a new subnet. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-creator) notes that subnet
creators can configure multiple incentive mechanisms to distribute emissions across different types
of work or evaluation criteria.

For readers, burn cost still names the TAO amount for subnet creation. A newly created subnet may
later run more than one incentive mechanism, but burn cost belongs to the creation step.
