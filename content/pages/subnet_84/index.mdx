ChipForge is Bittensor Subnet 84, a decentralized hardware-design subnet. Its public repository
describes a competitive system where miners submit digital hardware designs and validators evaluate
those submissions with electronic-design-automation tooling.

The subnet is organized around design challenges. Instead of asking miners for text, images, or
general compute, ChipForge asks them to produce hardware-design implementations for active
specifications. The repository describes current challenge areas such as RISC-V based processors and
AI accelerators, with submitted designs written in Verilog or SystemVerilog.

## What ChipForge Rewards

ChipForge rewards working hardware-design submissions. A miner downloads the current challenge,
builds a solution that satisfies the specification, and submits a packaged design for evaluation. At
the article level, the important artifact is the design itself: source files that can be tested
against the challenge requirements.

This places ChipForge in the physical-design and engineering side of the Bittensor ecosystem. The
subnet is not rewarding a hosted model endpoint or a content response; it is rewarding competitive
progress on digital circuit designs that can be checked against test benches and design metrics.

## Validator Evaluation

Validators evaluate miner submissions against the active challenge. The ChipForge README describes
validators downloading submissions, running them through EDA tools, scoring the results, and setting
weights from those scores.

The current public scoring description emphasizes functionality: whether the submitted design passes
test benches, matches the required behavior, and complies with the specification. The README also
lists area, delay, and power as future or planned scoring dimensions. Those dimensions matter
because a useful hardware design is not only correct, but also efficient in resources, timing, and
power once the evaluation pipeline supports those measures.

## Miner and Validator Roles

Miners are hardware-design competitors. They create Verilog or SystemVerilog solutions for active
challenges, package their submissions, and compete to produce the strongest verified design for a
challenge.

Validators are the verification layer. They retrieve batches of submissions, evaluate them with the
challenge test cases and EDA flow, submit scores, and set subnet weights so rewards follow the best
validated designs. Those weights are then processed through [[Yuma Consensus]] like other Bittensor
subnet weights.

## On-Chain Identity

Live SN84 subnet data is available on [TaoStats](https://taostats.io/subnets/84). The Finney
on-chain identity currently reports no registered name, description, GitHub URL, or website for
netuid 84 — the identity fields are blank. The subnet is publicly associated with the
`TatsuProject/ChipForge_SN84` repository, whose README describes the challenge, submission,
evaluation, and reward model used by the subnet and identifies it as ChipForge (SN84). That
repository was last updated in May 2026.
