# Stake Weight

Stake weight is the subnet-level stake measure used to describe validator influence in Bittensor
consensus. It is the computed stake value tied to consensus power and emissions inside a subnet
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The term connects staking vocabulary to evaluation vocabulary. It explains how stake-derived
influence affects weighted validator signals inside the consensus process.

## Consensus Influence

Stake weight explains how much influence a validator's evaluation carries inside a subnet. It is
more specific than a general stake amount because it belongs to subnet consensus rather than to
every account-balance context
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

This keeps stake weight tied to validation. Staking supplies economic backing, while Yuma Consensus
uses weighted validator signals to shape subnet outcomes.

## Alpha and TAO Components

Stake weight combines alpha stake, TAO stake, and TAO weight in subnet context
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)).

That framing matters because stake weight is not simply one visible stake amount copied into
consensus. It is the subnet-context value that combines stake components before validator influence
is interpreted.

## Effective Stake Boundary

Stake weight and effective stake are nearby staking terms, but they answer different questions.
Effective stake is a staking total, while stake weight is the value used for validator influence in
subnet consensus
([Glossary: Effective Stake](https://docs.learnbittensor.org/resources/glossary#effective-stake),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

Effective stake can help explain backing. Stake weight is the term for the influence measure used
inside subnet consensus.

## Validator Permit Boundary

Stake weight also sits near validator permits. A validator permit is the right to validate in a
subnet, while stake weight is the stake-based influence measure used in consensus context
([Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

The distinction is straightforward: stake weight is a measure, and a validator permit is a
validation right.

## Validator-Weight Interaction

Stake weight and validator weights work together in Yuma Consensus. Validator weights express a
validator's evaluation of miner work, while stake weight helps determine how much influence that
evaluation carries in subnet consensus
([Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

This keeps the two terms connected without collapsing them. Validator weights are evaluation
signals; stake weight is the stake-derived influence applied around those signals.

## Subnet Context

Stake weight should be read inside the subnet context that uses it. Validators evaluate miner work
according to subnet-specific standards, and stake weight helps determine how much influence each
validator signal carries in that setting
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

That makes stake weight a local consensus-influence term, not a universal comparison across every
Bittensor role or subnet.

## Reader Boundary

Use stake weight when the focus is validator influence in subnet consensus. Use effective stake for
a staking-total concept, TAO weight for the TAO component, and validator permit for validation
rights ([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit)).

The stable article-level point is that stake weight names the stake-derived influence measure used
to weight validator evaluation signals inside subnet consensus.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. The stake weight
concept applies across the Bittensor lifecycle: stake-derived influence measures can be observed in
localnet for isolated consensus testing, evaluated on testnet for shared non-production conditions,
and relied upon on mainnet for live operation with real emissions.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. Stake weight examples or consensus influence outcomes from
one environment should not be read as representing production subnet performance in another
environment.
