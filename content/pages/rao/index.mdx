# RAO

RAO is a small-denomination unit for TAO amounts in Bittensor. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#rao) defines RAO as one billionth of a
TAO, while [Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction) describes
TAO as the token used to incentivize subnet activity.

## Denomination Role

RAO is useful when Bittensor material needs to discuss small TAO-denominated amounts without losing
precision. Instead of changing what the token is, RAO changes the scale used to express the amount.
The [Glossary](https://docs.learnbittensor.org/resources/glossary#rao) gives the base conversion,
and [Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction) places TAO in
the broader incentive context of miners, validators, subnet creators, and stakers.

## Relationship to TAO

RAO and TAO are not separate token systems. RAO is a denomination of TAO, so the concept belongs
where Bittensor documentation discusses TAO-denominated incentives, emissions, staking context, or
subnet activity. [Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction)
describes TAO as the incentive token for subnet activity, and
[Emission](https://docs.learnbittensor.org/learn/emissions) describes emission as distributing newly
created TAO and subnet-specific alpha tokens across network roles.

## Emission Context

Emission articles and references sometimes need to distinguish the token being distributed from the
unit used to express an amount. RAO helps with that second part: it is a smaller unit for
TAO-denominated quantities, not a different reward category. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#rao) links RAO to emission context,
and [Emission](https://docs.learnbittensor.org/learn/emissions) provides the broader distribution
frame for TAO and subnet-specific alpha tokens.

## Reader Cautions

RAO should be read as a denomination, not as a separate token or subnet role. It also should not be
confused with subnet-specific alpha tokens: those are separate subnet-context tokens, while RAO is a
denomination of TAO. This distinction follows the
[Glossary](https://docs.learnbittensor.org/resources/glossary#rao) definition and the
[Emission](https://docs.learnbittensor.org/learn/emissions) distinction between TAO and
subnet-specific alpha tokens.
