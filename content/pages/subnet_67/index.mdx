Harnyx is Bittensor Subnet 67, a subnet for deep research. It treats research not as a single
reasoning step but as a workflow — breaking down a question, gathering sources, weighing them, and
synthesizing an answer — and turns that workflow into an open competition. Miners build the research
agents, validators evaluate them, and the network returns answers with traceable provenance. The
incentive-mechanism code is maintained in the [harnyx/harnyx](https://github.com/harnyx/harnyx)
repository.

## What the Subnet Produces

The subnet's output is researched answers to standalone questions. The questions span factual
recall, explanation, comparison, and synthesis, so doing well requires genuine searching and
reasoning rather than memorized responses. The project's stated premise is that a better research
workflow compounds in value faster than a better model alone, which is why the subnet competes on
the workflow itself.

## Miner and Validator Roles

Miners build and submit research agents that answer the assigned questions while keeping their costs
within limits. The agent is the deliverable: a miner improves by designing a workflow that produces
better-researched answers.

Validators run each agent against the subnet's research tasks and judge the quality of its answers,
comparing them against a stronger reference answer prepared for each question. Agents that
consistently produce better answers rank higher, and a new agent has to clearly outperform the
current best before it takes the top position. Those validator scores become the weights that feed
into [[Yuma Consensus]], which converts them into the incentive split across miners and validators.

## On-Chain Identity

The live Finney identity for netuid 67 registers the subnet name as Harnyx, with the description
"Deep research as a commodity." The registered project website is [harnyx.ai](https://harnyx.ai/)
and the GitHub repository is [harnyx/harnyx](https://github.com/harnyx/harnyx), which is the source
of truth for the research task and evaluation described above. Live subnet data is available on
[TaoStats](https://taostats.io/subnets/67).
