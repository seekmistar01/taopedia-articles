Quasar is Bittensor Subnet 24, a competitive language-model subnet operated by SILX Labs. Its public
site is [silxinc.com](https://silxinc.com), and its Finney identity describes Quasar as a subnet
built to push the long-context barrier.

## What Quasar Provides

The Quasar README describes a subnet where miners train Quasar-compatible language models, publish
them as public Hugging Face repositories, and commit pinned model revisions on chain. Validators
evaluate valid submissions and set weights toward the current strongest model.

In Bittensor terms, Quasar turns small-model training into a competitive subnet task. Miners improve
models, validators compare eligible models, and validator weights feed into [[Yuma Consensus]].

## Miner and Validator Roles

Miners train and publish models that match the Quasar base interface. The README describes
requirements around public model repositories, pinned revisions, safetensors weights, tokenizer
compatibility, and avoiding quantized or duplicate submissions.

Validators run the evaluation side of the subnet. The README describes checks for model validity and
a king-of-the-hill evaluation process that combines paired model comparisons with a broader
composite quality evaluator. Operators should treat the project documentation and source repository
as the source of truth for current setup details.

## On-Chain Identity

Live SN24 data is available on [TaoStats](https://taostats.io/subnets/24). Readers can also
reproduce the chain identity fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 24 --network finney`.

That Finney identity reports the subnet name as Quasar, the subnet URL as silxinc.com, and the
GitHub repo field as SILX-LABS/QUASAR-SUBNET.
