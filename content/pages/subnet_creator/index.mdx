# Subnet Creator

A subnet creator is the role that defines what a Bittensor subnet is meant to produce and how that
work should be evaluated. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-creator) describes the role
around defining the digital task for subnet miners, shaping incentive mechanisms, and providing
documentation for the subnet's work.

## Role in a Subnet

The creator role sits before miner work and validator evaluation. It gives the subnet a work target
and evaluation context so miners and validators are not operating against a generic network-wide
task. [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)
describes subnets as specialized markets, while the
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-creator) ties the creator role
to the task and incentive context for that market.

## Relationship to Miners and Validators

Subnet creators are not the same role as miners or validators. Miners produce work for the subnet,
and validators evaluate that work against subnet standards. The creator role shapes the context
those roles use.
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) describes
miners and validators as the producing and measuring communities inside a subnet, and the
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-creator) describes the creator
as responsible for the task and incentive context.

## Relationship to the Task

The subnet task is the work target; the subnet creator is the role that defines that target for the
subnet. Keeping those ideas separate avoids mixing a role with the work itself.
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)
places task, protocol, and scoring model beside each other as parts of subnet incentive design, and
the [Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-creator) connects the
creator role to that design context.

## Documentation Context

Official Bittensor docs describe subnet documentation as part of making a subnet understandable to
miners and validators. In this article, that point is only a reader-facing boundary: a subnet
creator provides context for the subnet's work and evaluation standards, while operational steps
belong in the Bittensor documentation.
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)
connects subnet documentation to miner and validator understanding.

## Relationship to Multiple Mechanisms

Subnet creator is usually discussed in a single-mechanism context, but subnets can also run multiple
mechanisms in parallel. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-creator) notes that subnet
creators can configure multiple incentive mechanisms to distribute emissions across different types
of work or evaluation criteria.

For readers, a subnet creator still shapes the work and evaluation context for a subnet, but that
design can include more than one mechanism. Design choices for one mechanism do not automatically
describe the rules for another when a subnet uses more than one evaluation system.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Subnet Creator](https://docs.learnbittensor.org/resources/glossary#subnet-creator)

## Relationship to Crowdloans

Subnet creator and crowdloans address related but different parts of Bittensor subnet creation
vocabulary. A subnet creator is the role that defines a subnet's work target and evaluation context
and performs the subnet registration that opens a new incentive market, while crowdloans are a
community-funded mechanism for paying a subnet's registration burn, where contributors supply TAO
collectively and in return become lease shareholders in the resulting leased subnet. The
[Glossary: Subnet Creator](https://docs.learnbittensor.org/resources/glossary#subnet-creator)
describes the creator as responsible for the task and incentive context, and the
[Crowdloans documentation](https://docs.learnbittensor.org/subnets/crowdloans) describes the
crowdloan as an alternative funding path for the subnet's registration.

The subnet creator can either self-fund the registration burn or use a crowdloan to raise the
required TAO from community contributors. When a crowdloan funds the registration, the creator
becomes the lease beneficiary — holding operational control through a beneficiary proxy — while
contributors become lease shareholders who receive a portion of the subnet's emissions pro-rata.
Subnet creator names the design and operational role; crowdloans name the community-funding
arrangement that can replace self-funding and introduce shared economic rights in the subnet's
output.

References:
[Glossary: Subnet Creator](https://docs.learnbittensor.org/resources/glossary#subnet-creator),
[Crowdloans documentation](https://docs.learnbittensor.org/subnets/crowdloans)

## Reader Boundary

Subnet creator should be read as a role concept, not as a guide to launching or managing a subnet.
The [Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-creator) defines the role,
and [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) provides
the broader subnet context without requiring procedure, account, or operational detail in this
article.
