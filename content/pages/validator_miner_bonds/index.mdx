# Validator-Miner Bonds

Validator-miner bonds are Bittensor consensus signals that connect validator evaluation to miner
outcomes over time. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds) defines bonds
as part of the relationship between a validator and a miner, while
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) explains how validator
evaluation, clipping, bonding, and emissions fit together.

## Consensus Bonding

In Yuma Consensus, validators evaluate miner work and those evaluations are resolved into subnet
outcomes. Bonds sit in that consensus flow after validator weights are compared with the consensus
benchmark. The [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) page places
bonding after clipping and before validator-side emission calculation.

## Smoothing Over Time

Validator-miner bonds are not just a one-moment signal. The
[Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema) page explains that
Bittensor uses EMAs of validator-miner bond strengths when determining emissions inside each subnet.
This smoothing keeps bond behavior tied to a running history rather than only the latest evaluation.

## Relationship to Evaluation

Subnets define useful work, miners produce that work, and validators evaluate it.
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) describes
that miner-and-validator model, while
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) explains how validator
rankings become inputs to consensus. Bonds help connect that evaluation path to validator-side
outcomes.

## Out-of-Consensus Penalty Context

Bonds can also reflect a penalty when a validator's evaluation moves away from consensus. The
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus#penalizing-out-of-consensus-bonds)
documentation explains that when a validator's weight for a miner exceeds that miner's consensus
benchmark, the bond-weight can be blended from the validator's raw weight toward the
consensus-clipped weight. A configurable penalty factor controls how far the bond-weight moves in
that direction, so the amount of above-consensus evaluation that survives into the bond depends on
how the subnet sets that factor.

For readers, the distinction is between the validator's submitted evaluation and the
consensus-aligned bond value that feeds emissions. Because the penalty is configurable, an
out-of-consensus evaluation may still contribute some bond strength, but the configured factor
determines how closely the bond-weight tracks consensus rather than the validator's raw opinion.

## Relationship to Multiple Mechanisms

Validator-miner bonds are usually discussed in a single-mechanism subnet context, but subnets can
also run multiple incentive mechanisms in parallel. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms) defines
multiple incentive mechanisms as independent evaluation systems, each with its own bond pool for
Yuma Consensus calculations.

For readers, bonds still describe how validator evaluation connects to subnet outcomes over time,
but the mechanism context matters. Bond behavior in one mechanism does not substitute for bond
context in another when a subnet runs parallel evaluation paths.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)

## Relationship to Yuma Consensus

Validator-miner bonds and Yuma Consensus address related but different parts of Bittensor subnet
incentive flow. Validator-miner bonds are smoothed consensus relationships between validators and
miners, while Yuma Consensus aggregates validator weight signals and resolves them into miner
incentives and validator dividends. The
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)
describes the bond relationship, and
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) describes clipping, bonding,
and emission calculation inside that process.

For readers, validator-miner bonds name the smoothed validator-miner relationship, while Yuma
Consensus names the broader aggregation step that uses bonds when allocating subnet outcomes.

References:
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)

## Relationship to Validator Weights

Validator-miner bonds and validator weights address related but different parts of subnet evaluation
structure. Validator weights are the per-validator evaluation signals validators submit for miners,
while validator-miner bonds describe the smoothed relationship that evolves from those evaluations
over time inside Yuma Consensus. The
[Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights)
describes the per-validator evaluation inputs, and the
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)
describes the bond values that connect validators to miners across epochs.

For readers, validator weights name the evaluation submissions, while validator-miner bonds name the
smoothed relationship those submissions help build inside consensus.

References:
[Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights),
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)

## Relationship to Dividends

Validator-miner bonds and dividends address related but different parts of validator-side subnet
outcomes. Validator-miner bonds describe the smoothed relationship between a validator and miners,
while dividends describe the validator-side emission result calculated from those bonds and miner
incentive values. The
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)
describes the bond relationship, and the
[Glossary: Dividends](https://docs.learnbittensor.org/resources/glossary#dividends) describes the
validator-side reward portion within emission distribution.

For readers, validator-miner bonds name the relationship being smoothed, while dividends name the
validator-side emission outcome that relationship helps produce.

References:
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds),
[Glossary: Dividends](https://docs.learnbittensor.org/resources/glossary#dividends)

## Relationship to Incentives

Validator-miner bonds and incentives address related but different parts of subnet consensus
outcomes. Validator-miner bonds connect validator evaluation to miners over time, while incentives
name the miner-side emission portion that results from aggregate miner rankings after consensus. The
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)
describes the validator-miner relationship, and the
[Glossary: Incentives](https://docs.learnbittensor.org/resources/glossary#incentives) describes the
miner-facing reward portion inside a subnet.

For readers, validator-miner bonds name the smoothed validator-miner connection, while incentives
name the miner-side emission result that consensus assigns after evaluations are aggregated.

References:
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds),
[Glossary: Incentives](https://docs.learnbittensor.org/resources/glossary#incentives)

## Relationship to Coinbase

Validator-miner bonds and coinbase are related but different parts of Bittensor subnet consensus
vocabulary. Validator-miner bonds are the smoothed validator-miner connections that Yuma Consensus
uses to allocate emissions — each bond records how consistently a validator has evaluated a given
miner, weighted using an exponential moving average across consensus rounds. Coinbase is the
per-block protocol mechanism that accumulates pending emissions each block and checks each subnet's
epoch boundary to trigger the Yuma Consensus round where bond state is read and emission outcomes
are calculated. The
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)
describes the smoothed validator-miner connection, and the
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)
documentation describes the per-block operation that detects when the next consensus round is due.

Bonds accumulate across consensus rounds; coinbase determines when each round runs. Each time
coinbase detects a subnet's epoch boundary and triggers a Yuma Consensus round, that round reads the
current bond state to calculate miner incentives and validator dividends for the epoch. Bonds name
the smoothed evaluation state that persists between rounds; coinbase names the per-block mechanism
that decides when the next round fires.

References:
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds),
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)

## Interpretation Boundaries

Validator-miner bonds should be read as a consensus concept, not as a social endorsement, a
standalone quality score, or a user-facing staking guide. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds) connects the
term to Yuma Consensus, and [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)
places it inside the broader clipping, bonding, and emission process.
