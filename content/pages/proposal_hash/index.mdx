# Proposal Hash

A proposal hash is the compact identifier used to refer to a proposal in Bittensor governance. The
[Proposal Hash glossary entry](https://docs.learnbittensor.org/resources/glossary#proposal-hash)
places the term in Senate voting, where reviewers need a stable way to identify the proposal under
consideration.

The hash is useful because governance discussions often need to point at the same proposal across
review, voting, and follow-up references. It gives the proposal a concise handle while the proposal
itself carries the substantive change.

## Governance Use

In Bittensor governance, a [proposal](https://docs.learnbittensor.org/resources/glossary#proposal)
is the item put forward for review, while the proposal hash is the identifier used to refer to that
item. This keeps the reference compact without replacing the proposal's full description
([Glossary: Proposal Hash](https://docs.learnbittensor.org/resources/glossary#proposal-hash),
[Glossary: Proposal](https://docs.learnbittensor.org/resources/glossary#proposal)).

That distinction is the main reason the term appears in Taopedia. A reader may see a proposal hash
in governance material and need to understand that it points to a proposal rather than summarizing
the proposal.

## Identifier Boundary

A proposal hash is identifier vocabulary, not a second proposal object. The proposal remains the
governance item being considered, while the hash is the compact reference used to identify it in
governance discussion and review
([Glossary: Proposal](https://docs.learnbittensor.org/resources/glossary#proposal),
[Glossary: Proposal Hash](https://docs.learnbittensor.org/resources/glossary#proposal-hash)).

This boundary prevents two common misreadings. A proposal hash does not by itself explain the change
being requested, and it does not replace the longer proposal context that governance reviewers
consider before voting.

That makes the hash a reference tool rather than a governance status. It can identify the item under
review, while the proposal and governance record explain what has happened to that item
([Glossary: Proposal](https://docs.learnbittensor.org/resources/glossary#proposal)).

## Senate Review Context

Bittensor governance separates proposal creation from Senate approval. Proposal review and voting
sit inside Senate context, so proposal hashes belong naturally beside Senate review vocabulary
([Governance Overview](https://docs.learnbittensor.org/governance),
[Senate](https://docs.learnbittensor.org/governance/senate)).

In that setting, the hash helps preserve continuity. The same proposal can be discussed before,
during, and after Senate review, and the hash is the reference that keeps those mentions tied to the
same governance item.

The hash also keeps proposal discussion compact. It lets governance prose refer back to the same
item without restating the entire proposal each time.

## Voting Boundary

The hash identifies the proposal being voted on; it is not the vote, the voting account, or the
final governance outcome. Senate documentation describes Senate voting as a governance process, and
the proposal hash is one reference point inside that process rather than the decision itself
([Senate](https://docs.learnbittensor.org/governance/senate),
[Glossary: Proposal Hash](https://docs.learnbittensor.org/resources/glossary#proposal-hash)).

That distinction matters when a hash appears beside voting language. The hash can keep the proposal
reference stable, but the surrounding governance record is what gives the vote context.

A proposal hash should be read together with the proposal and governance context around it. By
itself, the hash tells the reader which proposal is being referenced; the governance source explains
what the proposal concerns and where it sits in review
([Governance Overview](https://docs.learnbittensor.org/governance),
[Senate](https://docs.learnbittensor.org/governance/senate)).

That makes the term useful in citations and governance summaries, where a short reference needs to
lead readers back to the longer proposal record.

## Outcome Boundary

A proposal hash should not be read as proof that a proposal passed, failed, executed, or changed the
network. Governance references need the proposal context and the relevant Senate or governance
record to explain the status of the proposal
([Governance Overview](https://docs.learnbittensor.org/governance),
[Senate](https://docs.learnbittensor.org/governance/senate)).

The hash therefore works best as a pointer. It helps keep references to the same proposal aligned,
while approval status, implementation status, and governance effects require additional governance
evidence.

## Proposal Relationship

Proposal and proposal hash are paired terms. A proposal is the governance item under consideration;
the proposal hash is the reference used to identify that item compactly
([Glossary: Proposal](https://docs.learnbittensor.org/resources/glossary#proposal),
[Glossary: Proposal Hash](https://docs.learnbittensor.org/resources/glossary#proposal-hash)).

Keeping the two terms separate avoids overloading the hash with meaning it does not carry. The hash
can help locate a proposal, but the proposal text and governance process explain what is being
requested and how it is reviewed.

The pair is useful because governance references can be terse. The hash gives a compact label, and
the proposal supplies the substantive request that the Senate reviews
([Senate](https://docs.learnbittensor.org/governance/senate)).

## Network Scope

Bittensor documentation separates localnet, testnet, and mainnet environments
([Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)). Proposal-hash
examples need the environment attached because local, testnet, and mainnet governance references
have different scope.

A local example can illustrate mechanics. A testnet example can show shared non-production behavior.
A mainnet proposal hash refers to production governance context.

The identifier concept can be the same across environments, while the governance status attached to
that identifier depends on the environment and record being cited.

## Reader Boundary

Proposal hash should be read as proposal-identifier vocabulary. The stable reference point is a
compact governance identifier used during proposal review and voting
([Glossary: Proposal Hash](https://docs.learnbittensor.org/resources/glossary#proposal-hash),
[Senate](https://docs.learnbittensor.org/governance/senate)).

The proposal and governance record carry the actual requested change, review status, and outcome.
