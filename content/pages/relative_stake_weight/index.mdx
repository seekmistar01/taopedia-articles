# Relative Stake Weight

Relative stake weight is a subnet-local measure of validator influence in Bittensor. It expresses
one validator's stake weight as a proportion of the stake weight held by validators in the same
subnet
([Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

## What It Represents

Relative stake weight divides one validator's stake weight by total validator stake weight on the
same subnet. When delegation moves, the denominator can change even if one validator's own stake
weight stays flat; the result is always a share inside the current validator set, not an absolute
TAO or alpha amount
([Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight),
[Understanding Subnets: Validator stake weight](https://docs.learnbittensor.org/subnets/understanding-subnets#validator-stake-weight)).

Stake-weight proportion examples from one network environment should not be read as representing
production influence shares on another
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from TAO Weight

TAO weight is the protocol-level multiplier that sets how much root (TAO) stake counts relative to a
subnet's alpha stake when stake weight is computed. Relative stake weight expresses that merged
stake weight as a proportion of the validator set on the same subnet
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

- **TAO weight** — multiplier scaling how root TAO stake counts when stake weight forms.
- **Relative stake weight** — stake weight expressed as a share of the subnet's validator total.

## Distinction from Stake Weight

Stake weight is a validator's computed stake influence. Relative stake weight is that value divided
by the total stake weight of validators on the same subnet
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)).

- **Stake weight** — the validator's absolute computed stake influence.
- **Relative stake weight** — that influence as a share of the subnet's total.

## Distinction from Validator Weights

Validator weights express what a validator scored about miners. Relative stake weight expresses how
strongly that validator's signal counts inside the subnet's weighted aggregation
([Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)).

- **Validator weights** — what the validator evaluated about miners.
- **Relative stake weight** — how strongly that validator's signal counts.

## Distinction from Yuma Consensus

Relative stake weight names comparative influence among validators on one subnet. That proportional
scaling vocabulary is an input to aggregation, not the full mechanism that credits emission shares
at the epoch boundary
([Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

Yuma Consensus reads the weight matrix, scales submissions by stake weight, applies clipping and
bonding, and converts the result into miner incentives and validator dividends
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution)).

Readers should not treat relative stake weight as miner rank or a payout quote. It compares
validator influence inside one subnet market; credited outcomes still depend on the settlement pass
on that netuid
([Understanding Subnets: Validator stake weight](https://docs.learnbittensor.org/subnets/understanding-subnets#validator-stake-weight)).

- **Relative stake weight** — proportional stake-based influence within one subnet's validator set.
- **Yuma Consensus** — epoch-boundary settlement from stake-weighted weights to emission shares.

## Reader Boundary

Relative stake weight is proportional consensus vocabulary. Use stake weight for the computed
influence measure, validator weights for miner-evaluation signals, and Yuma Consensus for the
aggregation process
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).
