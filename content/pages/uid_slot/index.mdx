# UID Slot

A UID slot is a participant position within a Bittensor subnet. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#uid-slot) describes it as a position
occupied by a subnet miner or subnet validator and identified by a unique UID.

## Subnet Role

UID slots give subnet participants a position in the subnet context where miner and validator roles
are interpreted. The official [Miners](https://docs.learnbittensor.org/miners) and
[Validators](https://docs.learnbittensor.org/validators) pages describe those two roles at a high
level: miners produce subnet work, and validators evaluate miner responses.

## Slot Capacity and Limits

Each Bittensor subnet has a configurable maximum number of UID slots. The official
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters) reference
includes `max_allowed_uids` as the hyperparameter that sets this ceiling for a given subnet. When a
subnet's slots are all occupied, a new registration can succeed only by displacing an existing
participant.

The official
[Mining: Miner deregistration](https://docs.learnbittensor.org/miners#miner-deregistration)
documentation describes what happens when a full subnet receives a new registration: the participant
with the lowest pruning score is the candidate for displacement. The full selection criteria,
including how the immunity period interacts with the pruning score, how ties are resolved, and which
participants are exempt, are specified in that reference. For readers, the key concept is that the
slot ceiling is a real capacity constraint: once max_allowed_uids is reached, accepting a new
registrant requires removing an existing one according to the rules described in the official
documentation.

References:
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Mining: Miner deregistration](https://docs.learnbittensor.org/miners#miner-deregistration)

## Relationship to Multiple Mechanisms

A UID slot belongs to one subnet context. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism) notes that each
subnet has one or more incentive mechanisms.

For readers, a UID slot still identifies a participant position inside one subnet market. That
subnet may run more than one incentive mechanism, but the slot itself remains subnet-scoped.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Incentive Mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism)

## Reader Boundary

This article defines the slot concept. It does not list subnet participants, count occupied slots,
or describe current subnet state. Readers should use official subnet references for role-specific
details.
