# Crowdloans

A crowdloan is a Bittensor funding structure for pooling TAO toward a specified action. The funded
action can be an extrinsic execution or a balance transfer to an address, with subnet creation as
the main example in Bittensor's crowdloan documentation
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans)).

The term belongs to collective funding. It explains how a group can fund an on-chain action
together, rather than describing a general staking strategy or a guaranteed emission outcome
([Create a Subnet: Burn cost](https://docs.learnbittensor.org/subnets/create-a-subnet#burn-cost),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Collective Funding Context

Crowdloans spread a funding target across contributors by collecting TAO toward a campaign goal. For
subnet creation, that target is tied to the cost of bringing a subnet into existence
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans),
[Glossary: Burn Cost](https://docs.learnbittensor.org/resources/glossary#burn-cost)).

That makes crowdloan a funding-context term. It names the pooled contribution structure around the
action being funded, while burn cost names the TAO amount associated with subnet creation itself
([Create a Subnet: Burn cost](https://docs.learnbittensor.org/subnets/create-a-subnet#burn-cost)).

## Campaign Outcome Context

A crowdloan campaign can succeed or fail to finalize. Success performs the funded action, such as a
subnet creation extrinsic or transfer; failure leaves the campaign in a refund path rather than
turning the funding effort into the requested outcome
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans)).

This boundary is important for static reference prose. A crowdloan is conditional until its campaign
outcome is known. The article-level concept is pooled funding toward a specified action, not a claim
that every campaign becomes a subnet
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans)).

## Emission Sharing Context

When a crowdloan funds subnet creation, resulting emissions are shared according to contribution
share. That pro-rata framing is the subnet-creation example's tokenomics relationship between
funding and later emission flow ([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans),
[Emissions](https://docs.learnbittensor.org/learn/emissions)).

Emission sharing should be read as part of the documented subnet-creation crowdloan case, not as a
prediction about subnet performance. The actual emission context still depends on the subnet,
consensus outcomes, and network environment
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Subnet Lifecycle Context

Crowdloans sit near subnet creation because subnet creation is the primary documented example. They
also remain distinct from later lifecycle events such as subnet removal or dissolution
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans),
[Subnet Deregistration](https://docs.learnbittensor.org/subnets/subnet-deregistration)).

This keeps the lifecycle vocabulary separate. Crowdloan names a collective funding path at the start
of a funded action path, while deregistration names a removal process that can occur under separate
network rules for subnets
([Subnet Deregistration](https://docs.learnbittensor.org/subnets/subnet-deregistration)).

## Network Context

Network environment changes how crowdloan examples should be read. Localnet, testnet, and mainnet
are separate contexts, so a crowdloan example from one environment is evidence about that
environment rather than a universal statement for every Bittensor network
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Introduction to Bittensor: Subnet development](https://docs.learnbittensor.org/learn/introduction#subnet-development)).

The same boundary applies to campaign details. A cap, end block, contribution set, or emission
sharing example belongs to the campaign and network environment being discussed
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans)).

## Reader Boundary

Crowdloans are reference vocabulary for collective funding of an extrinsic or transfer. They are not
staking advice, yield estimates, campaign recommendations, or step-by-step funding instructions.
Concrete examples require the selected campaign, subnet context, network environment, and relevant
official source ([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).
