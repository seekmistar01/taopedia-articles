# Relative Stake Weight

Relative stake weight is a subnet-local measure of validator influence in Bittensor. It expresses
one validator's stake weight as a proportion of the stake weight held by validators in the same
subnet
([Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

The term matters because consensus influence is read inside a subnet. Relative stake weight tells
readers about proportional influence in that setting rather than about a validator's stake in
isolation.

## Proportional Influence

The core idea is proportion. Stake weight names the validator's computed stake influence, while
relative stake weight compares that influence against the validator set for the subnet
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)).

That makes relative stake weight a context term. A validator's stake weight can be meaningful on its
own, but relative stake weight explains how that value sits inside the subnet's validator set.

In the glossary formula, the validator's stake weight is divided by the total stake weight across
validators, so the result is a subnet-local share rather than an absolute stake amount
([Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)).

## Subnet Context

Relative stake weight should be read inside a subnet context. Subnets define work markets where
miners produce outputs and validators evaluate those outputs according to subnet-specific standards
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)).

This subnet context is part of the meaning. Relative stake weight is not a universal comparison
across every Bittensor role; it is a proportional influence measure for the relevant subnet.

## Consensus Context

Relative stake weight belongs near Yuma Consensus because validator influence matters when submitted
signals are aggregated. Yuma Consensus uses validator rankings of miner performance to shape miner
incentives and validator dividends
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)).

For readers, the useful point is that relative stake weight is an input-side influence concept. It
helps describe how validator signals are weighted before consensus results are understood.

## Validator-Weight Context

Relative stake weight and validator weights answer different questions. Validator weights express a
validator's miner evaluations, while relative stake weight describes the proportional influence of
that validator's signal inside the subnet
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)).

This keeps evaluation and influence separate. A validator weight says what was evaluated; relative
stake weight helps explain how much influence that validator has in the relevant consensus setting.

## The Denominator Is Total Validator Stake on the Subnet

The
[Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)
defines relative stake weight as one validator's
[stake weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) divided by the total
stake weight across validators on the same subnet. The result is a share of validator-side influence
inside that subnet rather than an absolute TAO or alpha amount.

When delegation moves on a subnet such as netuid 1, the denominator can change even if one
validator's own stake weight stays flat. Relative stake weight therefore tracks proportional
standing inside the current validator set, not a fixed balance read in isolation.

## The Same Absolute Stake Can Mean Different Relative Shares

[Understanding Subnets: Validator stake weight](https://docs.learnbittensor.org/subnets/understanding-subnets#validator-stake-weight)
documentation describes stake weight as the influence measure validators carry into consensus on a
subnet. Relative stake weight compares that measure against other validators on the same netuid.

Two validators with similar absolute stake weights on different subnets can therefore hold different
relative shares because each subnet's validator total differs. Relative vocabulary always needs the
subnet context that produced the proportion.

## Stake-Weighted Aggregation Uses Those Proportional Shares

[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) combines validator-submitted
signals with stake-weighted aggregation rather than treating every validator vote equally. Relative
stake weight names how much of that subnet-local weight pool one validator contributes before
[consensus score](https://docs.learnbittensor.org/resources/glossary#consensus-score) clipping and
[rank](https://docs.learnbittensor.org/resources/glossary#rank) aggregation proceed.

That placement keeps relative stake weight on the influence side of consensus vocabulary.
[Validator weights](https://docs.learnbittensor.org/resources/glossary#weight-vector) express what a
validator scored; relative stake weight expresses how strongly that validator's signal counts inside
the subnet's weighted merge.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For relative stake
weight on a subnet such as netuid 1, that sequence gives readers a boundary for interpreting
stake-weight examples and subnet-comparison notes.

In localnet, stake-weight examples are isolated and reflect local chain state, so they are useful
for controlled experiments rather than evidence of live Bittensor behavior.

On testnet, stake-weight examples add shared non-production conditions, which can reveal integration
behavior without touching mainnet state.

On mainnet, relative stake weight examples should be read as live production stake-weight behavior
on subnets such as netuid 1.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet, so outcomes from one environment should not be treated as
proof of behavior in another.

## Distinction from TAO Weight

Relative stake weight and tao weight act on validator influence at different stages. TAO weight is
the protocol-level multiplier that sets how much root (TAO) stake counts relative to a subnet's
alpha stake when stake weight is computed
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Because stake weight already combines alpha stake, TAO stake, and TAO weight, relative stake weight
expresses that merged value as a proportion of the validator set on the same subnet rather than
acting as a second TAO-side parameter
([Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)).

- **TAO weight** — the multiplier scaling how root TAO stake counts against subnet alpha when stake
  weight forms.
- **Relative stake weight** — that stake weight expressed as a share of the subnet's validator
  total.

## Distinction from Stake Weight

Relative stake weight and stake weight name a proportion and the absolute measure it is built from.
Stake weight is a validator's computed stake influence, while relative stake weight is that value
divided by the total stake weight of validators on the same subnet
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)).

- **Stake weight** — the validator's absolute computed stake influence.
- **Relative stake weight** — that influence as a share of the subnet's total.

## Distinction from Validator Weights

Relative stake weight and validator weights answer different consensus questions. Validator weights
express what a validator scored about miners, while relative stake weight expresses how strongly
that validator's signal counts inside the subnet's weighted aggregation
([Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)).

- **Validator weights** — what the validator evaluated about miners.
- **Relative stake weight** — how strongly that validator's signal counts.

## Distinction from Yuma Consensus

Relative stake weight expresses one validator's stake weight as a share of the validator set on a
subnet. Yuma Consensus is the mechanism that aggregates validator weight signals into emission
shares
([Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Relative stake weight** — proportional stake-based influence within one subnet.
- **Yuma Consensus** — aggregation of validator evaluations into incentives and dividends.

Proportional influence vocabulary and consensus-process vocabulary answer different questions on the
same subnet path ([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Reader Boundary

Relative stake weight is proportional consensus vocabulary. Use stake weight for the computed
influence measure, validator weights for miner-evaluation signals, and Yuma Consensus for the
aggregation process
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The stable article-level point is that relative stake weight explains a validator's proportional
stake-based influence within a specific subnet's consensus context.
