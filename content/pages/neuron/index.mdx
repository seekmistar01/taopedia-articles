# Neuron

A neuron is a registered participant in a Bittensor subnet. Official Bittensor documentation uses
the term when describing miners, validators, and the subnet metagraph state that records information
about subnet participants.

References: [Glossary: Neuron](https://docs.learnbittensor.org/resources/glossary#neuron),
[Understanding Neurons](https://docs.learnbittensor.org/learn/neurons)

## Subnet Participant

The term neuron is used inside subnet context. A subnet can contain miner and validator
participants, and those participants are represented in subnet state rather than as generic network
accounts. This helps distinguish the participant's subnet role from the wallet keys or account
addresses used to operate it.

Reference: [Understanding Neurons](https://docs.learnbittensor.org/learn/neurons)

## Metagraph Context

The subnet metagraph documentation describes the metagraph as a snapshot containing information
about neurons in a selected subnet. In that sense, a neuron is one of the participant records whose
state can appear in subnet-level views such as stake-related state, ranks, weights, incentives, and
activity state.

References: [The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph),
[Glossary: Metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph)

## Relationship to Multiple Mechanisms

A neuron is usually discussed for one incentive mechanism at a time, but subnets can also run
multiple mechanisms in parallel. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms) notes
that validators must evaluate miners separately for each mechanism.

For readers, a neuron still names one registered subnet participant, but the mechanism context
matters. Evaluation and performance readings for one mechanism should not be assumed to describe the
same neuron in another when a subnet uses more than one evaluation system.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)

## Reader Boundary

This page defines the term at a high level. It does not report current subnet membership, live
neuron counts, wallet ownership, or current rewards. Those values change with chain state and should
be checked from current Bittensor sources when needed.
