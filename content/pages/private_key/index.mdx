# Private Key

A private key is the protected side of a cryptographic key pair. The official
[Private Key glossary](https://docs.learnbittensor.org/resources/glossary#private-key) describes it
as the private component used for securing and authorizing transactions and operations within the
Bittensor network.

## Keypair Context

Private keys are read together with public keys. The
[Public Key glossary](https://docs.learnbittensor.org/resources/glossary#public-key) describes the
public side as publicly available key material used for verifying signatures, encrypting messages,
and identifying accounts. The private side is different: it is the protected authorization material
paired with that public side.

This distinction keeps the two terms from collapsing into a single wallet concept. Public key names
the shareable side of the pair, while private key names the side that must remain protected because
it authorizes activity.

## EdDSA Context

Bittensor glossary terminology connects public and private key pairs to EdDSA. The official
[EdDSA Cryptographic Keypairs glossary](https://docs.learnbittensor.org/resources/glossary#eddsa-cryptographic-keypairs)
describes EdDSA as the cryptographic algorithm used to generate public and private key pairs for
Bittensor coldkeys and hotkeys.

For Taopedia readers, EdDSA names the algorithm context, public key names the shareable side of the
resulting pair, and private key names the protected side of that same pair.

## Authorization Context

The private-key concept is useful because it separates account reference from authority. Public
material can identify or verify, while the private key is the component associated with
authorization. The official Private Key glossary supports this authorization framing, and the Public
Key glossary supports the contrast with public account-reference material.

The [Private Key glossary](https://docs.learnbittensor.org/resources/glossary#private-key) supports
the authorization concept, but this article stays at the concept level. It does not describe key
files, wallet setup, signing flows, or any procedure for handling keys.

## Reader Boundary

Private key should not be read as a public key, wallet address, balance, or account profile. It is
protected authorization material in a cryptographic key pair. Claims about a particular account,
key, wallet, or file need a direct source for that specific object.

## Relationship to Public Key

Private key and public key are related but different sides of Bittensor cryptographic keypair
vocabulary. A public key is shareable material used for verification and account reference, while a
private key is the protected component used to authorize transactions and operations. The
[Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key) describes the
public side of the pair, and the
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key) describes
the private side as protected authorization material.

For readers, public key names the shareable pair side, while private key names the protected side
that must remain secret because it authorizes activity.

References: [Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key),
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key)

## Relationship to Coldkeys

Private key and coldkey are related but different scopes in Bittensor wallet-key vocabulary. Private
key names the protected side of a cryptographic keypair, while coldkey names the security-sensitive
wallet key that controls TAO and stake. The
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key) describes
private key as protected authorization material, and the
[Glossary: Coldkey](https://docs.learnbittensor.org/resources/glossary#coldkey) describes the
coldkey as the key that controls TAO and stake.

For readers, private key names the keypair authorization concept, while coldkey names the specific
wallet role that uses protected key material to control funds and stake.

References: [Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key),
[Glossary: Coldkey](https://docs.learnbittensor.org/resources/glossary#coldkey)

## Relationship to Hotkeys

Private key and hotkey are related but different scopes in Bittensor wallet-key vocabulary. Private
key names the protected side of a cryptographic keypair, while hotkey names the operational wallet
key used for routine on-chain actions such as registration and weight submission. The
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key) describes
private key as protected authorization material, and the
[Glossary: Hotkey](https://docs.learnbittensor.org/resources/glossary#hotkey) describes the hotkey
as the operational key used by miners, validators, and subnet actors.

For readers, private key names the protected authorization concept, while hotkey names the
operational wallet role derived under coldkey control for subnet actions.

References: [Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key),
[Glossary: Hotkey](https://docs.learnbittensor.org/resources/glossary#hotkey)

## Relationship to Extrinsics

Private key and extrinsics are related but different parts of Bittensor chain action vocabulary.
Extrinsics name signed transactions and calls submitted to Subtensor, while private key names the
protected key material used to authorize those submissions. The
[Glossary: Extrinsics](https://docs.learnbittensor.org/resources/glossary#extrinsics) describes
signed transactions submitted to the chain, and the
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key) describes
private key as the protected component used to authorize transactions and operations.

For readers, extrinsics name submitted chain actions, while private key names the protected signing
material that can authorize those actions.

References: [Glossary: Extrinsics](https://docs.learnbittensor.org/resources/glossary#extrinsics),
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key)
