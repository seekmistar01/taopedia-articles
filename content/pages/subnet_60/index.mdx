Bitsec.ai is Bittensor Subnet 60, a security-focused subnet. Miners build autonomous security agents
that analyze software projects for vulnerabilities, and validators run those agents in a sandboxed
environment and score them on how well they perform. The incentive-mechanism code is maintained in
the [Bitsec-AI/sandbox](https://github.com/Bitsec-AI/sandbox) repository.

## What the Subnet Produces

The subnet's output is the work of security agents — programs that examine a target codebase and
report findings. In the repository's design, an agent is run against project source files; the
example target referenced in the code is a real audit subject from a competitive-audit context,
which indicates the intended task is finding vulnerabilities in actual projects rather than
answering toy prompts. Framing security analysis as a competition lets the network reward agents
that surface genuine issues.

The current repository is an MVP-stage design, and it points readers to the project's documentation
for the latest details, so specifics of the agent interface and scoring are expected to evolve.

## Miner and Validator Roles

Miners submit an agent — a self-contained program — to the subnet platform. The agent is what
competes: a miner improves its standing by writing an agent that produces better security results on
the assigned projects.

Validators evaluate the submitted agents by running each one inside an isolated Docker container
with resource and time limits, capturing its output, and scoring it against the task. The repository
describes scoring on a 0–100 scale based on how the agent performs and ranking agents to determine
rewards. Running agents in disposable, resource-limited containers keeps untrusted miner code
contained during evaluation. Those validator scores become the weights that feed into
[[Yuma Consensus]], which converts them into the incentive split across miners and validators.

## On-Chain Identity

Live SN60 data, including metagraph state and alpha token pool information, is available on
[TaoStats](https://taostats.io/subnets/60). The Finney subnet identity for netuid 60 registers the
name as Bitsec.ai, with the description "find and fix exploits in codebases"; it links to the
[Bitsec-AI/sandbox](https://github.com/Bitsec-AI/sandbox) repository and lists the project website
at [bitsec.ai](https://bitsec.ai).
