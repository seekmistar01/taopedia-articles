# Local Blockchain

A local blockchain is a private Bittensor chain used for development and testing. In Taopedia, the
term is useful because it separates isolated local experiments from public Bittensor network
activity
([Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

The concept should be read as environment vocabulary. It does not name a subnet role, a production
network, or a separate tokenomic mechanism.

## Private Chain Context

Local blockchain context means the chain is private to the development environment. Work performed
there can help test Bittensor components and subnet behavior without making claims about public
network state
([Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

That distinction is the point of the term. A local result can be useful evidence for a local test,
but it is not the same kind of evidence as behavior observed on mainnet or testnet.

## Relationship to Bittensor Networks

Bittensor network vocabulary separates mainnet, testnet, and localnet. Localnet is the local
environment label, while local blockchain names the private chain used in that environment
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)).

For readers, this keeps the environment and the chain concept aligned but not identical. Localnet is
one network context; local blockchain is the isolated chain that makes that context possible.

## Public Access Contrast

Local blockchain and Public Subtensor point in opposite directions. Public Subtensor is public
access to the Subtensor chain layer, while local blockchain is private local chain context
([Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor),
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)).

This contrast helps avoid mixing local test results with public chain access. A page or example that
uses local blockchain context should not be read as evidence about public Subtensor availability or
production network behavior.

## Block Timing Context

Fast blocks and non-fast blocks are local-chain timing terms. Fast blocks make local block
production faster for testing, while non-fast blocks keep the ordinary twelve-second local interval
([Glossary: Fast blocks](https://docs.learnbittensor.org/resources/glossary#fast-blocks),
[Glossary: Non-fast blocks](https://docs.learnbittensor.org/resources/glossary#non-fast-blocks)).

Both terms belong under local blockchain context. They describe timing behavior inside an isolated
local chain rather than changing the meaning of Bittensor mainnet or testnet.

## Mainchain and Subtensor Context

Mainchain and Subtensor are related public-network concepts, but they are not synonyms for local
blockchain. Mainchain names the production Bittensor network, while Subtensor names Bittensor's
chain layer and system of record
([Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor)).

Local blockchain instead names an isolated private chain for development and testing. It can model
or exercise Bittensor behavior, but it does not replace Mainchain or public Subtensor context.

## Reading the Term

Local blockchain should be read as a concept article term. It is not a tutorial, node-operation
guide, production status report, or source for public chain data
([Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

The stable article-level point is that a local blockchain is private, isolated, and useful for
development and testing while remaining separate from public Bittensor network behavior.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. The local
blockchain concept applies across the Bittensor lifecycle: a private chain is used in localnet for
isolated development and testing, while testnet provides shared non-production conditions, and
mainnet is the live production network with real emissions.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. Local blockchain examples or test outcomes from one
environment should not be read as representing production subnet behavior in another environment.
