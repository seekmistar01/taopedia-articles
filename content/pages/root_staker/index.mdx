# Root Staker

A root staker is a TAO holder who stakes on the Root Subnet through validators. The
[Root Staker glossary entry](https://docs.learnbittensor.org/resources/glossary#root-staker) anchors
that role, and the Root Subnet glossary places it on Subnet Zero
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

The term is useful because it names the staker role on Root Subnet without turning that role into a
validator role or an alpha-staker role.

## Root Subnet Staking

Root Subnet staking lets TAO holders stake through validators in a subnet-agnostic setting. This is
different from ordinary subnet staking, where alpha-side staking belongs to a particular subnet
context ([Glossary: Root Staker](https://docs.learnbittensor.org/resources/glossary#root-staker),
[Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

For readers, the practical distinction is asset and venue. A root staker is staking TAO on Root
Subnet, while alpha staking belongs to a subnet's local alpha-token context.

This makes the term most useful when a source needs to name who supplies TAO to Root Subnet
validators before later dividend handling is discussed.

## Delegation Context

A root staker participates by delegating TAO to a validator on the Root Subnet. The staking and
delegation overview places TAO holders and validators in that staking relationship, while the Root
Staker glossary supplies the role term
([Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation),
[Glossary: Root Staker](https://docs.learnbittensor.org/resources/glossary#root-staker)).

This keeps the article focused on the role. Root staker names the TAO holder in the relationship;
validator choice and staking steps belong to the relevant staking documentation.

## Dividends and Root Claims

Root-routed stake can earn alpha dividends from subnets. Root claims explain how those accumulated
dividends are handled, including Keep and Swap outcomes
([Root Claim Overview](https://docs.learnbittensor.org/staking-and-delegation/root-claims),
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution)).

That claim layer comes after the staking role. Root staker names who supplied TAO through Root
Subnet staking; root claims describe what can happen to dividends produced through that position.

## Root Proportion Context

Root proportion is nearby vocabulary because it measures how root-routed TAO appears in a subnet's
dividend context. Root staker is the role term, while root proportion is the subnet-level measure
([Glossary: Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion),
[Glossary: Root Staker](https://docs.learnbittensor.org/resources/glossary#root-staker)).

The relationship is simple: root staker names the TAO holder; root proportion helps describe the
subnet dividend context connected to root-routed TAO.

## Reader Boundary

Root staker should be read as a concise role term for Root Subnet staking. It does not by itself
select a validator, describe a claim preference, or summarize every staker role in Bittensor
([Glossary: Root Staker](https://docs.learnbittensor.org/resources/glossary#root-staker),
[Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

The stable article-level point is that root staker names a TAO holder who stakes on the Root Subnet
through validators.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For Root Staker,
this sequence gives readers a boundary for interpreting staking examples and root-network
participant notes.

Localnet examples are isolated and reflect local chain state, so they are useful for controlled
experiments rather than evidence of live Bittensor behavior. Testnet examples add shared
non-production conditions, which can reveal integration behavior without touching mainnet state.

On mainnet, Root Staker examples should be read as live production root staking behavior on the
production Bittensor network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet, so outcomes from one environment should not be treated as
proof of behavior in another.
