# TAO

TAO is the network token used in Bittensor incentive, staking, subnet reserve, and emissions
context. It sits at the center of Bittensor's incentive system
([Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction)), and emissions
include newly created TAO alongside subnet-specific alpha tokens
([Emission](https://docs.learnbittensor.org/learn/emissions)).

TAO reference language separates the network token from subnet alpha tokens, RAO denomination
language, and pool-specific TAO reserve language so that related terms are not collapsed into one
asset label.

## Network Token Role

TAO is the token of the Bittensor network as a whole. The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction) connects TAO to the
network's incentive design.

This makes TAO broader than any one subnet. A subnet can have its own alpha token, but TAO remains
the network-level token used across Bittensor.

The practical boundary is scope. TAO names the shared network token; a subnet alpha token names an
asset scoped to one subnet. That distinction is enough to keep subnet-token mechanics from being
read as general TAO mechanics.

## Emission Role

TAO is part of Bittensor's emission system. In the
[Emission](https://docs.learnbittensor.org/learn/emissions) reference, emissions include newly
created TAO and subnet-specific alpha tokens moving through the system.

That means TAO should be read separately from subnet alpha. Both can appear in the same emission
discussion, but they do different work: TAO is the network-token side of emission accounting, while
alpha is the subnet-token side.

In emission passages, the key distinction is which token is being named. Newly created TAO belongs
to TAO context; subnet-specific alpha movement belongs to alpha token context.

## Subnet Reserve Role

TAO also appears inside subnet pool language. The
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) reference
pairs a TAO reserve with a subnet-specific alpha reserve in subnet pools.

In that pool setting, TAO is the network-token side of the pair. Alpha reserve is the subnet-token
side. Dynamic TAO connects those two sides inside subnet tokenomics.

TAO reserve is therefore narrower than TAO itself. It names TAO held in the pool side of a subnet
liquidity-pool relationship, not every TAO balance and not the subnet's alpha reserve
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Staking Role

TAO can be used in staking context. Staking and delegation connect stake to validators in Bittensor
([Staking and Delegation](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

In subnet staking, TAO can appear beside subnet alpha through Dynamic TAO tokenomics. The staking
term itself should still be kept separate from the token label: staking describes the relationship
to validators, while TAO identifies the network token involved when the context names TAO.

## Alpha Token Contrast

TAO and alpha tokens have different scopes. The
[Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha) treats
alpha tokens as subnet-specific assets, while TAO is the network token.

That contrast is central to Dynamic TAO. TAO can sit beside alpha in subnet pools and emissions, but
the two names do not refer to the same asset.

## Term Placement

TAO names the network token. RAO names a smaller denomination of TAO. Alpha tokens name
subnet-specific assets. TAO reserve names the pool-held TAO side of a subnet pool
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

These terms are related because they often appear together in subnet tokenomics, but each one names
a different part of the system. TAO is the asset, RAO is a denomination of that asset, alpha is a
subnet asset, and TAO reserve is a pool role for TAO.

## Transfer Moves Existing TAO Between Addresses

A transfer sends TAO from one Bittensor wallet address to another. The
[Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer) describes that
movement as sending TAO tokens between wallet addresses on the network.

That action relocates TAO that already exists. It does not mint new TAO or attach value to a
validator position. The
[Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address) names
the destination identifier a sender targets when moving TAO to another account.

Transfer and emission answer different questions about the same network token. Emission names newly
created TAO entering the system, while transfer names existing TAO changing holder
([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Issuance Counts Circulating TAO Supply

Issuance names the total amount of TAO circulating in the Bittensor network. The
[Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance) includes TAO held
in wallets and TAO held in subnet liquidity pools.

That measure is broader than one wallet balance or one subnet pool line. A wallet balance names TAO
for one holder, while subnet documentation pairs a TAO reserve with an alpha reserve inside one
subnet pool
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Emission processes can change the circulating picture over time, but issuance names the counted
supply state those processes affect ([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Halving Cuts Daily TAO Emission Rate

Halving reduces Bittensor's daily TAO emission rate. The
[Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving) describes halving as
cutting the daily token emission rate in half, similar in spirit to Bitcoin's supply schedule.

Halving documentation ties TAO halving to supply-based thresholds rather than a fixed calendar date.
When the relevant issuance threshold is reached, the network TAO emission rate drops by fifty
percent ([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

TAO halving and alpha halving are separate supply schedules. A subnet alpha halving milestone tracks
that subnet's alpha issuance path and should not be read as the same event as a network TAO halving
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For TAO on a
subnet such as netuid 1, that sequence changes how readers should interpret balances, staking
amounts, and emission examples.

In localnet, TAO examples can be exercised in an isolated environment. Local balances and transfers
reflect local chain state rather than production token holdings.

On testnet, TAO activity can be observed in a shared, non-production network. Testnet balances and
staking context are separate from mainnet wallet state
([Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau)).

On mainnet, TAO is the live production incentive token used for staking, subnet economics, and
network activity on subnets such as netuid 1.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A TAO amount cited from one environment should not be read
as representing production balances or emissions in another environment.

## Distinction from Protocol Alpha

TAO and protocol alpha have different scopes. TAO is the Bittensor network token, while protocol
alpha names subnet-specific alpha tokens in Dynamic TAO tokenomics
([Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau),
[Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

- **TAO** — the shared network token used across Bittensor.
- **Protocol alpha** — a subnet-scoped token asset, not the network token.

## Distinction from RAO

TAO and RAO describe the same asset at different unit scales. TAO is the familiar whole-token label,
while RAO is the smaller denomination of that same TAO-denominated value
([Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau),
[Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao)).

- **TAO** — the network token at whole-unit scale.
- **RAO** — one billionth of a TAO for fine-grained accounting.

RAO does not introduce a second token system: it is the small-unit expression of TAO-denominated
value, so RAO references should still be understood as TAO references at a finer unit
([Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao)).

## Distinction from Alpha Tokens

TAO and alpha tokens name a network-scoped asset and a subnet-scoped asset. TAO is the Bittensor
network token used across the whole network, while alpha tokens give each subnet its own token
context under Dynamic TAO
([Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau),
[Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Emissions can mention both in one flow, but the labels answer different scope questions. Newly
created TAO names network-token emission, while subnet-specific alpha movement names the subnet
asset side ([Emission](https://docs.learnbittensor.org/learn/emissions)).

- **TAO** — the shared network token across Bittensor.
- **Alpha tokens** — subnet-scoped assets local to each subnet.

## Distinction from Yuma Consensus

TAO is the network token used in Bittensor incentive, staking, subnet reserves, and emissions
context. It names the primary network asset, not the on-chain step that turns validator weight
submissions into emission shares
([Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction),
[Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau)).

Yuma Consensus is the on-chain process that converts submitted validator weights into emission
shares after evaluation completes. Emissions can mint new TAO and subnet alpha, but Yuma Consensus
names weight aggregation and settlement rather than the token label itself
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Readers should keep network-token vocabulary separate from the settlement step. TAO answers which
asset incentives and staking use; Yuma Consensus answers how validator weights become emission
shares in a subnet ([Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **TAO** — primary Bittensor network token for incentives and staking.
- **Yuma Consensus** — on-chain weight aggregation that settles emissions.

## Reader Boundary

TAO names the Bittensor network token, not a subnet alpha token, a staking relationship by itself,
or a market-price label. RAO is a smaller denomination of TAO; alpha tokens are subnet-specific
assets; TAO reserve names the pool-held TAO side of a subnet pool
([Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau),
[Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Emission, staking, transfer, and halving passages can all mention TAO, but each names a different
movement or role. Newly created TAO belongs to emission context; existing TAO moved between wallet
addresses belongs to transfer context; threshold-based emission-rate reduction belongs to halving
context ([Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer)).

## Distinction from Emission

TAO and emission relate as an asset and the process that issues it. TAO is Bittensor's base token
used for staking, registration, and transfers, while emission is the per-block issuance that mints
new tokens and distributes them to subnet participants according to consensus
([Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau),
[Glossary: Emission](https://docs.learnbittensor.org/resources/glossary#emission),
[Emissions](https://docs.learnbittensor.org/learn/emissions)).

- **TAO** — Bittensor's base token.
- **Emission** — the per-block issuance that distributes new tokens.
