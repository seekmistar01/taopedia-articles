# Mainchain

Mainchain is the glossary term for the primary Bittensor blockchain network. The glossary describes
it as the primary Bittensor network used for production purposes
([Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

The term is network-context vocabulary. It should be read separately from localnet, testnet,
Subtensor as the chain layer, and public access surfaces that connect to the chain.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. Mainchain belongs
to the final production network context in that sequence
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

Localnet and testnet examples belong to separate chain histories. Mainchain marks production network
context rather than isolated development or shared non-production testing.

Reaching mainchain context places activity on the primary Bittensor network without, by itself,
claiming mechanism quality or reward outcomes.

## Testnet and Localnet Are Separate Chain Histories

Network documentation separates mainnet, testnet, and localnet as distinct environments. A subnet
observation, registration, or stake example from testnet or localnet therefore belongs to that
environment's chain history rather than to mainchain evidence by default
([Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)).

Readers comparing environments should keep both labels attached. The mainchain concept stays stable,
but the observed state belongs to the network where it was recorded.

## Distinction from Block

Mainchain and a block name a network and one unit of its ledger. Mainchain is the primary production
Bittensor network context, while a block is a single record of transactions within that chain
([Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[Glossary: Block](https://docs.learnbittensor.org/resources/glossary#block)).

- **Mainchain** — the primary production network context.
- **Block** — a single ledger record within the chain.

## Distinction from Metagraph

Mainchain and the metagraph describe the chain at different scopes. Mainchain is the primary
production Bittensor blockchain network, while the metagraph is a derived snapshot of one subnet's
neuron state recorded on that chain
([Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph)).

- **Mainchain** — the primary production Bittensor blockchain network.
- **Metagraph** — a derived snapshot of one subnet's neuron state on that chain.

## Distinction from Subnet

Mainchain and a subnet name the whole network and one market inside it. Mainchain is the primary
production Bittensor network, while a subnet is a specialized market that runs on that network
alongside many others
([Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[Glossary: Subnet](https://docs.learnbittensor.org/resources/glossary#subnet)).

- **Mainchain** — the primary production network.
- **Subnet** — one specialized market running on it.

## Distinction from Subtensor

Mainchain and Subtensor name related but distinct concepts. Mainchain names the primary production
network context, while Subtensor names Bittensor's layer 1 blockchain and system of record for
transactions and rankings
([Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor)).

- **Mainchain** — the primary production network context.
- **Subtensor** — the layer 1 blockchain and system of record.

## Distinction from Bittensor Networks

Bittensor Networks is the broader environment label, while Mainchain names the primary production
network context inside that picture
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain)).

- **Bittensor Networks** — the umbrella environment vocabulary (mainnet, testnet, localnet).
- **Mainchain** — the primary production network within that set.

## Distinction from Public Subtensor

Mainchain and Public Subtensor name different layers. Mainchain names the primary production network
context, while Public Subtensor names publicly reachable access to the Subtensor chain layer
([Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)).

- **Mainchain** — the primary production network context.
- **Public Subtensor** — public access surface for the Subtensor chain layer.

## Distinction from Local Blockchain

Mainchain and local blockchain name opposite sides of network context. Mainchain names the primary
production network, while local blockchain names a private chain used for development and testing
([Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)).

- **Mainchain** — the primary production network context.
- **Local blockchain** — private development and testing chain context.

## Distinction from Yuma Consensus

Mainchain in Bittensor context refers to the primary Subtensor ledger that records balances,
staking, subnet state, and governance. It names the base-chain layer vocabulary, not the on-chain
step that turns validator weight submissions into emission shares
([Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor)).

Yuma Consensus is the on-chain process that converts submitted validator weights into emission
shares after evaluation completes on subnets recorded by that ledger. Mainchain state holds the
inputs and outcomes Yuma Consensus settles each epoch
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Readers should keep ledger-layer vocabulary separate from the settlement step. Mainchain answers
which chain records Bittensor state; Yuma Consensus answers how validator weights become emission
shares in a subnet
([Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Mainchain** — primary Subtensor ledger for Bittensor state.
- **Yuma Consensus** — on-chain weight aggregation that settles emissions.

## Distinction from Network

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
lists mainnet alongside testnet and localnet. Mainchain belongs to the primary network side of that
vocabulary, while testnet and localnet name separate contexts.

## Distinction from Subnet Development Ends in Mainnet

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. Mainchain
therefore marks the final production stage in that sequence rather than an isolated technical label.

Reaching mainnet context changes what an example can prove about live participation. It places
activity on the production Bittensor network without, by itself, claiming mechanism quality or
reward outcomes.

Mainnet is the production environment in the
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) set, and the
[Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain) describes
mainchain as that primary, production-purpose network. In prose, mainnet is therefore the usual
deployment label for mainchain context, distinct from testnet and localnet.

That naming keeps network environment explicit. Mainchain vocabulary identifies production network
context rather than every Bittensor chain deployment.

## Reader Boundary

Mainchain does not name endpoint status, a connection guide, local development behavior, a subnet
quality claim, or every detail of the Subtensor layer
([Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

Keeping that boundary preserves the term's narrower meaning. Mainchain identifies the primary
Bittensor network context, while Subtensor, Public Subtensor, local blockchain, testnet, and
subnet-development references describe neighboring parts of the system.
