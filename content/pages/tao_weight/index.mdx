# TAO Weight

TAO weight is a Bittensor consensus parameter used when TAO stake is compared with alpha stake
inside a subnet's stake-weight context. The
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight) entry frames
it as the global parameter for TAO stake's relative influence against alpha stake, while
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) places
validator stake weight inside subnet evaluation.

## What TAO weight represents

TAO weight does not describe a new token or a separate staking role. It describes how TAO stake is
weighted when it is read alongside alpha stake for validator stake weight. The
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) connects
that combined stake weight to validator consensus power and emissions in a subnet.

## Relationship to stake weight

Stake weight is the broader consensus concept. TAO weight is one input to that concept: it tells
readers how TAO stake is treated relative to alpha stake before validator influence is interpreted
inside the subnet.
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) describes
subnet validators as measuring miner work, and the
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) ties that
measurement context back to stake-weighted influence.

## Relationship to subnet consensus

TAO weight matters because subnet outcomes depend on validator evaluations. In
[[yuma_consensus|Yuma Consensus]], validator signals are aggregated into miner and validator
emissions. [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) explains the
aggregation context, while
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) explains why
those validator signals are subnet-specific rather than universal across every subnet.

## Relationship to Root Proportion

TAO weight and root proportion answer different questions about TAO stake. TAO weight is the global
parameter for how TAO stake contributes relative to alpha stake, while root proportion is a
subnet-level measure of how much of a subnet's dividend context is attributed to root-staked TAO.
The [Glossary](https://docs.learnbittensor.org/resources/glossary#tao-weight) frames TAO weight as
the global stake-contribution parameter, and
[Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion) as the
subnet-level root-stake share.

For readers, TAO weight is network-wide, while root proportion is scoped to a single subnet's
dividend context. The two are related but should not be read as the same number.

References: [TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion)

## Interpretation boundaries

TAO weight should be read as stake-weight terminology, not as a staking recommendation, a live
parameter report, or a formula table. The
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight) provides the
term-level definition, and
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) provides the
subnet context needed to keep the term separate from broader staking and delegation decisions.

## Relationship to Multiple Mechanisms

TAO weight is usually discussed for one incentive mechanism at a time, but subnets can also run
multiple mechanisms in parallel. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#weight-matrix) notes that each
mechanism has its own weight matrix for independent consensus calculations.

For readers, TAO weight still describes how TAO stake is weighted relative to alpha stake inside a
subnet's consensus context, but the mechanism context matters. Stake-weight influence tied to one
mechanism's consensus should be read separately when a subnet uses more than one evaluation system.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix)

## Relationship to Protocol Alpha

TAO weight and protocol alpha address related but different parts of Bittensor subnet tokenomics.
TAO weight is the global parameter that determines how TAO stake contributes relative to alpha stake
when calculating a validator's stake weight inside a subnet, while protocol alpha names the alpha
token inventory the protocol accumulates as a result of chain-side TAO buys during reserve
injection, cached per subnet rather than immediately recycled. The
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight) describes TAO
weight as the global stake-contribution parameter for TAO stake, and the
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)
documentation describes reserve injection as the on-chain process that accumulates protocol alpha.

TAO weight operates in the consensus layer: it shapes how much influence a validator's TAO stake
contributes relative to their alpha stake before emissions are resolved through Yuma Consensus.
Protocol alpha accumulates in the liquidity layer when the protocol purchases alpha tokens with TAO
during reserve injection. The two are not part of the same calculation — TAO weight is a staking
parameter read during consensus, while protocol alpha is a pool accounting result from injection
events. TAO stake that feeds validator influence via TAO weight is separate from the alpha the
protocol holds as protocol alpha.

References: [Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)
