# Alpha Reserve

Alpha reserve is subnet alpha held inside a subnet liquidity pool. In Dynamic TAO, the pool pairs a
TAO reserve with a subnet-specific alpha reserve
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The term is a location label. It identifies pool-held alpha, not every alpha token associated with
the subnet.

## Pool-Side Meaning

Each subnet pool has two reserve sides: TAO reserve and alpha reserve
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Alpha reserve names the subnet-alpha side of that pair. It belongs to the automated-market-maker
pool context, while other alpha vocabulary can describe alpha outside the pool.

## Emission Context

The emission documentation uses alpha reserve injection for alpha added to the reserve side of the
pool
([Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)).

That separates reserve-side alpha from alpha allocated outside the pool. Alpha reserve injection
supports the pool side of subnet tokenomics, while outside-pool alpha belongs to other emission
contexts.

## Alpha Outstanding Contrast

Alpha reserve and alpha outstanding describe subnet alpha by location. Alpha reserve is pool-held
alpha; alpha outstanding is alpha outside the reserve context
([Emission](https://docs.learnbittensor.org/learn/emissions)).

This distinction prevents pool-reserve statements from being read as total-alpha statements. The
same subnet asset can appear in reserve-side and outside-reserve contexts, but those contexts are
not interchangeable.

## ADR Context

Alpha Distribution Ratio compares alpha emitted outside the pool with alpha injected into the
automated market maker pool
([Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)).

In that comparison, alpha reserve belongs to the pool-injection side. Alpha outstanding belongs to
the outside-pool side.

## Protocol Alpha Boundary

Protocol alpha is related to reserve injection, but it is not the same term. Protocol alpha names
alpha gathered by the protocol during reserve-injection activity, while alpha reserve names the
pool-held alpha side itself
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha)).

Keeping those terms separate makes the tokenomics vocabulary clearer. Alpha reserve identifies where
the subnet alpha sits; protocol alpha identifies alpha retained by the protocol.

## Network Context

Bittensor documentation separates localnet, testnet, and mainnet environments
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

Alpha reserve examples should keep that environment boundary attached. A localnet or testnet example
can illustrate reserve mechanics, but it is not evidence of a mainnet reserve outcome.

## Reader Boundary

Alpha reserve is a tokenomics term, not a separate token, staking guide, or pool strategy. It should
be used when the focus is subnet alpha inside the pool
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

When the focus is the paired TAO side, TAO reserve is more precise. When the focus is alpha outside
the pool, alpha outstanding is more precise. When the focus is the broader tokenomics system,
Dynamic TAO is more precise.

## Source Scope

The official references scope Alpha Reserve to a specific Bittensor context rather than a universal
account, market, or operational signal. The article's primary source is
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets), while
[Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)
supplies the adjacent system context used to interpret the term.

For readers, the useful boundary is that Alpha Reserve explains the referenced mechanism or concept
at the level the sources describe it. Current chain values, wallet state, subnet-specific outcomes,
or production measurements still belong to the relevant network, block, account, or tool output at
the time of observation.
