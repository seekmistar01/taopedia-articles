# Synapse

A synapse is the structured message object used in Bittensor subnet communication
([Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Synapse reference](https://docs.learnbittensor.org/python-api/html/autoapi/bittensor/core/synapse/)).

The term belongs to communication vocabulary. It is separate from the subnet role, subnet task,
scoring rule, and reward outcome that may interpret information carried by a message.

## Distinction from Axon

Synapse names the structured message object exchanged in subnet communication. Axon names the
server-side endpoint that receives incoming synapse objects
([Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon)).

Message-object vocabulary and endpoint vocabulary answer different questions on the same exchange.

## Distinction from Dendrite

Synapse names what is carried through subnet communication. Dendrite names the client-side component
that sends and receives those objects during the exchange
([Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Glossary: Dendrite](https://docs.learnbittensor.org/resources/glossary#dendrite),
[Understanding Neurons](https://docs.learnbittensor.org/learn/neurons#neuron-to-neuron-communication)).

Payload vocabulary and client-component vocabulary describe neighboring layers of the same path.

## Distinction from Neuron

A neuron is the UID-identified subnet participant. A synapse is the structured message object those
participants exchange across the axon-dendrite path
([Glossary: Neuron](https://docs.learnbittensor.org/resources/glossary#neuron),
[Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse)).

Participant identity and message-object vocabulary are separate labels in the same exchange.

## Distinction from Subnet Task

Subnet task names the work a mechanism is meant to evaluate. Synapse names the structured object
that can carry challenge and response fields for that exchange
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Task vocabulary describes what work means; synapse vocabulary describes the message shape used to
move that work between roles.

## Distinction from Subnet Protocol

Synapse names the structured message object carried between neurons. Subnet protocol names the
exchange rules for how validators request work and miners return responses
([Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol)).

- **Synapse** — structured message object in dendrite-to-axon exchange.
- **Subnet protocol** — rules governing request-and-response exchange.

Object vocabulary names what travels; protocol vocabulary names the exchange rules
([Understanding Neurons](https://docs.learnbittensor.org/learn/neurons#neuron-to-neuron-communication)).

## Distinction from Replay Attack

Synapse is the structured message object carried through dendrite-to-axon communication, while a
replay attack is the reuse of a valid signed message when freshness checks fail
([Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Understanding Neurons](https://docs.learnbittensor.org/learn/neurons#neuron-to-neuron-communication)).

Message-object vocabulary and message-reuse threat vocabulary answer different questions. Synapse
names the structured payload exchanged between neurons; replay attack names reuse of a previously
valid signed message.

- **Synapse** — the structured message object in dendrite-to-axon communication.
- **Replay attack** — reuse of a valid signed message when freshness checks fail.

## Distinction from Yuma Consensus

Synapse is the structured message object exchanged between dendrite and axon endpoints in subnet
communication on Bittensor. It names the payload format for miner-validator requests, not the
recurring settlement step that converts weight submissions into emission shares
([Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Validating: Dendrite and axon](https://docs.learnbittensor.org/validators#dendrite-and-axon)).

Yuma Consensus is the on-chain mechanism that runs at the epoch boundary after eligible validators
submit weights. It reads the weight matrix, applies clipping and bonding, and converts the result
into miner incentives and validator dividend shares
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Earlier sections separate synapse from replay attacks and from subnet task outputs. Validators send
synapses to miners and score responses into weights; Yuma settles the consensus outcome from
eligible submissions—not the message object itself
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task)).

Readers should treat synapse as off-chain communication vocabulary, and Yuma as the epoch-boundary
mechanism that turns aggregated weights into reward shares on the relevant netuid.

- **Synapse** — structured miner-validator message object.
- **Yuma Consensus** — epoch-boundary settlement from eligible validator weights to emission shares.

## Reader Boundary

Synapse does not name a miner role, a validator decision, a persistent chain record, a wallet
operation, or a complete subnet task design
([Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Subtensor Events](https://docs.learnbittensor.org/subtensor-api/events),
[Subtensor Storage](https://docs.learnbittensor.org/subtensor-api/storage)).

Synapse identifies what is exchanged, while axon, dendrite, subnet task, and incentive mechanism
references describe neighboring parts of the system.

## Distinction from Subnet Miner

A synapse and a subnet miner relate as a message and a participant that exchanges it. A subnet miner
is the neuron that performs the subnet's task and returns results, while a synapse is the structured
request-and-response object passed between validator and miner that carries the query and the
miner's answer ([Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

- **Subnet miner** — the neuron performing the task and returning results.
- **Synapse** — the request-and-response object passed between neurons.
