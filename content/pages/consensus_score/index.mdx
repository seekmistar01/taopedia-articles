# Consensus Score

A consensus score is the agreement threshold Yuma Consensus uses when validator weights are combined
into final rank calculations. It is framed as a stake-weighted median of validator-assigned weights,
which makes it a shared agreement signal rather than an individual validator opinion
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

The term matters because validator submissions can disagree. Consensus score names the threshold
used to limit outlier weights before the filtered result feeds rank calculations
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Agreement Threshold Context

Consensus score sits between submitted validator weights and final rank aggregation. Validators
submit weights, the stake-weighted median establishes the consensus threshold, and later steps use
that threshold to reduce outlier influence
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

Stake matters because the median is stake-weighted. The score summarizes where staked validator
support broadly aligns, so the threshold represents a stake-sensitive agreement point rather than a
simple unweighted average ([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Clipping Context

Yuma Consensus uses the consensus threshold as part of clipping. High weight submissions that exceed
the threshold can be clipped down before final aggregation, limiting the effect of isolated high
assignments ([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

That makes consensus score a filtering term. It is not another weight vector, a validator-quality
rating, or the final rank itself; it is the benchmark applied to validator inputs before rank is
calculated
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

## Rank and Trust Boundary

Consensus score, rank, and trust belong to the same Yuma vocabulary but answer different questions.
Consensus score names the threshold, rank names the aggregate result after filtering, and trust
reflects how much support remains after the clipping process
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank)).

Keeping those terms separate prevents the threshold from being mistaken for an outcome. A consensus
score helps shape final rank, but it is not the rank itself
([Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust)).

## Consensus-Based Weights Boundary

Consensus-based weights, also called liquid alpha, are related because they also depend on consensus
alignment. They describe a bond path that rewards validator alignment with consensus over time
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)).

Consensus score is narrower. It names the stake-weighted threshold around submitted validator
weights, while consensus-based weights describe how alignment influences bonds and incentives
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

## Multiple Mechanisms Context

When a subnet uses more than one incentive mechanism, each mechanism can have its own Yuma context.
That means consensus filtering should be read in the mechanism context being discussed, not as one
undifferentiated value for every evaluation path
([Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets)).

For readers, consensus score still means an agreement threshold over validator weights. The
mechanism context only clarifies which set of validator-weight submissions the threshold belongs to
([Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)).

## Reader Boundary

Consensus score should be read as a consensus-mechanism term. It is not a public subnet rating, a
reward promise, or a user-adjustable setting. Its stable meaning is the stake-weighted agreement
threshold used to filter validator weights before final rank calculations
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

This boundary also keeps it distinct from the weight matrix. The weight matrix names the structured
collection of validator weights; consensus score names a filtering benchmark applied to those
signals
([Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix)).
