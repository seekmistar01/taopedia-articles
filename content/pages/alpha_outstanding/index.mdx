# Alpha Outstanding

Alpha outstanding is subnet alpha outside a subnet's reserve pool. The
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) documentation
describes alpha outstanding as alpha held outside reserves and as total stake in a subnet.

The term is a location label for subnet alpha. It helps separate alpha inside the automated market
maker pool from alpha that sits outside that pool.

## Outside-Pool Meaning

A subnet has a reserve side and an outside-reserve side. The reserve side contains paired TAO and
alpha reserves; alpha outstanding names the subnet alpha outside that reserve context
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

This does not make alpha outstanding a new token. It describes where a portion of the same
subnet-specific alpha asset sits in the subnet tokenomics model.

## Emission Context

The [Emission](https://docs.learnbittensor.org/learn/emissions#alpha-outstanding-injection)
documentation describes alpha outstanding injection as alpha set aside for emission outside the
pool.

That separates it from alpha reserve injection. Alpha reserve injection supports the pool side,
while alpha outstanding injection supports the outside-pool side of subnet emissions.

## Reserve Contrast

Alpha reserve and alpha outstanding are easiest to read as opposite locations for subnet alpha.
Alpha reserve is pool-held alpha, while alpha outstanding is alpha outside the reserve pool
([Emission](https://docs.learnbittensor.org/learn/emissions)).

The distinction keeps reserve statements from being read as total-alpha statements. A subnet can
have alpha in reserve and alpha outstanding at the same time, but those phrases describe different
locations.

## Stake Context

The subnet overview connects alpha outstanding with total stake in a subnet
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)). In that
usage, alpha outstanding helps describe subnet alpha that is outside the pool and relevant to stake
context.

For a reference page, this is enough. The term does not need to become a staking walkthrough; it
only needs to identify the outside-reserve side of subnet alpha.

## ADR Context

Alpha outstanding also appears in Alpha Distribution Ratio context. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)
describes ADR through a comparison between alpha emitted outside the pool and alpha injected into
the automated market maker pool.

In that comparison, alpha outstanding belongs to the outside-pool emission side. Alpha reserve
belongs to the pool-injection side.

## Protocol Alpha Context

Protocol alpha is another outside-reserve category, but it is not the same as alpha outstanding. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#protocol-alpha) describes protocol
alpha as alpha gathered by the protocol during reserve-injection activity.

Alpha outstanding is useful for reading the broader outside-pool alpha context. Protocol alpha is
more specific: it identifies alpha retained by the protocol, not the whole outside-reserve category.

## Reading Boundaries

Alpha outstanding is a tokenomics term, not a separate token, a yield estimate, or an operating
guide. It should be used when the focus is subnet alpha outside the reserve pool
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

When the focus is pool-held alpha, alpha reserve is more precise. When the focus is the subnet asset
broadly, alpha tokens is more precise. When the focus is the wider system, Dynamic TAO is the
broader frame.
