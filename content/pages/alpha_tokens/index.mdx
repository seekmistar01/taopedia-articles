# Alpha Tokens

Alpha tokens are subnet-specific assets in Bittensor. They are separate from TAO, and each subnet
uses its own alpha token in the subnet's emission, staking, and liquidity-pool context. This is a
core part of [[dynamic_tao|Dynamic TAO]], where subnet-level assets help distinguish network-level
TAO from subnet-level tokenomics.

References: [Emission](https://docs.learnbittensor.org/learn/emissions),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)

## What alpha tokens represent

An alpha token represents value inside a specific subnet rather than across the whole Bittensor
network. TAO remains the network asset, while alpha tokens give each subnet a separate asset for
subnet-level emission and staking outcomes. This is why a claim about TAO and a claim about alpha
tokens can describe different parts of the same tokenomics system.

References: [Emission](https://docs.learnbittensor.org/learn/emissions),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)

## Emissions context

Bittensor emission distributes newly created TAO and subnet-specific alpha tokens. TAO emission
describes the network asset entering the system, while alpha-token emission describes subnet-level
assets used in reward distribution. Keeping those two assets separate avoids treating every emission
claim as a TAO-only claim.

References: [Emission](https://docs.learnbittensor.org/learn/emissions),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)

## Staking context

Outside the Root Subnet, stake can be represented in a subnet's alpha units. When a TAO holder
stakes into a subnet, the subnet pool context determines the amount of alpha-denominated stake
received. This means staking and delegation discussions need to be clear about whether they are
describing TAO, alpha-denominated stake, or the conversion between the two.

References:
[Staking/Delegation Overview](https://docs.learnbittensor.org/staking-and-delegation/delegation),
[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage)

## Liquidity pool context

Subnet pools hold TAO reserves and alpha reserves. Staking and unstaking interact with those
reserves, so the amount received from a pool action can differ from a simple one-for-one exchange.
For a concept article, the important point is that alpha tokens are part of the subnet pool design
rather than a calculation walkthrough.

References: [Understanding Slippage](https://docs.learnbittensor.org/learn/slippage),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)

## Supply and halving context

Alpha tokens also have their own halving context. Bittensor distinguishes TAO halvings from alpha
halvings: a TAO halving is network-scoped, while an alpha halving is subnet-scoped. That distinction
matters because an alpha-token supply statement is not the same as a TAO supply statement.

Reference: [Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)

## How to read alpha-token claims

An alpha-token claim should make clear which subnet scope and which token context it describes. Some
claims describe emissions, some describe alpha-denominated stake, and some describe a pool
conversion between TAO and alpha. Exact amounts, pool balances, and timing claims need the specific
source that reports those values.

References: [Emission](https://docs.learnbittensor.org/learn/emissions),
[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage),
[Staking/Delegation Overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)

## Relationship to Multiple Mechanisms

Alpha tokens belong to a specific subnet's tokenomics. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism) notes that when
multiple incentive mechanisms are used, each operates independently, allowing subnet creators to
distribute emissions across different types of work or evaluation criteria.

For readers, alpha tokens still name subnet-scoped assets for emission and staking. When a subnet
runs more than one incentive mechanism, emission context should be read with that subnet's mechanism
layout in mind.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Incentive Mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism)

## Relationship to Subnet Weights

Alpha tokens and subnet weights are related but different parts of subnet tokenomics vocabulary. The
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens) describes
subnet-specific tokens tied to individual subnets, while the
[Glossary: Subnet Weights](https://docs.learnbittensor.org/resources/glossary#subnet-weights)
defines cross-subnet emission allocation from net TAO flow.

For readers, alpha tokens name the subnet-level asset context that staking flows move through inside
a subnet, while subnet weights name how much cross-subnet emission share that subnet receives. A
subnet's alpha token is not the same concept as its subnet weight, even though both describe
subnet-scoped tokenomics.

References:
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens),
[Glossary: Subnet Weights](https://docs.learnbittensor.org/resources/glossary#subnet-weights)

## Relationship to Alpha Reserve

Alpha tokens and alpha reserve are related but distinct parts of Bittensor subnet tokenomics. Alpha
tokens name the subnet-specific asset that exists for a given subnet, while alpha reserve names the
pool-held side of that same asset inside the subnet's automated market maker. The
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) documentation
describes each subnet as an automated market maker with paired TAO and subnet-specific alpha
reserves, and the
[Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)
documentation describes alpha reserve injection as adding alpha to that pool side in proportion to
TAO injection.

For readers, alpha tokens name the asset, while alpha reserve names one location that asset can sit
in: inside the pool. Alpha can also sit outside the pool as participant-held stake or rewards in the
alpha-outstanding context. The two are not interchangeable: alpha tokens describe the subnet asset
itself, alpha reserve describes the pool-held side of that asset.

## Relationship to Autostaking

Alpha tokens and autostaking address related but different parts of miner emission and staking in
Bittensor. Alpha tokens are subnet-specific assets that exist in each subnet's emission and staking
context, while autostaking is a mechanism that automatically routes a miner's emissions into a
staking position—building an alpha token balance in the target validator's subnet without manual
staking steps after each reward arrives. The
[Auto Staking for Miners](https://docs.learnbittensor.org/miners/autostaking) documentation
describes autostaking as automatically staking a miner's emissions to a chosen validator, removing
the need for repeated manual staking transactions.

For readers, alpha tokens name the subnet-scoped asset that staking positions are denominated in,
while autostaking names one automated path that builds and grows a miner's alpha token position
inside a subnet. The [Emission](https://docs.learnbittensor.org/learn/emissions) documentation
provides the broader emission context in which miner rewards arrive and can be channeled into
staking through autostaking. A miner who enables autostaking incrementally accumulates alpha tokens
through each reward cycle without separately executing each staking transaction. The two concepts
are additive: autostaking is the mechanism, and the accumulated alpha token stake is the result that
the miner holds in the subnet's alpha context.

References: [Auto Staking for Miners](https://docs.learnbittensor.org/miners/autostaking),
[Emission](https://docs.learnbittensor.org/learn/emissions)

## Relationship to Conviction and Locked Stake

Alpha tokens and conviction both involve a subnet's alpha, but they describe different things. Alpha
tokens are the subnet-specific assets used in a subnet's emission, staking, and liquidity context,
as this article describes. Conviction and locked stake, by the
[Conviction and locked stake](https://docs.learnbittensor.org/staking-and-delegation/conviction-staking)
documentation, describe locking staked subnet alpha so that it builds a time-based commitment score
and the staked balance cannot be silently reduced below the locked amount.

For readers, alpha tokens name the asset itself, while conviction names a lock that can be placed on
staked alpha to signal commitment over time. The two are not interchangeable: alpha tokens are the
subnet currency that can be held, staked, or pooled, whereas conviction is an optional lock applied
to staked alpha that constrains its reduction. Holding or staking alpha tokens does not by itself
create conviction; conviction arises only when staked alpha is locked under the documented lock
behavior.

References:
[Conviction and locked stake](https://docs.learnbittensor.org/staking-and-delegation/conviction-staking),
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)

## Relationship to Coinbase

Alpha tokens and coinbase are related but different parts of subnet tokenomics. Alpha tokens are the
subnet-specific assets used in each subnet's emission, staking, and pool context. The
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)
documentation describes coinbase as the per-block protocol mechanism that drives TAO emission,
accumulates pending emissions, and triggers Yuma Consensus rounds at epoch boundaries, while the
[Emission](https://docs.learnbittensor.org/learn/emissions) documentation describes alpha tokens as
the subnet-level asset.

The distinction is asset versus mechanism. Alpha tokens are what a subnet's emissions and staking
are denominated in; coinbase is the per-block process that advances the emission cycle each epoch.
Alpha tokens name the subnet asset that participants hold and that subnet emission and staking are
measured in, while coinbase names the recurring engine that drives the emission process the subnet
operates within. One is a token, the other is the protocol mechanism that runs the emission cycle
around it.
