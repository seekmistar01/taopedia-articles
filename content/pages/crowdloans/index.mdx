# Crowdloans

A crowdloan is a Bittensor funding structure for pooling TAO toward a specified action. The
documented action can be an extrinsic execution or a balance transfer, with subnet creation as the
primary example in Bittensor's crowdloan documentation
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans),
[Crowdloan Lifecycle](https://docs.learnbittensor.org/subnets/crowdloans#crowdloan-lifecycle)).

## What It Represents

Crowdloans collect TAO toward a campaign-defined target. Contributors fund a named action under
crowdloan rules rather than placing TAO into a generic pool
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans)).

Exactly one finalization action — a call or a target-address transfer — is specified at creation and
cannot be changed mid-campaign
([Crowdloan Lifecycle](https://docs.learnbittensor.org/subnets/crowdloans#crowdloan-lifecycle)).

If a campaign finalizes successfully, the funded action runs. If it does not, contributors follow a
refund path instead of treating the attempt as a completed subnet or transfer. After the end block
passes without finalization, anyone can trigger refunds; dissolving the campaign afterward is
reserved for the creator and requires every contributor to be refunded first
([Crowdloan Lifecycle](https://docs.learnbittensor.org/subnets/crowdloans#crowdloan-lifecycle)).

Crowdloan examples depend on which network you connect to. Campaign outcomes from one environment
should not be read as production behavior on another
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from Burn Cost

Burn cost names the TAO amount required for subnet creation. A crowdloan names one collective
funding path that can help contributors pool TAO toward that creation action
([Glossary: Burn Cost](https://docs.learnbittensor.org/resources/glossary#burn-cost),
[Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans)).

- **Burn cost** — the dynamic TAO amount required to create a new subnet market.
- **Crowdloan** — collective funding toward a campaign-defined creation action.

## Distinction from Extrinsics

A crowdloan is the pooled-funding campaign. On finalization it runs exactly one specified action — a
call (an extrinsic) or a target-address transfer — chosen when the campaign is created
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans),
[Crowdloan Lifecycle](https://docs.learnbittensor.org/subnets/crowdloans#crowdloan-lifecycle)).

- **Crowdloan** — the pooled-funding campaign that triggers one finalization action.
- **Extrinsic** — the on-chain call a campaign can execute when it finalizes.

## Distinction from Subnet Creator

A crowdloan pools TAO toward a specified action such as subnet creation. A subnet creator is the
entity that registers and owns the resulting subnet
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans),
[Glossary: Subnet Creator](https://docs.learnbittensor.org/resources/glossary#subnet-creator)).

- **Crowdloan** — collective funding structure for a campaign-defined action.
- **Subnet creator** — the entity that registers and owns the resulting subnet.

## Distinction from Transfer

A crowdloan can finalize into a transfer rather than a call. The crowdloan is the funding campaign;
a transfer is a balance movement between accounts and is one of the two finalization-action types
the documentation allows ([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans),
[Crowdloan Lifecycle](https://docs.learnbittensor.org/subnets/crowdloans#crowdloan-lifecycle)).

- **Crowdloan** — the pooled-funding campaign that triggers one finalization action.
- **Transfer** — a balance movement between accounts, used as a finalization action.

## Distinction from Nominator

A nominator stakes TAO on a validator's hotkey as an ongoing position. A crowdloan pools
contributions toward one campaign-defined finalization action such as subnet creation
([Glossary: Nominator](https://docs.learnbittensor.org/resources/glossary#nominator)).

- **Crowdloans** — pool TAO toward a single campaign action that finalizes or refunds.
- **Nominator** — an account that stakes TAO on a validator's hotkey.

## Distinction from Yuma Consensus

Crowdloans pool TAO toward a specified extrinsic or transfer. Yuma Consensus is the epoch-boundary
mechanism that converts eligible validator weight submissions into emission shares on a subnet
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Crowdloans** — collective funding toward a specified on-chain action.
- **Yuma Consensus** — epoch-boundary settlement from validator weights to emission shares.

## Reader Boundary

Crowdloan examples should keep their campaign and network context attached. A cap, end block,
contribution set, refund path, or emission-sharing result belongs to the campaign where it was
observed, not to every possible Bittensor crowdloan
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

This article defines the concept, not a participation workflow. It should not be read as staking
advice, a yield estimate, a campaign recommendation, or a step-by-step funding guide.
