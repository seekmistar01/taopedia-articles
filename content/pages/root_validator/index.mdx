# Root Validator

A root validator is associated with the Root Subnet, also called Subnet Zero
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

The concept belongs to Bittensor's network-wide staking vocabulary. A root validator is connected to
Root Subnet stake, while an ordinary subnet validator belongs to a task-specific subnet with miner
evaluation.

## Root Subnet Role

The Root Subnet is a special subnet where TAO holders can stake to validators without binding that
stake to one task subnet. The glossary also states that miners do not operate there and no
validation work is performed there
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)).

That context explains why the term exists separately. Root validator points to the validator side of
Root Subnet staking, not to a validator scoring miner outputs inside a regular subnet.

## Stake and Weight Context

Root validators are connected to stake that carries weight across the wider Bittensor network. The
Root Subnet glossary explains that validator weight in a subnet includes both alpha-side stake and
TAO staked through Subnet Zero
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

This makes root validator a staking and weighting term. It helps explain how TAO support from the
Root Subnet can relate to validator weight outside the single-subnet context.

## TAO Staking Context

TAO holders can stake to validators on the Root Subnet. That staking relationship supplies the Root
Subnet support associated with root validators
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Glossary: Stake](https://docs.learnbittensor.org/resources/glossary#stake),
[Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

This is why the role can appear near stake and delegation discussions. The term identifies where the
cross-subnet TAO support is directed, while stake records and network context explain the current
effect of that support.

## Development Stage Context

Bittensor documentation separates localnet, testnet, and mainnet development contexts
([Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

Root validator examples need that environment context because local, testnet, and mainnet
observations have different scope.

Localnet examples can show isolated mechanics, testnet examples can show shared non-production
behavior, and mainnet references belong to production network context.

## Root Validators Do Not Score Miner Work

Because the Root Subnet has no miner-scoring task, a root validator does not set weights from miner
output the way a subnet validator does. Its standing instead comes from the TAO staked to it and the
network-wide weight context that stake carries
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

A root validator figure should therefore be read as stake and network-wide weight standing, not as a
miner-evaluation score produced inside a subnet
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

## Distinction from Stake Weight

A root validator and stake weight relate Root Subnet validation to its consensus weighting. A root
validator is a validator associated with the Root Subnet, also called Subnet Zero, while stake
weight is the stake-derived value that scales how much a validator's weights count, so a root
validator's influence is set by its stake weight like any other validator's
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Distinction from Root Proportion

A root validator and root proportion connect Root Subnet stake to its share of subnet dividends. A
root validator is a validator on the Root Subnet, also called Subnet Zero, that holds TAO stake not
bound to one mining subnet, while root proportion measures how much of a subnet's dividend context
is attributed to TAO staked through the root path
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Glossary: Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion)).

The two terms answer different questions. Root validator names the validator role that holds the
root-routed TAO stake, while root proportion names the subnet-level measure of how much of the
dividend context that root-routed stake accounts for, so the role describes who is staked through
Subnet Zero and the measure describes how much the root side weighs on a given subnet
([Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

## Distinction from Subnet Validator

Root validator and subnet validator are related terms, but they describe different validator
contexts. A subnet validator evaluates miner performance and sets weights from miner output, while
the Root Subnet has no miner-scoring work to evaluate
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

- **Subnet validator** — subnet-specific scoring and miner evaluation.
- **Root validator** — Root Subnet stake and network-wide weight context.

## Distinction from Senate

The root validator term is related to Senate membership because the Senate is formed from the top
delegate keys in Bittensor governance. The Senate glossary describes the body as elected from top
delegate keys and responsible for approving or disapproving proposals
([Glossary: Senate](https://docs.learnbittensor.org/resources/glossary#senate)).

- **Root validator** — Root Subnet staking and weighting.
- **Senate** — the governance body that approves or disapproves proposals.

## Distinction from Delegate

A delegate on a subnet such as netuid 1 is a subnet validator that receives nominated stake and
performs validation work inside that subnet's incentive mechanism, while a root validator is
associated with the Root Subnet, also called Subnet Zero, where miners do not operate and stake is
not bound to one task subnet
([Glossary: Delegate](https://docs.learnbittensor.org/resources/glossary#delegate),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

- **Delegate** — receiving subnet validator role on a selected netuid.
- **Root validator** — validator role on Subnet Zero without task-subnet mining.

## Distinction from Blockchain Validator vs Subnet Validator

Blockchain validator and subnet validator describe validation at the network layer and inside a
subnet market. A root validator is associated with the Root Subnet, also called Subnet Zero, where
miners do not operate and TAO stake is not bound to one task subnet
([Glossary: Blockchain Validator vs Subnet Validator](https://docs.learnbittensor.org/resources/glossary#blockchain-validator-vs-subnet-validator),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)).

- **Blockchain validator** — network-layer block production and transaction validation.
- **Subnet validator** — miner evaluation and weight submission inside a subnet.
- **Root validator** — validator role on Subnet Zero without task-subnet mining.

Root validator is a third scope: cross-subnet TAO staking rather than miner scoring or block
production
([Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

## Reader Boundary

Root validator identifies validators on the Root Subnet, where TAO stake is not bound to a
miner-validation subnet
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)).

When the focus is miner evaluation, subnet validator is the more precise term. When the focus is
Root Subnet stake and network-wide weight context, root validator is the more precise term.
