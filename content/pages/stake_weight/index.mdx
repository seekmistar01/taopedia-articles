# Stake Weight

Stake weight is a Bittensor consensus concept for describing how validator stake contributes to
influence inside a subnet. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#stake-weight) defines stake weight as
the computed stake value that determines a validator's consensus power and emissions in a subnet,
while [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) places
that idea in the broader subnet evaluation model.

## Consensus Influence

Stake weight helps determine how much a validator's evaluation contributes to subnet consensus. In
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus), validator rankings are
resolved into miner and validator emissions, and validator stake affects how those rankings are
weighted. Stake weight is the subnet-specific stake measure that sits behind that influence.

## Alpha Stake and TAO Stake

The [Glossary](https://docs.learnbittensor.org/resources/glossary#stake-weight) frames stake weight
as combining alpha stake and TAO stake through the TAO weight concept. The important Taopedia-level
point is that stake weight is not simply one asset balance. It is a subnet-context measure used when
validator influence is interpreted inside that subnet.

## Relationship to TAO Weight

Stake weight and TAO weight are related but not interchangeable. Stake weight is the computed
validator influence measure used inside subnet consensus, while TAO weight is the global parameter
that describes how TAO stake is weighted relative to alpha stake before that influence is computed.
The [Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)
connects stake weight to validator consensus power, and
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight) frames TAO
weight as the relative influence parameter for TAO stake.

For readers, TAO weight is an input to the stake-weight context. Stake weight names the resulting
subnet consensus influence; TAO weight names how one component of staked value is treated in that
calculation.

References:
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)

## Relationship to Subnets

Subnets define the work miners produce and the evaluation standards validators apply.
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) explains that
validators evaluate miner work according to subnet rules before those evaluations feed Yuma
Consensus. Stake weight belongs in that same subnet context because validator influence depends on
how stake is represented for the subnet being evaluated.

## Relationship to Validator Permit

Stake weight and validator permit refer to related but different parts of subnet validation. The
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) describes
stake weight as the computed stake value that determines a validator's consensus power and emissions
in a subnet. The
[Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit)
describes a validator permit as a boolean flag for whether a subnet participant has validation
rights.

For readers, stake weight names the stake-based measure of validator influence, while validator
permit names the permission flag for validation rights that is awarded by stake-weight position. A
stake weight should not be read as a permit itself, only as the input that permit allocation is
based on.

References:
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit)

## Relationship to Effective Stake

Stake weight and effective stake refer to related but different parts of validator staking. The
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) describes
stake weight as the computed stake value that determines a validator's consensus power and emissions
in a subnet. The
[Glossary: Effective Stake](https://docs.learnbittensor.org/resources/glossary#effective-stake)
describes effective stake as the total staked TAO amount of a delegate, including the delegate's own
TAO and TAO delegated by nominators.

For readers, effective stake names the combined staked TAO amount associated with a delegate, while
stake weight names the computed value that turns that stake into consensus influence inside a
subnet. Stake weight should not be read as the same amount as effective stake, only as the
subnet-level measure derived from it.

References:
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: Effective Stake](https://docs.learnbittensor.org/resources/glossary#effective-stake)

## Relationship to Multiple Mechanisms

Stake weight is usually discussed for one incentive mechanism at a time, but subnets can also run
multiple mechanisms in parallel. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#weight-matrix) notes that each
mechanism has its own weight matrix for independent consensus calculations.

For readers, stake weight still describes validator influence inside a subnet, but the mechanism
context matters. Influence tied to one mechanism's weight matrix should not be read as describing
another when a subnet uses more than one evaluation system.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix)

## Relationship to Conviction and Locked Stake

Stake weight and conviction both derive from staked positions, but they measure different things.
Stake weight is the computed stake value that determines a validator's consensus power and emissions
in a subnet, as the
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) defines
it. Conviction and locked stake, by the
[Conviction and locked stake](https://docs.learnbittensor.org/staking-and-delegation/conviction-staking)
documentation, describe locking subnet alpha so that it builds a time-based commitment score and the
staked balance cannot be silently reduced below the locked amount.

For readers, stake weight answers how much consensus influence a validator's stake carries, while
conviction answers whether a staked position has been locked to signal commitment over time. The two
are not interchangeable: stake weight is an influence measure computed for consensus, whereas
conviction is a separate lock-and-score property applied to staked alpha. A position contributes to
stake weight whether or not it is locked; conviction adds a commitment signal and a withdrawal
constraint that the stake-weight measure does not itself capture.

References:
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Conviction and locked stake](https://docs.learnbittensor.org/staking-and-delegation/conviction-staking)

## Interpretation Boundaries

Stake weight should not be read as a universal reputation score, a complete explanation of validator
behavior, or a standalone guide to staking decisions. It is one consensus input around validator
influence. The [Glossary](https://docs.learnbittensor.org/resources/glossary#stake-weight) connects
the term to validator influence, and
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) shows why weighted validator
rankings matter for subnet outcomes.

## Relationship to Coinbase

Stake weight and coinbase are related but different parts of subnet consensus and emissions. Stake
weight is the computed stake value that determines a validator's consensus power inside a subnet.
The
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)
documentation describes coinbase as the per-block protocol mechanism that drives TAO emission,
accumulates pending emissions, and triggers Yuma Consensus rounds at epoch boundaries, while the
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) describes
the consensus-power measure.

The connection is that stake weight is one of the inputs to the consensus rounds coinbase triggers,
while coinbase is the recurring engine that runs those rounds and advances emissions. Stake weight
shapes how much a validator's signal counts when Yuma Consensus runs; coinbase is the per-block
process that fires those epoch calculations. Stake weight describes a validator's standing going
into consensus, while coinbase describes the mechanism that drives the emission cycle that consensus
feeds — one is an input measure, the other is the emission engine.
