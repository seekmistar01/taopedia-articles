# Alpha Reserve

An alpha reserve is the pool-held side of a subnet's alpha token in Dynamic TAO tokenomics.
Bittensor subnets use paired reserve contexts: a TAO reserve and a subnet-specific alpha reserve
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The term is useful because it names where the alpha sits. Alpha reserve means alpha inside the
subnet pool, not alpha already outside the pool as stake, rewards, or other held balance.

## Pool Role

Each subnet pool is described as an automated market maker with two reserve sides. TAO reserve names
the TAO side, while alpha reserve names the subnet-alpha side
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

That makes alpha reserve a location-specific term. It does not mean every alpha token associated
with a subnet; it means the portion held in the pool that pairs with TAO.

## Ratio Context

The reserve pair gives readers a way to understand the pool relationship between TAO and alpha. The
[Emission](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection) documentation
uses the TAO reserve and alpha reserve as the two sides of that relationship.

For an article-level explanation, the important boundary is simple: alpha reserve is one side of the
pool ratio. The article does not need to reproduce short-term pool math or quote changing reserve
amounts.

## Emission Context

Subnet emission can add alpha to the reserve side. The
[Emission](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection) documentation
describes alpha reserve injection as alpha added in proportion to TAO injection.

This is separate from alpha that is allocated outside the pool. Alpha reserve injection supports the
pool side of the subnet economy, while other alpha flows describe balances held outside that reserve
context.

## Alpha Outstanding Contrast

Alpha reserve and alpha outstanding are two ways to describe subnet alpha by location. Alpha reserve
is pool-held alpha. Alpha outstanding is alpha outside that reserve context
([Emission](https://docs.learnbittensor.org/learn/emissions)).

The distinction prevents a common reading mistake. A pool reserve statement and an outside-pool
alpha statement can both be about the same subnet asset, but they do not describe the same location.

## ADR Context

Alpha reserve also appears in Alpha Distribution Ratio discussions. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)
describes ADR through the comparison between alpha emitted outside the pool and alpha injected into
the automated market maker pool.

In that comparison, alpha reserve belongs to the pool-injection side. It is the reserve-side context
that stands opposite alpha distributed outside the pool.

## Protocol Alpha Context

Protocol alpha is related to reserve injection but is not the same term. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#protocol-alpha) describes protocol
alpha as alpha gathered by the protocol during reserve-injection activity, while alpha reserve names
the pool-held alpha side itself.

Keeping those terms separate makes the page easier to read. Alpha reserve identifies the pool side;
protocol alpha identifies alpha retained by the protocol as a result of activity connected to that
side.

## Reading Boundaries

Alpha reserve is a tokenomics term, not a separate token, a staking guide, or a liquidity strategy.
It should be used when the focus is subnet alpha inside the pool
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

When the focus is the paired TAO side, TAO reserve is more precise. When the focus is alpha outside
the pool, alpha outstanding is more precise. When the focus is the broader system, Dynamic TAO is
the broader framing.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For alpha reserve,
that sequence changes how readers should interpret examples of subnet pool-held alpha and reserve
injection mechanics.

In localnet, alpha reserve mechanics can be observed in an isolated environment. Localnet alpha
reserve figures and injection outcomes reflect local chain settings and do not represent production
subnet pool state.

On testnet, alpha reserve can be observed in a shared, non-production network. Testnet alpha reserve
figures are separate from mainnet subnet pool state.

On mainnet, alpha reserve is the live pool-held alpha in a subnet's automated market maker in
production Bittensor subnet operations. The
[Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)
documentation describes the reserve injection mechanics that apply on the production network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. An alpha reserve figure from one environment should not be
read as representing pool state in another environment.
