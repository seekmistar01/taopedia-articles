# Alpha Halving

An alpha halving is a subnet-scoped emission-rate reduction for a subnet's alpha token. It applies
halving terminology to subnet alpha supply rather than to network-wide TAO supply
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha)).

## What It Represents

Each subnet has its own alpha token, so alpha halving is read in relation to one subnet's
alpha-token supply. TAO halving activates at network-level issuance thresholds; alpha halving uses
that subnet's own alpha issuance threshold
([Halving Mechanisms: Alpha Halvings](https://docs.learnbittensor.org/concepts/halving#alpha-halvings),
[Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

During an alpha halving, protocol alpha distributed to miners, validators, and subnet owners each
block is reduced by half uniformly within the affected subnet
([Alpha Halvings](https://docs.learnbittensor.org/concepts/halving#alpha-halvings)).

A subnet's alpha halving progress is tracked as an index separate from the network TAO halving
index. ADR uses the gap between those indexes as an input
([Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)).

Examples tied to netuid 1 illustrate one subnet's alpha halving path. They should not be read as
representing production tokenomics on another network
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from TAO Halving

TAO halving cuts network-wide TAO minting at issuance milestones. Alpha halving cuts one subnet's
alpha-token emission rate at that subnet's alpha issuance threshold
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Halving Mechanisms: Alpha Halvings](https://docs.learnbittensor.org/concepts/halving#alpha-halvings)).

- **TAO halving** — network-wide supply milestone that cuts TAO minting.
- **Alpha halving** — subnet-scoped milestone that cuts that subnet's alpha emission rate.

## Distinction from Emission

Emission names the ongoing reward and injection flow on a subnet. Alpha halving names the milestone
that reduces the alpha emission rate after a supply threshold
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

- **Emission** — ongoing reward and injection flow on a subnet.
- **Alpha halving** — step-down event on that subnet's alpha emission rate.

## Distinction from Issuance

Issuance names total circulating TAO used by TAO halving thresholds. Alpha halving belongs to subnet
alpha emission milestones on a named netuid
([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

- **Issuance** — network TAO circulating-supply measure.
- **Alpha halving** — subnet-scoped alpha emission-rate milestone.

## Distinction from ADR

ADR compares outside-pool alpha emissions with pool injections using TAO and alpha halving indexes.
Alpha halving supplies the subnet alpha halving index
([Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)).

- **Alpha halving** — subnet milestone that cuts alpha emission rate.
- **ADR** — ratio that reads the gap between TAO and alpha halving indexes.

## Distinction from Protocol Alpha

Protocol alpha is subnet alpha accumulated by the protocol during reserve injection. Alpha halving
reduces the rate at which new alpha is emitted to participants after a milestone
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

- **Protocol alpha** — alpha accumulated through reserve injection paths.
- **Alpha halving** — supply milestone that cuts participant alpha emission rate.

## Distinction from Yuma Consensus

Alpha halving changes the subnet alpha emission-rate input. Yuma Consensus converts validator weight
submissions into emission shares each tempo
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

- **Alpha halving** — subnet tokenomics input that lowers alpha emission rate.
- **Yuma Consensus** — consensus step that distributes incentives from validator weights.

## Reader Boundary

Alpha halving should not be read as a reward table, exact timing report, or subnet performance
score. It names a subnet-scoped supply milestone that reduces alpha emission rate
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).
