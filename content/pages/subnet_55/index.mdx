NIOME is Bittensor Subnet 55, a subnet for privacy-safe genomic intelligence. It rewards miners for
generating large-scale synthetic genomic data that is statistically similar to real human DNA but
contains no identifiable individual information. The aim is to give researchers usable genomic
datasets without the consent, privacy, and breach constraints that come with real patient data. The
incentive-mechanism code is maintained in the
[genomesio/subnet-niome](https://github.com/genomesio/subnet-niome) repository.

## What the Subnet Produces

The subnet's output is synthetic genome profiles. Real genomic data is expensive, slow to obtain,
and tightly regulated, which limits the scale of population and disease modeling. NIOME's premise is
that generative models can produce data that preserves the statistical structure of real DNA —
allele frequencies, linkage patterns, and pharmacogenomic variation — while carrying no real
individual's genome, so the data can be generated at large scale with low privacy risk.

Because the work is framed as a continuously evolving competition, the subnet can raise the
difficulty of its simulations over time, pushing miners toward higher biological realism rather than
toward a fixed benchmark.

## Miner and Validator Roles

The repository describes a backend-driven pipeline. The NIOME backend produces genomic simulation
tasks — population parameters and biological constraints — which validators fetch and broadcast to
miners, with every miner receiving the same challenge so submissions are comparable. Miners run
their generative models on the task and return a synthetic genome file that is expected to reflect
realistic allele frequencies, linkage disequilibrium, and pharmacogenomic variants while preserving
privacy.

Validators score the submissions using held-out datasets, statistical fidelity checks, and
biological-plausibility metrics, rewarding quality, novelty, and statistical fidelity rather than
volume. Those validator scores become the weights that feed into [[Yuma Consensus]], which converts
them into the incentive split across miners and validators. The repository notes that no specialized
hardware is required to participate as a miner.

## On-Chain Identity

Live SN55 data, including metagraph state and alpha token pool information, is available on
[TaoStats](https://taostats.io/subnets/55). Readers can reproduce the subnet's registered identity
fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 55 --network finney`.

The Finney subnet identity for netuid 55 points to the
[genomesio/subnet-niome](https://github.com/genomesio/subnet-niome) repository, which is the source
of truth for the generation and scoring process described above.
