# H160 Address

An H160 address is the Ethereum-style wallet address used by EVM wallets in Bittensor EVM. The
official [Bittensor EVM Smart Contracts](https://docs.learnbittensor.org/evm-tutorials) page says
Bittensor wallets use Polkadot-style SS58 addresses, while Ethereum wallets use H160 addresses.

The practical point is simple: an H160 address belongs to the Ethereum-compatible side of Bittensor
EVM. It is not the same thing as a native Bittensor coldkey address, hotkey address, or SS58 wallet
address.

## Why It Exists

Bittensor EVM lets Ethereum-style smart contracts run on the Bittensor blockchain. That makes it
possible to use familiar EVM wallet concepts in a Bittensor setting. H160 addresses are part of that
EVM wallet side.

The address can look familiar to someone who has used Ethereum tooling, but the chain context is
different. The official Bittensor EVM page emphasizes that Bittensor EVM smart contracts execute on
the Bittensor blockchain, not on the Ethereum blockchain.

For a reader, the safest interpretation is: H160 is the EVM wallet address format used inside
Bittensor EVM.

Reference: [Bittensor EVM Smart Contracts](https://docs.learnbittensor.org/evm-tutorials)

## H160 and SS58

H160 and SS58 are different address styles. H160 is associated with Ethereum-style wallets. SS58 is
associated with native Bittensor wallets and public wallet addresses.

The official Bittensor EVM documentation explains that a user can convert between H160 and SS58
representations. That conversion is useful when a transfer or account view needs to cross the
boundary between an EVM wallet and a Subtensor wallet view.

The same documentation gives the important warning: conversion does not produce the corresponding
private key. In other words, turning an address into another representation does not give signing
control over the other wallet.

References: [Bittensor EVM Smart Contracts](https://docs.learnbittensor.org/evm-tutorials),
[Convert Ethereum H160 Address to Substrate SS58](https://docs.learnbittensor.org/evm-tutorials/convert-h160-to-ss58)

## What It Identifies

An H160 address identifies an EVM wallet account. In Bittensor EVM, that is the account an
Ethereum-style wallet can use for EVM smart-contract activity on Bittensor.

That identity should not be stretched too far. An H160 address does not by itself identify a native
Bittensor hotkey. It does not by itself identify a native Bittensor coldkey. It also does not prove
that the holder can sign native Subtensor actions from a related SS58-looking address.

The address tells the reader which EVM account is being referenced. It does not settle every
question about native Bittensor wallet authority.

Reference: [Bittensor EVM Smart Contracts](https://docs.learnbittensor.org/evm-tutorials)

## Wallet Control

Wallet control follows the private key that can sign for the wallet. The Bittensor EVM overview
distinguishes EVM wallets from Bittensor wallets by what they can sign. EVM wallets sign EVM smart
contract interactions. Bittensor wallets sign native Bittensor transactions through Bittensor wallet
tools.

This is why address conversion needs careful wording. A converted address may help someone aim a
transfer or understand the corresponding representation, but it does not move the private key from
one wallet system into another.

For readers, the rule of thumb is: address format is not the same thing as wallet control.

Reference: [Bittensor EVM Smart Contracts](https://docs.learnbittensor.org/evm-tutorials)

## Transfer Context

The official conversion page describes moving across the boundary between EVM wallets and Subtensor
wallets on the Bittensor blockchain. In that context, converting an H160 address can help find the
SS58 representation needed for a transfer workflow.

This should be treated as an address-targeting step, not as wallet recovery or identity merging. The
user still needs the right wallet, the right network, the right destination, and the right signing
authority for the action being taken.

For a Taopedia reader, the main lesson is not how to run the conversion. It is why the distinction
matters: an H160 address and an SS58 address can be related in a workflow without being the same
kind of wallet authority.

Reference:
[Convert Ethereum H160 Address to Substrate SS58](https://docs.learnbittensor.org/evm-tutorials/convert-h160-to-ss58)

## Relationship to Bittensor EVM

H160 addresses appear wherever the EVM side of Bittensor needs Ethereum-style account references.
That includes smart-contract interactions and EVM wallet activity.

Native Bittensor concepts still keep their own meanings. Coldkeys, hotkeys, validators, miners,
staking relationships, and subnet roles are not renamed just because Bittensor also supports an EVM
layer. H160 should be read as an EVM wallet-address term.

Reference: [Bittensor EVM Smart Contracts](https://docs.learnbittensor.org/evm-tutorials)

## Common Confusions

An H160 address can be misunderstood in several ways:

- It is not an Ethereum mainnet activity marker just because it is Ethereum-style.
- It is not a native Bittensor SS58 wallet address.
- It is not a coldkey or hotkey label.
- It is not proof that someone controls a converted SS58 representation.
- It is not a replacement for checking the intended destination before moving funds.

These distinctions matter because wallet mistakes can be irreversible. Readers should keep the EVM
wallet address, native Bittensor wallet address, and signing key separate in their mental model.

## Reader Boundary

This article explains the term, not a setup procedure. It does not describe how to use an EVM
wallet, deploy a contract, or perform address-conversion steps.

An H160 address is best understood as the Ethereum-style address for the EVM wallet side of
Bittensor EVM. When an address crosses into SS58 representation, the user should treat that as an
address-format bridge, not as a transfer of private keys or native wallet authority.
