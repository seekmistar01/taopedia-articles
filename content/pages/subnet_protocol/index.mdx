# Subnet Protocol

A subnet protocol is the rule set that defines how validators and miners interact inside a Bittensor
subnet. The [Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-protocol) defines
it as the rules for validator-miner interaction, including how tasks are requested and responses are
provided. [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)
explains the larger setting: miners produce a subnet's digital commodity, and validators measure
that work.

## Interaction Rules

The protocol describes the interaction pattern around the work, not just the work itself. It helps
define how a validator can ask for work from a miner and how the miner returns a response for
evaluation. The [Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-protocol)
frames this as the rule set for validator-miner interactions, while
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)
places that interaction beside the task and evaluation method.

## Relationship To Miners And Validators

Subnet protocols sit between the two subnet roles. Miners produce responses within the subnet's
rules, and validators evaluate the returned work against the subnet's standards.
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) describes
miners as producing a specific digital commodity and validators as measuring miner work, which is
the role relationship the protocol helps organize.

## Relationship To The Task

The subnet task and subnet protocol are related but separate ideas. The task describes the work
target, while the protocol describes the interaction pattern used around that work.
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)
separates the protocol, task, and evaluation method as parts of an incentive mechanism, and the
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-protocol) keeps subnet protocol
focused on validator-miner interaction.

## Incentive Context

A subnet protocol matters because validator-miner interaction is part of how a subnet turns work
into evaluated output. It does not replace the task or the evaluation method; it provides the
interaction rules those pieces use.
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)
places protocol alongside task definition and evaluation, and
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) explains why
each subnet can have its own goals and standards.

## Relationship to Multiple Mechanisms

A subnet protocol is usually discussed for one incentive mechanism at a time, but subnets can also
run multiple mechanisms in parallel. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms) notes
that validators must evaluate miners separately for each mechanism.

For readers, a subnet protocol still names how miners and validators exchange work. When a subnet
evaluates different kinds of miner work under different mechanism rules, those interaction
expectations should be read per mechanism rather than assumed to apply across the whole subnet.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)

## Reader Boundary

Subnet protocol should be read as a concept about interaction rules. It is not a claim that every
subnet uses the same exchange pattern or evaluates work in the same way. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-protocol) defines the term at
the rule-set level, and
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)
contains broader design material that belongs outside this short encyclopedia entry.
