# Exponential Moving Averages

An exponential moving average, or EMA, is a smoothing method. It blends a new measurement with a
running value from earlier updates
([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

In Bittensor, EMAs are used around signals that should react over time instead of changing entirely
from one observation. Examples include emission flow and validator-miner bond behavior
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)).

## Smoothing Method

EMA smoothing gives newer measurements influence while keeping part of the earlier running value. A
larger smoothing factor reacts faster, while a smaller factor changes more slowly
([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema),
[Glossary: Exponential Moving Average](https://docs.learnbittensor.org/resources/glossary#exponential-moving-average-ema)).

That behavior makes EMAs useful when a Bittensor process should respond to sustained movement rather
than brief noise. The key question is always what signal is being smoothed.

## Signal Meaning

EMA is a method, not a complete explanation by itself. A smoothed subnet-flow signal, a smoothed
bond value, and another smoothed reward signal can all use EMA vocabulary while answering different
questions ([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

That is why an EMA claim should name both the smoothing method and the Bittensor signal it is
applied to. Without the signal, the term only says that a running value is being smoothed.

## Subnet Flow Context

Bittensor uses EMA-smoothed net TAO flow when comparing subnet-level emission allocation. Smoothing
helps a short staking or unstaking burst avoid dominating the flow signal
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Subnet Weights](https://docs.learnbittensor.org/resources/glossary#subnet-weights)).

In this context, EMA names the smoothing method. Emission vocabulary supplies the allocation meaning
attached to the smoothed signal.

## Bond Context

EMAs also appear in validator-miner bond smoothing. A bond can evolve from repeated evaluation
signals instead of jumping entirely from one measurement
([Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

That places EMA vocabulary inside the bond path. Validator weights are evaluation inputs; the EMA
helps smooth the bond value that develops from those inputs.

## Liquid Alpha Context

Liquid alpha changes the EMA smoothing factor for validator-miner bonds according to consensus
alignment. It is a Bittensor rule for varying smoothing inside the bond path rather than a separate
smoothing method
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

This distinction keeps the vocabulary precise. EMA names the general smoothing technique; liquid
alpha names a rule for selecting a smoothing response.

## Consensus Boundary

EMA smoothing can influence values used near consensus, but it is not the consensus rule itself.
Yuma Consensus explains how validator signals are aggregated into subnet outcomes, while EMA
describes smoothing behavior for selected values in that path
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

Keeping that boundary clear prevents the smoothing method from being treated as the full incentive
mechanism. EMA changes how a signal responds over time; the surrounding mechanism defines what the
signal means.

## Multiple-Mechanism Context

When a subnet uses multiple incentive mechanisms, EMA claims should stay tied to the mechanism being
discussed. A smoothed signal in one mechanism is separate from a smoothed signal in another
([Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)).

The smoothing method may be the same, but the signal and mechanism determine what the EMA means in
that article context.

## Reader Boundary

An EMA claim should identify what is being smoothed. In Bittensor, the phrase can refer to subnet
flow, validator-miner bonds, or another reward-related signal
([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

This article does not provide reward forecasts, validator recommendations, or point-in-time network
results. The stable point is the smoothing technique: an EMA blends new measurements with prior
history so Bittensor processes can respond to sustained changes.
