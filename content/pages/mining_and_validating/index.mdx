# Mining and Validating in Bittensor

Bittensor organizes work into specialized subnets. Within each subnet, miners produce work for the
subnet and validators evaluate that work according to the subnet's incentive mechanism
([Mining](https://docs.learnbittensor.org/miners),
[Validators](https://docs.learnbittensor.org/validators)).

The two roles are complementary but distinct. Mining is the production side of subnet work, while
validating is the evaluation side.

## Miners

Miners produce the work a subnet requests. The official mining documentation describes miners as the
role that performs subnet tasks, while subnet documentation frames subnets as specialized markets
for different digital commodities ([Mining](https://docs.learnbittensor.org/miners),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Miner work is subnet-specific. A miner's output only makes sense against the task, protocol, and
evaluation rules of the subnet where the work is being judged.

## Validators

Validators evaluate miner responses and produce weight signals. Official validator documentation
describes validators as measuring miner work, and incentive-mechanism documentation places that
evaluation inside each subnet's mechanism design
([Validators](https://docs.learnbittensor.org/validators),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Validation does not assign final emissions by itself. It contributes a view of miner performance
that later consensus logic can aggregate.

## Incentives and Emissions

The emission path links mining and validating without merging their roles. Miners compete for
miner-facing incentives by producing useful subnet work, while validators can receive validator-side
emissions for useful evaluation ([Emission](https://docs.learnbittensor.org/learn/emissions),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

This keeps vocabulary precise. Incentives point toward miner-side emissions, dividends point toward
validator-side emissions, and staking explains how stakers share in validator-side outcomes.

## Subnets and Mechanisms

Subnets define the task and incentive mechanism that make miner and validator behavior meaningful.
Incentive-mechanism documentation separates task, protocol, and scoring model as parts of subnet
design
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

That boundary explains why the same miner and validator labels can appear across subnets while the
actual work and evaluation standards differ.

## Multiple-Mechanism Context

Bittensor subnets can run more than one incentive mechanism in parallel. Official documentation
notes that miners and validators are evaluated separately for each mechanism context
([Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)).

For readers, miners still produce subnet work and validators still evaluate it, but the mechanism
context must stay attached to claims about performance or emissions.

## Reader Boundary

Mining and validating should be read as role vocabulary. This overview is not a subnet launch guide,
validator ranking, miner performance report, or emissions calculator
([Mining](https://docs.learnbittensor.org/miners),
[Validators](https://docs.learnbittensor.org/validators)).

The stable article-level point is that miners produce subnet work, validators evaluate it, and
consensus turns validator signals into subnet emission outcomes.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. Mining and
validating applies across the Bittensor lifecycle: miners can produce work and validators can
evaluate it in localnet for isolated development, testnet for shared non-production testing, and
mainnet for live operation with real emissions.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. Mining and validating examples or emission outcomes from
one environment should not be read as representing production subnet performance in another
environment.

## Role Scope

Mining and validating are complementary subnet roles, but the role labels do not replace the
subnet's incentive mechanism. Miner and validator documentation describe participants, while
incentive-mechanism documentation explains how task performance, validation, and emissions connect
([Miners](https://docs.learnbittensor.org/miners),
[Validators](https://docs.learnbittensor.org/validators),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

This keeps role vocabulary separate from the mechanism-specific rules that decide scoring and
rewards.
