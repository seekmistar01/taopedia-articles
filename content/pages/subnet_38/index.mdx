Colosseum is Bittensor Subnet 38. Its on-chain identity describes the subnet as strategic games on
Bittensor, and lists
[TAO-Colosseum/tao-colosseum-subnet](https://github.com/TAO-Colosseum/tao-colosseum-subnet) as the
subnet's GitHub repository.

## What Colosseum Provides

The TAO Colosseum README describes a Bittensor subnet connected to a peer-to-peer betting protocol
on Bittensor EVM. The game described by the repository is an underdog-style contract where
participants choose sides and the minority side wins the pool.

At the subnet level, Colosseum turns smart-contract game participation into a miner-scoring signal.
Validators read on-chain game activity, compute scores from recent betting volume, and publish
weights back to the Bittensor network through [[Yuma Consensus]].

## Miner and Validator Roles

Miners participate through the TAO Colosseum game contract. The repository describes miner scoring
as based on betting volume over a recent rolling window, with more recent activity weighted more
heavily.

Validators query the game contract, map miner activity to scores, and set subnet weights. The
repository describes validator responsibilities as checking activity, storing snapshots, computing
scores, and publishing weights. Operators should treat the repository as the source of truth for
current participant guidance.

## On-Chain Identity

Live SN38 data is available on [TaoStats](https://taostats.io/subnets/38). Readers can also
reproduce the chain identity fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 38 --network finney`.

That Finney identity reports the subnet name as colosseum, the subnet URL as taocolosseum.com, and
the GitHub repo as TAO-Colosseum/tao-colosseum-subnet.
