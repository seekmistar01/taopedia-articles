# Root Claims

Root claims handle alpha dividends earned through Root Subnet staking. They decide whether earned
alpha remains as subnet alpha or is converted into TAO and restaked on Root
([Root Claims](https://docs.learnbittensor.org/staking-and-delegation/root-claims),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## What It Represents

Root claims have two outcomes. Keep leaves earned alpha as the alpha of the subnet that produced it.
Swap converts the earned alpha into TAO and restakes that dividend output on Root
([Root Claims](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

Both outcomes apply to dividends, not to the original root stake. The initial TAO staked on the Root
Subnet remains staked on root in both modes
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

Claims can process automatically when accumulated alpha dividends exceed a minimum threshold
documented as 500,000 RAO (0.0005 TAO equivalent), or manually through the `claim_root` extrinsic
with up to five subnets per call
([Root Claims](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

Root-claim examples depend on which network you connect to. Dividend outcomes from one environment
should not be read as production behavior on another
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from Root Proportion

Root proportion is the relative weight a validator's stake carries through the root subnet. Root
claims handle the alpha dividends that root staking earns
([Glossary: Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion),
[Root Claims](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

- **Root proportion** — relative weight of root-routed stake.
- **Root claims** — handling rule for alpha dividends earned through Root staking.

## Distinction from TAO Weight

TAO weight is the global multiplier that scales how much root stake counts against a subnet's alpha
stake. Root claims route earned alpha dividends into either alpha or TAO
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Root Claims](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

- **Root claims** — the handling rule for alpha dividends earned through Root staking.
- **TAO weight** — the multiplier that scales how much root stake counts against alpha stake.

## Distinction from Issuance

Issuance is the circulating TAO supply counted across the network. Root claims decide the form
dividend output takes after rewards accrue
([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance),
[Root Claims](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

- **Root claims** — the rule that routes earned alpha dividends to alpha or TAO.
- **Issuance** — the circulating TAO supply measured network-wide.

## Distinction from Root Subnet

The Root Subnet is where validators register and TAO holders stake in a subnet-agnostic way. Root
claims exist because that stake path can receive alpha dividends from other subnets while the
underlying stake remains on Root
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Root Claims](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

- **Root Subnet** — the special subnet for subnet-agnostic root staking.
- **Root claims** — dividend-handling rule for alpha earned through that staking path.

## Distinction from Yuma Consensus

Root claims route alpha dividends earned through root-network staking. Yuma Consensus is the
epoch-boundary mechanism that converts eligible validator weight submissions into emission shares on
a task subnet ([Root Claims](https://docs.learnbittensor.org/staking-and-delegation/root-claims),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Root claims** — rule for routing earned alpha dividends to alpha or TAO on the root subnet.
- **Yuma Consensus** — epoch-boundary settlement from validator weights to emission shares on a task
  subnet.

## Reader Boundary

Root claims should be read as dividend-handling vocabulary for Root Subnet staking. The claim type
sets how accumulated alpha dividends are handled after they are claimable; it does not rename the
original root stake, guarantee future dividend amounts, or describe a general unstaking action
([Root Claims](https://docs.learnbittensor.org/staking-and-delegation/root-claims),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).
