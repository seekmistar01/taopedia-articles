Luminar Network is Bittensor Subnet 87. Its on-chain identity describes the subnet as "Video
Surveillance Agents" and lists [luminar.network](https://luminar.network/) as the subnet URL.

## What Luminar Network Provides

Luminar's documentation describes the subnet as a universal intelligence layer for video systems
that separates camera hardware from the AI that interprets it. Rather than relying on a single
vendor's analytics, any compatible camera feed can draw on an evolving pool of specialized vision
capabilities supplied by the network. The project frames this as turning existing, passive
surveillance footage into adaptive intelligence without requiring new hardware.

According to the documentation and website, the network runs as a permissionless marketplace for
vision intelligence: independent agents specialize in narrowly defined visual tasks, compete on
measurable performance, and the strongest capability is rewarded each epoch. Luminar's materials
list example tasks such as localized anomaly detection, semantic labeling and curation of raw video
into annotated datasets, and retroactive timeline reconstruction that correlates fragmented footage
from multiple cameras into a continuous, forensic-grade account of events.

## Miner and Validator Roles

Luminar's materials describe miners as the competing agents: they interpret live and historical
camera streams and return perception outputs such as detections, labels, and reconstructed event
narratives. Because the network is open, multiple agents specialize in different tasks and improve
through ongoing competition rather than being tuned by a central authority.

Validators score those outputs on measurable quality and set weights accordingly, so that the
best-performing capability earns the largest share of rewards each round. The website and
documentation should be treated as the source of truth for current participant guidance.

## On-Chain Identity

Live SN87 data is available on [TaoStats](https://taostats.io/subnets/87). Readers can also
reproduce the chain identity fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 87 --network finney`.

That Finney identity reports the subnet name as Luminar Network, the subnet URL as luminar.network,
and the description as "Video Surveillance Agents".
