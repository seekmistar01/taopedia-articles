# Subnet Task

A subnet task is the work target a Bittensor subnet asks miners to perform. It gives the subnet's
incentive design a concrete output to request and evaluate
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The term belongs to subnet design vocabulary. It names the requested work, while protocol, scoring,
and consensus vocabulary describe other parts of how that work is exchanged, evaluated, and
connected to incentives
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

That makes the task the first concept to identify before reading a subnet's protocol, scoring model,
or incentive mechanism. The task says what miners are trying to produce; the surrounding mechanism
explains how that work is requested, measured, and rewarded.

## Distinction from Subnet Miner

Subnet task names the work a subnet expects participants to perform. Subnet miner names the role
that performs that work
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Subnet-task vocabulary belongs to the requested work definition. Subnet-miner vocabulary belongs to
the performing role that produces that work.

## Distinction from Subnet Validator

Subnet task names the work a subnet expects participants to perform. Subnet validator names the role
that evaluates miner work inside the subnet
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Subnet-task vocabulary belongs to the requested work definition. Subnet-validator vocabulary belongs
to the evaluating role that scores that work.

## Distinction from Subnet Protocol

Subnet task names the work a mechanism asks miners to produce. Subnet protocol names the interaction
pattern for how validators request that work and miners return responses
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Task vocabulary describes what work means; protocol vocabulary describes how that work is exchanged.

## Distinction from Subnet Scoring Model

Subnet task names the work a mechanism asks miners to produce. Subnet scoring model names how
validators evaluate responses against that work target
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Task vocabulary describes what work means; scoring vocabulary describes how returned work is judged.

## Distinction from Synapse

Subnet task names the work a subnet expects miners to perform, while synapse names the structured
message object exchanged when validators request that work and miners return results
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Understanding Neurons](https://docs.learnbittensor.org/learn/neurons#neuron-to-neuron-communication)).

Work-target vocabulary and communication-object vocabulary answer different questions. Dendrite and
axon vocabulary name the client and server components that carry synapse objects during the exchange
([Glossary: Dendrite](https://docs.learnbittensor.org/resources/glossary#dendrite),
[Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon)).

- **Subnet task** — work a subnet expects miners to perform.
- **Synapse** — structured message object exchanged during validator-miner requests.

## Distinction from Multiple Incentive Mechanisms

Subnet task names the work target inside a subnet. Multiple incentive mechanisms names the case
where one subnet runs parallel evaluation tracks with separate weight matrices and bond pools
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms),
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Subnet-task vocabulary belongs to the requested work definition. Multiple-incentive-mechanisms
vocabulary belongs to mechanism-scope labels that identify which parallel evaluation track is in
view.

## Distinction from Register

Subnet task names the work a subnet expects miners to perform, while register names the subnet-entry
process that gives a participant a UID slot after paying the dynamic registration burn
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Register](https://docs.learnbittensor.org/resources/glossary#register),
[Mining: Miner registration](https://docs.learnbittensor.org/miners#miner-registration)).

Work-target vocabulary and subnet-entry vocabulary answer different questions. A participant must
register before miner or validator roles can perform or evaluate the task on a selected netuid
([Glossary: UID Slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

- **Subnet task** — work a subnet expects miners to perform.
- **Register** — subnet-entry process that obtains a UID slot.

## Distinction from UID Slot

Subnet task names the work target inside a subnet, while UID slot names the participant position
identified by a unique UID where subnet miner or subnet validator roles operate
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: UID Slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

Work-target vocabulary and participant-position vocabulary answer different questions. Task reading
identifies what work the subnet requests; UID slot reading identifies which indexed participant is
in scope ([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)).

- **Subnet task** — work target inside a subnet.
- **UID slot** — indexed participant position on a selected netuid.

## Distinction from Rank

A subnet task and rank describe a miner's path as work and result. The subnet task is the specific
work a subnet defines for miners to perform and be evaluated on, while rank is the consensus result
that orders miners after validators' clipped weights are merged, so task performance feeds the
weights that determine rank
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Distinction from Metagraph

Subnet task and metagraph are related but different subnet terms. Subnet task names the work target
inside a subnet, while metagraph names the subnet state snapshot for a selected netuid, including
participants and their subnet-level values
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph),
[The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph)).

Readers should use subnet task when the article covers expected work and metagraph when it reads
subnet participant state for a named netuid rather than the work definition alone
([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

- **Subnet task** — the work miners are asked to produce inside a subnet.
- **Metagraph** — the subnet state snapshot for a selected netuid at one block height.

## Distinction from Epoch

Subnet task and epoch are related but different subnet terms. Subnet task names the work miners are
asked to produce inside a subnet, while epoch names the consensus period in which validator signals
for that subnet are processed
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Epoch](https://docs.learnbittensor.org/resources/glossary#epoch),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Readers should use subnet task when the article covers expected work and epoch when it covers the
timing window in which subnet consensus groups validator evaluations on a selected netuid
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Glossary: Tempo](https://docs.learnbittensor.org/resources/glossary#tempo)).

- **Subnet task** — the work miners are asked to produce inside a subnet.
- **Epoch** — the consensus period in which validator signals for that subnet are processed.

## Distinction from Tempo

Subnet task names the work a subnet expects miners to perform. Tempo names the block-based cadence
that groups periodic subnet processes such as weight processing and reward distribution
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Tempo](https://docs.learnbittensor.org/resources/glossary#tempo),
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

The [Glossary: Tempo](https://docs.learnbittensor.org/resources/glossary#tempo) entry defines tempo
as a subnet hyperparameter measured in blocks, while the subnet task label stays on the requested
work evaluated inside that subnet
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Work-target vocabulary and timing vocabulary answer different reader questions. Use subnet task when
naming expected work on a netuid; use tempo when naming how many blocks pass between periodic subnet
processing steps on that netuid
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

- **Subnet task** — the work miners are asked to produce inside a subnet.
- **Tempo** — the block cadence for periodic subnet processing on that netuid.

## Distinction from Deregistration

Subnet task and deregistration are related but different subnet terms. Subnet task names the work
miners are asked to produce, while deregistration names the removal of a low-performing subnet miner
or subnet validator when capacity and replacement rules apply
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Deregistration](https://docs.learnbittensor.org/resources/glossary#deregistration),
[Mining: Miner deregistration](https://docs.learnbittensor.org/miners#miner-deregistration)).

Readers should use subnet task when the article covers expected work and deregistration when it
covers loss of a subnet position on a selected netuid rather than the work definition itself
([Glossary: UID Slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

- **Subnet task** — the work miners are asked to produce inside a subnet.
- **Deregistration** — removal of a subnet miner or validator when replacement rules apply.

## Distinction from Weight Setting

Subnet task names the work miners are asked to produce inside a subnet. Weight setting names the
validator process of assigning scores to that work before consensus runs
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Subnet-task vocabulary belongs to the work target miners return. Weight-setting vocabulary belongs
to the validator scoring step applied to that output
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)).

## Distinction from Yuma Consensus

Subnet task names the work miners produce and validators evaluate inside a subnet. Yuma Consensus
settles validator weight submissions into emission shares each tempo
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Subnet task** — work target miners supply and validators score.
- **Yuma Consensus** — settlement turning validator weights into emission shares.

Work-definition vocabulary and settlement vocabulary answer different subnet design layers
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Reader Boundary

Subnet task is requested-work terminology. It is not the scoring model, the protocol, the consensus
result, or a claim that every returned output is useful
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

A subnet task identifies the work miners are asked to produce inside a subnet or mechanism.
