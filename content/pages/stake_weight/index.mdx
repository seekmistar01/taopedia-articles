# Stake Weight

Stake weight is the subnet-level stake measure used to describe validator influence in Bittensor
consensus. It is the computed stake value tied to consensus power and emissions inside a subnet
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The term connects staking vocabulary to evaluation vocabulary. It explains how stake-derived
influence affects weighted validator signals inside the consensus process.

## Consensus Influence

Stake weight explains how much influence a validator's evaluation carries inside a subnet. It is
more specific than a general stake amount because it belongs to subnet consensus rather than to
every account-balance context
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

This keeps stake weight tied to validation. Staking supplies economic backing, while Yuma Consensus
uses weighted validator signals to shape subnet outcomes.

The influence role is the main boundary. Stake weight does not name the validator's evaluation
itself; it describes how strongly that evaluation can count inside consensus.

## Alpha and TAO Components

Stake weight combines alpha stake, TAO stake, and TAO weight in subnet context
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)).

That framing matters because stake weight is not simply one visible stake amount copied into
consensus. It is the subnet-context value that combines stake components before validator influence
is interpreted.

Alpha stake and TAO stake are neighboring inputs to the concept. Stake weight is the influence value
read by consensus after those components are combined in the subnet context.

## Effective Stake Boundary

Stake weight and effective stake are nearby staking terms, but they answer different questions.
Effective stake is a staking total, while stake weight is the value used for validator influence in
subnet consensus
([Glossary: Effective Stake](https://docs.learnbittensor.org/resources/glossary#effective-stake),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

Effective stake can help explain backing. Stake weight is the term for the influence measure used
inside subnet consensus.

Keeping the terms separate prevents a staking total from being read as the consensus influence value
without the subnet context that gives stake weight its meaning.

## Validator Permit Boundary

Stake weight also sits near validator permits. A validator permit marks validation eligibility in a
subnet, while stake weight is the stake-based influence measure used in consensus context
([Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

The two ideas are related but not interchangeable. Stake weight is a measure; validator permit
language concerns validation eligibility for the subnet.

This boundary keeps eligibility vocabulary separate from influence vocabulary. A permit term answers
whether validation is allowed in context, while stake weight answers how much stake-derived
influence the validator signal carries.

## Validator-Weight Interaction

Stake weight and validator weights work together in Yuma Consensus. Validator weights express a
validator's evaluation of miner work, while stake weight helps determine how much influence that
evaluation carries in subnet consensus
([Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

This keeps the two terms connected without collapsing them. Validator weights are evaluation
signals; stake weight is the stake-derived influence applied around those signals.

Weight vectors and weight matrices sit on the input side of the same process. Stake weight affects
how validator signals count, while vector and matrix vocabulary describes the structured evaluation
inputs ([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix)).

## Rank and Trust Context

Rank and trust are downstream of the weighted evaluation process. Rank names the miner-side result,
while trust describes how much miner-side support remains after clipping
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust)).

Stake weight sits earlier. It shapes validator influence around the signals that later contribute to
rank and trust readings.

Validator trust is also distinct. It describes validator-side influence after clipping, while stake
weight describes stake-derived influence used around validator signals
([Glossary: Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

## Subnet Context

Stake weight should be read inside the subnet context that uses it. Validators evaluate miner work
according to subnet-specific standards, and stake weight helps determine how much influence each
validator signal carries in that setting
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

That makes stake weight a local consensus-influence term, not a universal comparison across every
Bittensor role or subnet.

Use stake weight when the focus is validator influence in subnet consensus. Use effective stake for
a staking-total concept, TAO weight for the TAO component, and validator permit for validation
eligibility.

## Environment Boundary

Bittensor separates mainnet, testnet, and localnet environments. Stake-weight examples from one
environment should not be treated as evidence for another because subnet tasks, validators, and
stake conditions can differ
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Introduction to Bittensor: Subnet development](https://docs.learnbittensor.org/learn/introduction#subnet-development)).

Localnet examples are isolated development examples. Testnet examples are shared non-production
examples. Mainnet stake-weight interpretation concerns production Yuma Consensus behavior.
