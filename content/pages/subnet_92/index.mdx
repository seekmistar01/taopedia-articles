TensorClaw is Bittensor Subnet 92. Its on-chain identity describes the subnet as a decentralized
large language model inference subnet that aggregates high-quality LLM API nodes — such as OpenAI,
DeepSeek, Claude, and Llama — into a unified, highly available, and load-balanced service. The
identity lists [tensorclaw.ai](https://www.tensorclaw.ai) as the subnet URL and
[tensorclaw/tensorclaw](https://github.com/tensorclaw/tensorclaw) as the subnet's GitHub repository.

## What TensorClaw Provides

The TensorClaw README describes the subnet as a decentralized LLM inference network that pools many
model-serving backends behind a single API. A "Business API" forwards real user requests to
participating miners and feeds the resulting usage back into the network's incentive scores, so that
miners who handle more real, high-quality traffic earn a larger share of rewards.

The README notes that miners connect to the network as clients through a central router rather than
exposing public endpoints of their own, which removes the need for each miner to run a publicly
reachable server. When a miner serving a live request becomes unavailable, the request layer
reroutes that request to another miner so callers still receive a response.

## Miner and Validator Roles

The README describes miners as connecting a model backend to the network and serving the inference
requests routed to them. To keep miners from claiming capacity they do not have, validators and the
request layer send a short live prompt as a probe; a backend that fails to answer in time is treated
as offline, scored zero, and removed from the routing pool.

Validators score miners and set weights on-chain. According to the README, a miner's final score
combines a small base component — model availability, probe success, response time, and recent
uptime — with a much larger business component derived from the real request and token volume the
miner serves over a rolling window. Operators should treat the repository as the source of truth for
current participant guidance.

## On-Chain Identity

Live SN92 data is available on [TaoStats](https://taostats.io/subnets/92). Readers can also
reproduce the chain identity fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 92 --network finney`.

That Finney identity reports the subnet name as TensorClaw, the subnet URL as tensorclaw.ai, and the
GitHub repo as tensorclaw/tensorclaw.
