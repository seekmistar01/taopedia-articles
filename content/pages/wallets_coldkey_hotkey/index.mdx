# Wallets, Coldkeys, and Hotkeys

In Bittensor, a wallet represents your identity and signing authority. It manages two key types:

- **Coldkey** — The security-critical key that controls TAO and stake. Use for financial
  transactions (stake/unstake/transfer). Keep offline or in a secure environment.
- **Hotkey** — The operational key used by network participants to run miners/validators, register,
  and submit weights on-chain.

References: [Wallets (learnbittensor.org)](https://docs.learnbittensor.org/keys/wallets),
[Working with Keys](https://docs.learnbittensor.org/keys/working-with-keys),
[BTCLI Reference](https://docs.learnbittensor.org/btcli)

## Why two keys?

- **Security separation:** Coldkey protects funds; hotkey performs day-to-day on-chain actions for
  miners/validators.
- **Operational safety:** If a hotkey is compromised on a node, cold funds remain protected.

## Typical lifecycle

- Create/import wallet and generate a coldkey (store seed securely).
- Generate a hotkey for the role (mining/validating/subnet operations).
- Fund and manage stake with the coldkey; operate services using the hotkey.

## Best practices

- Store coldkey seed offline; use hardware-backed storage where possible.
- Never place coldkey on internet-exposed nodes.
- Validate you’re using official tools and documentation.

## Relationship to Coldkey Swap

The wallets, coldkeys, and hotkeys structure and coldkey swap address related but different parts of
Bittensor key management. Coldkeys and hotkeys form the two-layer key structure used for financial
operations and on-chain participation, while coldkey swap is the mechanism that moves a
participant's on-chain identity — including all associated hotkeys and their registrations — to a
new coldkey without resetting those registrations. The
[Wallets, Coldkeys and Hotkeys](https://docs.learnbittensor.org/keys/wallets) documentation
describes the coldkey as controlling TAO and stake, while the
[Coldkey Swap](https://docs.learnbittensor.org/keys/coldkey-swap) documentation describes the swap
as transferring that on-chain identity to a new coldkey.

For readers, the coldkey and hotkey structure describes how two keys divide signing authority in a
wallet, while coldkey swap names the operation that can change which coldkey holds that authority
without disturbing the hotkeys and registered positions already associated with it.

References: [Wallets, Coldkeys and Hotkeys](https://docs.learnbittensor.org/keys/wallets),
[Coldkey Swap](https://docs.learnbittensor.org/keys/coldkey-swap)
