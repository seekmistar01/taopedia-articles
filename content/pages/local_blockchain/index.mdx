# Local Blockchain

A local blockchain is a private Bittensor chain used for development and testing. Official Bittensor
documentation distinguishes local development environments from public network behavior, which makes
the term useful when discussing tools, subnet experiments, and test workflows.

References:
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)

## Development Context

The local blockchain context is isolated from production use. It lets developers exercise Bittensor
components and subnet behavior without treating the result as mainnet state. That separation is the
main point of the term: local behavior is useful for development feedback, but it is not the same as
live network behavior.

References:
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)

## Relationship to Fast Blocks

Fast blocks are described in official documentation as a development-only local-chain timing
configuration. Local blockchain context explains why that distinction matters: faster local block
production can help testing while remaining separate from public Bittensor network behavior.

Reference: [Glossary: Fast blocks](https://docs.learnbittensor.org/resources/glossary#fast-blocks)

## Reader Boundary

This page defines the term. It does not describe a live public network, a production subnet, or a
deployment recommendation. Readers should use current official setup documentation when they need
step-by-step local development instructions.
