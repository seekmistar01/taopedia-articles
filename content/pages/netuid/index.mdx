# Netuid

A netuid is the identifier used to select a Bittensor subnet context. The
[Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid) places netuid at the
subnet level, and Bittensor documentation uses netuid values when describing subnet-specific state,
including [metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph) data for a
selected subnet ([The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph)).

## Netuid and UID

Netuid and UID are distinct identifiers. A netuid identifies a subnet, while a UID identifies a
participant position within that subnet. The
[Glossary: UID Slot](https://docs.learnbittensor.org/resources/glossary#uid-slot) describes a UID
slot as a position within a subnet occupied by a miner or validator and identified by a unique UID.

Keeping the two separate matters because a participant's UID is scoped to a specific subnet: the
same UID number can appear in multiple subnets and may refer to entirely different participants in
each.

## Subnet Hyperparameters Belong to a Netuid

Each subnet's hyperparameter set is scoped to its netuid. The
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters) reference
describes configurable subnet settings such as
[tempo](https://docs.learnbittensor.org/resources/glossary#tempo), registration burn bounds, and
weights rate limits as properties of one subnet market rather than as global constants for the whole
network.

When readers encounter a hyperparameter name in subnet prose, they should ask which netuid's subnet
configuration is being discussed. Values such as `max_allowed_uids` or `activity_cutoff` belong to
one netuid's subnet context rather than to the whole Bittensor network at once. Tempo names the
block-based cadence that governs epoch transitions inside that selected subnet.

## Subnet Zero as a Netuid

Subnet Zero is itself a netuid with special rules. The
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)
describes Root as a special subnet where miners cannot register and no validation work is performed,
while validators can register and TAO holders can stake in a subnet-agnostic way
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

That makes netuid selection especially important at zero. Root shares the netuid identifier pattern
with other subnets, but its participant rules and staking role differ from an ordinary mining subnet
market.

## Distinction from UID

A netuid identifies a subnet market. A UID identifies a participant position inside the subnet
selected by that netuid
([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid),
[Glossary: UID Slot](https://docs.learnbittensor.org/resources/glossary#uid-slot)).

- **Netuid** — subnet selector identifier.
- **UID** — in-subnet participant position.

Subnet-selector vocabulary and in-subnet position vocabulary must stay paired in precise claims
([The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph)).

## Distinction from Metagraph

A netuid selects which subnet snapshot is in scope. A metagraph is the subnet state snapshot read
for that selected netuid ([The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph),
[Glossary: Metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph)).

- **Netuid** — selector for which subnet market is in scope.
- **Metagraph** — participant and hyperparameter snapshot for that selection.

Selector vocabulary names which market is discussed; metagraph vocabulary names observed fields
there ([Glossary: UID Slot](https://docs.learnbittensor.org/resources/glossary#uid-slot)).

For readers, a metagraph row without an attached netuid is incomplete: the same field name can
appear in more than one subnet, but its meaning belongs to the subnet identified by the netuid used
in the query ([The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph)).

## Distinction from Register

Register is subnet entry that grants a UID slot on a named netuid. Netuid vocabulary only selects
which subnet that entry targets
([Glossary: Register](https://docs.learnbittensor.org/resources/glossary#register),
[Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)).

Destination-subnet vocabulary is incomplete without the admission process that actually occupies a
slot.

## Distinction from Root Subnet

Root subnet, also called subnet zero, is a netuid with special participant rules. Netuid is the
general subnet identifier pattern shared by root and ordinary mining subnets
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Identifier vocabulary is shared; participant-role rules differ by which netuid is selected.

## Distinction from Emission

Emission names currency generation and allocation across the network. Netuid selects one subnet
market within that broader allocation picture
([Glossary: Emission](https://docs.learnbittensor.org/resources/glossary#emission),
[Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)).

Cross-network allocation vocabulary and single-subnet selector vocabulary answer different scope
questions.

## Distinction from Subnet Hyperparameters

Subnet hyperparameters are configurable per-netuid settings such as tempo and registration burn
bounds. Netuid selects which subnet's hyperparameter set is being discussed
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)).

Selector vocabulary names the subnet context; hyperparameter vocabulary names tunable values inside
that context.

## Distinction from Yuma Consensus

A netuid names the identifier used to select which Bittensor subnet context is in scope. That
selector vocabulary tells readers which market's state or rules apply, not how validator weights
settle inside it ([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Yuma Consensus is the on-chain mechanism that runs at the epoch boundary inside the selected subnet.
It reads the weight matrix from eligible validators on that netuid, applies clipping and bonding,
and converts the result into miner incentives and validator dividend shares
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Hyperparameters, registration rules, and scoring models can differ by netuid. Yuma is the shared
settlement layer that reads submitted weights on whichever netuid is selected—but the netuid itself
is only the selector that tells readers which subnet's table, metagraph rows, and consensus pass are
in scope ([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph)).

Readers should not treat netuid as a reward label or as proof of payout on a miner row. It scopes
which subnet's participants and epoch-boundary settlement are under discussion.

- **Netuid** — identifier that selects one subnet context for reads and rules.
- **Yuma Consensus** — epoch-boundary settlement from validator weights to emission shares on that
  netuid.

## Distinction from Subnet

A netuid selects which subnet market is in scope. A subnet is the broader market context identified
by that selector ([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

- **Netuid** — identifier that selects a subnet context.
- **Subnet** — market context for miner work, validators, and hyperparameters.

Selector vocabulary names which market is discussed; subnet vocabulary names the market itself
([Glossary: Metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph)).

## Reader Boundary

This page defines the term. It does not list current subnet names, live identities, registration
state, or current subnet counts. Those values change with chain state and should be checked from
current Bittensor sources when needed.

## Distinction from UID Slot

A netuid and a UID slot identify different levels of the network. A netuid is the number identifying
a whole subnet, while a UID slot is a position within one subnet that a single neuron occupies, so a
neuron is located by the pair of its netuid and its UID slot
([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid),
[Glossary: UID Slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph)).

- **Netuid** — the number identifying a whole subnet.
- **UID slot** — a neuron's position within a single subnet.
