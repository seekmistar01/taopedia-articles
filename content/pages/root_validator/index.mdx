# Root Validator

A root validator is the validator role associated with the Root Subnet, also called Subnet Zero
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Glossary: Root Network](https://docs.learnbittensor.org/resources/glossary#root-network)).

The role belongs to Bittensor's subnet-agnostic staking vocabulary. It should be read separately
from ordinary subnet validators that evaluate miner work inside task-specific subnets.

## Root Subnet Context

The Root Subnet is a special subnet where TAO holders can stake to validators in a subnet-agnostic
way. The official glossary also states that no miners operate there and no validation work is
performed there
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)).

That makes root validator a role in the root staking path rather than a miner-evaluation role. The
Root Subnet context is the reason the term exists separately from subnet validator.

## Network-Wide Weight Role

Root validators are connected to stake that carries weight across the wider Bittensor network. The
Root Subnet glossary explains that validator weight in a subnet includes both alpha-side stake and
TAO staked through Subnet Zero
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Glossary: Root Network](https://docs.learnbittensor.org/resources/glossary#root-network)).

For readers, the key point is scope. A root validator is tied to subnet-agnostic TAO support, while
an ordinary subnet validator is tied to the incentive mechanism of one subnet.

## Relationship to Subnet Validator

Root validator and subnet validator are related terms, but they describe different validator roles.
A subnet validator evaluates miner performance and sets weights from miner output, while the Root
Subnet has no miner work to evaluate
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

This distinction keeps root validator from being treated as a task-subnet evaluator. It is part of
the root staking and weighting path instead.

## TAO Staking Context

TAO holders can stake to validators on the Root Subnet. That staking relationship supplies the
subnet-agnostic support associated with root validators
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Glossary: Stake](https://docs.learnbittensor.org/resources/glossary#stake)).

Root validator is therefore also a staking-destination term. It does not name a separate asset, a
wallet type, or a governance title by itself.

## Relationship to Senate

The root validator role is related to Senate membership because the Senate is formed from the top
delegate keys in Bittensor governance. The Senate glossary describes the body as elected from top
delegate keys and responsible for approving or disapproving proposals
([Glossary: Senate](https://docs.learnbittensor.org/resources/glossary#senate),
[Glossary: Root Network](https://docs.learnbittensor.org/resources/glossary#root-network)).

This relationship does not make every root validator a senator. Root validator names the staking and
weighting role; Senate names the governance body selected from the highest-ranked delegates.

## Reader Boundary

Root validator should be read as a concept article term. It should not be used as a validator list,
a ranking report, a staking instruction, or a claim about a specific account's stake
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)).

The stable article-level point is that a root validator is the validator role on the Root Subnet,
where TAO stake is subnet-agnostic and no miner-validation task loop occurs.
