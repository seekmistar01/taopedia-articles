# Halving

Halving is a tokenomics term for a reduction in token emission rate. In Bittensor, the term belongs
beside emissions because it changes how quickly new tokens enter the reward-distribution path
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

The concept should be read as a supply-rate boundary. It is not a staking strategy, a validator
recommendation, or a complete explanation of subnet reward outcomes.

## Emission Rate Context

Halving changes the rate at which new tokens are emitted. Emissions describe the broader process for
creating and distributing reward material, while halving names one kind of rate change inside that
tokenomics context ([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

That distinction keeps halving from being overread. A halving event affects emission rate, but later
reward outcomes still depend on the relevant asset, subnet context, and consensus path.

## Issuance Threshold Context

Bittensor halving language is tied to supply thresholds. TAO halving reduces the emission rate when
the relevant issuance threshold is reached
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

That makes halving a supply milestone concept rather than a standalone rule for one subnet role. The
threshold matters because it explains when the rate-change vocabulary applies.

## TAO and Alpha Scope

Halving can apply at more than one tokenomics scope in Bittensor. TAO halving belongs to the
network-token side, while alpha halving belongs to subnet alpha-token emission
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

The shared word should not hide the asset boundary. A TAO halving and a subnet alpha halving both
concern emission-rate reduction, but they refer to different token contexts.

## Supply Vocabulary Boundary

Halving is different from burning. Halving reduces the rate at which new tokens are emitted, while
burning removes existing tokens from spendable circulation
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Glossary: Burning](https://docs.learnbittensor.org/resources/glossary#burning)).

Both concepts affect how readers think about token supply, but they operate at different points in
the token lifecycle. Halving is a rate-change concept; burning is a removal concept.

## Relationship to Emission Articles

Emission articles can discuss halving because halving changes an input to the emission path. The
[Emission](https://docs.learnbittensor.org/learn/emissions) and
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving) references together place
halving inside reward creation and distribution, not outside it.

For readers, this means halving is best treated as part of the emissions vocabulary. It clarifies
how token creation slows after a threshold rather than explaining every later distribution detail.

## Reader Boundary

Halving should be read as concept-level tokenomics vocabulary
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)). The stable point
is emission-rate reduction, with Bittensor using the idea around TAO and subnet alpha-token contexts
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

This article does not provide date forecasts, reward forecasts, market interpretation, staking
instructions, or a point-in-time supply report. Exact event timing and observed values belong to the
specific source and context where they are measured.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For Halving, that
sequence changes how readers should interpret emission examples, supply-schedule observations, and
tokenomics comparisons.

In localnet, Halving examples can be exercised in an isolated environment. Localnet observations
reflect local chain state and local configuration rather than production Bittensor behavior.

On testnet, Halving behavior can be observed in a shared, non-production network. Testnet examples
are useful for checking interactions under shared chain conditions while staying separate from
mainnet state.

On mainnet, Halving describes live production halving behavior on the production Bittensor network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. An example or outcome from one environment should not be
read as evidence of production behavior in another environment.

## Supply Threshold Scope

Halving should be read as supply-schedule vocabulary, not as a date forecast or market prediction.
The halving documentation explains the mechanism and threshold context, while emissions
documentation explains how issuance and distribution are accounted for elsewhere
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

That distinction keeps the concept focused on protocol supply behavior instead of expected price or
reward outcomes.
