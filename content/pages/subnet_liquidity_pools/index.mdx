# Subnet Liquidity Pools

The [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) and
[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage) docs frame subnet liquidity
pools as reserve context for subnet tokenomics in Bittensor. A subnet pool connects network-level
TAO with the subnet's own alpha token, so pool context matters when reading claims about
[[dynamic_tao|Dynamic TAO]], staking, unstaking, and emissions.

## What a subnet pool holds

A subnet pool has a TAO reserve and an alpha reserve, as described in
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) and the
[Emission](https://docs.learnbittensor.org/learn/emissions) docs. The TAO reserve reflects TAO
staked into the subnet, while the alpha reserve reflects alpha available in the pool. The same
subnet economy also tracks alpha outstanding, which is alpha held outside the reserve context and
should not be read as the same thing as pool liquidity.

## Relationship to alpha tokens

Subnet pools are one reason alpha tokens need separate wording from TAO. The
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) and
[Emission](https://docs.learnbittensor.org/learn/emissions) docs distinguish network-level TAO from
subnet-level alpha accounting. A pool claim should therefore say whether it is describing TAO
reserves, alpha reserves, or alpha outside the pool.

## Staking and unstaking context

When TAO is staked into a subnet, the
[Staking/Delegation Overview](https://docs.learnbittensor.org/staking-and-delegation/delegation) and
[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage) docs show why pool context
determines the alpha-denominated stake received. When stake is removed, the pool context is used
again to express alpha-denominated stake back into TAO terms. This is why
[[staking_and_delegation|Staking and Delegation]] discussions need to distinguish TAO,
alpha-denominated stake, and pool conversion context.

## Slippage context

[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage) describes the difference
between a simple expected exchange and the result produced by the pool context at the time of the
action. For Taopedia purposes, the important point is conceptual: staking and unstaking interact
with reserves, so pool state can affect the TAO or alpha amount received. Exact amounts need a
direct source for the values being used.

## Emissions context

Bittensor emissions also connect to subnet pools. The official
[Emission](https://docs.learnbittensor.org/learn/emissions) and
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) docs describe
new liquidity entering subnet pools and separate TAO reserve, alpha reserve, and alpha-outstanding
contexts. This means pool liquidity, subnet alpha tokens, and reward distribution are related, but
they are not interchangeable claims.

## How to read pool claims

A subnet-pool claim should identify the asset and scope. The
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage), and
[Staking/Delegation Overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)
docs separate TAO-reserve context, alpha-reserve context, and staking conversion context. Claims
about exact balances, rates, timing, or subnet examples need the specific source that reports those
values.
