Leoma is Bittensor Subnet 99. Its on-chain identity describes the subnet as a decentralized video
generation platform, lists [leoma.ai](https://leoma.ai/) as the subnet URL, and lists
[RendixNetwork/leoma](https://github.com/RendixNetwork/leoma) as the subnet's GitHub repository.

## What Leoma Provides

The Leoma README describes the subnet as an AI video network on Bittensor. In its current form,
miners run Text-Image to Video (TI2V) models: validators send a first frame and a text prompt, and
miners return a short generated video. The README notes that Text-to-Video and Image-to-Video
support are planned.

According to the README, validators score each generated video against a multi-aspect benchmark
covering first-frame fidelity, prompt adherence, temporal quality, and visual artifacts, recording
pass or fail results. Ranking is winner-take-all: the top miner receives full weight for the round
and earns the subnet's emissions for that round.

## Miner and Validator Roles

The README describes miners as publishing a TI2V model and exposing it through a callable inference
endpoint, then committing that reference on-chain so validators can evaluate it. Miners receive
generation challenges, and the strongest outputs win each round.

Validators run an evaluator and a weight-setter. The README describes them as polling the Leoma API
for the latest task, downloading the task data, scoring miner videos with a GPT-4o-based evaluation,
and submitting the results. Each epoch, validators read the computed ranking and set winner-take-all
weights on-chain. The subnet owner runs a separate sampler that creates tasks and an API that
computes the ranking and serves the weight vector. Operators should treat the repository as the
source of truth for current participant guidance.

## On-Chain Identity

Live SN99 data is available on [TaoStats](https://taostats.io/subnets/99). Readers can also
reproduce the chain identity fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 99 --network finney`.

That Finney identity reports the subnet name as Leoma, the subnet URL as leoma.ai, and the GitHub
repo as RendixNetwork/leoma.
