# Bittensor Governance

Bittensor governance is the proposal review process for privileged network changes. The model
separates proposal creation by the Triumvirate from approval by the Senate
([Governance Overview](https://docs.learnbittensor.org/governance)).

This gives governance a bicameral structure: one body brings privileged changes forward, and another
body reviews whether those changes should be approved
([Senate](https://docs.learnbittensor.org/governance/senate)).

## Proposal Review Context

A governance proposal represents a privileged change request. It should be read as a review item
that needs Senate approval before it can move through the governance process
([Governance Overview](https://docs.learnbittensor.org/governance)).

That review context is the main point of the article. Governance is not ordinary subnet operation;
it is the network-level review path for changes that need elevated authority
([Senate](https://docs.learnbittensor.org/governance/senate)).

## Triumvirate and Senate Roles

The Triumvirate and Senate have different responsibilities. The Triumvirate creates proposals, while
the Senate reviews and votes on them
([Governance Overview](https://docs.learnbittensor.org/governance)).

Keeping those roles separate avoids treating proposal creation and proposal approval as the same
action. A proposal can be brought forward without being approved, and Senate approval is the review
threshold ([Senate](https://docs.learnbittensor.org/governance/senate)).

## Visibility and Voting

Governance proposals can be inspected publicly, but voting is restricted to senators. That means
proposal visibility and proposal voting have different boundaries
([Senate](https://docs.learnbittensor.org/governance/senate)).

For readers, this distinction matters because governance records can be reviewed without implying
that every reader can cast a vote
([Governance Overview](https://docs.learnbittensor.org/governance)).

## Privileged Change Scope

Governance is scoped to privileged changes such as network parameters, upgrades, and subnet-related
administrative changes. It is not the everyday process by which miners produce work or validators
evaluate subnet output ([Governance Overview](https://docs.learnbittensor.org/governance)).

This scope keeps governance vocabulary separate from subnet incentive vocabulary. Subnet work is
evaluated through subnet mechanisms, while governance reviews privileged network changes
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Reader Boundary

Bittensor governance should be read as a proposal and approval process. It is not a general voting
tutorial, not a subnet scoring mechanism, and not the same thing as ordinary validator evaluation
([Governance Overview](https://docs.learnbittensor.org/governance)).

The stable article-level point is that privileged changes move through Triumvirate proposal and
Senate approval rather than relying on a single review point
([Senate](https://docs.learnbittensor.org/governance/senate)).
