# Consensus Score

A consensus score is the agreement threshold Yuma Consensus uses when validator weights are combined
into final rank calculations. It is a stake-weighted median of validator-assigned weights inside the
broader Yuma process
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The term matters because validator signals can disagree. Consensus score names the threshold used to
limit outlier weights before the filtered result contributes to rank.

## Agreement Threshold

Consensus score sits between submitted validator weights and final rank aggregation. Validators
provide weights, the consensus process establishes a stake-weighted threshold, and later steps use
that threshold to reduce outlier influence
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

That makes consensus score a filtering benchmark. It is not one validator's opinion, and it is not
the final miner-side result.

## Clipping Context

Yuma Consensus uses the consensus threshold as part of clipping. Weight values above the threshold
can be reduced before final aggregation, which limits isolated high assignments from dominating the
result ([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

For readers, the important boundary is that consensus score helps constrain inputs. It does not
replace the later rank result.

## Rank and Trust Boundary

Consensus score, rank, and trust are related Yuma terms with different roles. Consensus score names
the threshold. Rank names the aggregate miner-side result after filtering. Trust indicates how much
support remains after clipping
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust),
[Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

Keeping those terms separate prevents the threshold from being mistaken for an outcome. Consensus
score helps shape rank, but it is not the rank itself.

## Validator-Weight Boundary

Validator weights are the signals being evaluated by the consensus process. Consensus score is the
threshold produced from those signals for clipping and aggregation context
([Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights),
[Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

This distinction keeps input vocabulary separate from filtering vocabulary. A validator weight is a
signal; consensus score is an agreement threshold around the set of signals.

## Consensus-Based Weights Boundary

Consensus-based weights, also called liquid alpha, are related because they also depend on consensus
alignment. They cover bond adjustment around agreement with consensus over time
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

Consensus score is narrower. It names the stake-weighted threshold around submitted validator
weights, while consensus-based weights describe how alignment influences bonds and incentives.

## Reader Boundary

Consensus score should be read as a consensus-mechanism term. It is not a public subnet rating, a
reward promise, or a user-controlled parameter
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The stable article-level point is that consensus score identifies the agreement threshold Yuma
Consensus uses to filter validator weights before final rank results.
