# Subtensor Constants

Subtensor constants are named runtime values exposed by the official
[Subtensor Constants](https://docs.learnbittensor.org/subtensor-api/constants) reference. That
source describes the page as compile-time runtime constants for the Subtensor runtime and their
respective values.

## What Constants Represent

The constants reference lists named entries under runtime areas. For each entry, it gives the
constant name, data type, access path, value, and a short summary where one is available. This makes
the page a compact reference for runtime-defined values, not a full explanation of the feature that
uses each value.

Reference: [Subtensor Constants](https://docs.learnbittensor.org/subtensor-api/constants)

## Runtime Areas

The official reference is grouped by runtime areas such as aura, balances, commitments, contracts,
crowdloan, drand, proxy, registry, safe mode, scheduler, system, timestamp, utility, swap, and the
Subtensor module. That grouping helps readers identify the part of the runtime that owns a constant
before reading the entry name and summary.

Reference: [Subtensor Constants](https://docs.learnbittensor.org/subtensor-api/constants)

## Bittensor-Specific Context

The Subtensor module section contains Bittensor-specific constants for subnet defaults, registration
bounds, rate limits, weight-related defaults, Liquid Alpha status, and Yuma3 status. Other sections
cover broader runtime areas such as account balances, contracts, proxy behavior, scheduled calls,
system limits, and timestamp timing.

Reference: [Subtensor Constants](https://docs.learnbittensor.org/subtensor-api/constants)

## Relationship to Multiple Mechanisms

Subtensor constants are protocol parameters on the Bittensor chain layer. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism) notes that each
subnet has one or more incentive mechanisms.

For readers, constants still name chain-layer settings.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Incentive Mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism)

## Careful Interpretation

A constant entry shows the name and value exposed by the runtime reference. It should be read with
its runtime area and entry summary. Current network behavior can also depend on mutable storage,
events, and feature-specific documentation, so a constant alone should not be treated as a complete
description of the underlying mechanism.

Reference: [Subtensor Constants](https://docs.learnbittensor.org/subtensor-api/constants)
