# Alpha Distribution Ratio

Alpha Distribution Ratio, often shortened to ADR, is a Bittensor tokenomics measure for comparing
distributed subnet alpha with alpha injected into a subnet's automated market maker pool. The term
belongs to the split between alpha that leaves the pool-side path and alpha that remains part of
pool injection
([Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)).

The ratio is useful because Dynamic TAO has both distribution and injection stages. ADR gives those
two paths a compact comparison point inside a subnet's alpha-token accounting
([Emission: injection and distribution](https://docs.learnbittensor.org/learn/emissions#injection-and-distribution-two-stages-of-the-emissions-process)).

## Distribution and Injection Context

ADR compares two subnet-alpha paths. One path is distribution, where alpha is allocated outward as
part of subnet emissions. The other path is injection, where alpha is added to the subnet pool
alongside TAO reserve activity
([Emission: injection and distribution](https://docs.learnbittensor.org/learn/emissions#injection-and-distribution-two-stages-of-the-emissions-process)).

That makes ADR a relationship term rather than a balance, ranking, or payout table. It says how the
subnet's alpha-token flow is divided between distributed alpha and pool-injected alpha
([Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)).

## Halving Context

ADR is tied to halving vocabulary. The ADR glossary entry relates the ratio to the gap between the
global TAO halving index and a subnet's alpha halving index, so the term sits at the intersection of
system-wide TAO emissions and subnet-level alpha emissions
([Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)).

This keeps ADR separate from a simple emissions total. TAO halvings operate at system scope, while
alpha halvings operate at subnet scope; ADR is a subnet tokenomics ratio shaped by how those halving
contexts differ ([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

## Flow-Based Emissions Boundary

ADR should be read after the subnet context is already in view. Flow-based emissions compare subnets
for allocation, while ADR compares distribution and injection inside a subnet
([Emission: distribution across subnets](https://docs.learnbittensor.org/learn/emissions#distribution-across-subnets)).

That boundary avoids mixing two levels of analysis. Flow-based allocation answers how emission
enters subnet contexts; ADR answers how subnet alpha is split once the relevant subnet context is
being discussed
([Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)).

## Pool-Side Context

The injection side of ADR connects directly to alpha reserve. Alpha reserve is the pool-held alpha
side of a subnet market, and alpha reserve injection adds alpha to that pool-side reserve
([Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)).

For readers, ADR does not replace alpha reserve. Alpha reserve names where pool-side alpha sits; ADR
names the relationship between that pool-side injection path and the distribution path
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Protocol Alpha Context

Protocol alpha is related to ADR because it can arise from reserve-injection activity. During
coinbase-driven reserve injection, the protocol can accumulate alpha that is cached by subnet rather
than immediately recycled
([Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)).

That does not make ADR a protocol-alpha inventory. ADR remains the higher-level ratio between
distributed alpha and injected alpha, while protocol alpha names a specific retained-alpha category
within the broader injection-side mechanics
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha)).

## Reader Boundary

ADR should be read as a subnet tokenomics ratio. It is not a staking instruction, a yield promise,
or a universal comparison across every subnet. The term is about how subnet alpha is divided between
distribution and injection once the relevant subnet context is established
([Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)).

This boundary keeps ADR distinct from adjacent concepts. Halving indexes help shape it, flow-based
emissions establish allocation context, and alpha reserve describes the pool-side location, but ADR
itself is the distribution-versus-injection comparison
([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For alpha
distribution ratio, that sequence changes how readers should interpret examples of subnet emission
allocation and injection versus distribution splits.

In localnet, alpha distribution ratio mechanics can be tested in an isolated environment. Localnet
emission allocation and injection-versus-distribution outcomes reflect local chain settings and do
not represent production subnet emission state.

On testnet, alpha distribution ratio can be observed in a shared, non-production network. Testnet
emission allocation ratios are separate from mainnet subnet emission state.

On mainnet, alpha distribution ratio is the live subnet tokenomics measure that compares distributed
alpha with injected alpha within production Bittensor subnet emissions. The
[Emission](https://docs.learnbittensor.org/learn/emissions) documentation describes the emission
allocation mechanics that apply on the production network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. An alpha distribution ratio example from one environment
should not be read as representing emission allocation in another environment.
