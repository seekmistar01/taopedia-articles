# Weight Matrix

A weight matrix is the subnet-level collection formed from validator weight vectors. The Bittensor
glossary describes it as the matrix made from ranking weight vectors submitted by subnet validators
and used as an input to Yuma Consensus.

References: [Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)

## What the Matrix Represents

Each validator weight vector is one validator's relative evaluation of miner work inside a subnet.
The weight matrix puts those vectors into a shared structure so the consensus process can compare
validator signals at the subnet level instead of treating one validator's view as the whole result.

References: [Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix),
[Validator Weights](https://docs.learnbittensor.org/resources/glossary#weight-vector)

## Why It Matters

Yuma Consensus depends on many validator signals. The weight matrix is the organized input that lets
those signals be combined, filtered, and interpreted as a collective subnet result. This keeps the
article-level meaning focused on aggregation: the matrix is not a separate scoring rule, and it is
not a public quality label for a subnet.

References: [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)

## Relationship to Weight Vectors

A weight vector is the per-validator signal. A weight matrix is the set of those signals arranged
for the subnet's consensus step. This distinction matters because claims about one validator's
vector do not automatically describe the matrix result after other validators' vectors are included.

Reference: [Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector)

## Multiple Mechanisms

The glossary notes that when a subnet uses multiple incentive mechanisms, each mechanism has its own
weight matrix for independent consensus calculations. That means a matrix should be read in the
context of a particular subnet mechanism, not as a universal score across every task the subnet may
support.

References: [Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix),
[Multiple Incentive Mechanisms](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets)

## Careful Interpretation

A weight matrix is a consensus input, not an operating guide. It should not be used to infer exact
participant outcomes without the surrounding Yuma Consensus context, the subnet's incentive
mechanism, and the validator weights that formed the matrix.

References: [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Validator Weights](https://docs.learnbittensor.org/resources/glossary#weight-vector)
