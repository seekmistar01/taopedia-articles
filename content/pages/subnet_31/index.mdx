Recall is Bittensor Subnet 31, a decentralized retrieval-augmented generation (RAG) network. Its
on-chain identity describes it as a community-owned search engine with citations: miners run the
pieces of a RAG pipeline, validators judge how good the results are, and incoming queries are routed
to whichever participants answer best.

## What Recall Provides

Retrieval-augmented generation answers a question by first finding relevant source material and then
using a language model to write an answer grounded in that material. Recall turns that workflow into
a rewarded service on Bittensor. According to the subnet's on-chain description, miners serve
embedding models, vector search, and large language model inference, while validators independently
evaluate retrieval accuracy and answer quality.

The result is meant to behave like an always-improving search engine that returns answers with
citations rather than a static list of links. Because miners compete openly and the network routes
queries to the top performers, the quality of the pipeline is determined by ongoing evaluation
rather than fixed in advance.

## Miner and Validator Roles

The two roles map onto the standard Bittensor split between supplying a capability and scoring it.
Miners provide the retrieval-and-generation capability: turning content into embeddings, searching a
vector index for relevant passages, and producing an answer with a language model. Validators
provide the evaluation: they independently check how accurate the retrieval was and how good the
resulting answer is.

Validator assessments become weights, and those weights feed into [[Yuma Consensus]], which is how
the network converts many independent evaluations into the emissions that reward the strongest
miners. In plain terms, miners do the work of answering, and validators decide whose answers are
good enough to be rewarded and surfaced to users.

## On-Chain Identity

Live SN31 subnet data is available on [TaoStats](https://taostats.io/subnets/31), which identifies
the subnet as Recall. Readers can reproduce the subnet identity fields directly with the Bittensor
CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 31 --network finney`.

At the time of writing, the Finney identity for netuid 31 publishes the subnet name and a
description of the RAG mechanism but does not list a GitHub repository or project URL, so this
article describes the subnet from its on-chain identity rather than from a specific codebase. That
chain identity is the source of truth for what Subnet 31 is.
