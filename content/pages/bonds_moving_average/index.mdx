# Bonds Moving Average

Bonds moving average is a per-subnet hyperparameter, listed in the documentation as
`BondsMovingAverage`, that sets the moving-average behavior applied to
[validator-miner bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds) in
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus). The hyperparameter reference
describes it as the moving average of bonds and points readers to the bonding mechanics section of
the Yuma Consensus documentation
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

## What It Controls

The hyperparameter configures bond smoothing inside Yuma Consensus, not the subnet task or the raw
[validator weights](https://docs.learnbittensor.org/resources/glossary#weight-vector) validators
submit. Validator weights are evaluation inputs; bonds are smoothed validator-miner relationships
that evolve across consensus rounds and feed into
[validator dividends](https://docs.learnbittensor.org/resources/glossary#emission)
([Yuma Consensus: Bonding mechanics](https://docs.learnbittensor.org/learn/yuma-consensus#bonding)).

Yuma Consensus documentation describes each bond update as combining a new instant bond contribution
with the prior bond value through an exponential moving average. Bonds moving average is the
subnet-level hyperparameter that governs that smoothing behavior for bond state on the selected
subnet ([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

The reference also states that higher bonds yield to higher dividends for validators. That ties the
parameter to validator-side emission outcomes rather than to miner incentive ranking alone
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

## Documented Type, Default, and Setter

The [Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)
reference lists `BondsMovingAverage` as a `u64`. It is changed through the
`sudo_set_bonds_moving_average` extrinsic, exposed in tooling as
`btcli sudo set --param bonds_moving_avg`, and the reference marks the permission required to set it
as subnet owner
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

Because each subnet carries its own hyperparameters, the bond-smoothing setting is defined per
subnet and applies only within that subnet's Yuma Consensus path.

## Distinction from Validator-Miner Bonds

[Validator-miner bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)
name the smoothed relationship values that connect validator evaluation history to validator-side
consensus outcomes. Bonds moving average names the hyperparameter that configures how those bond
values are averaged over time
([Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Validator-miner bonds** — the smoothed relationship state used in consensus.
- **Bonds moving average** — the subnet hyperparameter that sets the moving-average smoothing for
  those bonds.

## Distinction from Liquid Alpha

[Consensus-based weights](https://docs.learnbittensor.org/concepts/consensus-based-weights), also
called liquid alpha, can vary the EMA smoothing factor for validator-miner bonds according to
consensus alignment. Bonds moving average is the documented bond-moving-average hyperparameter,
while liquid alpha is a separate modifier described in the consensus-based weights documentation
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

Readers should keep the hyperparameter name separate from subnet alpha token vocabulary. Bonds
moving average concerns bond-smoothing configuration; alpha-token language belongs to subnet
tokenomics.

- **Bonds moving average** — the subnet hyperparameter configuring bond smoothing.
- **Liquid alpha** — varies the EMA coefficient by consensus alignment.

## Distinction from Bonds Penalty

Bonds moving average and bonds penalty are different bond-related Yuma Consensus hyperparameters.
Bonds moving average sets the moving-average behavior applied to validator-miner bonds, while bonds
penalty sets the magnitude of the penalty applied when validator weights exceed the subnet consensus
benchmark ([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Bonds moving average** — the smoothing applied to validator-miner bonds over time.
- **Bonds penalty** — the penalty magnitude for out-of-consensus weights.

## Distinction from Alpha Sigmoid Steepness

Bonds moving average and alpha sigmoid steepness both touch the bond-smoothing path but set
different things. Bonds moving average sets the moving-average behavior applied to validator-miner
bonds across epochs, while alpha sigmoid steepness shapes how sharply the liquid alpha coefficient
responds to a validator's consensus alignment
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Bonds moving average** — the smoothing applied to bonds over time.
- **Alpha sigmoid steepness** — the sharpness of the liquid-alpha response curve.

## Distinction from EMA

EMA is the exponential moving-average technique that blends each bond update with prior bond state.
Bonds moving average is the per-subnet `BondsMovingAverage` hyperparameter that configures that
smoothing behavior on a selected netuid ([EMA](https://docs.learnbittensor.org/learn/ema),
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **EMA** — the smoothing technique applied to validator-miner bonds.
- **Bonds moving average** — the hyperparameter that configures that smoothing per subnet.

## BTCLI Reads the Live Value

Any user can read subnet hyperparameters for a selected netuid with
`btcli subnet hyperparameters --netuid 1`. The command output includes the live `BondsMovingAverage`
value for that subnet at query time
([Subnet Hyperparameters: View hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters#view-hyperparameters)).

Live values are network- and subnet-specific and can differ between subnets, so a bond-smoothing
claim should be checked against the hyperparameter table for the relevant netuid and network rather
than assumed from another subnet's configuration.

## Chain Reads for netuid 1

Readers can verify live hyperparameter values for the documented example netuid with
`btcli subnet hyperparameters --netuid 1 --network finney`
([Subnet Hyperparameters: View hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters#view-hyperparameters)).

That read path keeps live hyperparameter claims tied to a parseable netuid.

## Per-Subnet Live Value Boundary

Bonds moving average is per-subnet chain state. The bond-smoothing coefficient configured on one
netuid applies only inside that subnet's Yuma Consensus bonding path and can differ from reference
defaults and from the value on another netuid
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Yuma Consensus: Bonding mechanics](https://docs.learnbittensor.org/learn/yuma-consensus#bonding)).

This article's infobox uses netuid 1 as an example label when discussing bond smoothing on a single
subnet. That example helps readers separate mechanism vocabulary from live configuration on a chosen
netuid ([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

Bonds moving average configures how instant bond contributions blend into smoothed bonds; it does
not replace Yuma Consensus settlement itself
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Bonds moving average** — bond-smoothing hyperparameter inside Yuma bonding on a subnet.
- **Per-subnet boundary** — live smoothing coefficient is chain state on one netuid.

## Distinction from Yuma Consensus

Bonds moving average is a per-subnet hyperparameter that configures smoothing applied to
validator-miner bonds. Yuma Consensus runs at tempo boundaries on the selected netuid, using those
bonds when aggregating submitted weights and allocating miner incentives and validator dividends
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

- **Bonds moving average** — bond-smoothing hyperparameter in Yuma Consensus.
- **Yuma Consensus** — subnet consensus pass at tempo boundaries. Readers should keep hyperparameter
  smoothing separate from the full settlement pass. Bonds moving average sets how quickly
  validator-miner bonds update; Yuma aggregates submitted weights, applies clipping and bonding, and
  allocates miner incentives and validator dividends at epoch boundaries
  ([Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds),
  [Yuma Consensus: Validator emissions](https://docs.learnbittensor.org/learn/yuma-consensus#validator-emissions)).

The hyperparameter shapes bond persistence across epochs; Yuma reads those bonds when crediting
validator-side outcomes
([Glossary: Dividends](https://docs.learnbittensor.org/resources/glossary#dividends)).

- **Bonds penalty** — separate hyperparameter for out-of-consensus bond blending.

## Reader Boundary

Bonds moving average should not be read as a live bond value, a validator weight, or liquid alpha
itself. It names only the subnet hyperparameter that configures how validator-miner bonds are
smoothed across epochs on the selected netuid
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).
