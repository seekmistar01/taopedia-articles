BitSota is Bittensor Subnet 94. The live Finney identity currently leaves the subnet name, subnet
URL, Discord, description, logo, and additional fields as `pending...`, while keeping
[AlveusLabs/SN94-BitSota](https://github.com/AlveusLabs/SN94-BitSota) as the registered GitHub
repository. Public BitSota materials describe an open machine-learning research competition: instead
of a human hand-designing a model, miners use automated search to evolve algorithms that try to beat
the current best result on a shared evaluation task, and validators check those claims before any
reward is paid.

## What BitSota Rewards

The subnet is built around the idea of letting algorithms improve themselves. Miners run an
evolutionary search — automatically generating, testing, and refining candidate machine-learning
algorithms — and aim to discover one that performs better than the existing state of the art on a
fixed evaluation benchmark. The reward goes to genuine improvements: a submission earns when it can
beat the current best score on that task, which keeps the network pushing toward stronger algorithms
over time rather than rewarding repeated work.

Because the search is open-ended, the platform is described as problem-agnostic: the same
competitive structure can be pointed at different research problems rather than a single fixed
model. Participants can take part with different levels of hardware, either evolving algorithms on
their own or pooling their compute with others who share the work.

## Miner and Validator Roles

Miners run the algorithm-evolution process and submit candidates that beat the current benchmark.
The heavy lifting is the search itself — repeatedly mutating and evaluating algorithms locally until
one clears the bar — and a registered hotkey on netuid 94 is what ties a miner's submissions back to
them.

Validators are the independent check on those claims. When a miner submits an algorithm, validators
download it and re-evaluate it themselves rather than trusting the reported score, confirming that
it really does perform as claimed. Based on that verification they set weights on the network, which
is how reward flows through [[Yuma Consensus]]. At the article level the split is straightforward:
miners search for better algorithms, while validators reproduce and weight the ones that hold up.
