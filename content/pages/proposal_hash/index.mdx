# Proposal Hash

A proposal hash is the compact identifier used to refer to a proposal in Bittensor governance. The
[Proposal Hash glossary entry](https://docs.learnbittensor.org/resources/glossary#proposal-hash)
anchors the term in Senate voting, where reviewers need a stable way to identify the proposal under
consideration.

The hash is useful because governance discussions often need to point at the same proposal across
review, voting, and follow-up references. It gives the proposal a concise handle while the proposal
itself carries the substantive change.

## Governance Use

In Bittensor governance, a [proposal](https://docs.learnbittensor.org/resources/glossary#proposal)
is the item put forward for review, while the proposal hash is the identifier used to refer to that
item. This keeps the reference compact without replacing the proposal's full description
([Glossary: Proposal Hash](https://docs.learnbittensor.org/resources/glossary#proposal-hash),
[Glossary: Proposal](https://docs.learnbittensor.org/resources/glossary#proposal)).

That distinction is the main reason the term appears in Taopedia. A reader may see a proposal hash
in governance material and need to understand that it points to a proposal rather than summarizing
the proposal.

## Senate Review Context

Bittensor governance separates proposal creation from Senate approval
([Governance Overview](https://docs.learnbittensor.org/governance)). Proposal review and voting sit
inside Senate context, so proposal hashes belong naturally beside Senate review vocabulary
([Senate](https://docs.learnbittensor.org/governance/senate)).

In that setting, the hash helps preserve continuity. The same proposal can be discussed before,
during, and after Senate review, and the hash is the reference that keeps those mentions tied to the
same governance item.

## Reading Proposal Hashes

A proposal hash should be read together with the proposal and governance context around it. By
itself, the hash tells the reader which proposal is being referenced; the governance source explains
what the proposal concerns and where it sits in review
([Governance Overview](https://docs.learnbittensor.org/governance),
[Senate](https://docs.learnbittensor.org/governance/senate)).

That makes the term useful in citations and governance summaries, where a short reference needs to
lead readers back to the longer proposal record.

For Taopedia readers, the stable concept is simple: proposal hash is identifier vocabulary for
Bittensor governance. It helps name a proposal during Senate review and voting without trying to
describe the proposal's content or outcome on its own.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For Proposal Hash,
this sequence gives readers a boundary for interpreting hash examples and governance-reference
notes.

Localnet examples are isolated and reflect local chain state, so they are useful for controlled
experiments rather than evidence of live Bittensor behavior. Testnet examples add shared
non-production conditions, which can reveal integration behavior without touching mainnet state.

On mainnet, Proposal Hash examples should be read as live production proposal-reference behavior on
the production Bittensor network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet, so outcomes from one environment should not be treated as
proof of behavior in another.
