Actual is Bittensor Subnet 95.

## What Actual Provides

The on-chain description registers Subnet 95 as a "Heterogeneous inference network by Actual
Computer Inc." The registered project website is [actual.inc](https://actual.inc). Public technical
documentation for the specific subnet mechanism is not available through the on-chain identity
alone, as the registered GitHub repository is not publicly accessible at this time.

## Miner and Validator Roles

Subnet 95 operates under the standard Bittensor two-role structure. Miners supply a capability to
the network, and validators evaluate those contributions and set weights. Those weights feed into
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus), whose consensus weights then
help determine per-tempo [emissions](https://docs.learnbittensor.org/learn/emissions) for the
strongest performers.

## On-Chain Identity

The live Finney identity for netuid 95 registers the subnet name as Actual and the description as
"Heterogeneous inference network by Actual Computer Inc." The registered project site is
[actual.inc](https://actual.inc). The registered GitHub field points to
github.com/actual-computer/actual-subnet-95, though that repository is not publicly accessible. The
Discord field is blank. Live subnet data is available on [TaoStats](https://taostats.io/subnets/95).

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For Actual (SN95),
that sequence applies to the standard Bittensor lifecycle: localnet for isolated development,
testnet for shared non-production testing, and mainnet for live operation with real emissions.

On mainnet, Actual (SN95) is registered as the live production subnet at netuid 95. The
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. Participation examples or emission outcomes from one
environment should not be read as representing production subnet performance in another environment.

## Netuid 95 Identifies the Subnet On-Chain

Bittensor assigns every subnet a unique numeric identifier called a netuid, and Subnet 95 is the
subnet registered at netuid 95
([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)). The
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) reference
explains that each subnet runs its own incentive mechanism while sharing the same underlying
Subtensor chain, so the netuid is the stable handle that distinguishes Actual from every other
subnet.

For a reader, this means "Subnet 95" and "netuid 95" refer to the same on-chain slot. A claim about
Actual should be tied to that netuid rather than to the registered name alone, because the name
field can be changed on-chain while the netuid stays fixed.

## Distinction from Yuma Consensus

Subnet 95 (Actual) is the netuid registered as a heterogeneous inference network operated by Actual
Computer Inc. It names the inference-service subnet context, not the recurring settlement step that
converts validator weight submissions into emission shares at the epoch boundary
([actual.inc](https://actual.inc),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Yuma Consensus is the on-chain mechanism that runs at the epoch boundary on each subnet. It reads
the weight matrix from eligible validators, applies clipping and bonding, and converts the result
into miner incentives and validator dividend shares
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Earlier sections separate Actual on-chain identity, development stage boundaries, and netuid
registration from generic subnet slots. Validators score inference contributions into weights; Yuma
settles the consensus outcome from eligible submissions on netuid 95—not the heterogeneous inference
payload itself
([Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix),
[Weight Copying](https://docs.learnbittensor.org/learn/yuma-consensus#weight-copying)).

Readers should treat Subnet 95 as Actual's heterogeneous inference subnet, and Yuma as the
epoch-boundary mechanism that allocates rewards from consensus while validators remain active.

- **Subnet 95 (Actual)** — heterogeneous inference network subnet at netuid 95.
- **Yuma Consensus** — epoch-boundary settlement from eligible validator weights to emission shares.
