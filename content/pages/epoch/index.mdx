# Epoch

An epoch is a consensus period inside a Bittensor subnet. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#epoch) defines an epoch as the period
during which a subnet executes its consensus mechanism, with the period determined by the subnet's
tempo. [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) provides the broader
consensus context for how validator evaluation signals become subnet outcomes.

## Consensus Period

The useful way to read epoch is as a period of subnet consensus work. It is not the subnet task, the
miner output, or the validator's individual evaluation; it is the period in which the subnet's
consensus mechanism runs over the relevant signals. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#epoch) ties the term directly to
subnet consensus, and [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)
explains the consensus process that interprets validator signals.

## Relationship to Tempo

Epoch and tempo are related but not identical. Tempo describes the cadence that determines when
epochs run, while epoch names the consensus period itself. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#tempo) describes tempo as the
subnet-specific cadence for epochs, and the
[Glossary](https://docs.learnbittensor.org/resources/glossary#epoch) defines epoch as the period
that cadence produces.

## Relationship to Validator Signals

Epochs matter because Bittensor consensus uses validator evaluation signals rather than a single
validator's view. [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) describes
validator weights as inputs to subnet consensus, and the
[Glossary](https://docs.learnbittensor.org/resources/glossary#epoch) places epoch at the period
where the subnet executes that consensus mechanism.

## Subnet Context

An epoch is scoped to a subnet's consensus process. Different subnets can define different work and
standards, so the epoch is best understood as part of the subnet's own mechanism rather than a
standalone quality score.
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)
places task, protocol, and evaluation method inside subnet design, while
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) explains how evaluation
signals are resolved by consensus.

## Reader Boundary

Epoch should be read as a concept about consensus periods, not as a claim about exact duration or a
procedure for any particular subnet. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#epoch) provides the stable concept,
and the [Glossary](https://docs.learnbittensor.org/resources/glossary#tempo) provides the related
cadence term without requiring exact timing detail in this article.
