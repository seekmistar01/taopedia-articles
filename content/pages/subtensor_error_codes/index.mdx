# Subtensor Error Codes

Subtensor error codes are diagnostic signals from the blockchain layer beneath Bittensor. They help
readers understand why a network action failed without treating the error text as a complete guide
to the underlying rule. The [Subtensor Error Codes](https://docs.learnbittensor.org/errors) page
frames the topic as errors that can appear through Bittensor-facing interfaces, while the
[Subtensor Standard Errors](https://docs.learnbittensor.org/errors/subtensor) page describes
standard error categories from the Subtensor layer.

## What error codes describe

An error code usually points to the rule family that rejected an action. The official
[Subtensor Error Codes](https://docs.learnbittensor.org/errors) overview groups these failures into
standard Subtensor errors, custom errors, and errors inherited from the underlying Substrate
framework. That grouping matters because a short error message may identify the category of a
failure, but the surrounding action and source-specific context still determine what it means.

## Standard errors

Standard Subtensor errors are named conditions defined by the Bittensor chain layer. They cover
families such as subnet existence, balance and stake conditions, weight-setting constraints,
network-entry boundaries, and other protocol checks. The
[Subtensor Standard Errors](https://docs.learnbittensor.org/errors/subtensor) page presents this as
a high-level reference and points readers toward the fuller Subtensor error reference for complete
coverage.

## Custom errors

Custom errors are numeric conditions returned by Subtensor when a more specific rule fails. They can
describe staking limits, rate limits, liquidity limits, shielded-message parsing, and other
chain-layer checks. The [Subtensor Custom Errors](https://docs.learnbittensor.org/errors/custom)
page lists these numeric entries and their short descriptions. For readers, the important idea is
that the number is only a lookup key; it still needs the surrounding Bittensor action to be
interpreted correctly.

## Why source context matters

The same broad error family can appear around different Bittensor activities, including staking,
subnet participation, wallet use, and validator behavior. That is why error-code pages should be
read as diagnostic context, not as standalone operating instructions. The
[Subtensor Error Codes](https://docs.learnbittensor.org/errors) overview connects errors to several
Bittensor-facing interfaces, and the
[Subtensor Custom Errors](https://docs.learnbittensor.org/errors/custom) page shows how short codes
map back to named failure conditions.

## Relation to nearby concepts

Error codes sit close to [[staking_and_delegation|Staking and Delegation]],
[[dynamic_tao|Dynamic TAO]], and [[wallets|Wallets]]. Staking and Delegation explains one common
area where chain-layer rules can reject an action. Dynamic TAO explains the subnet-token and pool
context behind some staking-related outcomes. Wallets provide the user-facing key and balance
context around many network interactions. The
[Subtensor Standard Errors](https://docs.learnbittensor.org/errors/subtensor) and
[Subtensor Custom Errors](https://docs.learnbittensor.org/errors/custom) pages define the error
families themselves.
