oneoneone is Bittensor Subnet 111, a data subnet focused on user-generated content. The subnet turns
the work of gathering authentic online content — reviews, posts, comments, and similar public user
contributions — into a rewarded service and makes the result available for developers and
researchers to use. The codebase for the subnet is the
[oneoneone-io/subnet-111](https://github.com/oneoneone-io/subnet-111) repository. The live Finney
identity lists the subnet name as oneoneone and the GitHub repository, but does not currently list a
description, URL, or Discord field.

## What oneoneone Rewards

The subnet's product is fresh, authentic user-generated content collected from public platforms
around the web, such as map and business reviews. Rather than a single static dataset, the network
continuously gathers new content in response to requests, and rewards miners for returning data that
is genuine, complete, and recent.

To keep quality high, the collected content is enriched with automated analysis. The network applies
checks that help separate authentic contributions from spam or bot-generated text, and adds signals
such as the sentiment, intent, and language of each item. This enrichment is what makes the data
more useful than a raw scrape, and it gives validators a basis for judging whether a miner's
submission is trustworthy.

## Miner and Validator Roles

Miners collect, clean, and format user-generated content from public sources and return it in a
structured form. When the network asks for data on a particular topic or place, miners respond
within a time limit with results that are meant to be authentic and well formed. A registered hotkey
on netuid 111 is what ties a miner's submissions back to them.

Validators define the work and judge the results. They issue collection challenges — some synthetic,
generated to test miners, and some driven by real requests — and then assess each response for
authenticity, completeness, and how quickly and recently the data was gathered. From those
assessments they set weights on the network, which is how reward flows through [[Yuma Consensus]].
At the article level the split is straightforward: miners supply the content, while validators
measure its quality and weight each miner accordingly.

## On-Chain Identity

Live SN111 subnet data is available on [TaoStats](https://taostats.io/subnets/111), which identifies
the subnet as oneoneone. Readers can reproduce the subnet identity fields directly with the
Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 111 --network finney`. That Finney identity is the source of
truth for the subnet name and the GitHub repository associated with the subnet.
