# TAO Weight

TAO weight describes how TAO stake counts relative to alpha stake when Bittensor derives stake
weight inside a subnet
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

The term is part of stake-weight vocabulary. It is not a separate token, staking role, or validator
score by itself.

## Stake-Weight Role

Stake weight is the computed value used for validator consensus power and emissions. TAO weight
explains the relative role of the TAO-stake component inside that value
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)).

This keeps the concept narrow. TAO weight helps interpret stake weight before stake-derived
influence is read inside subnet consensus.

Stake weight combines stake components into validator influence. TAO weight only describes the
relative weighting of the TAO component in that calculation.

## TAO Stake and Alpha Stake

The term exists because validator influence in a subnet can involve both TAO stake and alpha stake.
TAO weight describes how the TAO side is weighted relative to the alpha side
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

That boundary matters because TAO stake and alpha stake are related inputs, not interchangeable
ones. TAO weight is the relative-weighting term between them.

Alpha stake belongs to the subnet-specific token side of Dynamic TAO, while TAO stake belongs to the
network asset side. TAO weight describes how the TAO side contributes when those inputs are read for
stake weight.

## Subnet Consensus Context

TAO weight matters because validator influence is used inside subnet consensus. Validators evaluate
miner work according to subnet rules, and Yuma Consensus aggregates weighted signals into subnet
outcomes ([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Within that setting, TAO weight helps explain how TAO stake contributes to a validator's influence.
It is not a standalone description of what miners do or how validators judge work quality.

The consensus context also keeps TAO weight separate from a general asset-amount statement. The term
is about stake-derived influence inside subnet consensus.

## Relative-Stake-Weight Boundary

TAO weight also differs from relative stake weight. TAO weight helps form the stake-weight value,
while relative stake weight describes validator influence as a proportional measure inside a
subnet's validator set
([Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight),
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)).

This boundary keeps component weighting separate from proportional influence. TAO weight belongs to
stake-weight interpretation; relative stake weight belongs to subnet-local comparison.

Relative stake weight is downstream of the component relationship. It is easier to interpret after
stake weight has already been formed for the subnet context.

## Validator-Influence Boundary

TAO weight should also stay separate from validator weights. TAO weight helps determine stake-based
influence, while validator weights express the validator's miner evaluations
([Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)).

This keeps influence and evaluation vocabulary separate. TAO weight concerns how stake contributes
to influence; validator weights concern what the validator evaluates.

Rank, trust, and dividends are later result terms. TAO weight sits earlier, as part of the
stake-weight context that shapes how validator signals are interpreted
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Glossary: Dividends](https://docs.learnbittensor.org/resources/glossary#emission)).

## Emission Weight Scope

TAO weight belongs to emission and weighting context, not to ordinary balance vocabulary.
[Emissions](https://docs.learnbittensor.org/learn/emissions) describe reward distribution, while
staking references explain how TAO stake participates in validator support
([Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

Use TAO weight when the focus is the relative role of TAO stake in stake weight. Use stake weight
when the focus is the combined validator influence value, and use root proportion when the focus is
that separate dividend-context measure.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For TAO weight on
a subnet such as netuid 1, that sequence changes how readers should interpret stake-weight
components, root-proportion inputs, and validator permit ranking examples.

In localnet, TAO-weight examples can be tested in an isolated environment. Local stake conditions
and root-stake proportions reflect local chain configuration rather than production subnet stake
context.

On testnet, TAO-weight observations can be checked in a shared, non-production network. Testnet
stake conditions and validator sets on a selected netuid are separate from mainnet subnet state
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)).

On mainnet, TAO weight describes live stake-weight inputs used in Yuma Consensus aggregation on the
production Bittensor network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A TAO-weight example from one environment should not be
read as representing production stake-weight conditions on another network.

## Stake Weight Combines Alpha and TAO Components

The [Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)
describes stake weight as a computed value that combines alpha stake, TAO stake, and TAO weight
inside a subnet. TAO weight names the relative role of the TAO side in that combination rather than
a separate influence score on its own
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)).

Readers comparing terms should keep component and total separate. TAO weight adjusts how TAO stake
counts; stake weight is the merged influence measure Yuma reads during aggregation.

## Yuma Rank Uses Stake-Weighted Post-Clip Weights

[Yuma Consensus: Miner emissions](https://docs.learnbittensor.org/learn/yuma-consensus#miner-emissions)
calculates each miner's aggregate rank as the stake-weighted sum of post-clip validator weights.
Stake-derived influence therefore scales how much each validator's evaluation can move the merged
miner result inside a subnet
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank)).

That formula connects TAO-weight context to outcomes. Component weighting helps form stake weight;
stake weight then scales how submitted evaluations count after clipping.

## Validator Permits Follow Stake-Weight Ranking

The
[Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit)
describes permits as awarded to the top neurons by stake weight within a subnet. TAO weight is
therefore part of the stake context that can affect not only aggregation influence but also which
neurons hold validation rights
([Understanding Subnets: Validator stake weight](https://docs.learnbittensor.org/subnets/understanding-subnets#validator-stake-weight)).

Permit vocabulary answers whether weight submission rights exist. Stake-weight ranking, shaped in
part by TAO weight, helps determine who can receive those rights.

## Clipping Scales Influence Through Post-Filter Weights

After stake weight is formed, Yuma Consensus clips outlier validator weights against a
stake-weighted [consensus score](https://docs.learnbittensor.org/resources/glossary#consensus-score)
benchmark before computing miner [rank](https://docs.learnbittensor.org/resources/glossary#rank)
([Yuma Consensus: Clipping](https://docs.learnbittensor.org/learn/yuma-consensus#clipping)).

The hyperparameter [kappa](https://docs.learnbittensor.org/subnets/subnet-hyperparameters#kappa)
sets how much validator stake must support a weight level before that level becomes the clipping
ceiling. TAO weight helps form the stake context that kappa reads, but kappa and clipping determine
how much of a validator's submitted signal survives into the post-clip aggregation pass.

## Relative Stake Weight Sits Downstream of Component Weighting

[Relative stake weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)
expresses one validator's stake weight as a proportion of the validator set on the same subnet.
Because stake weight already combines alpha stake, TAO stake, and TAO weight, relative stake weight
is a downstream comparison term rather than a second TAO-side parameter
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

Readers comparing TAO weight to relative stake weight should keep component weighting and
proportional influence separate. TAO weight adjusts how TAO stake counts; relative stake weight
compares the merged influence value against other validators.

## TAO Weight Is a Protocol Parameter, Not a Per-Validator Setting

A common misreading is that TAO weight is something an individual validator chooses. Documentation
describes it as a protocol-level parameter that governs how TAO stake counts relative to alpha stake
when stake weight is formed, so it applies uniformly rather than being tuned per validator
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

For readers, this is why TAO weight shapes the playing field rather than a single position. It sets
the relative pull of root-routed TAO versus subnet alpha for everyone forming stake weight, which is
different from a validator's own stake amount
([Emission](https://docs.learnbittensor.org/learn/emissions)).

That distinction matters when comparing it to root proportion. TAO weight is the parameter that
governs the counting; root proportion is the observed result of that counting in one subnet's
dividend context, so the two should not be read as the same lever
([Glossary: Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion)).

## Distinction from Stake Weight

TAO weight and stake weight sit at different layers of validator influence vocabulary. TAO weight
names the relative role of TAO stake when stake weight is formed, while stake weight is the computed
influence value that combines alpha stake, TAO stake, and TAO weight inside a subnet
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

- **TAO weight** — the protocol parameter that weights the TAO-stake component.
- **Stake weight** — the merged influence measure used in subnet consensus context.

## Distinction from Validator Weights

TAO weight and validator weights answer different questions. TAO weight helps determine
stake-derived influence, while validator weights express a validator's miner evaluations inside Yuma
Consensus ([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **TAO weight** — how TAO stake counts when forming stake weight.
- **Validator weights** — the validator's submitted miner evaluations.

## Distinction from Root Proportion

TAO weight and root proportion both involve TAO stake, but they answer different questions. TAO
weight is the relative-influence parameter for TAO stake when stake weight is formed, while root
proportion is a subnet-level dividend-context measure of how much of a subnet's dividend split is
attributed to TAO staked through Root
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Glossary: Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion)).

TAO weight is the parameter that governs the counting; root proportion is the observed result in one
subnet's dividend context. The two should not be read as the same lever.

- **TAO weight** — protocol parameter weighting the TAO-stake component in stake weight.
- **Root proportion** — subnet-level split reading for root-routed TAO dividends.

## Distinction from Relative Stake Weight

TAO weight is the protocol parameter that governs how TAO stake counts when stake weight is formed.
Relative stake weight expresses one validator's stake weight as a proportion of the validator set on
the same subnet
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Glossary: Relative Stake Weight](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

- **TAO weight** — parameter weighting the TAO-stake component in stake weight.
- **Relative stake weight** — proportional share of merged stake weight within the validator set.

Component weighting and downstream comparison answer different influence questions
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Distinction from Root Claims

Root claims and TAO weight both relate to Root Subnet staking but act on different things. Root
claims handle the alpha dividends earned through Root Subnet staking, deciding whether they stay as
alpha or convert to TAO and restake on Root, while TAO weight is the global multiplier that sets how
much root (TAO) stake counts relative to a subnet's own alpha stake when stake weight is computed
([Root Claims](https://docs.learnbittensor.org/staking-and-delegation/root-claims),
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)).

- **Root claims** — the handling rule for alpha dividends earned through Root staking.
- **TAO weight** — the multiplier that scales how much root stake counts against alpha stake.

## Distinction from Yuma Consensus

TAO weight names how TAO stake counts relative to alpha stake when Bittensor derives stake weight
inside a subnet. Yuma Consensus is the separate mechanism that uses those stake-weighted validator
signals when converting weights into emission shares each tempo
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

- **TAO weight** — relative role of TAO stake in stake-weight calculation.
- **Yuma Consensus** — mechanism that turns validator weights into emission shares.

## Reader Boundary

TAO weight names the relative role of TAO stake inside stake-weight computation, not validator
weights, rank, trust, dividends, or a standalone influence score
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

TAO weight is also a single global network parameter, not a per-subnet setting or a live validator
score: the same documented figure (currently 0.18) applies wherever stake weight is calculated,
rather than varying subnet by subnet
([Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)).
