Perturb is Bittensor Subnet 26, an adversarial-robustness subnet for image-model evaluation. Its
public site is [perturbai.io](https://www.perturbai.io), and the GitHub repository listed in the
subnet's Finney identity is [0xsigurd/Perturb](https://github.com/0xsigurd/Perturb).

## What Perturb Provides

The Perturb README describes a subnet where validators create adversarial-image challenges and
miners return perturbed images under bounded distortion constraints. The task is not simply to
change an image at random; miner outputs must remain close to the original image while changing how
the target classifier responds.

In Bittensor terms, Perturb turns adversarial robustness testing into a competitive subnet task.
Miners search for useful perturbations, validators verify and score the responses, and validator
weights feed into [[Yuma Consensus]].

## Miner and Validator Roles

Miners respond to image challenges with perturbed images. Strong miner work produces a response that
changes the classifier result while staying inside the subnet's distortion and image-quality
constraints.

Validators run the challenge and scoring side of the subnet. The README describes validators as
creating image challenges, checking miner responses, computing rewards, maintaining scoring history,
and setting on-chain weights. Operators should treat the repository as the source of truth for
current setup details.

## On-Chain Identity

Live SN26 data is available on [TaoStats](https://taostats.io/subnets/26). The live Finney identity
for netuid 26 registers the subnet name as Perturb, with the description "Decentralized adversarial
robustness network." The GitHub repository is
[0xsigurd/Perturb](https://github.com/0xsigurd/Perturb).

## Relationship to Multiple Mechanisms

Perturb validators create adversarial-image challenges and score miner responses off chain before
submitting weights. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model) and
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets)
docs note that each incentive mechanism has its own subnet scoring model.

For readers, challenge design and distortion scoring belong to this subnet's evaluation pipeline. A
parallel mechanism on the same netuid would need its own scoring model rather than reusing Perturb's
adversarial challenge rules by default.
