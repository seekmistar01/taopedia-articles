# Proposal Hash

A proposal hash is the compact identifier used to refer to a proposal in Bittensor governance. It
gives Senate review and voting a stable reference for the proposal under consideration
([Glossary: Proposal Hash](https://docs.learnbittensor.org/resources/glossary#proposal-hash),
[Governance Overview](https://docs.learnbittensor.org/governance)).

The hash is useful because governance discussions often need to point at the same item across
review, voting, and later references. It gives the proposal a concise handle while the proposal
itself carries the substantive request.

## Identifier Role

In Bittensor governance, a proposal is the item put forward for review, while the hash is the
identifier used to refer to that item
([Glossary: Proposal](https://docs.learnbittensor.org/resources/glossary#proposal),
[Glossary: Proposal Hash](https://docs.learnbittensor.org/resources/glossary#proposal-hash)).

This makes the hash reference vocabulary. The proposal explains the requested change; the hash gives
governance readers a compact way to keep referring to the same proposal.

## Senate Review Context

Bittensor governance separates proposal creation from Senate approval. Proposal review and voting
sit inside Senate context, so proposal hashes belong naturally beside Senate review vocabulary
([Governance Overview](https://docs.learnbittensor.org/governance),
[Senate](https://docs.learnbittensor.org/governance/senate)).

In that setting, the hash helps preserve continuity. The same proposal can be discussed before,
during, and after Senate review, and the hash is the reference that keeps those mentions tied to the
same governance item.

That continuity is the practical value of the term. It lets governance prose refer back to the same
item without restating the entire proposal each time.

This is especially helpful when a source moves between proposal description and Senate action. The
hash acts as the reference anchor, and the surrounding governance source supplies the reason the
proposal is being reviewed.

## Voting and Status Context

Senate voting uses governance context around the proposal being reviewed. A proposal hash identifies
the proposal being referenced, while the surrounding Senate record explains the vote, review state,
and later outcome ([Senate](https://docs.learnbittensor.org/governance/senate),
[Governance Overview](https://docs.learnbittensor.org/governance)).

That makes the hash useful in citations and governance summaries. It leads readers back to the
proposal record, but status and implementation details still come from the governance source that
uses the hash.

When a hash appears with little surrounding explanation, the narrow reading is usually best: it
identifies which proposal is in view, and the neighboring governance material explains why that
proposal matters.

## Network Scope

Bittensor documentation separates localnet, testnet, and mainnet environments
([Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)). Proposal-hash
examples need the environment attached because local, testnet, and mainnet governance references
have different scope.

The identifier concept can be the same across environments, while the governance status attached to
that identifier depends on the environment and record being cited. A local example can illustrate
mechanics, a testnet example can show shared non-production behavior, and a mainnet hash belongs to
mainnet governance context.

## Reading the Term

Proposal hash should be read as proposal-identifier vocabulary. It is the compact governance
reference used during proposal review and voting, while the proposal and governance record carry the
requested change, review status, and outcome
([Glossary: Proposal Hash](https://docs.learnbittensor.org/resources/glossary#proposal-hash),
[Glossary: Proposal](https://docs.learnbittensor.org/resources/glossary#proposal),
[Senate](https://docs.learnbittensor.org/governance/senate)).
