# Alpha Tokens

Alpha tokens are subnet-specific assets in Bittensor. They are separate from TAO and give each
subnet its own tokenomics context for emissions, staking, and liquidity pools
([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Subnet Asset

An alpha token represents value inside a specific subnet rather than across the whole Bittensor
network. TAO remains the network asset, while alpha tokens belong to subnet-level tokenomics
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

This keeps TAO and alpha-token language separate. A TAO supply or emission statement is not
automatically a statement about a subnet's alpha-token context.

A reader should also ask which subnet's alpha token is meant. Two subnets can both have alpha-token
context without sharing the same pool, reserve, staking, or emission setting. The subnet label is
therefore part of the asset context, not a minor detail.

## Emissions and Pools

Bittensor emission uses both TAO and subnet-specific alpha tokens. The
[Emission](https://docs.learnbittensor.org/learn/emissions) documentation describes TAO entering
subnet contexts and alpha tokens appearing in subnet-level reward and reserve flows.

Subnets use paired TAO and alpha reserve contexts. The
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) documentation
describes each subnet as an automated market maker with TAO and subnet-specific alpha reserves. The
[Emission](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection) documentation
describes alpha reserve injection on that alpha side.

That means alpha tokens are not only reward vocabulary. They are also part of the subnet pool design
that supports Dynamic TAO tokenomics. TAO is network-scoped; alpha tokens are subnet-scoped assets
used inside each subnet's emission and pool context.

Alpha reserve is a related but narrower idea: it points to the pool-held side of a subnet's alpha
context, while alpha tokens name the subnet-specific asset more broadly
([Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)).

## Staking and Delegation

Outside the root subnet, stake can be represented in a subnet's alpha units. The
[Delegation](https://docs.learnbittensor.org/staking-and-delegation/delegation) documentation places
subnet alpha in staking and delegation context, while
[Slippage](https://docs.learnbittensor.org/learn/slippage) explains why pool conditions matter when
TAO and alpha interact.

This keeps alpha-token staking language scoped. It describes subnet-denominated stake context, not
TAO itself. When TAO and alpha interact through a pool, the pool context matters for how the
relationship is read.

## Halving and Asset Scope

Alpha tokens also have subnet-level halving context. The
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving) documentation distinguishes
network-level TAO halving from subnet-level alpha halving.

That distinction matters because the word "halving" can apply at different asset scopes. A TAO
halving statement and an alpha halving statement should name the asset and subnet context being
discussed.

This is another place where alpha-token language needs precision. Alpha halving belongs to the
subnet-specific alpha side, while TAO halving belongs to the network asset schedule
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

## Practical Limits

Alpha tokens are the right term when the focus is the subnet-specific asset. If the focus is
network-scoped TAO, TAO is more precise. If the focus is the pool-held side of the subnet asset,
[alpha reserve](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection) is more
precise. If the focus is cross-subnet emission share,
[subnet weights](https://docs.learnbittensor.org/resources/glossary#subnet-weights) is more precise.

Those distinctions keep alpha tokens focused on subnet asset vocabulary, not a staking walkthrough,
pool calculation, or full explanation of Dynamic TAO. When an example includes observed balances or
pool values, the network and subnet context should stay attached
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

The term also should not be used as a shortcut for every alpha-side concept. Alpha price, alpha
reserve, alpha outstanding, and alpha halving each name a more specific part of the alpha-token
system.
