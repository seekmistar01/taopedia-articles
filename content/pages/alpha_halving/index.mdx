# Alpha Halving

An alpha halving is a subnet-scoped emission-rate reduction for a subnet's alpha token. It applies
halving vocabulary to subnet alpha supply rather than to network-wide TAO supply
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

The concept belongs to Bittensor tokenomics because emissions include both TAO and subnet-specific
alpha-token flows ([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Subnet Scope

Each subnet has its own alpha token, so alpha halving is read in relation to one subnet's
alpha-token supply. It should not be treated as a network-wide event affecting every subnet in the
same way ([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

That scope is the main difference from TAO halving. TAO halving is network-token supply language;
alpha halving is subnet-token supply language.

## Emission Reduction

A halving reduces the rate at which new tokens are emitted after a supply threshold is reached. For
alpha halving, the reduced emission is the alpha-token emission for the relevant subnet
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

For readers, the useful point is narrow: alpha halving slows future alpha creation for that subnet.
It does not describe a transfer action, validator evaluation, or reward-quality score.

## Alpha Token Context

Alpha tokens are subnet-specific assets used inside subnet tokenomics. Alpha halving matters because
it changes the emission rate of that subnet-specific asset
([Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

This keeps the vocabulary separate: alpha token names the asset, while alpha halving names an
emission-rate reduction for that asset.

## TAO Halving Boundary

Alpha halving and TAO halving use the same broad supply-threshold idea, but they apply at different
scopes. One belongs to a subnet's alpha-token supply, and the other belongs to the network's TAO
supply ([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

Keeping that distinction avoids reading a subnet-level alpha event as a network-level TAO event. The
shared word is halving; the asset and scope are different.

## Emission and Incentive Boundary

Alpha halving changes an emission-rate input. It does not by itself explain miner evaluation,
validator rewards, or subnet incentive design
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Those later outcomes still depend on the subnet's incentive mechanism and consensus context. Alpha
halving names the supply-rate change for the subnet alpha token.

## Reader Boundary

Alpha halving should be read as concept-level subnet tokenomics vocabulary
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)). The stable point is that
alpha-token emission can be reduced at the subnet level while remaining distinct from broader TAO
halving.

This article does not provide date forecasts, reward forecasts, staking instructions, or
point-in-time emission reports. Exact timing and observed values belong to the source and subnet
context where they are measured.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For Alpha Halving,
that sequence changes how readers should interpret alpha-emission examples, subnet timing, and
tokenomics comparisons.

In localnet, Alpha Halving examples can be exercised in an isolated environment. Localnet
observations reflect local chain state and local configuration rather than production Bittensor
behavior.

On testnet, Alpha Halving behavior can be observed in a shared, non-production network. Testnet
examples are useful for checking interactions under shared chain conditions while staying separate
from mainnet state.

On mainnet, Alpha Halving describes live subnet alpha-token behavior on the production Bittensor
network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. An example or outcome from one environment should not be
read as evidence of production behavior in another environment.
