# Validator Permit

A validator permit is a Bittensor permission flag for validation rights inside a subnet. It answers
whether validation rights exist in that subnet context
([Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit),
[Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit)).

The term is scoped. It should not be read as a universal validator label across every subnet.

## Subnet Permission Flag

Validator permit is a one-subnet permission concept. It tells readers whether validation rights
exist for the relevant subnet, not whether the same validator has rights everywhere else
([Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit),
[Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit)).

This makes validator permit useful as a local rights term. The surrounding subnet context is part of
the meaning.

## VPermit Scope

Validator permit and VPermit describe nearby ideas at different scopes. A validator permit is the
one-subnet flag. VPermit is the broader view of where validation is authorized
([Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit),
[Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit)).

The distinction keeps the vocabulary readable. Validator permit answers a subnet-level permission
question, while VPermit summarizes validator authorization across subnet contexts.

## Stake-Weight Context

Validator permit is connected to stake-weight position. Stake weight is the computed stake value
used for validator consensus power and emissions inside a subnet
([Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

For readers, this means a validator permit is not a reputation score by itself. It is a validation
permission flag connected to stake-weight eligibility inside the subnet context.

## Stake Weight Boundary

Stake weight and validator permit describe different parts of the validation picture. Stake weight
is a computed value used for consensus influence, while validator permit records whether validation
rights exist for the subnet being discussed
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit)).

The two ideas are related but not interchangeable. Stake weight helps explain the eligibility and
influence setting; the permit term names the rights state.

## Reader Boundary

Use validator permit when the focus is permission to validate inside a subnet. Use stake weight when
the focus is stake-derived validator influence, and use VPermit when the focus is a broader
authorization view
([Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit)).

This page is a reference article for the permission flag. It is not a full validator guide, a
general ranking, or a description of every subnet role.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. The validator
permit concept applies across the Bittensor lifecycle: validation rights can be tested in localnet
for isolated subnet testing, verified on testnet for shared non-production conditions, and relied
upon on mainnet for live operation with real emissions.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. Validator permit observations or validation-rights
outcomes from one environment should not be read as representing production subnet performance in
another environment.
