ForeverMoney is Bittensor Subnet 98. Its on-chain identity describes the subnet as decentralized
intelligence for advanced liquidity management, lists [forevermoney.ai](https://forevermoney.ai) as
the subnet URL, and lists
[SN98-ForeverMoney/forever-money](https://github.com/SN98-ForeverMoney/forever-money) as the
subnet's GitHub repository.

## What ForeverMoney Provides

The ForeverMoney README describes the subnet as decentralized automated liquidity management on
Bittensor. It optimizes concentrated-liquidity provision on Uniswap V3 and Aerodrome through
competitive strategies: miners propose dynamic rebalancing decisions, validators evaluate
performance through forward simulations, and the winning strategies are executed on-chain on Base.

According to the README, validators run multiple liquidity-management jobs concurrently, each tied
to a specific pool. Rounds operate in two modes: evaluation rounds, in which all miners compete in
forward simulations from the current chain state, and live rounds, in which winning miners execute
strategies on-chain after an eligible participation period. Miners build per-job reputation for
specific trading pairs, and the README notes a participation requirement before a miner becomes
eligible for live execution.

## Miner and Validator Roles

The README describes miners as implementing a rebalance handler that responds to a validator's
rebalance queries: a miner accepts or refuses a job and returns its desired positions, choosing when
and how to adjust a position or to keep the current one. Miners are scored on protecting inventory
and growing the managed position's value, with reputation accumulated through exponential moving
averages.

Validators evaluate miner strategies through forward simulations, score miners per job, and execute
the winning strategies on-chain. Operators should treat the repository as the source of truth for
current participant guidance.

## On-Chain Identity

Live SN98 data is available on [TaoStats](https://taostats.io/subnets/98). Readers can also
reproduce the chain identity fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 98 --network finney`.

That Finney identity reports the subnet name as ForeverMoney, the subnet URL as forevermoney.ai, and
the GitHub repo as SN98-ForeverMoney/forever-money.
