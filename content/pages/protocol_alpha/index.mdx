# Protocol Alpha

Protocol alpha is subnet alpha accumulated by the Bittensor protocol during reserve injection. It is
cached per subnet instead of being immediately recycled, and it becomes important during pro-rata
settlement when a subnet dissolves
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha)).

The term belongs to subnet tokenomics. It names protocol-held alpha rather than staker-held alpha,
ordinary staking position, or the alpha reserve side of the liquidity pool
([Glossary: Stake](https://docs.learnbittensor.org/resources/glossary#stake),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Distinction from Alpha Halving

Protocol alpha is subnet alpha the Bittensor protocol accumulates during reserve injection and
caches per subnet. Alpha halving is a scheduled reduction in a subnet's alpha emission rate at
supply milestones
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)).

Use protocol alpha for the protocol-held alpha balance. Use alpha halving for the scheduled slowdown
in how fast new subnet alpha is emitted; one names an amount the protocol holds, the other names a
rate change over time.

## Distinction from Slippage

Protocol alpha is subnet alpha accumulated by the Bittensor protocol during reserve injection.
Slippage names the received-amount gap when TAO and subnet alpha move through a pool conversion
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage)).

Accounting attribution and pool conversion outcomes are separate questions on the same subnet.

## Distinction from Alpha Outstanding and Alpha Reserve

Protocol alpha names protocol-held alpha cached per subnet. Alpha outstanding names subnet alpha
outside the reserve pool, while alpha reserve names pool-held alpha inside the paired reserves
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Protocol alpha is one protocol-attributed balance; outstanding and reserve are location labels for
broader subnet alpha accounting. The alpha reserve in particular is the alpha-side balance that,
paired with the TAO reserve, sets the alpha price, whereas protocol alpha is held by the protocol
itself rather than sitting in that tradeable pool reserve
([Understanding Subnets: Liquidity pools](https://docs.learnbittensor.org/subnets/understanding-subnets#liquidity-pools),
[Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)).

## Distinction from Alpha Distribution Ratio

Protocol alpha is a cached protocol-held amount from reserve injection. ADR compares outside-pool
alpha emissions with pool injections on a subnet
([Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)).

One names a balance the protocol holds; the other names a ratio over alpha flow paths.

## Distinction from Stake

Stake names tokens delegated to a validator in a subnet. Protocol alpha names alpha attributed to
the protocol itself during reserve-injection activity
([Glossary: Stake](https://docs.learnbittensor.org/resources/glossary#stake)).

Delegated support and protocol-held alpha should not be folded into one label.

## Distinction from Coinbase and Emission

Protocol alpha names alpha attributed to the protocol during reserve injection. Coinbase advances
per-block emission processing; emission names the broader subnet reward and injection flow
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

- **Protocol alpha** — protocol-held accounting result from reserve injection.
- **Coinbase / emission** — processing step and broader reward flow.

Balance vocabulary and flow vocabulary answer different accounting questions
([Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)).

## Distinction from Subnet Deregistration

Cached protocol alpha becomes important during pro-rata settlement when a subnet dissolves. Protocol
alpha therefore belongs to reserve-injection and dissolution vocabulary, not to ordinary staking
positions ([Subnet Deregistration](https://docs.learnbittensor.org/subnets/subnet-deregistration),
[Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha)).

Dissolution settlement is when readers most often need the protocol-alpha label attached to the
subnet lifecycle.

## Distinction from Alpha Tokens

Alpha tokens name the subnet-specific asset in tokenomics. Protocol alpha names alpha accumulated by
the Bittensor protocol itself during reserve injection and cached per subnet
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The general subnet asset label and the protocol-held balance label answer different accounting
questions.

## Distinction from Yuma Consensus

Protocol alpha is subnet alpha accumulated by the Bittensor protocol during reserve injection and
cached per netuid. It names protocol-held supply from coinbase mechanics, not the recurring
settlement step that converts validator weight submissions into emission shares
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)).

Yuma Consensus is the on-chain mechanism that runs at the epoch boundary on each subnet. It reads
the weight matrix from eligible validators, applies clipping and bonding, and converts the result
into miner incentives and validator dividend shares
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Earlier sections separate protocol alpha from general alpha tokens and from TAO. Reserve injection
builds protocol-held balances; Yuma settles evaluation signals from eligible validators on that
netuid—not the protocol cache ledger entry itself
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Readers should treat protocol alpha as tokenomics supply held by the protocol, and Yuma as the
epoch-boundary mechanism that allocates subnet rewards from consensus while validators remain
active.

- **Protocol alpha** — protocol-held subnet alpha from reserve injection.
- **Yuma Consensus** — epoch-boundary settlement from eligible validator weights to emission shares.

## Distinction from TAO

TAO and protocol alpha have different scopes. TAO is the Bittensor network token, while protocol
alpha names subnet-specific alpha tokens in Dynamic TAO tokenomics
([Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau),
[Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

- **TAO** — the shared network token used across Bittensor.
- **Protocol alpha** — a subnet-scoped token asset, not the network token.

## Reader Boundary

Protocol alpha is reference vocabulary for subnet alpha held by the protocol. It identifies the
protocol-attributed amount created around reserve injection and relevant to subnet settlement
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha)).

A protocol-alpha statement needs the subnet and lifecycle context attached. During reserve
injection, the term explains protocol-side alpha attribution. During dissolution, it explains how
that cached amount participates in settlement.
