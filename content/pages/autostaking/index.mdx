# Autostaking

Autostaking is a Bittensor feature that automatically routes a miner's emissions into stake with a
validator selected by that miner. It turns newly earned mining income into delegated stake without
requiring each emission event to be handled as a separate manual staking action
([Auto Staking for Miners](https://docs.learnbittensor.org/miners/autostaking)).

The concept belongs to reward handling. Mining produces emissions, staking puts value behind a
validator, and autostaking connects those two ideas by making the miner's new emissions flow into
the staking path
([Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

## Emission Routing Context

Autostaking acts on mining emissions after they are earned. The relevant source describes the
feature as automatic staking of mining income to a validator of the miner's choice, so the durable
idea is emission routing rather than a new emissions category
([Auto Staking for Miners](https://docs.learnbittensor.org/miners/autostaking)).

That makes autostaking different from ordinary reward accounting. Emission names the reward flow;
autostaking names one treatment of that flow after the miner receives it
([Emissions](https://docs.learnbittensor.org/learn/emissions)).

## Delegation Context

Autostaking uses the existing staking-and-delegation vocabulary. In Bittensor, stake can be
delegated to validators, and a delegate is the validator-side recipient in that relationship
([Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

Autostaking does not create a separate validator role or a separate reward type. It is an automatic
path that sends new miner emissions into the same delegation context used elsewhere in staking
([Glossary: Delegate](https://docs.learnbittensor.org/resources/glossary#delegate)).

## Miner-Side Boundary

The feature is framed around miners because miners are the actors receiving the mining emissions
being routed. A subnet miner performs work for a subnet and can receive emissions from that subnet's
reward process
([Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner)).

For readers, this means autostaking is not a generic name for every form of staking. It is a
miner-side automation for handling newly earned emissions, with the destination set by the miner
([Auto Staking for Miners](https://docs.learnbittensor.org/miners/autostaking)).

## Validator Destination Boundary

The destination in autostaking is a validator chosen by the miner, but the term itself is not a
validator endorsement. Autostaking names the routing mechanism; validator selection remains a
separate delegation decision
([Auto Staking for Miners](https://docs.learnbittensor.org/miners/autostaking)).

This distinction keeps the article from turning a concept into advice. Autostaking explains what
happens to new miner emissions after the feature is enabled, while validator evaluation belongs to
the broader staking and delegation context
([Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

## Tokenomics Boundary

Autostaking and subnet stake burn both involve subnet-level tokenomics, but they act on different
sides of the system. Autostaking routes miner emissions into delegated stake; subnet stake burn is a
subnet-side mechanism for adding TAO to a subnet reserve while burning corresponding alpha
([Subnet Stake Burn](https://docs.learnbittensor.org/concepts/stake-burn)).

Keeping those terms separate prevents a routing feature from being confused with a reserve or burn
mechanism. Autostaking grows delegated stake from emissions; subnet stake burn changes pool-side
token accounting through a different action path
([Auto Staking for Miners](https://docs.learnbittensor.org/miners/autostaking)).

## Reader Boundary

Autostaking should be read as a concept for automatic emission routing. It is not an earnings
guarantee, a validator recommendation, or an instruction manual. The article-level point is that
miner emissions can be automatically sent into validator staking once the feature is in use
([Auto Staking for Miners](https://docs.learnbittensor.org/miners/autostaking)).

That boundary also keeps autostaking distinct from staking itself. Staking names the destination
activity; autostaking names the automation that keeps sending new miner emissions into that activity
([Glossary: Staking](https://docs.learnbittensor.org/resources/glossary#staking)).
