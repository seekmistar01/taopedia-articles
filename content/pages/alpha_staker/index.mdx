# Alpha Staker

An alpha staker is a participant who holds subnet alpha through staking in a specific Bittensor
subnet. The official
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution) documentation
describes emissions to stakers holding subnet alpha in a mining subnet separately from emissions to
stakers on the Root Subnet. Alpha staker names the subnet-local staking side of that distinction.

## Subnet Alpha Context

Subnet alpha is a subnet-specific asset in Dynamic TAO tokenomics. The
[Glossary: Stake](https://docs.learnbittensor.org/resources/glossary#stake) describes stake as
currency tokens delegated to a validator UID in a subnet, and alpha staking places that support in
the subnet's alpha context rather than through the Root Subnet TAO path.

For readers, alpha staker should be read as a role tied to one subnet market and its alpha token,
not as a network-wide TAO balance label or a wallet file location.

## Contrast with Root Staker

Bittensor staking prose distinguishes subnet-local alpha staking from Root Subnet TAO staking. The
[Glossary: Root Subnet](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)
describes Subnet Zero as the subnet-agnostic TAO staking path, while alpha staker language belongs
to stake held in an ordinary mining subnet's alpha context.

This keeps two ideas separate: root staker names TAO staked through Root Subnet validators, while
alpha staker names participation through subnet alpha in a specific subnet.

## Stake and Emissions Context

Alpha stakers participate in the subnet emission picture described in official distribution
references. The [Emission](https://docs.learnbittensor.org/learn/emissions) documentation treats TAO
and subnet alpha as separate assets in reward flow, and the
[Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)
frames how TAO holders attach support to validators inside subnet markets.

This article does not quote live balances, slippage outcomes, or staking steps. Those details belong
in official staking documentation and live chain data.

## Relationship to Multiple Mechanisms

An alpha staker supplies stake inside one subnet market. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms) notes
that subnets may run more than one incentive mechanism, each with separate bond pools for Yuma
Consensus calculations.

For readers, alpha staker still names subnet-alpha staking in one subnet. When that subnet runs more
than one incentive mechanism, evaluation and emissions context should be read per mechanism rather
than as one blended alpha-staking label.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)

## Reader Boundary

Alpha staker should not be read as a validator recommendation, a live portfolio report, or a synonym
for nominator in every context. It is a role term for subnet-alpha staking participation. Exact
balances, fees, slippage, and subnet-specific behavior belong in current official staking
references.
