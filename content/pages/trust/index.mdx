# Trust

Trust is a Bittensor consensus metric for the miner side of evaluation. It describes how much miner
support remains after validator weights are clipped by consensus
([Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The term belongs to Yuma Consensus vocabulary because it is read after validator weight signals have
been filtered and aggregated. It is not a standalone miner reputation label.

## Consensus Clipping Context

Validators set weight signals after evaluating miner work. Consensus filtering can reduce outlier
support before the final miner-side result is read
([Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust)).

Trust describes the miner-side effect of that filtering. It is therefore about consensus-adjusted
support, not a separate label outside the consensus process.

The clipping context is the main boundary. Trust does not name the original validator weights and
does not name final emission allocation; it describes what remains of miner-side support after
consensus filtering
([Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust)).

## Rank and Pre-Rank

Rank and trust are connected but not identical. Rank names the consensus-adjusted miner result,
while trust describes how much support is preserved between pre-rank and final rank
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust)).

Rank says where the miner lands after aggregation. Trust says how much clipping changed the
pre-rank-to-rank relationship.

That relationship keeps the result and the comparison separate. Rank is the adjusted miner-side
outcome, while trust describes the preservation of support between the pre-clipping signal and final
rank ([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Validator Trust Boundary

Trust is read from the evaluated miner's side of the process. Validator trust is read from the
validator side, describing validator influence after clipping
([Glossary: Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust),
[Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust)).

Keeping the two terms separate avoids treating trust as a universal quality score for every role in
a subnet.

The boundary is role direction. Trust concerns support for an evaluated miner. Validator trust
concerns the validator-side influence reflected by clipped weights
([Glossary: Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust),
[Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights)).

## Weight and Score Context

Validator weights are the input signals, and consensus score is the benchmark used to clip outlier
weights. Trust is read after those filtering steps affect miner-side support
([Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights),
[Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

This makes trust downstream of weight input language. A validator weight can contribute to the
process, but trust describes the miner-side relationship after consensus filtering.

Weight matrix terminology is also earlier than trust. The matrix collects validator weight vectors,
while trust is interpreted after the Yuma process has filtered and aggregated those signals
([Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix),
[Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust)).

## Same-Context Comparison

Trust comparisons are most meaningful when the compared values come from the same subnet consensus
context. Yuma Consensus operates over validator weights inside a subnet, so miner-side trust values
are clearest when read against the process that produced them
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

Across different subnets, the task, validator set, incentive mechanism, and weight pattern can
differ. Trust keeps the same definition, but the surrounding consensus setting can change what a
particular value is telling the reader.

This is why same-context comparisons are safer than cross-subnet comparisons. A trust value is most
informative when the subnet task, validator set, and consensus process remain part of the comparison
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

## Incentive Context

Trust belongs near emissions vocabulary because miner-side support, rank, and clipping all feed into
the broader Yuma Consensus path for incentives
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

That does not make trust a reward table. It explains a consensus relationship between pre-rank and
rank, while emissions describe the broader reward-flow context.

## Environment Boundary

Bittensor separates mainnet, testnet, and localnet environments. Trust keeps the same miner-side
clipping meaning across those environments, but a concrete trust value belongs to the environment,
subnet task, validator set, and consensus context where it was produced
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Introduction to Bittensor: Subnet development](https://docs.learnbittensor.org/learn/introduction#subnet-development)).

Localnet examples are isolated development examples. Testnet examples are shared non-production
examples. Mainnet trust interpretation concerns production Yuma Consensus behavior.
