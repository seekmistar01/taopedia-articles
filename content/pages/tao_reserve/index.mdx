# TAO Reserve

A TAO reserve is the TAO side of a Bittensor subnet liquidity pool. In Dynamic TAO, each subnet pool
pairs a TAO reserve with a subnet-specific alpha reserve, and that pair forms the reserve context
for staking conversions and subnet liquidity
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The term is a pool-location concept. TAO names the network token, while TAO reserve names where TAO
sits when it is part of a subnet's paired reserve system
([Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao)).

## Pool Pair Context

The TAO reserve is paired with an alpha reserve. The TAO side contains TAO associated with the
subnet pool, while the alpha side contains that subnet's alpha token. Together, the two reserves
make the pool usable for Dynamic TAO staking and unstaking context
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

This keeps TAO reserve separate from a general TAO holding. A reserve is the pool side used by the
subnet market; the token itself can appear in other Bittensor contexts outside that pool
([Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao)).

## Reserve Ratio Context

Reserve ratio is the relationship between the TAO reserve and the paired alpha reserve. That ratio
is the pool relationship used when discussing alpha token pricing context, while TAO reserve names
only one side of the pair
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The distinction helps avoid overloading the reserve term. TAO reserve identifies the TAO quantity
inside the pool, alpha reserve identifies the subnet-token side, and reserve ratio names the
relationship between them
([Emission: TAO reserve injection](https://docs.learnbittensor.org/learn/emissions#tao-reserve-injection),
[Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)).

## Emission Injection Context

Emission can add TAO into subnet reserves. TAO reserve injection is part of the emission process
that allocates TAO across subnet pools, while alpha reserve injection grows the paired alpha side in
the related pool context
([Emission: TAO reserve injection](https://docs.learnbittensor.org/learn/emissions#tao-reserve-injection),
[Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)).

That places TAO reserve inside the injection stage of tokenomics. It does not by itself name miner
incentives, validator dividends, staking returns, or creator allocations. Those are later allocation
outcomes in the broader emission flow
([Emissions](https://docs.learnbittensor.org/learn/emissions)).

## Alpha and Protocol Context

TAO reserve is easiest to understand beside alpha reserve and protocol alpha. Alpha reserve is the
subnet-token side of the pool, while protocol alpha is alpha accumulated by the protocol during
reserve-injection activity
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha),
[Emission: TAO reserve injection](https://docs.learnbittensor.org/learn/emissions#tao-reserve-injection)).

The three terms answer different questions. TAO reserve names where TAO sits in the pool, alpha
reserve names the paired subnet-token side, and protocol alpha names protocol-held alpha produced
around the injection process
([Emission: Alpha reserve injection](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection)).

## Liquidity Context

TAO reserve is part of the core subnet pool, while liquidity positions are additional ranges that
can add depth to the same pool context. The reserve pair is the baseline TAO-and-alpha structure;
liquidity positions sit around that structure rather than replacing it
([User Liquidity Positions](https://docs.learnbittensor.org/liquidity-positions),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

This makes TAO reserve a core pool term rather than a strategy term. A liquidity position can refer
to how depth is supplied around a pool range, but TAO reserve names the pool-held TAO side used in
the subnet's tokenomics model
([User Liquidity Positions](https://docs.learnbittensor.org/liquidity-positions)).

## Coinbase and Network Context

Coinbase is the per-block mechanism that advances emission processing, including reserve injection.
TAO reserve is the pool-side value affected by that injection; coinbase is the mechanism that runs
the emission step
([Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase),
[Emission: TAO reserve injection](https://docs.learnbittensor.org/learn/emissions#tao-reserve-injection)).

Network environment also matters. Mainnet, testnet, and localnet are separate contexts, so a TAO
reserve example from one environment is evidence about that environment rather than a universal
subnet-liquidity statement
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Introduction to Bittensor: Subnet development](https://docs.learnbittensor.org/learn/introduction#subnet-development)).

## Reader Boundary

TAO reserve is a reference term for the pool-held TAO side of a subnet liquidity pool. It is not a
staking tutorial, a subnet ranking table, a yield estimate, or a recommendation about liquidity
positioning. Exact amounts and concrete examples require the selected subnet, network environment,
and source for the pool context
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).
