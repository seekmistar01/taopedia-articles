# Root Claim Type

Root claim type is the Keep or Swap setting for handling alpha dividends earned through Root Subnet
staking. Keep leaves processed dividends as originating-subnet alpha, while Swap converts processed
dividends to TAO and restakes them on Root
([Root Claim overview](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

The term belongs to dividend-handling vocabulary. It names the selected outcome for processed root
claims, not a separate staking action, claim trigger, or balance report.

## Claim Setting

Root claim type is a standing setting applied when accumulated root-claim dividends are processed.
The setting chooses between the Keep and Swap dividend outcomes
([Root Claim overview](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

That makes root claim type different from claim timing. Claim timing answers when accumulated
dividends are processed; root claim type answers how those dividends are handled when processing
occurs.

## Keep Mode

Keep mode leaves earned alpha on the subnet that generated the dividend
([Root Claim overview](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

For readers, Keep is the alpha-retention outcome. It does not change the definition of Root Subnet
staking, and it does not by itself describe the quality of the subnet that produced the dividend.

## Swap Mode

Swap mode converts accumulated alpha dividends to TAO and restakes the resulting TAO on the Root
Subnet ([Root Claim overview](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

For readers, Swap is the TAO-restaking outcome. It should not be confused with unstaking from Root,
moving stake between validators, or changing the underlying Root Subnet staking position.

## Claim Processing Boundary

Root claims name the broader process for handling accumulated dividends. Root claim type is the
setting used inside that process
([Root Claim overview](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

This boundary keeps the vocabulary precise. A root claim is the processing event, while root claim
type is the Keep or Swap setting that determines the dividend outcome during processing.

## Root Vocabulary Context

Root Subnet is the subnet-agnostic staking context where TAO can be delegated, and root claim type
names what happens to dividends produced through that position
([Glossary: Root Subnet](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Root Claim overview](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

The setting also belongs to the root-staker role. Root staker names who holds the Root staking
position; root claim type names how claim dividends from that position are handled
([Glossary: Root Staker](https://docs.learnbittensor.org/resources/glossary#root-staker),
[Root Claim overview](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

Root proportion is adjacent but different vocabulary. Root proportion is about weight influence,
while root claim type is about Keep or Swap outcomes for dividends
([Glossary: Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion),
[Root Claim overview](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

## Network Evidence Boundary

Root claim type examples should keep their network environment attached. Mainnet, testnet, and
localnet are separate environments for interpreting observed claim behavior
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

The concept remains the Keep or Swap setting, but observed claim behavior belongs to the network
environment where it was observed. Static article prose should not turn one environment's example
into a general claim about every Root Subnet context.

## Reader Boundary

Root claim type should be read as a concept for dividend handling. It does not describe how to set
the preference, initiate a claim, quote claim thresholds, or report exact conversion outcomes
([Root Claim overview](https://docs.learnbittensor.org/staking-and-delegation/root-claims)).

The stable article-level point is that root claim type names the Keep or Swap setting that
determines whether processed Root Subnet dividends remain as subnet alpha or are converted to TAO
and restaked on Root.
