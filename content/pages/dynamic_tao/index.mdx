# Dynamic TAO (dTAO)

Dynamic TAO, often shortened to dTAO, is Bittensor's subnet-aware tokenomics system. It connects
network-level TAO with subnet-specific alpha tokens, subnet liquidity pools, emission routing, and
validator stake weight
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The main change is that subnet economics are no longer described only through network-wide TAO. Each
subnet has an alpha side as well as a TAO side, so staking, pool liquidity, and emission
distribution all need subnet-specific wording.

## Subnet Alpha Model

Dynamic TAO gives each subnet its own alpha token. The
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) documentation
describes subnet pools with TAO reserves, alpha reserves, and alpha outstanding.

That creates a two-asset subnet model. TAO remains the network asset, while alpha describes the
subnet-specific asset used in that subnet's pool, stake, and reward calculations.

## Reserve and Outstanding Alpha

The alpha side is split by location. Alpha reserve is the subnet alpha inside the liquidity pool,
paired with the subnet's TAO reserve. Alpha outstanding is alpha outside that reserve side and is
connected to subnet stake and rewards ([Emission](https://docs.learnbittensor.org/learn/emissions)).

This makes Dynamic TAO easier to read. A pool statement usually concerns TAO reserve and alpha
reserve. A reward or stake statement usually concerns alpha outstanding.

## Emission Routing

The [Emission](https://docs.learnbittensor.org/learn/emissions) documentation describes emitted TAO
and alpha moving through subnet-specific paths. TAO enters subnet reserve paths, alpha can enter the
reserve side, and alpha outstanding is set aside for subnet rewards.

Dynamic TAO therefore links emission to subnet liquidity. Emission is not only a network supply
event; it also changes the TAO-and-alpha structure that each subnet uses.

## Validator Stake Weight

Validator stake weight is one concrete place where Dynamic TAO combines TAO and alpha. The
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets#validator-stake-weight)
documentation defines a validator's subnet stake weight as alpha stake plus TAO stake multiplied by
the `tao_weight` parameter.

In plain terms, validator stake weight lets subnet alpha stake and TAO stake both contribute to a
validator's influence in a subnet, with TAO scaled by the subnet's `tao_weight`. This is more
specific than saying TAO "matters differently"; the difference comes from the documented stake
weight formula.

## Cross-Subnet Allocation

Dynamic TAO also affects how emissions are compared across subnets. The
[Emission](https://docs.learnbittensor.org/learn/emissions#distribution-across-subnets)
documentation describes distribution across subnets through flow-based emission logic.

This connects subnet demand signals with emission share. Dynamic TAO is the system in which TAO
flows, subnet alpha, and subnet weights interact rather than a separate asset by itself.

## Yuma Consensus Role

Dynamic TAO and Yuma Consensus describe different layers of the system. Dynamic TAO describes the
TAO-and-alpha tokenomics across subnets, while
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) describes how validator
signals are combined inside a subnet to determine reward outcomes.

The two are connected through emissions. Dynamic TAO provides the subnet tokenomics environment;
Yuma Consensus resolves within-subnet reward allocation.

## Term Placement

Dynamic TAO is the broad term for Bittensor's subnet-aware tokenomics. Alpha reserve names the
pool-held alpha side. Alpha outstanding names alpha outside that reserve side. Validator stake
weight names the formula that combines subnet alpha stake and TAO stake for validator influence.
Subnet weights describe cross-subnet emission allocation
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Those terms are related because they all describe parts of the same Dynamic TAO design, but each one
answers a different question about subnet economics.
