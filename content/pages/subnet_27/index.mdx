Nodexo is Bittensor Subnet 27, described in its on-chain identity as a decentralized AI compute
platform. In plain terms, the subnet turns access to computing power into a rewarded network
service: participants contribute compute, the network checks that the compute is genuine and useful,
and rewards flow to the contributors who provide the most value.

## What Nodexo Provides

The on-chain identity for netuid 27 names the subnet Nodexo and describes it as a decentralized AI
compute platform. Rather than training a single model, this kind of subnet provides a shared
resource — compute capacity that applications, researchers, and other Bittensor subnets can draw on.

This places Nodexo in the infrastructure layer of the ecosystem. Its value proposition is access:
instead of renting capacity from one centralized provider, demand is matched against a competitive
pool of independent operators, and the network's incentives push that pool toward reliable, genuine
compute.

## Miner and Validator Roles

The two roles follow the standard Bittensor split between supplying a capability and scoring it.
Miners supply the capability — they contribute compute resources to the network. Validators provide
the evaluation — they assess what miners contribute and decide how much each contribution is worth.

Validator assessments become weights, and those weights feed into [[Yuma Consensus]], which converts
many independent evaluations into the emissions that reward the strongest miners. In plain terms,
miners supply compute and validators decide whose compute is genuine and useful enough to be
rewarded. Operators should treat the subnet's own documentation and tooling as the source of truth
for current setup and scoring details.

## On-Chain Identity

Live SN27 subnet data is available on [TaoStats](https://taostats.io/subnets/27), which identifies
the subnet as Nodexo. Readers can reproduce the subnet identity fields directly with the Bittensor
CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 27 --network finney`.

At the time of writing, the Finney identity for netuid 27 publishes the subnet name (Nodexo), the
description above, and `nodexo.ai` as the project site. The identity also lists a GitHub repository
field, but that repository path no longer resolves, so this article describes the subnet from its
current on-chain identity rather than from a specific codebase. That chain identity is the source of
truth for what Subnet 27 is.
