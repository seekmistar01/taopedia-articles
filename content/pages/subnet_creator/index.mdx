# Subnet Creator

A subnet creator is the role that defines what a Bittensor subnet is meant to produce and how that
work should be evaluated
([Glossary: Subnet Creator](https://docs.learnbittensor.org/resources/glossary#subnet-creator),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The term belongs to subnet design vocabulary. It should be read separately from miner and validator
roles that operate inside the subnet.

## Role in a Subnet

The creator role sits before miner work and validator evaluation. It gives the subnet a work target
and evaluation context so miners and validators are not operating against a generic network-wide
task ([Glossary: Subnet Creator](https://docs.learnbittensor.org/resources/glossary#subnet-creator),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

For readers, the creator shapes the context. Miners produce work inside that context, and validators
evaluate the work inside that context.

## Digital Task Context

A subnet creator defines the digital task miners are asked to perform. That task gives miners a
concrete work target inside the subnet rather than a generic assignment
([Glossary: Subnet Creator](https://docs.learnbittensor.org/resources/glossary#subnet-creator),
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task)).

Task definition is the reason subnet creator is a design role. It establishes what useful work means
for the subnet.

## Incentive Mechanism Context

Subnet creators also shape incentive mechanisms for the subnet. Incentive-mechanism documentation
separates task, protocol, and scoring model as related parts of the design
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Glossary: Subnet Creator](https://docs.learnbittensor.org/resources/glossary#subnet-creator)).

This keeps creator vocabulary tied to evaluation design rather than day-to-day mining or validating.
The creator defines the context those roles use.

## Relationship to Miners and Validators

Subnet creators are not the same role as miners or validators. Miners produce work for the subnet,
and validators evaluate that work against subnet standards
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Subnet Creator](https://docs.learnbittensor.org/resources/glossary#subnet-creator)).

The creator role shapes the rules and work target. Miner and validator roles operate inside the
resulting subnet design.

## Multiple-Mechanism Context

A subnet creator can shape more than one incentive mechanism inside a subnet. Multiple-mechanism
documentation describes separate mechanism contexts that can operate in parallel
([Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Subnet Creator](https://docs.learnbittensor.org/resources/glossary#subnet-creator)).

For readers, one mechanism's task or scoring model should not automatically be treated as the whole
subnet's design.

## Reader Boundary

Subnet creator should be read as a role concept. It is not a subnet launch guide, operations
checklist, funding explainer, or claim about one specific subnet's quality
([Glossary: Subnet Creator](https://docs.learnbittensor.org/resources/glossary#subnet-creator),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The stable article-level point is that a subnet creator defines a subnet's work and evaluation
context.
