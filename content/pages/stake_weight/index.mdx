# Stake Weight

Stake weight is the subnet-level stake measure used to describe validator influence in Bittensor
consensus. It is the computed stake value tied to consensus power and emissions inside a subnet
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The term is a bridge between staking and evaluation. It explains how stake-derived influence affects
weighted validator signals in the consensus process.

## Consensus Role

Stake weight explains how much influence a validator's evaluation carries inside a subnet. It is not
only a count of staked TAO, and it is not a general reputation score
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

This keeps stake weight tied to consensus. Staking supplies economic backing, while Yuma Consensus
uses weighted validator signals to shape subnet outcomes.

## Alpha and TAO Components

Stake weight combines alpha stake, TAO stake, and TAO weight in subnet context
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)).

That framing matters because stake weight is not simply one visible stake amount copied into
consensus. It is the subnet-context value that combines stake components before validator influence
is interpreted.

## Effective Stake Boundary

Stake weight and effective stake are easy to confuse because both concern validators and staked TAO.
Effective stake is a staking total, while stake weight is the value used for subnet consensus
influence
([Glossary: Effective Stake](https://docs.learnbittensor.org/resources/glossary#effective-stake),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

Effective stake can help explain validator backing, but stake weight is the term for the
subnet-specific influence measure.

## Validator Permit Boundary

Stake weight also sits near validator permits, but it should not be read as the permit itself. A
permit is the right to validate, while stake weight is the stake-based influence measure used in
consensus context
([Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

The distinction is straightforward. Stake weight is a measure; a validator permit is a validation
right.

## Mechanism Boundary

Stake weight should be read inside the subnet and mechanism context that uses it. Validators
evaluate miner work according to subnet rules, and some subnets can separate evaluation into more
than one mechanism
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Multiple Incentive Mechanisms](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets)).

That means stake weight is best understood as validator influence for the relevant consensus path,
not as a universal comparison across unrelated evaluation systems.

## Reader Boundary

Use stake weight when the focus is validator influence in subnet consensus. Use effective stake for
a staking-total concept, TAO weight for the relative role of TAO stake, and validator permit for
validation rights
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit)).

This page is a reference article for the stake-derived consensus influence measure. It is not a
staking guide, a validator ranking, or a full explanation of reward outcomes.
