# Bittensor Governance

Bittensor governance is the proposal and approval path for privileged network changes. The process
separates proposal creation by the Triumvirate from review and approval by the Senate
([Governance Overview](https://docs.learnbittensor.org/governance),
[Senate](https://docs.learnbittensor.org/governance/senate)).

## What It Represents

Governance applies to privileged network changes such as parameter changes, upgrades, and related
administrative actions. It is not a validator-scoring mechanism, a staking flow, or an ordinary
subnet work loop ([Governance Overview](https://docs.learnbittensor.org/governance)).

A proposal executes only after two conditions: the Senate reaches a majority of approvals, and a
Triumvirate member closes the proposal. Senate approval alone is not execution
([Governance Overview](https://docs.learnbittensor.org/governance)).

Anyone can view proposals before the Senate through public commands, while senator voting is
restricted to senators who meet documented stake and registration requirements
([Senate](https://docs.learnbittensor.org/governance/senate)).

Sudo names the earlier single-key administration model. Governance names the review path meant to
broaden control over privileged changes
([Glossary: Sudo](https://docs.learnbittensor.org/resources/glossary#sudo),
[Governance Overview: security](https://docs.learnbittensor.org/governance#security)).

Governance examples depend on which network you connect to. Proposal outcomes from one environment
should not be read as production behavior on another
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from Senate

Governance names the overall proposal-and-approval process. The Senate is the elected body that
reviews proposals for approval or rejection
([Senate](https://docs.learnbittensor.org/governance/senate),
[Glossary: Senate](https://docs.learnbittensor.org/resources/glossary#senate)).

- **Governance** — the overall proposal-and-approval process.
- **Senate** — the reviewing and voting body in that process.

## Distinction from Triumvirate

The Triumvirate creates proposals that the Senate reviews. Governance vocabulary covers both steps
without collapsing them into one role
([Glossary: Triumvirate](https://docs.learnbittensor.org/resources/glossary#triumvirate),
[Governance Overview](https://docs.learnbittensor.org/governance)).

- **Governance** — the full proposal-and-approval path.
- **Triumvirate** — the proposal-origin body that creates items for review.

## Distinction from Proposal

A proposal is the privileged change package under review. Governance names the wider process that
includes creation, review, and execution gates
([Glossary: Proposal](https://docs.learnbittensor.org/resources/glossary#proposal)).

- **Governance** — the process that moves proposals through review and execution.
- **Proposal** — the specific item under review.

## Distinction from Sudo

Sudo names the earlier single-key administration model. Governance names the staged review path that
replaced unilateral control
([Glossary: Sudo](https://docs.learnbittensor.org/resources/glossary#sudo),
[Governance Overview: security](https://docs.learnbittensor.org/governance#security)).

- **Governance** — proposal review and approval for privileged network changes.
- **Sudo** — legacy single-key administrative authority pattern.

## Distinction from Bicameral Legislature

Bicameral legislature names the two-body structure that pairs proposal creation with proposal
review. Governance names the active process that runs through that structure
([Glossary: Bicameral Legislature](https://docs.learnbittensor.org/resources/glossary#bicameral-legislature),
[Governance Overview](https://docs.learnbittensor.org/governance)).

- **Bicameral legislature** — structural label for the two-body governance system.
- **Governance** — the proposal-and-approval process within that system.

## Distinction from Yuma Consensus

Governance names the Triumvirate-and-Senate path for privileged network changes. Yuma Consensus
converts validator weight submissions into emission shares each tempo on a subnet
([Governance Overview](https://docs.learnbittensor.org/governance),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Governance** — proposal review and approval for privileged network changes.
- **Yuma Consensus** — aggregation that turns validator weights into emission shares.

## Reader Boundary

Bittensor governance should be read as proposal-and-approval vocabulary. It does not provide voting
instructions, proposal strategy, operational steps, or point-in-time governance results
([Governance Overview](https://docs.learnbittensor.org/governance),
[Senate](https://docs.learnbittensor.org/governance/senate)).
