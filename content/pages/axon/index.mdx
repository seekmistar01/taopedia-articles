# Axon

An axon is the server-side endpoint in Bittensor's Axon-dendrite communication model. The official
[Glossary](https://docs.learnbittensor.org/resources/glossary#axon) describes axons as receiving
incoming Synapse objects and as the entry point commonly advertised by a subnet miner so subnet
validators can communicate with that miner. The official
[Axon reference](https://docs.learnbittensor.org/python-api/html/autoapi/bittensor/core/axon/)
places the term on the server-side interface that services incoming requests.

## Endpoint Context

Axon is useful as a communication term. It names the endpoint that can receive subnet messages,
while dendrite names the client-side component that contacts axon endpoints and handles responses.
At Taopedia level, the distinction is simple: dendrite initiates the exchange, and axon is the
endpoint being contacted.

This endpoint framing is narrower than a miner, validator, or complete subnet protocol. An axon can
be associated with a miner-facing service path, but the term itself describes the communication
entry point rather than the full work performed by a subnet participant.

## Relationship to Dendrites

The paired term is dendrite. The official
[Dendrite reference](https://docs.learnbittensor.org/python-api/html/autoapi/bittensor/core/dendrite/)
describes Dendrite as a network client that sends requests to axon endpoints and processes their
responses. Read together, axon and dendrite give Bittensor's request-and-response vocabulary around
subnet communication.

This does not mean every axon has the same behavior. The subnet protocol still defines what request
types matter, what response is expected, and how the result is evaluated. Axon only names the
server-side contact point in that path.

## Subnet Communication Context

The glossary places axons near subnet miners and subnet validators because those roles often meet
through this communication path. A validator-side process can use a dendrite to contact miner
endpoints, while the miner-side axon receives the incoming Synapse objects relevant to the subnet's
protocol.

For readers, the concept helps separate transport vocabulary from scoring vocabulary. Axon explains
where a subnet message is received; it does not by itself explain the subnet task, scoring model, or
validator judgement.

## Relationship to Multiple Mechanisms

An axon is usually discussed for one incentive mechanism at a time, but subnets can also run
multiple mechanisms in parallel. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#subnet-task) notes that when multiple
incentive mechanisms are used within a subnet, each mechanism can define different tasks for miners
to perform.

For readers, an axon still names a server-side communication endpoint, but the mechanism context
matters. The task context for one mechanism should be read separately when a subnet uses more than
one evaluation system.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task)

## Reader Boundary

Axon should not be read as a complete miner implementation, a live endpoint listing, or a guarantee
that a participant is reachable. Claims about current endpoint details, availability, ports, or a
specific subnet's serving behavior need a current chain source or that subnet's own documentation.

## Relationship to Subtensor

Axon and Subtensor are related but different parts of Bittensor chain vocabulary. Subtensor names
Bittensor's layer 1 blockchain and system of record, while axon names the server-side endpoint used
in off-chain subnet communication before validator evaluation is recorded on that chain layer. The
[Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor) describes the
blockchain layer, and the [Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon)
describes axons as receiving incoming synapse objects.

For readers, Subtensor names the chain layer, while axon names the server-side communication
endpoint used in subnet request-and-response work.

References: [Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor),
[Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon)

## Relationship to Netuid

Axon and a netuid are related but different parts of Bittensor vocabulary. A netuid identifies a
subnet on the network, while axon names the server-side endpoint a subnet miner can advertise inside
that subnet context. The
[Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid) places netuid at the
subnet level, and the [Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon)
describes axons as the entry point commonly advertised by a subnet miner.

For readers, a netuid selects which subnet is in scope, while axon names the server-side endpoint
inside that subnet.

References: [Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid),
[Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon)

## Relationship to Synapse

Axon and a synapse object are related but different parts of Bittensor subnet communication
vocabulary. A synapse object names the structured message exchanged between subnet validators and
subnet miners, while axon names the server-side endpoint that receives those incoming messages. The
[Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse) describes synapse
objects as the main vehicle for information exchange, and the
[Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon) describes axons as
receiving incoming synapse objects.

For readers, a synapse object names the message format, while axon names the endpoint that receives
it.

References: [Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon)

## Relationship to Subnet Miner

Axon and a subnet miner are related but different parts of Bittensor subnet role vocabulary. A
subnet miner produces work for a subnet task and receives validator evaluation, while axon names the
server-side endpoint commonly advertised by that miner for subnet communication. The
[Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner) describes
the miner role, and the [Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon)
describes axons as the entry point commonly advertised by a subnet miner.

For readers, a subnet miner names the work-production role, while axon names the communication
endpoint associated with that role.

References:
[Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner),
[Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon)
