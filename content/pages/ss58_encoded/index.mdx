# SS58 Encoded

SS58 encoded values are compact representations used for Bittensor wallet addresses. The
[SS58 Encoded glossary](https://docs.learnbittensor.org/resources/glossary#ss58-encoded) places the
term in wallet-address context, where public-key-derived address strings identify TAO destinations.

## Address Representation

An SS58 encoded value is not a separate asset or account role. It is an address representation. The
[Wallet Address glossary](https://docs.learnbittensor.org/resources/glossary#wallet-address)
describes wallet addresses as public-key-derived identifiers used as TAO destinations in the
Bittensor network.

## Public-Key Context

The public-key side of the term matters because SS58 encoded values represent public-key-derived
addresses. A [public key](https://docs.learnbittensor.org/resources/glossary#public-key) is
shareable material used for verification, encryption, and account reference in Bittensor; private
signing material is outside the SS58 encoded address-format concept.

## Wallet Context

In reader-facing Taopedia prose, SS58 encoded should be read as address-format vocabulary. It
connects the compact representation described in the
[SS58 Encoded glossary](https://docs.learnbittensor.org/resources/glossary#ss58-encoded) with the
destination-address meaning described for
[wallet addresses](https://docs.learnbittensor.org/resources/glossary#wallet-address).

## Reading the Term

An SS58 encoded value should not be treated as a balance, a staking position, or a live account
claim by itself. The
[SS58 Encoded glossary](https://docs.learnbittensor.org/resources/glossary#ss58-encoded) supports
the address-representation concept, not instructions for creating, securing, or using a wallet.
