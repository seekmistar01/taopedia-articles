# Alpha Distribution Ratio

Alpha Distribution Ratio, often shortened to ADR, is a Bittensor tokenomics measure that compares
alpha tokens emitted to participants with alpha tokens injected into a subnet's automated market
maker pool. The
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)
entry defines the term as a comparison between alpha held by participants and alpha remaining in the
pool.

## What ADR compares

ADR separates two paths for subnet alpha tokens. One path is participant emission: alpha set aside
for subnet participants through the emission process. The other path is pool injection: alpha added
to the subnet's pool alongside TAO liquidity. The
[Emission](https://docs.learnbittensor.org/learn/emissions#injection-and-distribution-two-stages-of-the-emissions-process)
docs describe emission as having an injection stage and a distribution stage, which is the context
needed to read ADR.

This makes ADR a relationship between two parts of subnet tokenomics, not a standalone price,
staking recommendation, or reward table.

## Halving Index Gap

The [Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)
states that, under the current protocol, Alpha Distribution Ratio tracks the difference between the
global TAO halving index and a subnet's alpha halving index. That gap links ADR to halving
vocabulary at two scopes: network-wide TAO halving and subnet-level alpha halving. Readers should
treat ADR as a subnet tokenomics ratio shaped by those indexes rather than as a wallet action or
staking command.

The [Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving) documentation explains
that TAO halvings operate at network scope while alpha halvings operate at subnet scope. ADR sits in
the vocabulary that compares participant-side emission with pool-side injection after that halving
background is understood.

References:
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)

## Emissions Relative to Injections

The glossary summarizes ADR as emissions relative to injections for a subnet. That wording names a
ratio between alpha allocated toward participant emission and alpha added through pool injection,
not a separate asset or a live market quote. The
[Emission](https://docs.learnbittensor.org/learn/emissions#injection-and-distribution-two-stages-of-the-emissions-process)
documentation describes injection and distribution as two stages of the emissions process, which is
the process context ADR compares.

Readers should keep ADR on the relationship between those subnet alpha paths. It does not by itself
report a participant balance, a staking rank, or a recommended trade.

References:
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio),
[Emission](https://docs.learnbittensor.org/learn/emissions)

## Subnet Tokenomics After Context Is Fixed

ADR compares participant emission with pool injection inside one subnet that is already in view.
Network-wide comparisons of subnets for injection share belong to flow-based emission vocabulary
rather than to the ADR definition itself. The
[Emission: Distribution across subnets](https://docs.learnbittensor.org/learn/emissions#distribution-across-subnets)
documentation describes cross-subnet injection comparison, while ADR names an intra-subnet ratio
between participant-side and pool-side alpha paths.

For readers, ADR answers how alpha is split between those two paths within a subnet's tokenomics.
Live ADR values, discount mechanics, and formula tables belong in official emission references
rather than in this concept article. Halving schedules and live subnet indexes can change and should
be read from current official halving and emission documentation.

References:
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio),
[Emission](https://docs.learnbittensor.org/learn/emissions)

## Relationship to alpha emissions

Alpha emitted to participants is the numerator side of the ADR idea. In the emission docs, alpha can
be allocated to alpha outstanding and later distributed to subnet participants. The same docs also
describe alpha reserve injection into the subnet pool, which is the pool-side comparison point for
ADR.

The glossary summarizes this distinction by saying ADR is calculated as emissions relative to
injections:
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio).

## Relationship to Alpha Reserve

Alpha reserve is the pool-side context behind the injection side of ADR. The
[Emission](https://docs.learnbittensor.org/learn/emissions#alpha-reserve-injection) documentation
describes alpha reserve injection as adding alpha to the subnet's reserve, while ADR compares
participant-side alpha emission with that pool-side injection.

For readers, alpha reserve names where the injected pool-side alpha sits. ADR names the relationship
between that pool-side path and alpha emitted to participants.

## Halving context

ADR is tied to the relationship between network-level TAO halvings and subnet-level alpha halvings.
The glossary states that, under the current protocol, ADR tracks the difference between the global
TAO halving index and a subnet's alpha halving index. The
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving) docs explain the necessary
background: TAO halvings operate at network scope, while alpha halvings operate at subnet scope.

This is why ADR should be read as a subnet-level tokenomics ratio rather than a general statement
about all emissions in Bittensor.

## Reader boundary

ADR is not the same concept as subnet weights. Subnet weights describe how emissions are allocated
across subnets under the flow-based model. ADR describes the relationship between alpha emitted to
participants and alpha injected into a subnet pool after the subnet context is already in view.

ADR also should not be treated as a live market quote. The glossary notes that higher ADR changes
the discount context for liquidating alpha relative to spot price, but the term's core meaning is
the emissions-versus-injections relationship.

## Flow-Based Emissions Boundary

ADR should be read inside a single subnet's tokenomics, separate from network-wide subnet
comparison. The official
[Emission: Distribution across subnets](https://docs.learnbittensor.org/learn/emissions#distribution-across-subnets)
documentation describes how subnets are compared for injection share through flow-based allocation,
while ADR compares alpha emitted to participants with alpha injected into that subnet's pool.

For readers, flow-based emissions answer how subnets compete for injection context; ADR answers how
alpha is split between participant emission and pool injection after the subnet context is already
in view.

References:
[Emission: Distribution across subnets](https://docs.learnbittensor.org/learn/emissions#distribution-across-subnets),
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)

## Relationship to Multiple Mechanisms

Alpha Distribution Ratio compares alpha emitted to participants with alpha injected into a subnet
pool. The [Glossary](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism) notes
that when multiple incentive mechanisms are used, each operates independently, allowing subnet
creators to distribute emissions across different types of work or evaluation criteria.

For readers, ADR still describes a subnet-level ratio between participant emission and pool
injection. Multiple mechanisms change how participant-side alpha emission is earned, not what ADR
compares.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Incentive Mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism)

## Relationship to TAO Weight

Alpha Distribution Ratio and TAO weight both relate to how alpha and TAO interact in Bittensor
subnet emissions, but they name different parts of that relationship. ADR is a subnet-level ratio
that compares alpha emitted to participants with alpha injected into the pool. TAO weight is a
network-level parameter that describes how TAO stake contributes relative to alpha stake in the
emission context. The
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight) describes the
global weight given to TAO stake alongside alpha stake, while
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)
describes the split between participant emission and pool injection for alpha.

For readers, TAO weight is a network-scope parameter that shapes how TAO stake is counted; ADR is a
subnet-scope ratio that measures how alpha is split after the emission context is set. They operate
at different scopes and answer different questions about the same emission system.

References: [Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight),
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)

## Relationship to Protocol Alpha

Alpha Distribution Ratio and protocol alpha address related but different parts of Bittensor subnet
tokenomics. ADR is a subnet-level ratio comparing alpha emitted to participants with alpha injected
into the subnet's pool, while protocol alpha names the alpha token inventory the protocol
accumulates from chain-side TAO buys during reserve injection, cached per subnet rather than
immediately recycled. The
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)
defines the ratio as comparing participant-side emission with pool-side injection, and the
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)
documentation describes reserve injection as the process that accumulates protocol alpha.

Protocol alpha belongs to the pool-injection side of the ADR comparison. When the protocol buys
alpha with TAO during reserve injection, the resulting alpha accumulates as protocol alpha — cached
per subnet rather than recycled immediately. ADR tracks the relationship between participant
emission and pool injection as a ratio, but it does not further decompose how much of the injection
side is cached as protocol alpha versus made available in other ways. ADR names what alpha does at
the subnet level; protocol alpha names a specific inventory category that accumulates within the
injection side of that comparison.

References:
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio),
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from local testing to testchain and then mainchain. For Alpha
Distribution Ratio, that sequence changes how readers should interpret evidence about the
participant-emission versus pool-injection balance in a subnet.

In local testing, ADR can show how alpha is split between participant emission and pool injection
inside an isolated subnet environment. That is useful for checking the ratio mechanics, but it
remains isolated feedback from a local environment and does not reflect production tokenomics.

On testchain, ADR behavior can be observed in a shared testing network. This gives stronger evidence
about how the participant-emission and pool-injection split responds to halving and emission events
than a private local run, while still keeping the result separate from production mainchain values.

On mainchain, ADR reflects live subnet tokenomics tied to actual alpha emission and pool injection
activity. The
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)
defines ADR as a comparison between alpha held by participants and alpha remaining in the pool, so
production ADR context is tied to real subnet emission state on the active network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. An ADR value observed in one environment should not be
read as identical evidence about the participant-emission versus pool-injection split in another
environment.

For readers, this keeps ADR examples from sounding stronger than their environment supports. Local,
testchain, and mainchain contexts can all describe the participant-emission versus pool-injection
relationship, but they do not carry the same interpretive weight.
