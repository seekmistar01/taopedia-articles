# Weight Vector

A weight vector is one validator's structured set of miner evaluations inside a Bittensor subnet.
Each element represents a weight assigned to a subnet miner according to that validator's evaluation
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The scope is narrow. A single vector is one validator's signal before aggregation, not the whole
subnet result. The individual values inside the set are validator weights
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector)).

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For weight vector
examples on a subnet such as netuid 1, that sequence changes how readers should interpret examples
tied to subnet state, validator activity, or chain observations.

In localnet, weight vector examples can be tested in an isolated environment. Local chain state
reflects local configuration rather than production network behavior.

On testnet, weight vector can be observed in a shared, non-production network. Testnet readings on a
selected netuid are separate from mainnet state
([Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector)).

On mainnet, weight vector applies on the live production Bittensor network for subnets such as
netuid 1 and the environment context being discussed.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A weight vector example from one environment should not be
read as representing production behavior on another network.

## Distinction from Weight Matrix

A weight vector is one validator's miner weight assignments in a single submission. A weight matrix
is the combined validator-to-miner assignment table across the validator set on the selected subnet
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix)).

Weight-vector vocabulary names one submission. Weight-matrix vocabulary names the grouped input Yuma
reads before consensus aggregation.

## Distinction from Consensus Score

Consensus score is the stake-weighted benchmark used to clip outlier weights. A weight vector is the
per-validator input evaluated before that benchmark is formed on the selected subnet
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus: Clipping](https://docs.learnbittensor.org/learn/yuma-consensus#clipping)).

Weight-vector vocabulary belongs to submitted signals. Consensus-score vocabulary belongs to the
agreement benchmark derived from those signals.

## Distinction from Rank

Rank is the miner-side standing after validator weights are filtered and aggregated. A weight vector
is one validator's raw evaluation input before that aggregation on the selected subnet
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector)).

Weight-vector vocabulary belongs to input signals. Rank vocabulary belongs to miner-side outcomes
after processing.

## Distinction from Stake Weight

Stake weight names how much validator influence each submitted weight carries in consensus math. A
weight vector names which miners a validator scored, not the stake backing that validator
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector)).

Stake-weight vocabulary belongs to influence scaling. Weight-vector vocabulary belongs to evaluation
targets in one submission.

## Distinction from Weight Setting

Weight setting is the per-validator action that submits one weight vector. The weight vector is the
structured payload that action records on the selected subnet
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Implementation of the Yuma Consensus Epoch](https://docs.learnbittensor.org/navigating-subtensor/epoch)).

Weight-setting vocabulary names the submission act. Weight-vector vocabulary names the evaluation
content inside that submission.

## Distinction from Trust

A weight vector is one validator's submitted evaluation signals, while trust is interpreted after
the full validator set is filtered and aggregated into miner-side outcomes
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust)).

The weight vector is an input one validator submits; trust is a downstream miner-side reading of how
that evaluation survived aggregation.

## Distinction from Validator Trust

A weight vector is one validator's submitted evaluation signals, while validator trust summarizes
that validator's clipped influence after the full weight matrix is filtered
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust)).

Weight-vector vocabulary belongs to the input signals; validator-trust vocabulary belongs to the
validator-side measure of how much of that input survived clipping.

## Distinction from Validator-Miner Bonds

A weight vector is one validator's miner assignments in a single round. Validator-miner bonds are
the smoothed relationships built as those assignments repeat across rounds
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)).

## Distinction from Weights Version Key

The weight vector is the scored targets a validator submits. The weights version key is the
per-subnet hyperparameter that sets the minimum version a `set_weights` call must carry
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

## Distinction from Subnet Scoring Model

A subnet scoring model is the subnet-defined method a validator uses to judge returned miner work. A
weight vector is the structured set of miner scores that method produces before on-chain submission
on the selected subnet
([Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Scoring-model vocabulary names the evaluation rule; weight-vector vocabulary names the per-validator
signal that rule becomes before consensus aggregation.

## Distinction from Yuma Consensus

A weight vector is one validator's structured miner evaluations before subnet consensus runs. Yuma
Consensus aggregates those vectors into emission shares each tempo
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Weight vector** — one validator's miner evaluations before aggregation.
- **Yuma Consensus** — mechanism that turns validator weights into emission shares.

Input vocabulary names one validator signal; Yuma vocabulary names subnet settlement
([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Reader Boundary

Weight vector names per-validator consensus-input vocabulary. It describes one validator's
structured miner-evaluation set before subnet-level matrix and consensus processing, not stake
weight, rank, or a complete reward model
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Specific examples belong to the subnet, mechanism, network, and source that produced the vector.
Validator weights, weight matrices, consensus score, rank, and stake weight describe neighboring
parts of the same consensus process.
