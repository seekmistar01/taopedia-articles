# Burn Cost

Burn cost is the TAO amount associated with creating a new Bittensor subnet. In Bittensor
vocabulary, the term refers to TAO that is recycled when a new subnet is created
([Glossary: Burn Cost](https://docs.learnbittensor.org/resources/glossary#burn-cost)).

The phrase matters because it is narrower than a general fee. Burn cost belongs to the subnet
creation path, not to every transfer, validator action, or user interaction. It gives readers a
specific name for the TAO amount tied to bringing a subnet into existence
([Create a Subnet: Burn cost](https://docs.learnbittensor.org/subnets/create-a-subnet#burn-cost)).

## Subnet Creation Context

Burn cost is dynamic rather than fixed. The subnet creation documentation presents burn cost as a
value that can decrease over time and rise when additional subnets are created, so the durable
concept is the pricing mechanism rather than a quoted number
([Create a Subnet: Burn cost](https://docs.learnbittensor.org/subnets/create-a-subnet#burn-cost)).

That makes burn cost a useful boundary term for readers. The article can explain that the amount is
part of subnet creation, changes with creation demand, and is denominated in TAO without trying to
preserve a value that may already be stale by the time a reader sees it
([BTCLI: subnets burn-cost](https://docs.learnbittensor.org/btcli/#btcli-subnets-burn-cost)).

## Recycling Boundary

The name includes "burn," but Bittensor documentation treats subnet creation cost as recycled TAO.
The local subnet creation guide calls out that the label remains "burn cost" even though the TAO is
technically recycled rather than destroyed
([Local subnet creation: Subnet creation cost](https://docs.learnbittensor.org/local-build/create-subnet#subnet-creation-cost)).

This distinction keeps the term from being overread. Burn cost is the familiar subnet-creation
label; recycling is the accounting treatment for the TAO involved. Readers should not infer from the
word "burn" alone that the amount is permanently removed from all future issuance
([Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)).

## Subnet Funding Context

Burn cost names the amount required by subnet creation, while crowdloans name one community funding
path that can help cover that amount. In a crowdloan, contributors provide TAO toward subnet
creation and later receive TAO distributions based on the subnet's converted alpha emissions
([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans)).

Those concepts answer different questions. Burn cost identifies what subnet creation requires;
crowdloans explain how a group may share the funding burden and later participate in the resulting
subnet economics. Keeping the two separate avoids treating a funding structure as the cost itself
([Glossary: Burn Cost](https://docs.learnbittensor.org/resources/glossary#burn-cost)).

## Network and Development Context

The meaning of burn cost is also affected by environment. Bittensor documentation separates local
development, testnet-style testing, and mainnet use, so a burn cost observation from one environment
should not be treated as evidence for all others
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

Local subnet work can show the creation mechanism in an isolated setting, while shared testing
environments can show more realistic movement without carrying production meaning. The stable
article-level point is that burn cost is environment-bound evidence about subnet creation, not a
single universal number
([Introduction to Bittensor: Subnet development](https://docs.learnbittensor.org/learn/introduction#subnet-development)).

## Supply Flow Context

Burn cost sits on the input side of subnet creation: TAO is required for a new subnet and is treated
as recycled. Coinbase sits elsewhere in the supply flow, as the per-block mechanism involved in TAO
emission and epoch-boundary processing
([Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)).

The contrast is helpful because both ideas involve TAO movement but at different moments. Burn cost
is tied to the creation of a subnet, while coinbase concerns ongoing emission mechanics after the
network is producing blocks and distributing rewards
([Glossary: Burn Cost](https://docs.learnbittensor.org/resources/glossary#burn-cost)).

## Reader Boundary

Burn cost should be read as a conceptual and accounting term, not as timing advice. A reader can use
it to understand why subnet creation has a TAO requirement and why the amount may move, but the
article should not recommend when to create a subnet or repeat a live interface value
([BTCLI: subnets burn-cost](https://docs.learnbittensor.org/btcli/#btcli-subnets-burn-cost)).

This boundary also keeps the article source-stable. The
[Create a Subnet: Burn cost](https://docs.learnbittensor.org/subnets/create-a-subnet#burn-cost)
reference can explain the concept and where it appears, while live values belong in tools that query
the relevant environment at the time of use
([Create a Subnet: Burn cost](https://docs.learnbittensor.org/subnets/create-a-subnet#burn-cost)).

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For burn cost,
that sequence changes how readers should interpret examples of subnet creation costs and TAO
recycling amounts.

In localnet, burn cost mechanics can be tested in an isolated environment. Localnet subnet creation
costs reflect local chain settings and do not represent production subnet creation costs.

On testnet, burn cost can be observed in a shared, non-production network. Testnet burn cost amounts
are separate from mainnet subnet creation state.

On mainnet, burn cost is the live dynamic TAO amount recycled when creating a new subnet in
production Bittensor operations. The
[Create a Subnet: Burn cost](https://docs.learnbittensor.org/subnets/create-a-subnet#burn-cost)
documentation describes the burn cost mechanics that apply on the production network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A burn cost figure from one environment should not be read
as representing subnet creation costs in another environment.
