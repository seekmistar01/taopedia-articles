# VPermit

VPermit is a Bittensor validation term for the subnets where a delegate is authorized to validate.
The [Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit) defines VPermit
as a list of subnet IDs that indicate which subnets a delegate is authorized to validate on.

## Distinction from Validator Permit

A validator permit and a VPermit differ in scope. The glossary describes a validator permit as the
per-subnet permission flag awarded to the top neurons by stake weight inside one subnet, while a
VPermit is the delegate-level list of subnet IDs that aggregates those individual permits across
subnets
([Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit),
[Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit)).

- **Validator permit** — the validation-rights flag inside one subnet.
- **VPermit** — the delegate-level list of subnets where those permits apply.

## Distinction from Delegate

A delegate and a VPermit name an entity and one of its attributes. The glossary describes a delegate
as a validator that can receive nominated stake, while a VPermit is the list of subnet IDs that
delegate is authorized to validate on
([Glossary: Delegate](https://docs.learnbittensor.org/resources/glossary#delegate),
[Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit)).

- **Delegate** — the validator entity that can receive nominated stake.
- **VPermit** — the per-delegate list of subnets it is authorized to validate on.

## Distinction from Stake Weight

Individual validator permits follow stake-weight ranking inside each subnet. VPermit summarizes
which subnets carry validation authorization for a delegate
([Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

- **VPermit** — delegate-level list of authorized subnet IDs.
- **Stake weight** — influence ranking inside one subnet that can affect permit eligibility.

Cross-subnet summary vocabulary and per-subnet influence vocabulary answer different scope questions
([Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit)).

## Distinction from Yuma Consensus

VPermit is the delegate-level list of subnet IDs where a hotkey is authorized to validate. It names
cross-subnet authorization scope in wallet-facing views, not the settlement step that turns weight
submissions into emissions on any one netuid
([Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit),
[Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit)).

Yuma Consensus is the on-chain mechanism that runs at the epoch boundary inside each active subnet.
It reads the weight matrix from eligible validators, applies clipping and bonding, and converts the
result into incentive and dividend shares
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Earlier sections separate VPermit from delegate identity and from per-subnet stake weight. A subnet
appearing on a VPermit list means validation may occur there; Yuma still settles only the weights
that pass eligibility and consensus rules on that netuid.

Readers should treat VPermit as a scope summary across subnets, and Yuma as the per-subnet mechanism
that allocates rewards from accumulated pools
([Yuma Consensus: Validator emissions](https://docs.learnbittensor.org/learn/yuma-consensus#validator-emissions)).

- **VPermit** — delegate-level list of subnet IDs authorized for validation.
- **Yuma Consensus** — epoch-boundary settlement from eligible validator weights to emission shares.

## Distinction from Coinbase

VPermit records where a delegate is authorized to validate across subnets. Coinbase is the per-block
emission mechanism that drives TAO injection and triggers Yuma epochs
([Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit),
[Coinbase documentation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)).

- **VPermit** — delegate-level authorization across subnet IDs.
- **Coinbase** — per-block emission mechanism on active subnets.

Authorization vocabulary names validation scope; coinbase vocabulary names emission cadence
([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Distinction from Child Hotkeys

VPermit is the delegate-level list of subnet IDs a delegate is authorized to validate on. A child
hotkey is a separate authorized key that validates on a parent's behalf using routed stake
([Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit),
[Child Hotkeys](https://docs.learnbittensor.org/validators/child-hotkeys)).

- **VPermit** — which netuids a delegate may validate on.
- **Child hotkey** — key that carries out validation using routed stake.

Authorization vocabulary names permitted subnets; child-hotkey vocabulary names delegated execution
([Glossary: Delegate](https://docs.learnbittensor.org/resources/glossary#delegate)).

## Distinction from UID Slot

VPermit and a UID slot identify validation context at different levels: the glossary describes
VPermit as the list of subnet IDs a delegate is authorized to validate on, while a UID slot is the
registration position a neuron occupies inside a single subnet
([Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit),
[Glossary: UID slot](https://docs.learnbittensor.org/resources/glossary#uid-slot)).

- **VPermit** — the set of subnets where a delegate may validate.
- **UID slot** — the registration position a neuron holds within one subnet.

## Reader Boundary

This article defines the term at a high level. It does not list live permissions, rank validators,
describe staking or registration steps, or restate validator requirements. Current permission lists
belong to the relevant delegate, network, and tooling at the time of observation
([Glossary: VPermit](https://docs.learnbittensor.org/resources/glossary#vpermit)).
