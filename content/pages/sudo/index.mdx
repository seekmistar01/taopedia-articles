# Sudo

Sudo is Bittensor governance vocabulary for privileged administrative authority. In Bittensor's
governance model, the term points to an older single-key authority pattern that governance replaces
with proposal review ([Glossary: Sudo](https://docs.learnbittensor.org/resources/glossary#sudo),
[Governance Overview](https://docs.learnbittensor.org/governance)).

## What It Represents

Sudo names the legacy single-key pattern that could enact privileged changes unilaterally. Bittensor
governance replaces that model with a staged review path: the Triumvirate originates proposals, the
Senate supplies an approval gate, and privileged calldata runs only after both steps complete
([Governance Overview](https://docs.learnbittensor.org/governance),
[Glossary: Triumvirate](https://docs.learnbittensor.org/resources/glossary#triumvirate),
[Senate](https://docs.learnbittensor.org/governance/senate)).

The governance security section contrasts the old sudo private-key model with distributed review. A
malicious actor would need to compromise a Triumvirate member and control a Senate majority rather
than rely on one privileged key alone
([Governance Overview: security](https://docs.learnbittensor.org/governance#security)).

Sudo examples depend on which network you connect to. Localnet, testnet, and mainnet governance
activity should not be read as interchangeable
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from Triumvirate

Sudo names unilateral administrative authority. The Triumvirate is the proposal-origin body in the
governance model that replaced it
([Glossary: Sudo](https://docs.learnbittensor.org/resources/glossary#sudo),
[Glossary: Triumvirate](https://docs.learnbittensor.org/resources/glossary#triumvirate)).

- **Sudo** — the legacy single-key authority that could act unilaterally.
- **Triumvirate** — the proposal-origin body in the governance model that replaced it.

## Distinction from Senate

Sudo names legacy single-key control. The Senate is the elected body that votes to approve or reject
proposals in the governance model that replaced it
([Glossary: Sudo](https://docs.learnbittensor.org/resources/glossary#sudo),
[Senate](https://docs.learnbittensor.org/governance/senate)).

- **Sudo** — the legacy single-key authority that acted unilaterally.
- **Senate** — the elected body that votes on proposals in its place.

## Distinction from Bicameral Legislature

Sudo names the legacy single-authority model. A bicameral legislature is the two-body structure that
separates proposal creation from proposal approval
([Glossary: Sudo](https://docs.learnbittensor.org/resources/glossary#sudo),
[Glossary: Bicameral Legislature](https://docs.learnbittensor.org/resources/glossary#bicameral-legislature)).

- **Sudo** — the legacy single-authority model being moved away from.
- **Bicameral legislature** — the two-body structure that replaced it.

## Distinction from Proposal

Sudo names the legacy administrative pattern governance replaced. A proposal is the governance item
senators review and the Triumvirate closes before privileged calldata runs
([Glossary: Sudo](https://docs.learnbittensor.org/resources/glossary#sudo),
[Glossary: Proposal](https://docs.learnbittensor.org/resources/glossary#proposal)).

- **Sudo** — legacy single-key administrative authority pattern.
- **Proposal** — governance item requiring Senate majority and Triumvirate close.

## Distinction from Yuma Consensus

Sudo names elevated administrative control for protocol maintenance on Subtensor. Yuma Consensus is
the epoch-boundary mechanism that converts eligible validator weight submissions into emission
shares on a subnet ([Glossary: Sudo](https://docs.learnbittensor.org/resources/glossary#sudo),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Sudo** — elevated administrative control for protocol maintenance.
- **Yuma Consensus** — epoch-boundary settlement from validator weights to emission shares.

## Reader Boundary

Sudo should be read as historical authority vocabulary inside Bittensor governance. It does not
identify an administrative key holder, expose key material, or provide instructions for privileged
actions ([Glossary: Sudo](https://docs.learnbittensor.org/resources/glossary#sudo)).

When the focus is proposal creation, Triumvirate is more precise. When the focus is voting and
approval, Senate is more precise. When the focus is the item under review, proposal is more precise.
