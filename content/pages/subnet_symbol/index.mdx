# Subnet Symbol

In Bittensor, the network currency and each subnet's token are written with symbols. Official
documentation denotes TAO with the tau sign and refers to a subnet's own token as its alpha. A
subnet symbol is the mark used to show which token a quantity belongs to.

## TAO and Alpha

The documentation presents TAO as the currency of the Bittensor network, written with the Greek
letter tau. Each subnet then has its own dynamic token, and in the abstract the documentation refers
to any subnet's token as that subnet's alpha. So tau marks the shared network token, while alpha is
the general name for a per-subnet token.

## Per-Subnet Symbols in Tooling

Beyond the abstract alpha label, interfaces show each subnet with its own symbol next to its name.
The documentation's command-line examples list subnets such as one shown with an alpha symbol,
another with a gamma symbol, and another with an iota symbol, each paired with a subnet name. The
symbol there identifies which subnet's token a row refers to rather than standing for a generic
alpha.

In pool and balance views, that symbol marks which subnet's
[alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha) liquidity is being quoted
beside the subnet name and [netuid](https://docs.learnbittensor.org/resources/glossary#netuid), so a
reader can see whether a quantity refers to one subnet's token or another's before reading price,
pool depth, or swap direction
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Why Symbols Matter

Because many subnets each issue their own token, a quantity of alpha is ambiguous without knowing
which subnet it came from. A symbol, read together with the subnet's netuid and name, ties a balance
or price to a specific subnet's token and keeps it separate both from other subnets' tokens and from
TAO. This is the practical role of the symbol: disambiguation, not a new asset type.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For Subnet Symbol,
that sequence changes how an example should be read, because the surrounding network state differs
at each stage.

In localnet, subnet Symbol can be exercised in an isolated development environment, where the
surrounding chain state reflects local configuration rather than production history.

On testnet, subnet Symbol can be observed in a shared, non-production network whose state is kept
separate from mainnet.

On mainnet, subnet Symbol applies on the live, production Bittensor network, where the surrounding
state is real and persistent.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A subnet Symbol example from one environment should not be
read as representing another environment.

## The Metagraph Exposes Symbol Beside Subnet Name

[The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph) lists `symbol` as the
subnet token symbol and `name` as the subnet name among its snapshot fields. The symbol therefore
travels with the subnet's identity data rather than being a free-floating label, and it appears next
to the subnet name in metagraph-derived views.

That placement reinforces the disambiguation role. When a metagraph snapshot reports stake or
emission values in a subnet's alpha token, the `symbol` field marks which subnet's token those
quantities belong to alongside its `netuid` and `name`
([The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph)).

## Distinction from Alpha Tokens

A subnet symbol and a subnet's alpha tokens are the label and the asset it labels. Each subnet has
its own alpha token under Dynamic TAO, and the subnet symbol is the written character used to denote
that token in interfaces. The symbol is how the token is shown; the alpha token is the actual subnet
currency it stands for
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha)).

Official
[Managing Your Stakes](https://docs.learnbittensor.org/staking-and-delegation/managing-stake-sdk)
guidance discusses staking and unstaking in terms of both network
[TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau), written with the tau sign, and
subnet-specific alpha balances. Interface examples therefore keep TAO rows distinct from rows that
show a particular subnet's token symbol.

That separation matters when a reader compares portfolio views. TAO names the shared network token,
while a subnet symbol names which subnet market a staked or swapped alpha position belongs to. The
two labels should not be read as interchangeable even when both appear in one wallet screen.

## Distinction from Netuid

A subnet symbol and a netuid are two ways to identify a subnet. The subnet symbol is the display
character shown beside a token in interfaces, while the netuid is the authoritative numeric
identifier used for a subnet's chain state and hyperparameters
([The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph),
[Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)).

- **Subnet symbol** — the human-facing display character for a subnet's token.
- **Netuid** — the authoritative numeric subnet identifier.

A subnet symbol is a display convenience, not a substitute for
[netuid](https://docs.learnbittensor.org/resources/glossary#netuid) in precise reading. Official
subnet documentation uses netuid as the numeric subnet identifier on
[Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor), while symbols such as
gamma or iota appear in examples to make human-facing tables easier to scan.

When two subnets could use similar names in conversation, netuid and symbol should be read together
with the subnet name. The symbol disambiguates token rows in tooling; netuid disambiguates which
subnet's chain state and hyperparameters are in scope.

## Distinction from TAO

A subnet symbol and TAO name a per-subnet token's mark and the network token. A subnet symbol marks
which subnet's alpha token a quantity belongs to, while TAO is the shared network currency written
with the tau sign
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau)).

- **Subnet symbol** — the mark for a specific subnet's alpha token.
- **TAO** — the shared network token written with the tau sign.

## Distinction from Yuma Consensus

Subnet symbol is the display character for a subnet's alpha token in wallets, explorers, and other
interfaces. It names how one netuid's alpha is shown to readers, not the recurring settlement step
that converts validator weight submissions into emission shares
([Glossary: Subnet Symbol](https://docs.learnbittensor.org/resources/glossary#subnet-symbol),
[The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph)).

Yuma Consensus is the on-chain mechanism that runs at the epoch boundary on each subnet. It reads
the weight matrix from eligible validators, applies clipping and bonding, and converts the result
into miner incentives and validator dividend shares
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Earlier sections separate subnet symbol from TAO and from netuid identifiers. A symbol helps users
recognize one subnet's alpha in interfaces; Yuma settles evaluation signals from eligible validators
on that netuid—not the display glyph itself
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Readers should treat subnet symbol as interface labeling for one alpha token, and Yuma as the
epoch-boundary mechanism that allocates subnet rewards from consensus while validators remain
active.

- **Subnet symbol** — display mark for a subnet's alpha token.
- **Yuma Consensus** — epoch-boundary settlement from eligible validator weights to emission shares.

## Reader Boundary

This page explains the role of token symbols at a high level. It does not list which symbol is shown
for any particular subnet or claim a fixed registry of symbol-to-netuid mappings, since how a token
is displayed is an interface and chain-state detail. For the symbol of a specific subnet, such as
netuid 1, check current Bittensor tooling
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).
