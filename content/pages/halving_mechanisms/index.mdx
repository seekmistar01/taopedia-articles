# Halving Mechanisms

Halving mechanisms reduce token emission rates after supply thresholds are reached. In Bittensor,
halving vocabulary applies to both network-level TAO emission and subnet-level alpha-token emission
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

The important reader boundary is scope. A halving claim should name the asset and context it is
about, because TAO and subnet alpha tokens sit in different parts of Bittensor tokenomics
([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Scope Context

TAO halving is network-scoped, while alpha halving is subnet-scoped. Both use the broad idea of an
emission-rate reduction after a supply threshold, but the asset and scope are different
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

That difference prevents one halving statement from being generalized too far. A subnet alpha-token
halving is not the same claim as a network TAO halving.

## TAO Halving

TAO halving concerns the network token. It reduces TAO emission after the relevant TAO supply
threshold is reached ([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

For readers, the useful point is asset scope. TAO halving belongs to network-token emission, not to
the full reward logic inside every subnet.

## Alpha Halving

Alpha halving concerns subnet alpha tokens. It reduces a subnet alpha token's emission rate after
the relevant subnet alpha supply threshold is reached
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

This makes alpha halving a subnet-token concept. It shares the halving pattern with TAO halving, but
it should remain attached to subnet-specific tokenomics.

## Emission Relationship

Halving changes emission rate; emissions describe the broader creation and distribution path for
reward material. The [Emission](https://docs.learnbittensor.org/learn/emissions) reference places
TAO and subnet alpha tokens inside Bittensor's reward-distribution context.

The distinction matters because halving is not the whole reward system. It names a supply-rate
change, while subnet outcomes still depend on consensus and incentive context
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Dynamic TAO Context

Dynamic TAO gives subnet alpha tokens their own tokenomics context alongside TAO. That is why
halving mechanism articles need to keep TAO emission, alpha emission, and subnet liquidity context
separate ([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage)).

For readers, these are related tokenomics ideas, not interchangeable claims. A halving statement
answers an emission-rate question, not every question about subnet token behavior.

## Reader Boundary

Halving mechanisms should be read as supply-threshold rules for emission-rate reduction
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)). The stable point is that
Bittensor uses halving vocabulary at both the TAO scope and the subnet alpha-token scope.

This article does not provide date forecasts, reward forecasts, market interpretation, staking
instructions, or point-in-time supply reports. Exact observed values belong to the specific source
and context where they are measured.
