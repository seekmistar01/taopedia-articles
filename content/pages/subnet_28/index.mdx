lists contact, website, and description metadata.

Readers should treat this page as a current identity record for the subnet metadata exposed on

older metadata statements.

## Current Status

The useful public fact about Subnet 28 is its registered identity state. Active subnet articles can
usually cite a project repository or documentation that explains how miners produce work and how
validators score it. Subnet 28's current identity provides lightweight project metadata through its
contact, URL, and description fields.

If the operator later updates the identity fields or publishes documentation, the subnet may need a

Live SN28 data is available on [TaoStats](https://taostats.io/subnets/28). Readers can reproduce the

## Netuid 28 Identifies the Subnet On-Chain

Bittensor assigns every subnet a unique numeric identifier called a netuid, and Subnet 28 is the
subnet registered at netuid 28
([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)). The
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) reference
explains that each subnet runs its own incentive mechanism while sharing the same underlying
Subtensor chain, so the netuid is the stable handle that distinguishes gm from every other subnet.

For a reader, this means "Subnet 28" and "netuid 28" refer to the same on-chain slot. A claim about
gm should be tied to that netuid rather than to the registered name alone, because the name field
can be changed on-chain while the netuid stays fixed.

## Miner and Validator Roles

Subnet 28 operates under the standard Bittensor two-role structure. Miners supply the subnet's
capability and validators evaluate those contributions and set weights. Reward distribution follows
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus), whose consensus weights then
help determine per-tempo [emissions](https://docs.learnbittensor.org/learn/emissions).

## Distinction from Yuma Consensus

gm on netuid 28 is registered with lightweight identity metadata and no published mechanism
documentation. Yuma Consensus is the separate mechanism that converts validator weight submissions
into emission shares each tempo
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

- **Subnet 28 (gm)** — identity-record subnet with say-gm description.
- **Yuma Consensus** — mechanism that turns validator weights into emission shares.
