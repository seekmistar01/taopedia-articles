# Bittensor Emissions

Bittensor emissions describe how newly created TAO and subnet-specific alpha tokens enter reward and
subnet tokenomics contexts ([Emission](https://docs.learnbittensor.org/learn/emissions)).

The topic includes both the network asset and subnet alpha tokens, so an emissions claim should name
which asset and scope it is describing
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Asset Scope

Emission is not only a TAO release schedule. Bittensor emissions include TAO and the alpha token
associated with each subnet ([Emission](https://docs.learnbittensor.org/learn/emissions)).

That distinction matters because TAO belongs to the network-wide side of tokenomics, while alpha
tokens belong to subnet-specific contexts
([Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

## Injection and Distribution

Emissions can be read in two broad stages. Injection describes how newly created assets enter subnet
contexts, while distribution describes how rewards are allocated after that context exists
([Emission](https://docs.learnbittensor.org/learn/emissions)).

Keeping those stages separate prevents every emissions statement from being reduced to one
network-level number or one subnet reward outcome
([Flow-Based Emissions](https://docs.learnbittensor.org/learn/emissions#flow-based-emissions)).

## Subnet Role Context

Inside a subnet, miners produce work and validators evaluate that work. Emissions make those roles
economically meaningful by connecting useful work and evaluation to reward outcomes
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Yuma Consensus explains how validator evaluation is aggregated into miner incentives and
validator-side outcomes ([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Staking Context

Staking connects token holders to validator reward outcomes. Emissions can therefore be discussed
with staking, but the article-level term still refers to the broader token distribution process
([Emission](https://docs.learnbittensor.org/learn/emissions)).

For readers, this means staking outcomes should be read as one part of emissions, not as the whole
emission system
([Staking and Delegation](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

## Halving and Flow Context

Halving changes emission rates, while flow-based emissions describe one way subnet-level injection
is compared. Both ideas sit inside the broader emissions vocabulary
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

The distinction is useful: halving explains supply-rate reductions, flow-based emissions explain
subnet injection comparison, and Bittensor emissions names the broader distribution topic
([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Reader Boundary

Bittensor emissions should be read as a tokenomics and reward-distribution concept. It is not a
single TAO-only number, a validator recommendation, or a staking guide
([Emission](https://docs.learnbittensor.org/learn/emissions)).

The stable article-level point is that emissions include newly created TAO and subnet alpha tokens,
then connect those assets to subnet roles and reward outcomes
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For Bittensor
emissions, that sequence changes how readers should interpret examples of TAO and alpha token reward
flows and emission outcomes.

In localnet, emission mechanics can be tested in an isolated environment. Localnet emission outcomes
and reward flows reflect local chain settings and do not represent production network emission
state.

On testnet, emission behavior can be observed in a shared, non-production network. Testnet emission
outcomes are separate from mainnet subnet reward state.

On mainnet, Bittensor emissions are the live reward flows that create TAO and subnet alpha tokens
and distribute them to subnet roles in production Bittensor operations. The
[Emission](https://docs.learnbittensor.org/learn/emissions) documentation describes the emission
mechanics that apply on the production network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. An emission example from one environment should not be
read as representing reward flows or emission outcomes in another environment.
