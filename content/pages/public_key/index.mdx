# Public Key

A public key is the shareable side of a cryptographic keypair in Bittensor wallet-key terminology.
The [Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key) describes
it as publicly available key material used for verifying signatures, encrypting messages, and
identifying accounts in the Bittensor network.

## Signature Verification Role

The [Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key) describes
public key material as used for verifying signatures on Bittensor network activity. That
verification role is separate from private-key authorization: the public side confirms which key
pair an account reference belongs to, while the private side actually signs submissions.

In Bittensor, a public key is used for verification and identification; it is not a recovery secret
and does not authorize signing. The matching private key is the protected signing material
([Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key),
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key)).

## Identification Without Authorization

Public key material can identify an account reference, but it does not by itself authorize
transfers, staking actions, or subnet registrations. The
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key) describes
the protected component used to authorize transactions and operations on the network.

That split helps readers keep shareable reference material separate from protected signing material
when reading wallet-security documentation alongside key vocabulary.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. The public key
concept applies across the Bittensor lifecycle: key pairs can be generated and used for account
identification in localnet for isolated development, testnet for shared non-production
participation, and mainnet for live operation with real emissions.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. Public key examples or account reference outcomes from one
environment should not be read as representing production subnet participation in another
environment.

## Distinction from Wallet Address

A wallet address is the public destination identifier derived from public-key material. Public key
names the shareable cryptographic pair side itself
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address),
[Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key)).

Address vocabulary names where TAO can be sent; public key vocabulary names the shareable key
material behind that reference.

The [Working with Keys](https://docs.learnbittensor.org/keys/working-with-keys) guide shows that a
coldkey's `coldkeypub.txt` file stores the `accountId`, `publicKey`, and `ss58Address` while leaving
the private fields null. The `ss58Address` is the SS58 representation of that public key and serves
as the coldkey's public address for receiving TAO.

That layout matches the public-key role described here: the shareable side of the pair is what gets
written to the public keyfile and shared as a destination, while the protected private material is
kept out of it. The [Wallets, Coldkeys and Hotkeys](https://docs.learnbittensor.org/keys/wallets)
documentation adds that the public key is mathematically derived from the private key, so the
shareable side follows from the protected side rather than the reverse.

## Distinction from SS58 Encoded

SS58 encoded names the compact display format for public-key-derived addresses. Public key names the
shareable keypair side that those addresses represent
([Glossary: SS58 Encoded](https://docs.learnbittensor.org/resources/glossary#ss58-encoded),
[Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key)).

Encoding-format vocabulary and keypair-side vocabulary sit on adjacent but different layers.

## Distinction from Private Key

Public key is the shareable pair side used for verification and account reference. Private key is
the protected side that authorizes signing
([Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key),
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key)).

Shareable reference material and protected signing material must stay in separate handling
categories.

## Distinction from Coldkeys

Public key names shareable keypair material. Coldkey names the custody role that controls TAO and
hotkeys under Bittensor wallet structure
([Glossary: Coldkey](https://docs.learnbittensor.org/resources/glossary#coldkey),
[Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key)).

Keypair-side vocabulary is structural; coldkey vocabulary names a specific wallet authority role
built on protected key material.

## Distinction from Hotkeys

Public key names the shareable half of a cryptographic pair. Hotkey names the operational wallet
role used for routine subnet actions
([Glossary: Hotkey](https://docs.learnbittensor.org/resources/glossary#hotkey),
[Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key)).

Reference vocabulary names shareable material; hotkey vocabulary names the operational signing role
derived under coldkey control.

## Distinction from Address Poisoning Scams

Public key material produces the wallet addresses users recognize. Address poisoning names the scam
pattern where a lookalike destination is mistaken for the intended one
([Bittensor address-poisoning guide](https://docs.learnbittensor.org/keys/address-poisoning-scams),
[Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key)).

Key-reference vocabulary and social-engineering risk vocabulary answer different safety questions.

## Distinction from EdDSA Cryptographic Keypairs

EdDSA cryptographic keypairs name the key format used by Bittensor wallets. Public key names the
shareable side of that pair
([Glossary: EdDSA Cryptographic Keypairs](https://docs.learnbittensor.org/resources/glossary#eddsa-cryptographic-keypairs),
[Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key)).

Format vocabulary describes the pair structure; public key vocabulary names one side of it.

Those pairs connect to coldkeys and hotkeys, which are the operational wallet roles built on top of
the underlying keypair vocabulary rather than replacements for it
([Glossary: EdDSA Cryptographic Keypairs](https://docs.learnbittensor.org/resources/glossary#eddsa-cryptographic-keypairs)).

## Distinction from Extrinsics

Extrinsics and the public key are related but different parts of Bittensor chain action vocabulary.
Extrinsics name signed transactions and calls submitted to Subtensor, while the public key is the
shareable key material used to identify accounts and verify the signatures on those submissions
([Subtensor Extrinsics](https://docs.learnbittensor.org/subtensor-api/extrinsics),
[Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key)).

- **Extrinsic** — the submitted action sent to Subtensor.
- **Public key** — the shareable account reference that signs and identifies that submission.

## Distinction from Yuma Consensus

A public key is shareable account material used to identify accounts and verify signatures. Yuma
Consensus is the on-chain mechanism that converts validator weight submissions into emission shares
each tempo and is not key-derivation vocabulary
([Public Key](https://docs.learnbittensor.org/resources/glossary#public-key),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Public-key vocabulary belongs to wallet identity and signature verification. Yuma Consensus
vocabulary belongs to validator weight settlement on a subnet.

- **Public key** — shareable account identifier derived from key material.
- **Yuma Consensus** — on-chain mechanism that turns validator weights into emission shares.

## Distinction from Keypair

Public keys are read together with private keys. The public side can be shared for account reference
or verification, while the
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key) describes
the private side as the protected component used to authorize transactions and operations.

This distinction keeps public key from being treated as a secret. It is still part of the same
cryptographic keypair vocabulary, but it has a different role from private signing material.

## Distinction from Wallet-Key

In Bittensor, public keys belong to wallet-key terminology rather than naming a separate network
role or token. The
[Glossary: EdDSA Cryptographic Keypairs](https://docs.learnbittensor.org/resources/glossary#eddsa-cryptographic-keypairs)
connects public and private key pairs with coldkeys and hotkeys, which are Bittensor wallet-key
roles.

For readers, the useful point is structural: a public key is part of the wallet-key system used to
identify and verify, not a staking position, subnet role, or balance.

## Reader Boundary

Public key should not be read as a recovery phrase, a private key, or a complete wallet by itself.
It is a shareable public component used in identification and verification context. This article
does not describe how to create, store, recover, rotate, or use wallet keys.

## Distinction from Seed Phrase Security

A public key and seed phrase security sit at opposite ends of key handling. A public key is the
shareable half of a keypair used to verify signatures and derive a wallet address, while seed phrase
security is the set of practices for protecting the secret mnemonic from which the entire keypair —
public and private — is derived
([Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key),
[Handle your Seed Phrase/Mnemonic Securely](https://docs.learnbittensor.org/keys/handle-seed-phrase),
[Glossary: Mnemonic](https://docs.learnbittensor.org/resources/glossary#mnemonic)).

- **Public key** — the shareable half of a keypair, safe to distribute.
- **Seed phrase security** — practices protecting the secret mnemonic behind the keypair.
