# Wallet Address

A wallet address is the public-facing identifier used as a TAO destination in Bittensor. The
official
[Wallet Address glossary](https://docs.learnbittensor.org/resources/glossary#wallet-address)
describes it as a unique identifier derived from a public key and used for sending and receiving TAO
tokens.

## Address Context

A wallet address is address material, not private signing material. It identifies where TAO can be
sent within Bittensor, while the protected key material that authorizes account activity is a
separate wallet-key concept.

## Signing Authority Boundary

A wallet address can be shared as a destination identifier, but it does not authorize movement from
that address. Authorization belongs to protected private-key material, while public-key-derived
address material supports account reference and receiving context
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address),
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key)).

## SS58 Address Format

Bittensor wallet addresses use SS58 encoding as the compact public-facing representation derived
from a public key. The
[Glossary: SS58 Encoded](https://docs.learnbittensor.org/resources/glossary#ss58-encoded) describes
SS58 as the encoding format for those addresses, while the
[Wallet Address glossary](https://docs.learnbittensor.org/resources/glossary#wallet-address) defines
the address as the TAO destination identifier derived from that key material.

## Transfer Destinations Use Wallet Addresses

The [Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer) describes
transfer as sending TAO tokens between wallet addresses. A wallet address is therefore the
destination identifier a sender targets when moving TAO on Subtensor.

Readers should read wallet address as the receiving-side label in transfer vocabulary. The transfer
action names the movement; the wallet address names where the TAO is sent.

## Sharing an Address to Receive TAO

A wallet address can be shared so someone else knows where to send TAO. The
[Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key) describes
public keys as shareable material used for account reference, and wallet addresses are derived from
that public-side material for destination use.

Sharing an address is not the same as sharing signing authority. The
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key) remains the
protected material needed to authorize outbound actions from the account.

## Transfers Require an Exact Destination String

The [Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer) describes
moving TAO between wallet addresses. Transfer vocabulary therefore depends on a precise destination
identifier rather than on a similar-looking prefix alone
([Address Poisoning Scams](https://docs.learnbittensor.org/keys/address-poisoning-scams)).

For readers, sharing or quoting an address is not the same as completing a transfer. Submission
still requires the sender to specify the exact destination and authorize the move.

## Public Address Material Is Not Signing Authority

Wallet documentation keeps public-side address material separate from protected private signing
material. A wallet address can identify where TAO should be sent without exposing authorization to
move funds from that account
([Wallets, Coldkeys and Hotkeys](https://docs.learnbittensor.org/keys/wallets),
[Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key)).

Readers should not infer custody or spend authority from an address string alone.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. The wallet address
concept applies across the Bittensor lifecycle: wallet addresses can be generated and used in
localnet for isolated development, testnet for shared non-production participation, and mainnet for
live operation with real emissions. Each network environment produces distinct address contexts.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. Wallet address examples or account-reference outcomes from
one environment should not be read as representing production subnet participation in another
environment.

## Distinction from External Wallet

A wallet address names a public TAO destination, while an external wallet names where custody or
signing control is managed
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address),
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)).

The address is shareable routing information; the external wallet names where the keys that
authorize spending from that address are held.

## Distinction from Seed Phrase

A wallet address is a public destination identifier, while a seed phrase is secret recovery material
that must never be shared like an address
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address),
[Glossary: Seed Phrase](https://docs.learnbittensor.org/resources/glossary#seed-phrase)).

Sharing an address only lets others send TAO; sharing a seed phrase hands over control of the wallet
the address belongs to.

## Distinction from Wallet Location

A wallet address names a public TAO destination, while a wallet location names a local filesystem
path where wallet files are stored
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address),
[Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location)).

The address identifies where value is sent on-chain; the location identifies where the wallet's
files sit on a machine.

## Distinction from Address Poisoning Scams

Wallet addresses are the destination identifiers that address-poisoning scams target. The
[Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address)
describes a wallet address as a unique identifier derived from a public key and used for sending and
receiving TAO tokens, and the
[Bittensor address-poisoning guide](https://docs.learnbittensor.org/keys/address-poisoning-scams)
warns that similar-looking addresses can appear familiar during routine wallet use.

A wallet address names the destination identifier itself, while address poisoning names the risk
that a lookalike identifier gets treated as the intended destination. The address stays
cryptographically valid; the harm comes from mistaking one valid address for another.

- **Wallet address** — the public TAO destination identifier derived from key material.
- **Address poisoning scam** — the risk of treating a lookalike address as the intended destination.

## Distinction from Public Key

A public key is shareable cryptographic key material used for verification and account reference,
while a wallet address is the public-key-derived identifier used as a TAO destination
([Glossary: Public Key](https://docs.learnbittensor.org/resources/glossary#public-key),
[Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address)).

Key-material vocabulary and destination-identifier vocabulary answer different questions. Public key
names the shareable keypair material; wallet address names the destination identifier derived from
that material.

- **Public key** — shareable key material used for verification and account reference.
- **Wallet address** — the TAO destination identifier derived from that material.

## Distinction from SS58 Encoded

A wallet address is the TAO destination identifier derived from public-key material, while SS58
encoded names the compact representation format used for those addresses
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address),
[Glossary: SS58 Encoded](https://docs.learnbittensor.org/resources/glossary#ss58-encoded)).

Destination vocabulary and display-format vocabulary answer different questions. Wallet address
names the destination concept; SS58 encoded names the compact address format used to display it.

- **Wallet address** — the TAO destination identifier concept.
- **SS58 encoded** — the compact display format for public-key-derived addresses.

That SS58 format is separate from Ethereum-style H160 addresses used on the Bittensor EVM side, so a
wallet address in native Bittensor prose usually means the SS58 destination derived from a Subtensor
public key rather than an EVM account label
([Glossary: SS58 Encoded](https://docs.learnbittensor.org/resources/glossary#ss58-encoded),
[Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address)).

## Distinction from TAO

TAO is the network incentive token, while a wallet address identifies where TAO can be sent within
the network ([Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau),
[Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address)).

Asset vocabulary and destination-identifier vocabulary answer different questions. TAO names the
asset being moved; wallet address names the destination identifier for that movement.

- **TAO** — the network incentive token being transferred.
- **Wallet address** — the destination identifier where TAO is sent or received.

## Distinction from Transfer

A transfer names the act of sending TAO between wallet addresses, while a wallet address names the
destination identifier used in that movement
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer),
[Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address)).

Action vocabulary and destination-identifier vocabulary answer different questions. Transfer names
the movement action; wallet address names the destination identifier that action requires.

- **Transfer** — the action of sending TAO between wallet addresses.
- **Wallet address** — the destination identifier used in that movement.

## Distinction from Coldkeys

A coldkey and a wallet address name a signing key and its public identifier. The coldkey is the
security-critical key that controls TAO and stake, while the wallet address is the coldkey's
public-key-derived identifier used to receive TAO
([Glossary: Coldkey](https://docs.learnbittensor.org/resources/glossary#coldkey),
[Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address)).

- **Coldkey** — the security-critical signing key controlling TAO and stake.
- **Wallet address** — the coldkey's public identifier used to receive TAO.

## Distinction from Hotkey

A hotkey and a wallet address name an operational signing key and a public identifier. The hotkey is
the key used for frequent network operations such as validation and mining, while the wallet address
is the public-key-derived identifier used to receive TAO
([Glossary: Hotkey](https://docs.learnbittensor.org/resources/glossary#hotkey),
[Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address)).

- **Hotkey** — the signing key used for frequent network operations.
- **Wallet address** — the public-key-derived identifier used to receive TAO.

## Distinction from Yuma Consensus

A wallet address is the public-facing identifier used as a TAO destination in Bittensor. It names
address material derived from a public key, not the on-chain step that turns validator weight
submissions into emission shares
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address)).

Yuma Consensus is the on-chain process that converts submitted validator weights into emission
shares after evaluation completes. Wallet addresses identify where TAO is sent; Yuma Consensus
settles scoring outcomes on a subnet
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Readers should keep destination-address vocabulary separate from the settlement step. Wallet address
answers which public identifier receives TAO; Yuma Consensus answers how validator weights become
emission shares in a subnet
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Wallet address** — public identifier used as a TAO destination.
- **Yuma Consensus** — on-chain weight aggregation that settles emissions.

## Distinction from Public-Key

The public-key relationship matters because the glossary defines wallet addresses as derived from
public keys. The
[Public Key glossary](https://docs.learnbittensor.org/resources/glossary#public-key) describes
public keys as shareable material used for verification, encryption, and account reference.

## Distinction from Wallet

Bittensor wallet documentation is the broader context for wallet and key concepts. In Taopedia
prose, a wallet address should be read as destination-address vocabulary rather than a wallet
operation guide, balance claim, or live account-status claim
([Bittensor wallet documentation](https://docs.learnbittensor.org/keys/wallets)).

## Reader Boundary

Wallet address should not be read as a private key, a completed transfer, or proof that funds were
received. It names the public destination identifier derived from key material and used when sending
or receiving TAO
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address)).

## Distinction from Private Key

A wallet address and a private key are the public and secret halves of a keypair. A wallet address
is the public SS58-encoded identifier derived from the public key that others use to send funds,
while a private key is the secret that signs transactions and must never be shared
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address),
[Glossary: Private Key](https://docs.learnbittensor.org/resources/glossary#private-key),
[Glossary: SS58 Encoded](https://docs.learnbittensor.org/resources/glossary#ss58-encoded)).

- **Wallet address** — the public SS58 identifier used to receive funds.
- **Private key** — the secret that signs transactions and stays hidden.
