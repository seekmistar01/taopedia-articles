# External Wallet

An external wallet refers to wallet software or key custody outside the local Bittensor wallet path.
Official Bittensor documentation distinguishes wallet concepts when explaining how keys and wallet
files are managed.

References:
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet),
[Wallets](https://docs.learnbittensor.org/keys/wallets)

## Wallet Context

The term is useful because wallet location and key custody affect how a user protects access to TAO
and Bittensor identities. An external wallet should be understood as part of the broader wallet and
key-management model rather than as a subnet role or consensus mechanism.

References:
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet),
[Working with Keys](https://docs.learnbittensor.org/keys/working-with-keys)

## Custody Distinction

External wallet describes custody context, not a different token or subnet role. The
[External Wallet glossary](https://docs.learnbittensor.org/resources/glossary#external-wallet)
places the distinction around where wallet control is managed: local files on the user's machine or
some wallet handling outside that local path.

Reference: [Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet)

## Coldkey and Hotkey Still Apply

External wallet names where wallet software or key custody lives, not a replacement for Bittensor's
coldkey and hotkey roles. The
[Working with Keys](https://docs.learnbittensor.org/keys/working-with-keys) documentation still
frames Bittensor participation through those wallet-key roles even when signing or storage happens
outside a local wallet file path.

Readers should therefore keep external wallet as a custody-location label. Coldkey and hotkey still
describe which key performs fund control versus routine subnet actions, regardless of whether the
wallet interface is local or external.

References:
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet),
[Working with Keys](https://docs.learnbittensor.org/keys/working-with-keys)

## Sending and Receiving TAO Addresses

External wallets still use Bittensor destination addresses when moving TAO. The
[Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer) describes
transferring TAO between coldkeys, and the [Wallets](https://docs.learnbittensor.org/keys/wallets)
documentation covers how wallet software manages those addresses regardless of custody location.

That keeps external wallet inside wallet-management vocabulary rather than inside subnet task or
consensus vocabulary. The term explains where wallet control is handled, not how validators score
miners or how emissions are calculated.

References: [Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer),
[Wallets](https://docs.learnbittensor.org/keys/wallets)

## Hardware and Third-Party Custody Paths

Official wallet documentation discusses options beyond default local files, including external
signing and custody setups that place key handling outside the default local path. The
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)
captures that broader custody category without naming one vendor or one device as the only valid
external path.

Readers evaluating wallet security should compare custody location, backup practices, and signing
flow using current official key documentation for the setup they actually use, whether local or
external.

External wallet therefore marks a custody path choice inside wallet documentation, not a separate
Bittensor token, subnet role, or consensus mechanism.

References:
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet),
[Wallets](https://docs.learnbittensor.org/keys/wallets)

## Relationship to Address Poisoning Scams

External wallets use destination addresses for sending and receiving TAO, so the address-poisoning
risk applies regardless of where wallet control is managed. The
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)
describes an external wallet as wallet software or key custody outside the local Bittensor wallet
path, and the
[Bittensor address-poisoning guide](https://docs.learnbittensor.org/keys/address-poisoning-scams)
frames address review as part of wallet safety before value-bearing actions.

For readers, external wallet names the custody context, while address poisoning names the risk that
a lookalike destination address gets treated as the intended one. Wallet custody changes where keys
are stored, but the destination-verification step belongs to every wallet type.

References:
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet),
[Address Poisoning Scams guide](https://docs.learnbittensor.org/keys/address-poisoning-scams)

## Relationship to Local Wallet

External wallet and local wallet are related but different parts of Bittensor wallet-storage
vocabulary. A local wallet describes wallet files stored on a user's own machine under their direct
custody, while an external wallet refers to wallet software or key custody outside that local
Bittensor wallet path, such as hardware wallets or browser-based wallet software. The
[Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet) describes
wallet files created and stored on the user's machine, and the
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)
describes wallet handling outside that local path.

For readers, local wallet and external wallet both describe where wallet control is managed and who
has custody, not differences in token access or on-chain authority. Both require appropriate
security practices for their context — the distinction is about storage and custody location, not
about which approach is inherently safer.

References:
[Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet),
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)

## Relationship to Wallets

External wallet and wallets are related but different scopes in Bittensor wallet vocabulary. A
wallet is the cryptographic identity formed by coldkey and hotkey pairs used to access TAO and sign
transactions, while an external wallet refers to where that wallet's key custody is managed —
outside the local Bittensor wallet path through hardware devices or separate wallet software. The
[Wallets](https://docs.learnbittensor.org/keys/wallets) documentation describes the wallet as the
identity structure, and the
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)
describes external wallet as the custody location for that identity.

For readers, the wallet names the cryptographic identity and on-chain control, while external wallet
names one possible custody context for managing that identity's keys. A wallet's identity remains
the same whether it's accessed through local or external storage.

References: [Wallets](https://docs.learnbittensor.org/keys/wallets),
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)

## Reader Boundary

This page defines the term. It does not endorse a specific wallet provider, ask readers to enter
keys, or provide a transfer workflow. Readers should use current official wallet documentation for
security-sensitive wallet actions.
