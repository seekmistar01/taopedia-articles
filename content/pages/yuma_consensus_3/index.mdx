# Yuma Consensus 3

Yuma Consensus 3, or YC3, is a refinement of Bittensor's Yuma Consensus mechanism. The official
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)
article describes YC3 as an evolution focused on validator fairness, validator-miner bonds, and
faster recognition of useful miner work.

## What YC3 Addresses

Yuma Consensus turns validator evaluations into subnet incentive allocation. The YC3 article
describes earlier issues around small validator treatment, stale bond allocation, and slow
recognition of miners that only some validators had identified. YC3 narrows those issues by making
bond movement more responsive to individual validator-miner relationships.

Reference:
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Per-Bond EMA Scaling

A central YC3 idea is per-bond EMA scaling. Instead of treating every validator-miner bond with the
same adjustment rate, YC3 lets each pair move according to its own relationship to subnet consensus.
This gives the mechanism a more local view of how a validator's evaluation relates to a miner over
time.

Reference:
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Early Recognition

The official YC3 article emphasizes earlier recognition of promising miners. Under YC3, validators
can begin building bonds with miners before those miners are broadly recognized by the validator
set. This helps distinguish independent evaluation from simply following already established
consensus.

Reference:
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Liquid Alpha Context

YC3 is described alongside Liquid Alpha because both features concern how validator evaluations
respond to changing subnet consensus. The YC3 article says the mechanisms can work together to
support independent evaluation while preserving the broader anti-manipulation goals of Yuma
Consensus. Taopedia readers should treat Liquid Alpha as related context rather than as a separate
operating step inside YC3.

Reference:
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Relationship to Multiple Mechanisms

Yuma Consensus 3 refines validator-miner bonding inside subnet consensus. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms) notes
that each mechanism operates with its own bond pool for Yuma Consensus calculations.

For readers, YC3 still adjusts bonding inside a subnet that may run more than one incentive
mechanism.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)

## Careful Interpretation

YC3 is not a separate subnet and not a guide for changing subnet settings. It is a consensus
refinement within Bittensor's validator-miner incentive machinery. Exact activation status,
settings, and subnet-specific behavior should be checked against current official documentation and
chain state.

Reference:
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Relationship to Yuma Consensus

Yuma Consensus 3 and Yuma Consensus address related but different parts of Bittensor consensus
vocabulary. Yuma Consensus names the aggregation process that turns validator evaluations into miner
incentives and validator dividends, while Yuma Consensus 3 names a refinement focused on
validator-miner bond scaling inside that process. The
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) documentation describes the
consensus flow, and the
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)
article describes YC3 as an evolution of that bonding behavior.

For readers, Yuma Consensus names the broader consensus process, while Yuma Consensus 3 names a
bond-scaling refinement within that process.

References: [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Relationship to Liquid Alpha

Yuma Consensus 3 and liquid alpha address related but different parts of bond responsiveness inside
Yuma Consensus. Liquid alpha varies the bond-smoothing factor per validator-miner pair based on
consensus alignment, while Yuma Consensus 3 focuses on per-bond EMA scaling so individual
validator-miner relationships can move at different rates. The
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)
documentation describes liquid alpha, and the
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)
article describes YC3 alongside liquid alpha as related bond-refinement context.

For readers, liquid alpha names a per-pair smoothing adjustment, while Yuma Consensus 3 names a
broader bond-scaling refinement that can work with that adjustment inside consensus.

References:
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Relationship to Validator-Miner Bonds

Yuma Consensus 3 and validator-miner bonds address related but different parts of Yuma Consensus
vocabulary. Validator-miner bonds describe the smoothed relationship between a validator and miners
over time, while Yuma Consensus 3 refines how those bonds scale so earlier independent evaluations
can be reflected more fairly. The
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)
describes the bond relationship, and the
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)
article describes YC3 as focused on validator-miner bond movement.

For readers, validator-miner bonds name the smoothed relationship, while Yuma Consensus 3 names a
refinement to how those bonds respond inside consensus.

References:
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds),
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Relationship to Exponential Moving Averages

Yuma Consensus 3 and exponential moving averages address related but different parts of bond
smoothing inside Yuma Consensus. Exponential moving averages provide the general smoothing method
for bond updates, while Yuma Consensus 3 applies per-bond EMA scaling so each validator-miner pair
can adjust at its own rate. The
[Glossary: Exponential Moving Average (EMA)](https://docs.learnbittensor.org/resources/glossary#exponential-moving-average-ema)
describes EMA smoothing, and the
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)
article describes per-bond EMA scaling as a central YC3 idea.

For readers, exponential moving averages name the general smoothing technique, while Yuma Consensus
3 names a per-bond scaling refinement applied inside that smoothing context.

References:
[Glossary: Exponential Moving Average (EMA)](https://docs.learnbittensor.org/resources/glossary#exponential-moving-average-ema),
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)
