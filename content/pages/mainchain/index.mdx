# Mainchain

Mainchain is the glossary term for the primary Bittensor blockchain network. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#mainchain) describes it as the primary
Bittensor network used for production purposes.

## Network Context

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
lists mainnet alongside testnet and localnet. In Taopedia, Mainchain is best read as the production
chain context, while testnet and localnet name non-production contexts for testing and development.

## Relationship to Subtensor

Mainchain and Subtensor name related but distinct concepts. The
[Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain) describes
Mainchain as the primary Bittensor network used for production purposes, while the
[Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor) describes
Subtensor as Bittensor's layer 1 blockchain and the system of record for transactions and rankings.

For readers, Mainchain is the production network context, while Subtensor is the chain layer that
the production network runs. Reading the two terms together keeps the network-context meaning of
Mainchain distinct from the chain-layer meaning of Subtensor.

References: [Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor)

## Relationship to Bittensor Networks

Mainchain and Bittensor Networks name related but different network-context ideas. The
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference groups
mainnet, testnet, and localnet, while the
[Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain) describes the
primary Bittensor network used for production purposes.

For readers, Bittensor Networks is the broader environment label, while Mainchain names the
production subset inside that picture. Mainchain should be read as production context, not as the
full list of network environments.

References: [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain)

## Relationship to Public Subtensor

Mainchain and Public Subtensor name related but different parts of production chain access. The
[Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain) describes the
primary Bittensor network used for production purposes, while the
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)
describes publicly reachable access to the Subtensor chain layer.

For readers, Mainchain is the production network-context label, while Public Subtensor names public
chain access within that live network picture. Public Subtensor does not replace Mainchain or rename
Subtensor.

References: [Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)

## Relationship to Local Blockchain

Mainchain and local blockchain name opposite sides of Bittensor network context. The
[Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain) describes the
primary Bittensor network used for production purposes, while the
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)
describes a private chain used for development and testing.

For readers, Mainchain is production network context, while local blockchain is isolated development
context. Neither term should be read as interchangeable with the other.

References: [Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain),
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)

## Subnet Development Endpoint Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
places mainchain at the end of the subnet development path: after local testing and testchain use, a
subnet goes live by connecting to the Bittensor mainchain. That framing is useful because Mainchain
is not just another testing label. It is the production network context.

For readers, the sequence separates preparation from live participation. Local testing can help a
developer understand an incentive mechanism, and testchain can exercise the work in a public testing
environment. Mainchain is the stage where the same kind of subnet activity belongs to production
Bittensor network behavior.

That distinction also narrows what a Mainchain article should imply. Calling something mainchain
context does not prove that a subnet's mechanism is well designed or that a participant should join
it. It only places the activity in the production network rather than in local or testchain
development context.

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
says subnet participants should become familiar with the incentive mechanism before joining or
creating a subnet. Mainchain context is therefore best read as the final network stage after that
familiarity and earlier testing.

This keeps Mainchain separate from both local blockchain and testchain vocabulary. Mainchain names
the live production setting; the earlier stages explain how a subnet can be tried and evaluated
before its behavior is treated as production network behavior.

References:
[Introduction to Bittensor: Subnet development](https://docs.learnbittensor.org/learn/introduction#subnet-development),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain)

## Reader Boundary

This article defines a network context. It does not list connection details, endpoint status, local
environment details, or current network conditions. Readers should use official network references
for network-specific details.

## Relationship to Coinbase

Mainchain and coinbase are related but different in scope: one is the network, the other is a
routine that runs inside it. Mainchain is the primary Bittensor network used for production. The
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)
documentation describes coinbase as the per-block protocol mechanism that drives TAO emission,
accumulates pending emissions, and triggers Yuma Consensus rounds at epoch boundaries, while the
[Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain) describes the
production network.

On Mainchain, coinbase executes as part of ordinary block production — each block the network
produces advances coinbase's emission processing. Mainchain names the production environment in
which that happens, while coinbase names the per-block routine doing the emission work. The same
coinbase mechanism also runs in non-production environments, so coinbase is not unique to Mainchain;
Mainchain is simply the production context in which it runs live. One names where blocks are
produced, the other names a process those blocks carry out.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For Mainchain,
that sequence is the definition: Mainchain is the production network stage itself. Examples of
Mainchain activity belong to that final production environment.

In localnet, there is no Mainchain. Local blockchain activity runs in an isolated environment that
does not represent production Bittensor network behavior.

On testnet, there is no Mainchain. Testnet activity runs in a shared, non-production environment
separate from mainnet state.

On mainnet, Mainchain is the live primary Bittensor network where production block production,
emission, and subnet participation occur. The
[Glossary: Mainchain](https://docs.learnbittensor.org/resources/glossary#mainchain) describes
Mainchain as the primary Bittensor network used for production purposes.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A Mainchain example describes production Bittensor network
behavior and should not be read as representing localnet or testnet activity.

## Network Scope

Mainchain references should be kept separate from localnet and testnet examples. Bittensor network
documentation distinguishes production mainnet behavior from shared testnet and isolated localnet
contexts ([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)).

This keeps a mainchain claim from being supported only by development-environment evidence.
