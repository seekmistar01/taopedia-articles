# Halving

Halving is a tokenomics term for a reduction in token emission rate. In Bittensor, it belongs to
emission mechanics because it changes how quickly new tokens enter the reward-distribution path. The
core reading is an emission-rate reduction tied to a threshold, not a price forecast or reward
allocation model ([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

## Emission-Rate Role

Halving changes the rate side of emissions.
[Emission](https://docs.learnbittensor.org/learn/emissions) describes creation and allocation over
time, while halving identifies the reduction point in that emission schedule. It sits upstream of
later reward distribution.

That keeps halving separate from a full reward calculation. A lower emission rate changes the input
to downstream reward flows, but later outcomes still depend on the token, subnet, incentive
mechanism, and consensus process involved.

## Threshold Trigger

Bittensor halving is tied to issuance thresholds. The
[halving mechanism](https://docs.learnbittensor.org/concepts/halving) reduces emission rate when the
relevant supply threshold is reached. That makes the threshold part of the tokenomics meaning, not
just a timing detail.

A useful halving reference therefore needs the asset and the threshold context. Without those, the
term can collapse into a vague statement about slower issuance instead of the named rate reduction
associated with a supply milestone
([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

## TAO and Alpha Scope

Halving language can appear at more than one Bittensor tokenomics scope. TAO halving belongs to the
network-token side, while alpha halving belongs to subnet alpha-token emission. The shared word does
not erase the asset distinction.

This matters in Dynamic TAO discussions because alpha tokens are subnet-specific while TAO is the
network token. A clear halving reference identifies whether the reduced emission rate belongs to TAO
or to a subnet's alpha token
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Supply Accounting

Halving changes issuance pace; burning and recycling address different movements in token supply.
Burning removes existing tokens from spendable circulation, while Bittensor recycling vocabulary
describes tokens returned to the protocol supply path.

These terms can all affect supply interpretation, but they operate at different moments. Halving is
about how quickly new tokens enter the system. Burning and recycling describe what happens to tokens
that already exist. Issuance sits nearby as the circulating-supply measure
([Glossary: Burning](https://docs.learnbittensor.org/resources/glossary#burning),
[Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)).

## Maximum Supply and Block Reward Pipeline

Halving sits between issuance milestones and the per-block mint figure that feeds emission. Each
supply-triggered halving lowers the network
[block reward](https://docs.learnbittensor.org/learn/emissions) — the amount of new TAO minted each
block — as cumulative [issuance](https://docs.learnbittensor.org/resources/glossary#issuance)
crosses documented thresholds
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

[Maximum supply](https://docs.learnbittensor.org/concepts/halving) names the protocol's 21 million
TAO lifetime ceiling. Halving does not change that cap; it slows how quickly new TAO approaches it.
Emission material documents the first such cut lowering the per-block network reward from 1 TAO to
0.5 TAO, and later thresholds continue the same halving pattern as issuance climbs
([Emission](https://docs.learnbittensor.org/learn/emissions)).

For readers, the pipeline order is: issuance crosses a threshold, halving cuts the emission rate,
block reward steps down, and the broader emission process continues injecting and distributing value
at the reduced rate. Block reward is the active per-block mint figure; halving is the supply event
that lowers it.

Unlike Bitcoin, which halves on block numbers, Bittensor implements halvings based on total token
supply, so the trigger is a supply milestone rather than a fixed block height
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)). Anything that
changes the pace toward the next supply milestone, such as recycled TAO returning to issuance,
shifts when the next halving lands rather than moving a pre-set block target.

Each halving event cuts Bittensor's daily token emission rate in half, reducing how quickly new TAO
enters circulation. The effect is on the inflow of new tokens, not on TAO balances that already
exist.

## Emission and Reward Flow

Halving sits upstream of reward distribution. Emission covers how emitted value moves through
Bittensor reward paths, while halving covers the rate-reduction mechanism that changes the supply
entering those paths ([Emission](https://docs.learnbittensor.org/learn/emissions),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

That order matters because a halving event can affect supply entering the system without explaining
miner incentives, validator dividends, subnet reserves, or market behavior. The same separation
applies to subnet alpha: alpha halving changes an alpha-token emission rate, while incentive
mechanisms and consensus explain later reward outcomes.

A halving event does not by itself describe every downstream subnet allocation rule, because
emission vocabulary names the ongoing release path rather than the supply threshold alone.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For halving on a
subnet such as netuid 1, that sequence changes how readers should interpret supply totals, block
reward figures, and subnet alpha halving behavior.

In localnet, halving thresholds and emission rates can be tested in an isolated environment. Local
issuance totals and alpha halving examples reflect local chain configuration rather than production
supply schedules on the selected netuid.

On testnet, TAO and alpha halving behavior can be observed in a shared non-production network.
Testnet issuance and subnet alpha totals on a selected netuid are separate from mainnet chain state
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

On mainnet, halving describes live production issuance rules on the connected Bittensor network.
Observed timing or supply values depend on current emission, recycling, and halving history on the
active network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A halving example from one environment should not be read
as representing production issuance behavior on another network.

## Reader Boundary

Halving is supply-schedule vocabulary. It is not a date forecast, price forecast, staking strategy,
or complete reward model. The durable concept is emission-rate reduction at a threshold
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

TAO halving belongs to the network-token side; alpha halving belongs to subnet alpha-token emission.
Burning and recycling address different supply movements than halving, which changes how quickly new
tokens enter the system.

## Relationship to Issuance

Halving and issuance are related but different tokenomics terms. Halving names the supply-triggered
emission-rate cut that lowers the per-block mint as cumulative issuance crosses documented
thresholds, while issuance names the circulating amount of TAO counted in the network at a given
moment ([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

For readers, halving is the threshold event that slows future minting and issuance is the live
circulating measure that rises toward the documented TAO ceiling as new TAO enters circulation
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

A halving event does not by itself quote current circulating totals, because issuance vocabulary
names the measured circulating amount rather than the supply threshold alone
([Maximum Supply](https://docs.learnbittensor.org/concepts/halving)).

Official halving documentation keeps threshold-triggered rate-cut vocabulary separate from
circulating-supply accounting vocabulary so readers can tell which term applies to issuance slowdown
and which applies to the live circulating total
([Emission](https://docs.learnbittensor.org/learn/emissions)).
