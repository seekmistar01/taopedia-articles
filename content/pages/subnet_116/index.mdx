Subnet 116 is a Bittensor subnet slot that currently has no registered identity returned by the
Finney identity query. Because the live identity lookup does not return a subnet name, GitHub
repository, project URL, contact field, description, logo, or additional metadata, this page treats
SN116 as an unknown identity record.

The article therefore does not describe a public miner task, validator task, scoring function, or
operator workflow for Subnet 116. Those claims would need a current identity record, operator
documentation, or another direct source that can be reproduced by readers.

## Current Status

For active subnet articles, Taopedia normally describes what miners do, how validators score them,
and which repository or documentation defines the mechanism. Subnet 116 does not currently expose
that kind of public source through its live identity state.

A future operator could register identity metadata and publish project documentation for this
netuid. Until that happens, readers should verify the live Finney state before relying on any older
or third-party project-specific description.

## Miner and Validator Roles

Without registered identity metadata or a current public codebase, the subnet-specific miner and
validator roles cannot be described reliably. The general [[Mining and Validating]] pattern still
applies at the Bittensor level: miners compete for incentive and validators submit weights that feed
into [[Yuma Consensus]].

For SN116 specifically, operators should use live chain state and current operator documentation, if
published, rather than stale profile text.

## On-Chain Identity

Live SN116 subnet data is available on [TaoStats](https://taostats.io/subnets/116). Readers can
check the registered identity fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 116 --network finney`.

At the time this article was prepared, that command returned that no existing subnet identity was
found for subnet 116 on Finney. This means the article intentionally avoids claims about a subnet
name, GitHub repository, website, or technical mechanism.
