# Consensus-Based Weights

Consensus-based weights, also called liquid alpha, are a Bittensor consensus mechanism that makes
validator-miner bond smoothing depend on how a validator's weights align with consensus. The
mechanism sits inside the Yuma Consensus bond path between submitted validator weights and
validator-side outcomes
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)).

The core idea is alignment-sensitive smoothing. When a validator's weights are closer to consensus,
the related bond response can be more direct; weaker alignment keeps the response more conservative
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Liquid Alpha Context

Liquid alpha is the alternate name for consensus-based weights. The name highlights that the alpha
term in bond smoothing can vary rather than staying fixed across every validator-miner pair
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)).

For readers, liquid alpha and consensus-based weights should be treated as the same mechanism term.
They do not name separate reward types or separate miner-scoring systems
([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

## Bond Smoothing Context

Bittensor uses exponential moving averages to smooth validator-miner bonds, so bond state does not
change entirely from one evaluation step to the next. Consensus-based weights make the smoothing
factor responsive to consensus alignment
([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

That places the mechanism after validator weights are compared with consensus. It does not replace
weight submission, clipping, rank calculation, or Yuma Consensus itself
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)).

## Validator Dividend Context

Consensus-based weights affect the bond path that feeds validator dividends. Yuma Consensus
describes validator emissions through bonds to miners weighted by miner emission results, and liquid
alpha adjusts how quickly those bonds respond to consensus-aligned evaluation
([Yuma Consensus: validator emissions](https://docs.learnbittensor.org/learn/yuma-consensus#validator-emissions)).

The result is still validator-side consensus behavior, not a new miner-ranking rule. The mechanism
changes bond smoothing so validator outcomes can track useful consensus-aligned evaluation more
closely
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)).

## Outlier and Consensus Boundary

Consensus-based weights are adjacent to clipping but not identical to it. Clipping limits outlier
validator weights before final rank calculations, while liquid alpha adjusts how bond history
responds after weights are compared with consensus
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

For readers, clipping is a filtering step and liquid alpha is a bond-smoothing step. Both sit inside
the consensus path, but they answer different questions about validator submissions
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Weight Copying Context

Consensus-based weights are often discussed near the weight-copying problem because copied or stale
weights can diverge from consensus. The mechanism can make bond response more sensitive to whether
submitted weights remain aligned
([The Weight Copying Problem](https://docs.learnbittensor.org/concepts/weight-copying-in-bittensor)).

This does not make liquid alpha an accusation model. Weight copying names an evaluation-quality
problem; consensus-based weights name a bond-path response to consensus alignment
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)).

## Multiple Mechanisms Context

When a subnet uses multiple incentive mechanisms, bond and consensus context should be read per
mechanism. Each mechanism can have its own Yuma bond pool and evaluation path
([Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets)).

Consensus-based weights still mean alignment-sensitive bond smoothing, but the mechanism context
clarifies which evaluation path produced the relevant validator-miner bond updates
([Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)).

## Reader Boundary

Consensus-based weights should be read as a Yuma Consensus bond-smoothing concept. They are not a
standalone user feature, a separate subnet token, or a complete explanation of validator rewards
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)).

The stable meaning is the role of consensus alignment in smoothing validator-miner bonds. Exact
formula details and parameter values belong in the Bittensor consensus references rather than in a
concept summary
([Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)).

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For
consensus-based weights, that sequence changes how readers should interpret examples of weight
adjustment and validator-miner bond smoothing.

In localnet, consensus-based weight mechanics can be tested in an isolated environment. Localnet
weight adjustment and bond smoothing outcomes reflect local chain settings and do not represent
production subnet state.

On testnet, consensus-based weights can be observed in a shared, non-production network. Testnet
weight adjustment outcomes are separate from mainnet subnet validator state.

On mainnet, consensus-based weights are the live weight adjustment mechanism that smooths validator
weights toward consensus over successive Yuma Consensus rounds in production Bittensor subnets. The
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)
documentation describes the mechanics that apply on the production network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A consensus-based weights example from one environment
should not be read as representing weight adjustment or bond outcomes in another environment.
