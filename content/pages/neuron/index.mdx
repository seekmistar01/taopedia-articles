# Neuron

A neuron is a registered participant in a Bittensor subnet
([Glossary: Neuron](https://docs.learnbittensor.org/resources/glossary#neuron),
[Understanding Neurons](https://docs.learnbittensor.org/learn/neurons)). The term appears when
describing miners, validators, and metagraph state for subnet participants.

## Distinction from Metagraph, UID Slot, and Registration

Neuron names the registered participant record inside a subnet. Metagraph names the subnet state
table; UID slot names the indexed position; register names the entry process that grants that slot
([Glossary: Neuron](https://docs.learnbittensor.org/resources/glossary#neuron),
[The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph)).

- **Neuron** — registered participant record tied to a UID slot.
- **Metagraph / UID slot / register** — state table, position, and entry process.

Participant vocabulary names the record; metagraph and entry vocabulary name table and admission
([Understanding Neurons](https://docs.learnbittensor.org/learn/neurons)).

## Distinction from Subnet Miner and Subnet Validator

Subnet miner and subnet validator name participant roles inside a subnet. Neuron names the
registered participant record that can hold one of those roles on a netuid
([Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)).

Role vocabulary says what the participant does; neuron vocabulary says the participant is registered
in subnet state.

## Distinction from Hotkey-Coldkey Pair

A hotkey-coldkey pair names the wallet keys that authenticate operations. A neuron ties that
identity to one subnet's participant record and UID position
([Glossary: Hotkey-Coldkey Pair](https://docs.learnbittensor.org/resources/glossary#hotkey-coldkey-pair),
[Glossary: Neuron](https://docs.learnbittensor.org/resources/glossary#neuron)).

- **Hotkey-coldkey pair** — wallet keys that authenticate operations.
- **Neuron** — registered participant record tied to a UID on one netuid.

Key-pair vocabulary and participant-record vocabulary answer different identity layers
([Understanding Neurons](https://docs.learnbittensor.org/learn/neurons)).

## Distinction from Axon and Dendrite

Axon and dendrite name serving and querying endpoints in subnet communication. Neuron names the
registered participant record in subnet state
([Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon),
[Glossary: Dendrite](https://docs.learnbittensor.org/resources/glossary#dendrite),
[Glossary: Neuron](https://docs.learnbittensor.org/resources/glossary#neuron)).

- **Axon and dendrite** — server and client endpoints for subnet communication.
- **Neuron** — registered participant record in subnet state.

Endpoint vocabulary and participant-record vocabulary answer different layers on the same path
([Understanding Neurons: Neuron-to-neuron communication](https://docs.learnbittensor.org/learn/neurons#neuron-to-neuron-communication)).

## Distinction from Subnet Protocol

Subnet protocol names the rule set for how validators and miners exchange work inside a subnet.
Neuron names the UID-identified participant record registered to operate within that subnet exchange
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Glossary: Neuron](https://docs.learnbittensor.org/resources/glossary#neuron),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Exchange-rule vocabulary and participant-registration vocabulary answer different questions on the
same netuid.

## Distinction from Yuma Consensus

A neuron is the registered participant record on a netuid. Yuma Consensus is the tempo mechanism
that aggregates validator weights into miner incentives and validator dividends
([Glossary: Neuron](https://docs.learnbittensor.org/resources/glossary#neuron),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Neuron** — registered participant record on a netuid.
- **Yuma Consensus** — subnet consensus pass at tempo boundaries.

Participant-record vocabulary and settlement vocabulary answer different subnet layers
([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Distinction from Netuid

Netuid names which subnet is in scope. Neuron names a registered participant inside that subnet's
state table ([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)).

Every neuron claim should carry the netuid it belongs to because participant records are per-subnet,
not network-wide.

## Distinction from Deregistration

Deregistration removes a neuron from a UID slot. Neuron names the participant record while that
registration is active in subnet state
([Glossary: Deregistration](https://docs.learnbittensor.org/resources/glossary#deregistration)).

A deregistration event changes the next metagraph snapshot; it does not erase the neuron vocabulary
for historical reads at earlier blocks.

## Distinction from Incentive Mechanism

An [incentive mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism)
covers the task, protocol, and evaluation method that define how a subnet connects work to rewards.
A [subnet task](https://docs.learnbittensor.org/resources/glossary#subnet-task) names the work
miners are expected to perform inside that subnet's incentive design.

A neuron names who is registered in a subnet; an incentive mechanism names the evaluation rules that
govern how that participant's role is judged.

## Reader Boundary

This page defines the term at a high level. It does not report current subnet membership, live
neuron counts, wallet ownership, or current rewards. Those values change with chain state and should
be checked from current Bittensor sources when needed.

## Distinction from Axon

A neuron and an axon describe a participant and the server it exposes. A neuron is the registered
participant occupying a UID slot in a subnet, identified by its hotkey-coldkey pair and acting as a
miner or validator; an axon is the network server a neuron (typically a miner) runs to receive and
respond to requests over the subnet protocol
([Glossary: Neuron](https://docs.learnbittensor.org/resources/glossary#neuron),
[Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon),
[Understanding Neurons](https://docs.learnbittensor.org/learn/neurons)).

- **Neuron** — the registered participant holding a UID slot in a subnet.
- **Axon** — the network server a neuron exposes to receive requests.
