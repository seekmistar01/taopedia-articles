# Root Staker

A root staker is a TAO holder who stakes through validators on the Root Subnet, also called Subnet
Zero
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

The term names the TAO-holder role on Bittensor's subnet-agnostic staking path. It should be read
separately from alpha-side staking inside an ordinary subnet.

## Root Subnet Context

The Root Subnet is the special subnet where TAO holders can stake to validators in a subnet-agnostic
way. The glossary also distinguishes that path from ordinary subnet miner and validator work
([Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero),
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution)).

Root staker therefore names the person supplying TAO through that path, not the validator role and
not a subnet-local alpha position.

## Root Staker vs Alpha Staker

Bittensor staking prose separates root-side TAO staking from subnet-local alpha staking. The
emission documentation describes emissions to stakers on the root subnet separately from emissions
to stakers holding subnet alpha in a mining subnet
([Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

For readers, the useful boundary is asset and path. Root staker describes TAO staked through Root
Subnet validators; alpha staker language belongs to a subnet's alpha context.

## Delegation Boundary

A root staker commits TAO by delegating to a validator on the Root Subnet. The staking and
delegation overview describes TAO holders staking through validators, while the Root Subnet glossary
places that relationship on Subnet Zero
([Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

This article keeps the role definition separate from validator selection. Root staker names who
supplies TAO on the root path; it does not rank validators or explain staking steps.

## Dividends and Root Claims

Root-routed stake can earn alpha dividends that are later handled through the root-claim process.
The Root Claim documentation describes keep and swap handling for accumulated dividends from Root
Subnet stake
([Root Claim Overview](https://docs.learnbittensor.org/staking-and-delegation/root-claims),
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution)).

Root staker names the staking role before that dividend-handling layer. Root claims and root claim
type describe what happens when accumulated dividends are processed.

## Relationship to Root Proportion

Root staker and root proportion describe different levels of the same staking context. Root staker
names the TAO-holder role, while root proportion measures how much of a subnet's dividend context is
attributed to root-routed TAO
([Glossary: Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

One term names who is staking through the Root Subnet. The other names a subnet-level proportion
that reflects root-side stake in that subnet's dividend context.

## Reader Boundary

Root staker should be read as a concept article term. It is not a live balance report, validator
recommendation, wallet guide, or synonym for every staker in every Bittensor context
([Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation),
[Glossary: Root Subnet/Subnet Zero](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)).

The stable article-level point is that root staker names a TAO holder using the Root Subnet staking
path.
