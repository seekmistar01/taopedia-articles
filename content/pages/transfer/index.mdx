# Transfer

A transfer is Bittensor terminology for sending TAO from one wallet address to another
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer)).

## What It Represents

A transfer moves existing TAO on chain from one coldkey wallet address to another. It does not
create new emission proceeds, change subnet registration state, or attach TAO to a validator hotkey
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer),
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)).

Coldkeys authorize transfers as sensitive financial operations. Destination addresses are
public-key-derived identifiers; transfer vocabulary names the movement, not the signing key material
([Glossary: Coldkey](https://docs.learnbittensor.org/resources/glossary#coldkey),
[Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address)).

Transfer examples from one network environment should not be read as representing production wallet
balances on another
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from Staking

A transfer sends free TAO between wallet addresses. Staking attaches TAO to a validator hotkey to
support it inside a subnet
([Glossary: Staking](https://docs.learnbittensor.org/resources/glossary#staking),
[Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

- **Transfer** — sending free TAO between wallet addresses.
- **Staking** — attaching TAO to a validator hotkey inside a subnet.

## Distinction from Moving Stake

Moving stake relocates an existing stake position from one hotkey to another without first returning
it to free balance
([Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

- **Transfer** — sending free TAO between coldkey wallet addresses.
- **Moving stake** — relocating stake between hotkeys without unstaking.

## Distinction from Coldkey Swap

Coldkey swap migrates on-chain identity to a new coldkey while preserving hotkeys and registrations.
A transfer only relocates free TAO between addresses
([Coldkey Swap](https://docs.learnbittensor.org/keys/coldkey-swap)).

- **Transfer** — token movement between addresses.
- **Coldkey swap** — identity migration to a new coldkey.

## Distinction from Address Poisoning Scams

Address poisoning plants a lookalike wallet address so a transfer reaches the wrong recipient.
Transfer names the legitimate send action itself
([Address Poisoning Scams](https://docs.learnbittensor.org/keys/address-poisoning-scams)).

- **Transfer** — sending TAO to a chosen wallet address.
- **Address poisoning scam** — deception that corrupts which address is chosen.

## Distinction from SS58 Encoded

SS58 encoded names the compact address format for public-key-derived destinations. Transfer names
the movement action
([Glossary: SS58 Encoded](https://docs.learnbittensor.org/resources/glossary#ss58-encoded)).

- **Transfer** — action of sending TAO to a destination address.
- **SS58 encoded** — address format that identifies that destination.

## Distinction from Coinbase

Coinbase is the per-block emission mechanism that produces and routes new TAO. A transfer relocates
TAO that already exists
([Glossary: Coinbase](https://docs.learnbittensor.org/resources/glossary#coinbase)).

- **Transfer** — wallet-to-wallet send of existing TAO.
- **Coinbase** — per-block emission mechanism that produces new TAO.

## Distinction from Crowdloans

A crowdloan is a pooled-funding campaign. A transfer is one allowed finalization action that moves
balances between accounts ([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans)).

- **Crowdloan** — pooled-funding campaign with a finalization action.
- **Transfer** — balance movement used as a finalization action.

## Distinction from Yuma Consensus

Transfer names ledger balance movement on chain. Yuma Consensus converts submitted validator weights
into emission shares after evaluation completes
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Transfer** — moving TAO between accounts.
- **Yuma Consensus** — on-chain settlement from validator weights to emission shares.

## Reader Boundary

Transfer should not be read as staking, subnet registration, or coinbase emission. It names
wallet-to-wallet movement of existing TAO between addresses
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer)).

## Distinction from Unstaking

Transfer and unstaking both change where TAO sits, but at different layers. A transfer moves free
TAO from one coldkey address to another without touching stake, while unstaking removes stake from a
hotkey — converting staked alpha back to TAO — and returns it to the owner's free balance rather
than sending it to another account
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer),
[Glossary: Unstaking](https://docs.learnbittensor.org/resources/glossary#unstaking),
[Staking and Delegation](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

- **Transfer** — moves free TAO between coldkey addresses.
- **Unstaking** — withdraws stake back into the owner's free balance.
