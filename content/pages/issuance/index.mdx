# Issuance

Issuance is the circulating amount of TAO in the Bittensor network. It includes directly held TAO
and TAO held in subnet liquidity pools
([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

The term is useful because it names a supply state, not a reward category. Issuance tells readers
what TAO amount is counted as circulating before more specific emission, halving, recycling, or
burning context is applied.

That makes issuance a supply-accounting term. It should be read beside emissions, halving,
recycling, and burning, but it does not replace any of those more specific token-flow concepts
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

## Circulating TAO Context

Issuance is a supply measure for TAO. It should not be reduced to only direct holdings, only subnet
pool reserves, or only newly emitted tokens
([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

The broader circulating-TAO picture matters because Bittensor has both direct TAO holdings and
subnet pool context. Issuance names the counted circulating amount, while other articles explain
where token movement comes from and where it goes.

This also keeps issuance separate from an account-level balance. Issuance is network-level
tokenomics vocabulary, not a statement about one holder or one subnet position.

## Emission Accounting Context

Emissions describe token creation and allocation over time. Issuance is the circulating TAO amount
that those processes affect ([Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

This keeps a statement about circulating TAO separate from a statement about who receives newly
created token amounts. Emission flow changes supply context; issuance is the resulting TAO
circulation measure being tracked.

Issuance answers how supply is accounted for, while emission answers how protocol rewards are
distributed. A precise sentence should identify which side of that boundary it is using
([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

## Halving Context

TAO halving rules use issuance milestones. A network-level TAO halving reduces TAO emission after
total TAO issuance reaches the relevant supply milestone
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

That means issuance is part of the halving trigger context. It is not a date, countdown, or short
term timing signal.

The distinction keeps halving and issuance separate. Issuance names the counted supply context;
halving mechanisms name the emission-rate rules that use supply milestones
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

Alpha halving is related but separate. It concerns subnet alpha-token emission rate, while issuance
is the TAO circulating-supply context used by network-level TAO halving
([Glossary: Alpha Halving](https://docs.learnbittensor.org/resources/glossary#alpha-halving),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

## Recycling and Burning Context

Recycling adjusts issuance accounting by allowing deducted amounts to return to future emission.
Recycled tokens are subtracted from the issuance record so the same quantity can be emitted again
([Glossary: Recycling](https://docs.learnbittensor.org/resources/glossary#recycling),
[Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)).

Issuance names the supply record, while recycling describes a supply-flow adjustment that can return
amounts to future emission.

Burning removes tokens from spendable circulation without making that amount available for future
emission under the recycling rule
([Glossary: Burning](https://docs.learnbittensor.org/resources/glossary#burning),
[Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)).

Together, recycling and burning show why issuance is not the same as ordinary balance movement.
Recycling can make a deducted amount available again, while burning removes spendable circulation
without the same return mechanism.

## Supply Boundary

Issuance is high-level Bittensor tokenomics terminology. It is not a reward rule, subnet scoring
concept, or report of exact network totals
([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Use issuance when the focus is circulating TAO supply context. Use emission for creation and
allocation flow, halving for emission-rate rules, recycling for return-to-emission accounting, and
burning for removal from spendable circulation.

The narrow use keeps supply language precise. Issuance helps frame emission, halving, recycling, and
burning without replacing those concepts.

## Environment Boundary

Bittensor separates mainnet, testnet, and localnet environments. Issuance examples from one
environment should not be treated as evidence for another because supply state and testing
conditions can differ
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Introduction to Bittensor: Subnet development](https://docs.learnbittensor.org/learn/introduction#subnet-development)).

Localnet examples are isolated development examples. Testnet examples are shared non-production
examples. Mainnet issuance interpretation concerns production Bittensor tokenomics behavior.
