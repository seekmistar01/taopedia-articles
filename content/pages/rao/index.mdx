# RAO

RAO is a small-denomination unit for TAO amounts in Bittensor. One RAO equals one billionth of a TAO
([Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao)).

The term is useful when Bittensor material needs more precision than a whole-TAO amount. TAO is the
token used around subnet incentives, while RAO is the smaller unit used to express TAO-denominated
quantities ([Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction),
[Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao)).

## Denomination Role

RAO changes the scale used to express a TAO amount; it does not create a separate token category.
The conversion boundary is denomination-first: RAO is a fractional unit of TAO
([Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao)).

That boundary matters whenever an article discusses small quantities, rewards, or accounting
figures. RAO keeps the unit precise without changing the asset being described.

## Display Precision Boundary

RAO is helpful when a TAO amount needs to be shown below whole-token precision. Because one RAO is
one billionth of a TAO, the unit lets small TAO-denominated balances or rewards be written without
changing the underlying token label
([Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao)).

This is a presentation boundary, not a separate accounting class. A value shown in RAO remains a
TAO-denominated amount; the unit only changes how finely that amount is expressed.

## Distinction from TAO

RAO and TAO are two scales for the same TAO-denominated value. TAO is the incentive token used
around subnet activity, and RAO is a fractional unit of that token
([Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction),
[Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao),
[Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau)).

- **TAO** — the network token at whole-unit scale.
- **RAO** — the smaller denomination of the same TAO-denominated value.

Because the glossary fixes one RAO at one billionth of a TAO, converting between the two scales is a
fixed denomination change rather than a market exchange, which is why RAO appears in fee quotes and
emission accounting without introducing a second token
([Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao),
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction)).

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For Rao, this
sequence gives readers a boundary for interpreting unit-conversion examples and balance-accounting
notes.

Localnet examples are isolated and reflect local chain state, so they are useful for controlled
experiments rather than evidence of live Bittensor behavior. Testnet examples add shared
non-production conditions, which can reveal integration behavior without touching mainnet state.

On mainnet, Rao examples should be read as live production balance behavior on the production
Bittensor network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet, so outcomes from one environment should not be treated as
proof of behavior in another.

## Transaction Fees Use RAO for Precise Charges

[Understanding Fees](https://docs.learnbittensor.org/learn/fees) documentation describes extrinsic
fees using RAO units for length and weight components. RAO therefore appears in fee vocabulary as
the fine-grained TAO denomination for charging encoded transaction size and execution weight, not as
a separate token class ([Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao)).

Readers comparing fee quotes should keep RAO inside TAO-denominated accounting. A fee stated in RAO
still describes a charge against TAO balances on the network.

## Existential Deposit Thresholds Can Use RAO Units

Existential-deposit material can express minimum balance thresholds in RAO when documentation needs
sub-whole-TAO precision. The
[Glossary: Existential Deposit](https://docs.learnbittensor.org/resources/glossary#existential-deposit)
describes the minimum balance required to keep an account active, while RAO supplies the smaller
unit for stating that threshold precisely
([Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao)).

That usage keeps account-activation vocabulary aligned with TAO balances. RAO changes the display
scale; it does not introduce a second network token.

## Distinction from Protocol Alpha

RAO and protocol alpha answer different token questions. RAO is a TAO denomination for fine-grained
TAO-denominated amounts, while protocol alpha names subnet-specific alpha tokens in Dynamic TAO
tokenomics ([Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao),
[Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha)).

- **RAO** — a fractional unit of TAO for precise TAO-denominated accounting.
- **Protocol alpha** — a subnet-scoped token asset, not a TAO denomination.

## Distinction from Stake

RAO and stake name different things in TAO-denominated material. RAO is a denomination unit equal to
one billionth of a TAO, while stake is the amount of currency tokens delegated to a validator within
a subnet, including both self-stake and stake delegated from others
([Glossary: Stake](https://docs.learnbittensor.org/resources/glossary#stake)).

- **RAO** — a fractional unit for expressing a TAO amount at finer precision.
- **Stake** — a delegated quantity of tokens held against a validator, which RAO can express but
  does not define.

## Distinction from Tempo

RAO and tempo describe different axes of subnet accounting. RAO is a TAO denomination that sets the
scale for expressing token amounts, while tempo is a subnet hyperparameter that determines how
frequently epochs run automatically, measured as a number of blocks
([Glossary: Tempo](https://docs.learnbittensor.org/resources/glossary#tempo)).

- **RAO** — a unit of token amount, fixing how finely a TAO value is expressed.
- **Tempo** — a unit of timing cadence in blocks, governing how often epochs run.

## Distinction from Yuma Consensus

RAO is a TAO denomination unit equal to one billionth of a TAO. It expresses TAO-denominated amounts
at finer precision; it does not name the on-chain process that turns validator weights into emission
shares ([Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao)).

Yuma Consensus is the on-chain process that converts submitted validator weights into emission
shares after evaluation completes. Emission amounts may be quoted in RAO for precision, but Yuma
Consensus names the settlement mechanism, not the unit scale
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Readers should keep denomination vocabulary separate from consensus settlement. RAO answers how
finely a TAO amount is expressed; Yuma Consensus answers how validator weights become emission
shares in a subnet ([Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **RAO** — fractional TAO denomination for precise accounting.
- **Yuma Consensus** — on-chain weight aggregation that settles emissions.

## Distinction from Emission

Emission context often needs exact units because rewards can be divided across subnet roles.
Bittensor emissions distribute reward material through the network's tokenomics, while RAO supplies
the smaller TAO-denominated unit for precise amounts
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao)).

RAO therefore helps with precision around emissions without changing the emission concept itself.
Emission describes reward creation and distribution; RAO describes the unit scale for TAO amounts in
that discussion.

## Reader Boundary

RAO is best read as unit vocabulary: it expresses small TAO-denominated amounts in contexts such as
incentives and emissions ([Glossary: RAO](https://docs.learnbittensor.org/resources/glossary#rao),
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

This article does not provide market interpretation, reward forecasts, account analysis, or
point-in-time network summaries. Exact observed amounts belong to the source and context where they
were measured.
