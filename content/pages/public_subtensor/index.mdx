# Public Subtensor

Public Subtensor refers to publicly reachable Bittensor chain access. Official Bittensor
documentation uses Subtensor to describe the blockchain layer that supports Bittensor network state.

References:
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor),
[Subtensor](https://docs.learnbittensor.org/subtensor-nodes)

## Network Context

The term is useful when distinguishing public chain access from local development context. Public
Subtensor belongs to the live network side of that distinction, while local blockchain context is
private and development-oriented.

References:
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from local testing to testchain and then mainchain. Public
Subtensor should be read against that staged path because it names public chain access, not the
private local development environment.

In local testing, a developer can observe chain behavior in an isolated local blockchain. That can
be useful for understanding state changes, subnet mechanics, or tool behavior, but it should not be
described as Public Subtensor because the chain is not publicly reachable network access.

On testchain, public access can expose shared testing-network state. That gives stronger evidence
about public endpoint behavior than a private local run, while still keeping the result separate
from production mainchain state.

On mainchain, Public Subtensor belongs to live production chain access. The
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)
defines it as publicly reachable access to the Subtensor chain layer, so production readings should
keep the network, endpoint context, queried state, and block timing attached.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A Public Subtensor example should therefore make clear
which public network context is being discussed, rather than treating public access as a single
environment-independent signal.

This distinction matters because Public Subtensor names an access context, not a guarantee about the
meaning of every value read from the chain. The network environment explains where the state is
observed, while the relevant protocol or subnet article explains what that state means.

For readers, this keeps public chain examples from sounding broader than their environment supports.
Testnet and mainnet can both involve public Subtensor access, but local blockchain examples remain
private development context and should be interpreted separately.

## Relationship to Subtensor

Public Subtensor and Subtensor are related but name different layers of the same picture. The
[Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor) describes
Bittensor's layer 1 blockchain and the system of record for transactions and rankings, while the
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)
describes publicly reachable access to that chain layer.

For readers, Subtensor is the chain layer itself, while Public Subtensor names live public access to
it. The term does not rename Subtensor and should not be read as a separate blockchain.

References: [Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor),
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)

## Relationship to Local Blockchain

Public Subtensor and local blockchain name opposite sides of Bittensor chain access. The
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)
describes publicly reachable chain access, while the
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)
describes a private chain used for development and testing.

For readers, Public Subtensor belongs to live network access context, while local blockchain belongs
to isolated development context. The two terms should not be read as interchangeable network labels.

References:
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor),
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)

## Relationship to Bittensor Networks

Public Subtensor and Bittensor Networks name related but different parts of live chain context. The
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference groups
mainnet, testnet, and localnet, while the
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)
describes publicly reachable access to the Subtensor chain layer.

For readers, Bittensor Networks is the environment label, while Public Subtensor names public access
to Subtensor inside mainnet and testnet context. It does not describe private local blockchain
development.

References: [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)

## Reader Boundary

This page defines the term. It does not provide endpoint recommendations, connection steps, or
node-operation guidance. Readers should use current official Bittensor documentation when they need
operational details.

## Relationship to Coinbase

Public Subtensor and coinbase are related but different in scope: one is access to the chain, the
other is a mechanism that runs inside it. Public Subtensor is publicly reachable access to the
Subtensor chain layer. The
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)
documentation describes coinbase as the per-block protocol mechanism that drives TAO emission,
accumulates pending emissions, and triggers Yuma Consensus rounds at epoch boundaries, while the
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)
describes the public chain-access concept.

Coinbase runs as part of the Subtensor runtime that Public Subtensor exposes access to. Through a
public endpoint, an observer can read chain state that coinbase's per-block execution produces, but
the access path is not the emission mechanism itself. Public Subtensor names how the chain is
reached; coinbase names the per-block process executing within that chain. One is a connectivity
concept, the other is an emission mechanism — accessing the chain and producing its emissions are
different things.
