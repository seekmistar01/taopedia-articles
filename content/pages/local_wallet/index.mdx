# Local Wallet

A local wallet refers to Bittensor wallet files stored on a user's own machine. Official Bittensor
documentation discusses wallet files and key material as part of wallet and key management.

References:
[Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet),
[Wallets](https://docs.learnbittensor.org/keys/wallets)

## Wallet Security Context

The term is useful because local storage affects how wallet access is protected. A local wallet is
part of the user's key-management responsibility, so the machine, wallet files, and recovery
material all belong to the same security context.

References:
[Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet),
[Working with Keys](https://docs.learnbittensor.org/keys/working-with-keys)

## Local and External Wallets

Local wallet describes where the wallet files are stored. It does not mean the wallet is less
important than an external wallet, only that custody and protection depend on the user's local
machine and backup practices. This distinction helps readers separate wallet location from the
underlying need to protect sensitive wallet material.

Reference:
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)

## Wallet Location Context

The [Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet)
describes a local wallet as a Bittensor wallet created on the user's machine. The related
[Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location) term
names the directory path where generated wallet files are stored locally, so local wallet and wallet
location describe custody and storage context rather than an on-chain role.

References:
[Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet),
[Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location)

## Relationship to Coldkey Swap

Local wallet and coldkey swap are related but different parts of Bittensor wallet vocabulary. A
local wallet describes Bittensor wallet files stored on a user's machine — including the coldkey
files that hold the cryptographic material for controlling hotkeys and their on-chain registrations.
Coldkey swap is the on-chain identity migration that moves all hotkeys associated with one coldkey
to a new coldkey address without resetting their subnet registrations or removing existing stake.
The [Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet)
describes wallet files in local custody, and the
[Coldkey Swap](https://docs.learnbittensor.org/keys/coldkey-swap) documentation describes how
on-chain identity can be migrated to a new coldkey.

After a coldkey swap, the relationship between local wallet files and on-chain control changes.
Before the swap, the coldkey files in the original local wallet are the controlling identity for a
set of hotkeys. After the swap completes on-chain, the new coldkey's wallet files become the
controlling identity — the old coldkey files no longer control those hotkeys even if they remain on
disk. Local wallet names where key files are kept and whose custody they are under; coldkey swap
names the on-chain event that determines which local wallet's coldkey actually holds control.

References:
[Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet),
[Coldkey Swap](https://docs.learnbittensor.org/keys/coldkey-swap)

## Reader Boundary

This page defines the term. It does not ask readers to expose keys, enter a recovery phrase, or
follow a specific wallet action. Readers should use current official wallet documentation for
security-sensitive wallet actions.
