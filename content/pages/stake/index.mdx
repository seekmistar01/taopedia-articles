# Stake

Stake is the amount of currency tokens associated with a validator UID in a Bittensor subnet. The
[Glossary: Stake](https://docs.learnbittensor.org/resources/glossary#stake) entry defines it as the
currency tokens delegated to a validator UID, including both the validator's own stake and stake
delegated from others.

## Validator Context

Stake is read in relation to validator support. The official
[Staking/delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)
explains that TAO holders can stake liquidity to a validator and that the validator's total stake,
including delegated stake, helps determine consensus power and emissions. The same page also frames
staking and delegation as closely related terms.

This makes stake a position-linked amount rather than a generic wallet balance. It describes support
associated with a validator in a subnet context.

## Subnet Context

Stake is local to subnet state. The
[Staking into Subnets](https://docs.learnbittensor.org/staking-and-delegation/delegation#staking-into-subnets)
section explains that stake is always local to a subnet and is held in subnet alpha-token
denominations.
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets#liquidity-pools)
uses the term alpha outstanding for alpha held by subnet participants and describes it as total
stake in the subnet.

This subnet boundary matters because stake is not a single network-wide number that means the same
thing in every context. It depends on the subnet and the validator position being discussed.

## Relationship to Stake Weight

Stake and stake weight are related but not identical. Stake names the associated token amount, while
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) defines
stake weight as the computed total stake value used for validator consensus power and emissions.
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets#validator-stake-weight)
places that computed measure in the validator stake-weight context.

For a reader, the distinction is simple: stake is the underlying associated amount; stake weight is
the subnet-specific measure derived from stake for consensus interpretation.

## Staking Process Context

Bittensor uses separate terms for validator support action and validator support amount.
[Glossary: Staking](https://docs.learnbittensor.org/resources/glossary#staking) names the process of
attaching TAO to a validator hotkey, while stake names the token amount already associated with a
validator UID in a subnet. That distinction helps readers separate action language from amount
language when reading staking and consensus articles.

References: [Glossary: Staking](https://docs.learnbittensor.org/resources/glossary#staking),
[Glossary: Stake](https://docs.learnbittensor.org/resources/glossary#stake)

## Relationship to Multiple Mechanisms

Stake is usually discussed for one incentive mechanism at a time, but subnets can also run multiple
mechanisms in parallel. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism) notes that each
subnet has one or more incentive mechanisms.

For readers, stake still names validator-associated tokens inside one subnet, but the mechanism
context matters when reading how that subnet evaluates work and allocates consensus influence.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Incentive Mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism)

## Relationship to Dividends

Stake and dividends are related but different parts of Bittensor vocabulary. Stake names the TAO
associated with a validator that gives the validator influence in a subnet, while dividends name the
validator-facing emission share that flows back to that stake. The
[Glossary: Stake](https://docs.learnbittensor.org/resources/glossary#stake) describes stake as
backing a validator's weight, and the
[Glossary: Dividends](https://docs.learnbittensor.org/resources/glossary#dividends) describes
dividends as the validator-side reward distributed from emissions.

For readers, stake names the committed input that gives a validator standing, while dividends name
the reward outcome that standing can earn. Stake is what is committed; dividends are what may be
returned.

References: [Glossary: Stake](https://docs.learnbittensor.org/resources/glossary#stake),
[Glossary: Dividends](https://docs.learnbittensor.org/resources/glossary#dividends)

## Reader Boundary

Stake should not be read as a live balance report, a validator recommendation, or a complete
explanation of staking operations. It is a term for the amount associated with validator support
inside subnet staking and consensus. Current balances, staking actions, fee effects, and slippage
depend on current official tooling and subnet state.
