# EMA

EMA stands for exponential moving average, a way of smoothing a value over time by blending each new
observation with the running average instead of replacing it. In Bittensor, the
[EMA](https://docs.learnbittensor.org/learn/ema) documentation describes this smoothing as how
[validator-miner bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)
are updated across epochs rather than recomputed from scratch each time.

## Distinction from Liquid Alpha

The EMA is the smoothing method; liquid alpha is a feature that varies the EMA's alpha coefficient
by how well a validator aligns with consensus, instead of holding it fixed. The alpha low and alpha
high bounds set the range that coefficient may take
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

- **EMA** — the moving-average technique that blends new and prior values.
- **Liquid alpha** — varies the EMA coefficient by consensus alignment.

## Distinction from Validator-Miner Bonds

Validator-miner bonds are the values being smoothed; the EMA is the rule for updating them. One is
the quantity, the other is the method applied to it across epochs
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Validator-miner bonds** — the accumulated validator-to-miner relationship.
- **EMA** — how those bonds are updated each epoch.

## Distinction from a Simple Moving Average

An exponential moving average weights recent observations more heavily and lets older values fade
gradually. A simple moving average takes the equal-weighted mean of a fixed recent window
([EMA](https://docs.learnbittensor.org/learn/ema)).

- **EMA** — recency-weighted blend where older values fade gradually.
- **Simple moving average** — equal-weighted mean of a fixed recent window.

Bond smoothing favors gradual accumulation over abrupt window resets
([Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)).

## Distinction from Bonds Moving Average

The EMA is the smoothing technique applied to validator-miner bonds. Bonds moving average is the
per-subnet `BondsMovingAverage` hyperparameter that configures that smoothing on a selected netuid
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **EMA** — the smoothing technique applied to bonds.
- **Bonds moving average** — the per-subnet hyperparameter that configures that smoothing.

## Distinction from Yuma Consensus

EMA names the smoothing technique applied to validator-miner bonds across epochs. Yuma Consensus
settles validator weight submissions into emission shares each tempo
([EMA](https://docs.learnbittensor.org/learn/ema),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **EMA** — smoothing technique for bond updates across epochs.
- **Yuma Consensus** — settlement turning validator weights into emission shares.

Smoothing vocabulary names bond updates over time; Yuma vocabulary names reward settlement
([Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)).

## Distinction from Alpha Sigmoid Steepness

The alpha that the steepness curve assigns is the EMA smoothing coefficient used when updating
validator-miner bonds. EMA is the technique that blends each epoch's new bond value with the prior
one, while alpha sigmoid steepness shapes how the coefficient feeding that EMA is set from a
validator's consensus alignment under the liquid alpha feature
([EMA](https://docs.learnbittensor.org/learn/ema),
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)).

- **EMA** — the moving-average technique that blends new and prior bond values.
- **Alpha sigmoid steepness** — shapes the coefficient that EMA uses under liquid alpha.

## Reader Boundary

EMA should not be read as a subnet hyperparameter value or as liquid alpha itself. It names the
exponential moving-average technique Bittensor uses to update validator-miner bonds across epochs
([EMA](https://docs.learnbittensor.org/learn/ema),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Live bond-smoothing behavior on a subnet also depends on that netuid's hyperparameter state,
including bonds moving average and any liquid-alpha settings
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).
