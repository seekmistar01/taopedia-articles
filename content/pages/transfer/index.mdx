# Transfer

A transfer is Bittensor terminology for sending TAO from one wallet address to another. The
[Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer) describes it as
the process of sending TAO tokens between wallet addresses in the Bittensor network.

## Address Context

Transfers rely on wallet addresses as destinations. The
[Wallet Address glossary](https://docs.learnbittensor.org/resources/glossary#wallet-address)
describes a wallet address as a public-key-derived identifier used as a TAO destination.

## Wallet Context

In Taopedia prose, transfer should be read as wallet-movement vocabulary. It is not a staking
position, subnet role, or live balance claim. It also does not describe the protected key material
that authorizes account activity.

## Transfer Relocates Existing TAO Balances

A transfer moves TAO that already exists on chain from one wallet address to another. It does not
create new emission proceeds or change subnet registration state by itself
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer),
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)).

Readers should keep supply-creation vocabulary separate from wallet-to-wallet movement vocabulary.

## Destination Address Labels the Receiver

The [Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address)
describes the public-key-derived identifier used as a TAO destination. Transfer actions cite that
address label; they do not by themselves prove possession of the receiving key material
([Bittensor wallet documentation](https://docs.learnbittensor.org/keys/wallets)).

## Transfer Vocabulary Excludes Operation Steps

This article is a concept boundary. It does not provide transaction steps, CLI commands, or wallet
operation guidance. Wallet and fee documentation covers how to submit transfers on chain
([Bittensor wallet documentation](https://docs.learnbittensor.org/keys/wallets),
[Transaction Fees in Bittensor](https://docs.learnbittensor.org/learn/fees)).

Readers should keep procedural claims tied to those references rather than to transfer vocabulary
alone.

## State-Changing Transaction Context

Sending TAO between wallet addresses is a state-changing chain action rather than a passive read.
The [Transaction Fees](https://docs.learnbittensor.org/learn/fees) documentation separates free
chain reads from state-changing actions that the runtime must process, which is the context in which
wallet transfers are submitted on chain.

## Moving TAO Between Wallet Addresses

A transfer sends TAO from one Bittensor wallet address to another. The
[Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer) describes that
movement in simple wallet terms: one address sends TAO, and another address receives it.

Readers should read transfer as free TAO movement between addresses. It is not the same action as
staking TAO to a validator or registering on a subnet.

## Submitted as an On-Chain Transaction

Sending a transfer is a state-changing action on Subtensor. The
[Transaction Fees in Bittensor](https://docs.learnbittensor.org/learn/fees) documentation groups
wallet transfers with other submitted actions that can incur base chain fees when processed by the
runtime.

That means a transfer has both a destination and a submission cost. This article names the movement;
fee documentation names what executing the transfer can cost on chain.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For transfer, that
sequence changes how readers should interpret evidence about TAO movement between addresses.
Transfer is a network-level coldkey operation rather than a subnet-specific action on a subnet such
as netuid 1, but the environment separation still applies to every balance movement.

In localnet, transfers can be tested in an isolated environment. Localnet transfer outcomes reflect
local chain state and do not represent real TAO value or production wallet balances.

On testnet, transfers operate in a shared, non-production network with testnet TAO that is separate
from mainnet balances.

On mainnet, a transfer is a live Subtensor operation that moves real TAO between coldkey addresses.
The [Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer) describes the
TAO transfer operation that applies on the production network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A transfer example from one environment should not be read
as representing wallet balances or TAO amounts in another environment.

## Coldkey Authority Signs Transfers

[Bittensor wallet documentation](https://docs.learnbittensor.org/keys/wallets) describes coldkeys as
the keys that control TAO balances and authorize sensitive financial operations. Transfer vocabulary
therefore belongs to coldkey-signed balance movement rather than to hotkey operational actions.

Readers should not cite a destination address alone as proof that a transfer occurred; submission
still requires the controlling key material for the sending wallet.

## Distinction from Moving Stake

A transfer and moving stake both relocate value, but they act on different balances. A transfer
sends free, unstaked TAO from one coldkey wallet address to another, while moving stake relocates an
existing stake position from one hotkey to another without first returning it to free balance, so a
transfer changes which address holds free TAO and moving stake changes which hotkey a stake supports
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer),
[Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

- **Transfer** — sending free TAO between coldkey wallet addresses.
- **Moving stake** — relocating an existing stake position between hotkeys without unstaking.

## Distinction from Staking

A transfer and staking move TAO toward different ends. A transfer sends free TAO from one wallet
address to another, while staking attaches TAO to a validator hotkey to support it inside a subnet
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer),
[Glossary: Staking](https://docs.learnbittensor.org/resources/glossary#staking),
[Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)).

A transfer only relocates free balance between addresses. Staking places TAO behind validator
support on a subnet and does not happen automatically when two addresses exchange TAO in a send
example.

- **Transfer** — sending free TAO between wallet addresses.
- **Staking** — attaching TAO to a validator hotkey to support it inside a subnet.

## Distinction from Coldkey

A transfer and a coldkey swap change different things. A transfer changes which address holds free
TAO, while a coldkey swap migrates an on-chain identity to a new coldkey, reassigning its hotkeys
and preserving their registrations
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer),
[Coldkey Swap](https://docs.learnbittensor.org/keys/coldkey-swap)).

- **Transfer** — a token movement between addresses.
- **Coldkey swap** — an identity migration to a new coldkey.

## Distinction from Address Poisoning Scams

Address poisoning and a transfer describe a deception and the action it targets. Address poisoning
names a destination-selection scam that plants a lookalike wallet address, while a transfer is the
wallet action that moves TAO to a chosen destination address
([Address Poisoning Scams](https://docs.learnbittensor.org/keys/address-poisoning-scams),
[Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer)).

Transfer names the legitimate action of sending funds to an address; address poisoning corrupts
which address that action targets rather than breaking the transfer itself.

- **Transfer** — sending TAO to a chosen wallet address.
- **Address poisoning scam** — planting a lookalike address so the transfer reaches the wrong
  recipient.

## Distinction from SS58 Encoded

A transfer sends TAO between wallet addresses, while SS58 encoded names the compact address format
used to identify those public-key-derived destinations
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer),
[Glossary: SS58 Encoded](https://docs.learnbittensor.org/resources/glossary#ss58-encoded)).

Transfer names the movement action; SS58 encoded names the address format a transfer targets. The
two terms should not be read as interchangeable wallet concepts.

- **Transfer** — the action of sending TAO to a destination address.
- **SS58 encoded** — the address format that identifies that destination.

## Distinction from Coinbase

A transfer is a user-initiated send of existing TAO between wallet addresses. Coinbase is the
per-block protocol mechanism that drives TAO emission, injects TAO into emitting subnets' pool
reserves, and checks epoch boundaries to trigger Yuma Consensus rounds
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer),
[Glossary: Coinbase](https://docs.learnbittensor.org/resources/glossary#coinbase)).

A transfer relocates TAO that already exists and changes no token supply. Coinbase is a
protocol-level runtime process that generates and routes new emission proceeds each block.

- **Transfer** — a wallet-to-wallet send of existing TAO.
- **Coinbase** — the per-block emission mechanism that produces and routes new TAO.

## Distinction from Crowdloans

A crowdloan can finalize into a transfer rather than a call. The crowdloan is the funding campaign,
while a transfer is a balance movement between accounts and is one of the two finalization-action
types the documentation allows ([Crowdloans](https://docs.learnbittensor.org/subnets/crowdloans),
[Crowdloan Lifecycle](https://docs.learnbittensor.org/subnets/crowdloans#crowdloan-lifecycle),
[Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer)).

- **Crowdloan** — the pooled-funding campaign that triggers one finalization action.
- **Transfer** — a balance movement between accounts, used as a finalization action.

## Reader Boundary

Transfer should not be read as staking, subnet registration, or coinbase emission. It names
wallet-to-wallet movement of existing TAO between addresses
([Glossary: Transfer](https://docs.learnbittensor.org/resources/glossary#transfer)).
