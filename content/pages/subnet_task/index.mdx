# Subnet Task

A subnet task is the work target miners are expected to perform inside a Bittensor subnet
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The term is narrower than the whole incentive mechanism. It names what work is being produced, not
the full scoring or emission process around that work.

## Work Target

The task is the subnet's work definition. Different subnets can ask miners to produce different
outputs, so the task keeps subnet vocabulary tied to the market being discussed
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task)).

This makes subnet task a useful boundary term. It answers what work the subnet asks for before
moving to who performs it or how it is evaluated.

## Miner and Validator Context

Miners produce work for the task, while validators evaluate that work. Subnet miner and subnet
validator glossary entries define those two roles around task performance and evaluation
([Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator)).

The task is not the same as either role. It is the work target that makes those roles meaningful
inside the subnet.

## Incentive Mechanism Context

An incentive mechanism combines the task, protocol, and evaluation design. The subnet task is the
work component of that larger mechanism
([Glossary: Incentive Mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

A task can say what miners should produce, but it does not by itself explain every scoring rule,
validator behavior, or emission outcome.

## Relationship to Subnet Protocol

Subnet task and subnet protocol are related but different parts of incentive design. The task names
the work target, while the protocol defines how miners and validators exchange requests and
responses around that work
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task)).

This distinction keeps the work target separate from the interaction rules used to evaluate it.

## Weights and Consensus Context

Validator weights are evaluation signals submitted after miners have produced work and validators
have evaluated it. Yuma Consensus then aggregates those signals into emission outcomes
([Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The subnet task sits before those consensus outcomes. It defines the work being evaluated.

## Multiple-Mechanism Context

Some subnets can run multiple incentive mechanisms, and each mechanism can define separate task
context. A task for one mechanism should not be treated as the task for every mechanism in the same
subnet
([Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)).

This keeps task claims scoped to the mechanism and subnet being discussed.

## Reader Boundary

Subnet task should be read as reference vocabulary for requested work inside a subnet. It is not a
miner performance report, validator strategy, reward guarantee, or proof that every subnet uses the
same evaluation standard
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The stable article-level point is that a subnet task defines the work miners produce and validators
evaluate.
