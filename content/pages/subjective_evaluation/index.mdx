# Subjective Evaluation

Subjective evaluation is the challenge that much of the work a subnet asks for has no single
objectively correct answer, so two validators can reasonably score the same submission differently.
Bittensor handles this not by checking work against an answer key, but by treating the agreement of
validators, weighted by stake, as the standard the network accepts. On a subnet such as netuid 1,
that consensus is what turns many individual judgments into one result.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For subjective
evaluation, that sequence changes how readers should interpret stake-weighted consensus and scoring
examples.

In localnet, consensus-as-standard mechanics can be tested in an isolated environment. Localnet
validator agreement does not represent production scoring outcomes.

On testnet, subjective evaluation flows can be exercised in a shared non-production network. Testnet
consensus results are separate from mainnet subnet state.

On mainnet, subjective evaluation concerns live production validator agreement on the selected
subnet. Observed consensus values depend on that subnet's current chain state
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A subjective evaluation example from one environment
should not be read as representing production consensus behavior in another environment.

## Distinction from Clipping

Subjective evaluation names the challenge that subnet work often has no single correct answer.
Clipping is the Yuma step that trims validator weights that depart too far from stake-weighted
agreement ([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Clipping keeps one validator from setting the standard alone when judgments diverge.

- **Subjective evaluation** — scoring work with no single objectively correct answer.
- **Clipping** — trimming weights that depart from stake-weighted agreement.

## Distinction from Rank

Subjective evaluation names why validators may disagree on open-ended work. Rank is the consensus
standing that divides miner-side rewards after validator weights are aggregated
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Yuma Consensus: Miner emissions](https://docs.learnbittensor.org/learn/yuma-consensus#miner-emissions)).

The consolidated consensus outcome becomes an emission input rather than a label on one submission.

- **Subjective evaluation** — no single answer key for open-ended work.
- **Rank** — consensus standing that drives miner-side emission shares.

## Distinction from Weight Copying

Subjective evaluation names the challenge that validators must contribute distinct judgments when no
objective answer exists. Weight copying is when validators reuse visible weight signals instead of
forming independent evaluations
([Weight Copying Problem](https://docs.learnbittensor.org/concepts/weight-copying-in-bittensor),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector)).

Copying imitates visible scores rather than adding independent judgment.

- **Subjective evaluation** — open-ended work with no single correct answer.
- **Weight copying** — reusing visible weight signals instead of independent scoring.

## Distinction from Subnet Scoring Model

Subjective evaluation and the subnet scoring model meet where judgment becomes a defined method.
Subjective evaluation is the challenge that subnet work often has no single correct answer, so
validators may score the same submission differently, while the subnet scoring model is the
subnet-defined method that turns returned work into the scores validators submit
([Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

A scoring model can be defined even when the underlying judgment stays subjective and consensus
resolves the differences
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

- **Subjective evaluation** — the challenge that work has no single correct answer.
- **Subnet scoring model** — the subnet-defined method for turning work into scores.

## Distinction from Weight Setting

Subjective evaluation is the challenge that subnet work often has no single correct answer, so
validators may score the same submission differently. Weight setting is the action each validator
uses to submit its own scores, so consensus over those submitted weights, not an answer key, is what
resolves the subjectivity
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Subjective evaluation explains why validators may disagree. Weight setting is how each validator
records its scores for consensus to merge.

- **Subjective evaluation** — scoring work that has no single objectively correct answer.
- **Weight setting** — the action each validator uses to submit its own scores.

## Distinction from Validator Trust

Subjective evaluation is the challenge that subnet work often has no single correct answer, so
validators may score the same submission differently. Validator trust measures how consistently a
validator's clipped weights align with the consensus outcome that resolves those differing judgments
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Glossary: Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Validator trust measures how closely a validator's weights aligned with the consensus result after
clipping.

- **Subjective evaluation** — no single answer key; validators may disagree on scores.
- **Validator trust** — alignment with the consensus outcome after clipping.

## Distinction from Yuma Consensus

Subjective evaluation names why open-ended work has no single correct answer. Yuma Consensus is the
mechanism that aggregates validator weight submissions into emission shares each tempo
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

- **Subjective evaluation** — the challenge of scoring work without an answer key.
- **Yuma Consensus** — on-chain settlement that turns validator weights into emission shares.
  Subjective evaluation names why open-ended work has no answer key; Yuma processes validator
  weights into rewards
  ([Yuma Consensus: Validator emissions](https://docs.learnbittensor.org/learn/yuma-consensus#validator-emissions)).

## Reader Boundary

This page describes the concept at a high level. How a particular subnet defines and scores quality
depends on its own incentive mechanism, which varies between subnets and changes over time, so the
exact standard is subnet-specific. The durable point is the principle: where there is no objective
answer, stake-weighted validator consensus becomes the accepted one.
