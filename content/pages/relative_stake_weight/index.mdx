# Relative Stake Weight

Relative stake weight is a subnet-local measure of validator influence in Bittensor. The
[Relative Stake Weight glossary entry](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)
frames it as one validator's stake expressed as a proportion of stake held by active validators in
the same subnet.

## Proportional Influence

The key idea is proportion. A validator's stake weight is meaningful on its own, but relative stake
weight compares that value against the validator set in the same subnet. The
[Stake Weight glossary entry](https://docs.learnbittensor.org/resources/glossary#stake-weight)
provides the broader stake-weight term, while relative stake weight narrows it to a subnet-local
share.

## Subnet Context

Relative stake weight should be read inside a subnet context. In
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets), validators
measure miner work according to subnet-specific standards before those evaluations feed consensus.
Relative stake weight helps describe how much influence a validator signal carries within that
subnet setting.

## Consensus Context

The measure belongs near Yuma Consensus because validator influence matters when submitted signals
are aggregated. [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) provides the
broader consensus frame for miner incentives and validator dividends, while relative stake weight
names a proportional input-side concept inside subnet consensus.

## Difference from Stake Weight

Stake weight and relative stake weight are related but not identical. Stake weight names the
computed stake influence, while relative stake weight expresses that influence as a share of the
relevant validator set. The
[Relative Stake Weight glossary entry](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)
keeps that comparison attached to the selected subnet.

## Multiple-Mechanism Context

When a subnet uses more than one incentive mechanism, mechanism context still matters. The
[multiple incentive mechanisms](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets)
material separates mechanism-specific evaluation systems, so relative stake weight should be read
alongside the validator and mechanism context being discussed.

## Reading the Term

Relative stake weight is proportional consensus vocabulary. It is not a validator recommendation,
standalone miner-quality score, or universal network ranking. The
[Relative Stake Weight glossary entry](https://docs.learnbittensor.org/resources/glossary#relative-stake-weight)
anchors the term, with stake-weight and Yuma references supplying the surrounding consensus
structure.
