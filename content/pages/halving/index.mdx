# Halving

Halving describes a reduction in token emission rate. In Bittensor, it is a tokenomics term for
scheduled supply-rate changes
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)).

The term should be read alongside emissions because halving affects how quickly new tokens enter the
emission path ([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Emission Rate Context

Halving changes the rate at which new tokens are emitted. It does not by itself describe staking
outcomes, validator evaluation, or subnet work quality
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)).

For readers, the useful boundary is simple: halving names a supply-rate change, while emissions
describe the broader distribution process affected by that rate
([Emission](https://docs.learnbittensor.org/learn/emissions)).

## Issuance Threshold Context

Bittensor halving language is tied to supply thresholds. TAO halving is described as reducing the
emission rate when the relevant TAO issuance threshold is reached
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

That makes halving a supply milestone concept rather than a standalone reward rule for one subnet
role ([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

## TAO and Alpha Scope

Halving can be discussed at more than one scope in Bittensor tokenomics. TAO halving belongs to the
network asset, while alpha halving belongs to subnet alpha-token emission
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

This distinction keeps a TAO halving from being confused with a subnet alpha halving. The word is
shared, but the asset and scope must be named
([Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

## Supply Vocabulary Context

Halving is different from burning. Burning removes tokens that already exist, while halving reduces
the rate at which new tokens are emitted
([Glossary: Burning](https://docs.learnbittensor.org/resources/glossary#burning)).

The difference matters because both terms relate to supply, but they act at different points in the
token lifecycle ([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)).

## Reader Boundary

Halving should be read as a concept-level tokenomics term. It is not a staking guide, a validator
guide, or a full explanation of reward distribution
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)).

The stable article-level point is that halving names an emission-rate reduction, with Bittensor
using the idea for TAO and subnet alpha-token contexts
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For halving, that
sequence changes how readers should interpret examples of emission rate thresholds and halving
events.

In localnet, halving behavior can be tested in an isolated environment. Localnet emission rate
thresholds reflect local chain settings and do not represent production TAO or alpha supply state.

On testnet, halving behavior can be observed in a shared, non-production network. Testnet emission
rate milestones are separate from mainnet supply state.

On mainnet, halving is the live mechanism that reduces TAO and subnet alpha emission rates when
production supply thresholds are reached. The
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving) documentation describes the
halving mechanics that apply on the production network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A halving example from one environment should not be read
as representing emission rate or supply state in another environment.
