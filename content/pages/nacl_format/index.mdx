# NaCl Format

NaCl Format is Bittensor wallet-file terminology for a secure encryption format. The official
[NaCl Format glossary](https://docs.learnbittensor.org/resources/glossary#nacl-format) describes it
as a format that uses the NaCl library and is used when improving security for legacy Bittensor
wallets.

## Format Context

NaCl Format is not a Bittensor network role, subnet role, asset type, or address format. In Taopedia
context, the term is best read as encryption-format vocabulary for wallet-file protection.

## Wallet Context

Bittensor documentation treats wallets and keys as the surrounding context for wallet-file security.
NaCl Format sits beside that vocabulary as a file-protection term: it concerns the format used for
legacy wallet security, not a new wallet category.

## Wallet File, Seed Phrase, and Chain Identity

NaCl Format is useful when a reader needs to separate local wallet-file protection from the wallet's
on-chain identity. The wallet docs describe a Bittensor wallet through key pairs, balances, stake,
transactions, and subnet interactions; the NaCl glossary entry instead names a secure encryption
format used when updating legacy Bittensor wallets. In that sense, NaCl Format is about how
sensitive wallet material is protected locally, not about changing a wallet address, coldkey role,
hotkey role, or staking record.

It is also different from a seed phrase. A seed phrase is recovery material for a wallet's
cryptographic key pair, so losing it can mean losing access and leaking it can give someone else
control. NaCl Format may describe protected wallet-file storage, but it does not replace
recovery-phrase handling or make an exposed phrase safe.

For Taopedia readers, the short version is: use NaCl Format for the wallet-file security format, use
seed phrase or mnemonic for recovery material, and use wallet, coldkey, hotkey, or address when
talking about network identity and signing authority.

## Reading the Term

This article is a concept boundary. It does not provide wallet handling steps. For a reader, the
important point is that NaCl Format names an encryption format used in Bittensor wallet security
context.

## Relationship to Wallets

NaCl Format and wallets are related but different parts of Bittensor wallet vocabulary. A wallet is
the cryptographic identity formed by coldkey and hotkey pairs used to access TAO and sign
transactions, while NaCl Format describes the secure encryption format used when improving security
for legacy Bittensor wallet files. The
[Bittensor wallet documentation](https://docs.learnbittensor.org/keys/wallets) describes wallets as
the key-pair identity and on-chain control, while the
[Glossary: NaCl Format](https://docs.learnbittensor.org/resources/glossary#nacl-format) describes
NaCl Format as the encryption format for local wallet-file protection.

For readers, the wallet names the cryptographic identity and network authority, while NaCl Format
names the file-protection layer used to secure how that wallet material is stored locally.

References: [Bittensor wallet documentation](https://docs.learnbittensor.org/keys/wallets),
[Glossary: NaCl Format](https://docs.learnbittensor.org/resources/glossary#nacl-format)

## Relationship to Mnemonic

NaCl Format and mnemonic are related but different parts of Bittensor wallet-security vocabulary. A
mnemonic is the recovery phrase (seed phrase) from which wallet key pairs can be derived and which
must be protected as the ultimate recovery material, while NaCl Format describes the secure
encryption format used for local wallet-file storage when improving security for legacy wallets. The
[Glossary: Mnemonic](https://docs.learnbittensor.org/resources/glossary#mnemonic) describes the
mnemonic as the human-readable representation of the wallet seed, and the
[Glossary: NaCl Format](https://docs.learnbittensor.org/resources/glossary#nacl-format) describes
NaCl Format as an encryption format for wallet-file protection.

For readers, the mnemonic is the recovery material that should be stored offline and never exposed,
while NaCl Format is the encryption layer applied to wallet files stored on disk. NaCl Format
protects local file storage but does not replace mnemonic security practices — an exposed mnemonic
remains a custody risk regardless of wallet-file encryption.

References: [Glossary: Mnemonic](https://docs.learnbittensor.org/resources/glossary#mnemonic),
[Glossary: NaCl Format](https://docs.learnbittensor.org/resources/glossary#nacl-format)
