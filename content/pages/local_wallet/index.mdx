# Local Wallet

A local wallet is a Bittensor wallet whose coldkey and hotkey files are kept on the user's machine
through the local Bittensor installation
([Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet),
[Wallets, Coldkeys and Hotkeys](https://docs.learnbittensor.org/keys/wallets)).

## What It Represents

A local wallet names wallet files stored on the user's workstation, not browser-extension or
website-based custody under a different trust model
([Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet),
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)).

Local storage does not remove the need to protect coldkey and hotkey material or verify transaction
destinations before signing
([Working with Keys](https://docs.learnbittensor.org/keys/working-with-keys),
[Handle your Seed Phrase/Mnemonic Securely](https://docs.learnbittensor.org/keys/handle-seed-phrase)).

Local wallet describes where files are stored, not which Bittensor network they connect to. A local
wallet is not the same thing as localnet
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from External Wallet

An external wallet keeps wallet software or custody outside the local Bittensor wallet path. A local
wallet keeps files under the user's own local installation
([Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)).

- **Local wallet** — wallet files kept through the local installation.
- **External wallet** — custody or software outside that local path.

## Distinction from Wallet Location

Wallet location names the directory path where generated files are stored. Local wallet names the
wallet category that uses that local storage model
([Glossary: Wallet Location](https://docs.learnbittensor.org/resources/glossary#wallet-location)).

- **Local wallet** — wallet category using local file storage.
- **Wallet location** — directory path for generated wallet files.

## Distinction from Local Blockchain

Local blockchain names a private isolated chain for development. Local wallet names key material
stored on the user's machine
([Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)).

- **Local wallet** — on-machine wallet file storage.
- **Local blockchain** — private isolated development chain.

## Distinction from NaCl Format

NaCl format names how some legacy wallet files may be encrypted on disk. Local wallet names where
those files are kept
([Glossary: NaCl Format](https://docs.learnbittensor.org/resources/glossary#nacl-format)).

- **Local wallet** — wallet files stored on the user's machine.
- **NaCl format** — encryption format for some legacy wallet files.

## Distinction from Staking

Staking attaches TAO to a validator hotkey on-chain. Local wallet names where signing material is
stored on disk ([Glossary: Staking](https://docs.learnbittensor.org/resources/glossary#staking)).

- **Local wallet** — on-machine storage for wallet files and keys.
- **Staking** — on-chain act of attaching TAO to a validator hotkey.

## Distinction from Subtensor

Subtensor is Bittensor's layer-1 blockchain. Local wallet names client-side file storage for keys
([Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor)).

- **Local wallet** — wallet files stored locally on the user's machine.
- **Subtensor** — layer-1 blockchain that records transactions and rankings.

## Distinction from Yuma Consensus

Local wallet names on-machine custody for coldkey and hotkey material. Yuma Consensus converts
submitted validator weights into emission shares after evaluation completes
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Local wallet** — on-machine storage for coldkey and hotkey material.
- **Yuma Consensus** — on-chain weight aggregation that settles emissions.

## Reader Boundary

Local wallet should not be read as localnet, an on-chain wallet identity, or proof that keys are
stored only on one machine forever. It names a wallet created through the local Bittensor
installation on the user's workstation
([Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet)).
