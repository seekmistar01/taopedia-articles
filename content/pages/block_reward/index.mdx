# Block Reward

The block reward is the amount of new TAO the network creates each block and shares across the whole
network. Bittensor emission material puts the network emission at 1 TAO per block, roughly every 12
seconds, until the first halving, after which it fell to 0.5 TAO per block
([Emission](https://docs.learnbittensor.org/learn/emissions)).

## A Network-Wide Amount per Block

The reward is minted per block for the network as a whole rather than separately for each subnet.
That network-wide amount of TAO is then spread across subnets and their participants. So the block
reward describes how fast new TAO enters the system overall, before any split between subnets is
applied ([Emission](https://docs.learnbittensor.org/learn/emissions),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

## Halving the Reward

The block reward does not stay constant. At supply-based thresholds the emission documentation
describes the reward being cut by 50 percent. The first such halving lowered it from 1 TAO to 0.5
TAO per block, and later halvings will continue to halve it. Each cut slows how quickly new TAO is
issued, which is how issuance tapers as it approaches the documented 21 million TAO ceiling
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Halving names the supply-triggered rate cut; block reward names the active per-block mint figure
before and after each threshold
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

## Effect on Subnets

Because the per-block TAO is injected across subnets, a change in the block reward flows through to
them. When a halving cuts the reward, it also lowers the daily TAO emission and cuts the TAO
injected into subnet pools on a subnet such as netuid 1. Each subnet additionally emits its own
alpha each block, which is tracked separately from this network-wide TAO reward
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For block reward
and its effect on a subnet such as netuid 1, that sequence changes how readers should interpret
per-block TAO issuance and halving examples.

In localnet, block reward and coinbase cadence can be tested in an isolated environment. Localnet
issuance amounts do not represent production supply schedules.

On testnet, per-block emission can be exercised in a shared non-production network. Testnet halving
thresholds and block rewards are separate from mainnet chain state.

On mainnet, block reward describes live production TAO issuance per block across the network.
Observed amounts depend on how many halvings have occurred and current chain state
([Emission](https://docs.learnbittensor.org/learn/emissions)).

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A block-reward example from one environment should not be
read as representing production issuance in another environment.

## Coinbase Advances Emission Each Block

[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)
documentation describes coinbase as the per-block mechanism that advances emission processing on
[Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor). The block reward names
how much new [TAO](https://docs.learnbittensor.org/resources/glossary#tao-tau) enters the network in
that block; coinbase names the runtime step that carries that issuance forward through the emission
pipeline.

That distinction keeps the reward label separate from the processing label. Block reward answers how
much TAO is created per block for the whole network, while coinbase answers how the chain advances
emission accounting as each [block](https://docs.learnbittensor.org/resources/glossary#block) is
produced
([Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

## Each TAO Halving Advances the Index ADR Depends On

A block-reward halving leaves a footprint beyond the per-block mint rate. The
[Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)
entry uses the global TAO halving index as one of two terms in its formula for comparing a subnet's
distributed alpha against pool-injected alpha, alongside that subnet's own alpha halving index
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

That means each TAO-level halving that cuts the block reward also advances the shared index every
subnet's ADR calculation references, separate from whatever alpha halving progress an individual
subnet has made on its own.

## Distinction from Emission

Emission is the full create-and-allocate flow for TAO and subnet alpha. Block reward names only the
per-block TAO mint rate for the whole network before that amount is split across subnets
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

- **Block reward** — new TAO created per block network-wide.
- **Emission** — injection, tempo distribution, and participant payouts together.

[Emission](https://docs.learnbittensor.org/learn/emissions) documentation separates the emissions
process into injection and distribution. Injection adds new liquidity to each subnet's pools every
block, while distribution waits until the end of each
[tempo](https://docs.learnbittensor.org/resources/glossary#tempo) before participants extract the
accumulated value.

The [Glossary: Emission](https://docs.learnbittensor.org/resources/glossary#emission) states the
same rhythm in plain terms: every block, TAO is injected into each subnet, and every tempo the
accumulated value is extracted by miners, validators, stakers, and subnet creators through
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus).

Block reward vocabulary therefore belongs to the per-block issuance side of that pipeline. A lower
block reward slows how much TAO enters the injection stage each block, even though
participant-facing payouts are still grouped and settled on the tempo boundary
([Emission: Injection and distribution](https://docs.learnbittensor.org/learn/emissions#injection-and-distribution-two-stages-of-the-emissions-process)).

## Distinction from Coinbase

Block reward answers how much new TAO enters the network in one block. Coinbase is the per-block
runtime step that advances emission processing through that block on Subtensor
([Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase),
[Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor)).

For netuid 1 readers, a lower block reward slows how much TAO reaches the injection stage each
block. Coinbase vocabulary belongs when the article covers how the chain processes that issuance
step rather than the mint amount itself
([Emission: Injection and distribution](https://docs.learnbittensor.org/learn/emissions#injection-and-distribution-two-stages-of-the-emissions-process)).

## Distinction from Tempo

Block reward is measured every block. Tempo is the subnet epoch boundary where accumulated injection
is distributed to miners, validators, stakers, and subnet creators through Yuma Consensus
([Glossary: Tempo](https://docs.learnbittensor.org/resources/glossary#tempo),
[Emission](https://docs.learnbittensor.org/learn/emissions),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

A halving cuts the per-block mint rate immediately. Participant-facing payouts still settle on the
tempo cadence for the selected subnet, so block-reward and tempo vocabulary answer different timing
questions on netuid 1
([Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution)).

## Distinction from Halving

Halving is the supply-triggered event that cuts the per-block mint rate by 50% at documented
thresholds. Block reward is the active per-block TAO amount before and after each such cut
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Halving vocabulary names the threshold crossing that steps future mint rates down. Block-reward
vocabulary names the per-block figure in force before and after that crossing on the production
network ([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

A block-reward reading states how much TAO enters each block now. Halving vocabulary names when the
next supply threshold will cut that rate, not the live mint figure alone
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

- **Block reward** — the active per-block TAO mint figure for the whole network.
- **Halving** — the supply-threshold event that cuts that mint figure by 50%.

## Distinction from Recycling

Block reward and recycling describe new issuance and returned supply. Block reward is the amount of
new TAO the network mints each block before any split across subnets, while recycling describes TAO
returned to the protocol's supply path rather than newly minted at the block reward rate
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)).

- **Block reward** — the per-block mint rate that adds new TAO toward cumulative issuance.
- **Recycling** — TAO returned to the protocol supply path for future emission.

Recycled TAO re-entering issuance can change how quickly the network reaches the next halving
threshold that steps the block reward down
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

## Distinction from Maximum Supply

Maximum supply is the documented 21 million TAO lifetime ceiling. Block reward is the per-block mint
rate that moves cumulative issuance toward that ceiling over time
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Maximum Supply](https://docs.learnbittensor.org/resources/glossary#maximum-supply)).

- **Maximum supply** — the fixed lifetime ceiling on total TAO issuance.
- **Block reward** — the per-block mint rate that approaches that ceiling.

Maximum supply bounds total issuance. Block reward names how fast new TAO enters each block before
subnet splits and tempo distribution
([Emission: Injection and distribution](https://docs.learnbittensor.org/learn/emissions#injection-and-distribution-two-stages-of-the-emissions-process)).

## Distinction from Issuance

Issuance is the cumulative amount of TAO created across all blocks so far, while block reward is the
per-block rate at which that running total grows
([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

- **Block reward** — the per-block TAO mint figure that adds to issuance.
- **Issuance** — the running total of TAO created across all blocks to date.

Each block reward adds to issuance, and halvings cut the block reward as cumulative issuance
approaches the documented 21 million TAO ceiling, so the two terms describe the rate and the running
total of the same supply ([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Maximum Supply](https://docs.learnbittensor.org/resources/glossary#maximum-supply)).

## Distinction from Yuma Consensus

Block reward names the per-block TAO emission produced by the coinbase mechanism. Yuma Consensus is
the separate subnet-level process that converts validator weight submissions into miner and
validator emission shares each tempo
([Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

- **Block reward** — per-block TAO emission from coinbase.
- **Yuma Consensus** — on-chain settlement that turns validator weights into emission shares. Block
  reward names per-block TAO emission; Yuma processes validator weights into rewards
  ([Yuma Consensus: Validator emissions](https://docs.learnbittensor.org/learn/yuma-consensus#validator-emissions)).

## Reader Boundary

This page defines the concept at a high level. The exact per-block reward in force depends on how
many halvings have occurred; after the first halving it is 0.5 TAO per block, and it will fall
further at later thresholds. Current emission figures and the timing of the next halving are live
chain state and should be read from current Bittensor sources
([Emission](https://docs.learnbittensor.org/learn/emissions)).

Block reward is the per-block mint rate, not the maximum-supply ceiling itself
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).
