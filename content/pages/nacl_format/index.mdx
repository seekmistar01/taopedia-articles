# NaCl Format

NaCl Format is Bittensor terminology for a secure encryption format used around legacy wallet files.
The [NaCl Format glossary entry](https://docs.learnbittensor.org/resources/glossary#nacl-format)
places the term in wallet-security vocabulary rather than subnet, tokenomics, or consensus
vocabulary.

## Format Role

NaCl Format names an encryption-format role. It is not a Bittensor account role, address format, or
network feature. In the
[NaCl Format glossary entry](https://docs.learnbittensor.org/resources/glossary#nacl-format), the
important boundary is that the term points to a format using the NaCl library for wallet-file
security context.

## Legacy Wallet Context

The term is tied to legacy wallet security. Bittensor's
[wallet documentation](https://docs.learnbittensor.org/keys/wallets) gives the broader wallet
context, while the NaCl glossary entry narrows the subject to an encryption format used for older
wallet-file protection. That makes NaCl Format a compatibility and security term, not a new wallet
type.

## Encryption Boundary

Encryption format and wallet behavior are related but different ideas. NaCl Format concerns how
wallet-file material is protected in a legacy format; it does not define staking, emissions, subnet
roles, or address ownership. The
[NaCl Format glossary entry](https://docs.learnbittensor.org/resources/glossary#nacl-format) keeps
the term at that file-format level.

## Relationship to Wallets

Wallets are the surrounding concept, because wallet files and wallet security provide the setting
where NaCl Format appears. The [wallets guide](https://docs.learnbittensor.org/keys/wallets)
provides the broader Bittensor wallet context, while NaCl Format remains narrower: it is one
encryption-format label inside that context.

## Relationship to Public and Private Keys

NaCl Format sits near public-key and private-key terminology because wallet files protect key
material. A [public key](https://docs.learnbittensor.org/resources/glossary#public-key) is shareable
material for account reference, while the
[NaCl Format glossary entry](https://docs.learnbittensor.org/resources/glossary#nacl-format)
concerns the protection format around legacy wallet files. The terms belong to the same security
area but answer different questions.

## Reading the Term

NaCl Format is best read as format vocabulary. It does not name a recovery process, a wallet
recommendation, a subnet action, or a balance state. For article-level reading, the
[NaCl glossary entry](https://docs.learnbittensor.org/resources/glossary#nacl-format) is the anchor
for the term, with Bittensor wallet documentation supplying the surrounding context.
