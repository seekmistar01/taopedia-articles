# Bittensor Wallet

A Bittensor wallet is the identity and ownership container for a participant on the network. The
official
[Glossary: Bittensor Wallet](https://docs.learnbittensor.org/resources/glossary#bittensor-wallet)
describes it as a digital wallet that holds core ownership in the Bittensor network and underlies
user operations across staking, delegation, and subnet participation.

## Keys and Ownership Context

Wallet vocabulary in Bittensor separates long-term ownership from operational keys. Official wallet
documentation describes coldkeys as the security-critical key that controls TAO and stake, and
hotkeys as operational keys used for day-to-day network roles. A Bittensor wallet is the container
that organizes those keys rather than a single key file label.

For readers, wallet names the ownership package, while coldkey and hotkey name specific key roles
inside that package.

References: [Wallets](https://docs.learnbittensor.org/keys/wallets),
[Working with Keys](https://docs.learnbittensor.org/keys/working-with-keys)

## Staking and Delegation Context

Bittensor wallet ownership also frames staking participation. The
[Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)
describes how TAO holders stake to validators, and wallet material is what authorizes those
financial operations on chain. Wallet therefore belongs to the identity side of staking vocabulary,
not to validator performance or subnet emission formulas.

This article does not describe how to create, import, or back up a wallet. Those steps belong in
official wallet documentation.

## Local and External Wallet Boundaries

Bittensor documentation also distinguishes where wallet material is stored, such as local software
wallets versus external wallet integrations. Those labels describe storage and tooling context
around the same ownership idea rather than separate network identities.

For readers, Bittensor wallet still names the participant ownership container. Local wallet,
external wallet, and wallet location are adjacent tooling terms, not replacements for the core
wallet concept.

References:
[Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet),
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)

## Reader Boundary

Bittensor wallet should not be read as a balance report, a validator recommendation, or a command
tutorial. It is a concept term for the identity container behind Bittensor key and stake operations.
Exact addresses, balances, and backup steps belong in official wallet references and live chain
data.

## Relationship to Coldkeys

Bittensor wallet and coldkey are related but distinct terms in Bittensor's key model. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#bittensor-wallet) defines a Bittensor
wallet as the identity and ownership container for a participant, while the
[Coldkey](https://docs.learnbittensor.org/resources/glossary#coldkey) article names the
security-critical key inside that container that controls TAO balances and stake authority.

For readers, the wallet names the ownership package, and the coldkey is the high-security key within
that package. Financial operations on the network — staking, governance votes, hotkey management —
require the coldkey, not just knowledge of the wallet name. This boundary helps readers distinguish
wallet-level identity concepts from the specific key-level operations that coldkeys authorize.

References:
[Glossary: Bittensor Wallet](https://docs.learnbittensor.org/resources/glossary#bittensor-wallet),
[Glossary: Coldkey](https://docs.learnbittensor.org/resources/glossary#coldkey),
[Wallets, Coldkeys and Hotkeys in Bittensor](https://docs.learnbittensor.org/keys/wallets)

## Relationship to Coldkey Swap

Bittensor wallet and coldkey swap address related but different parts of Bittensor key management
vocabulary. A Bittensor wallet is the identity and ownership container that holds a participant's
coldkey and hotkeys, while coldkey swap is the mechanism that transfers a participant's on-chain
identity — including all associated hotkeys and their registrations — to a new coldkey without
resetting those registrations. The
[Glossary: Bittensor Wallet](https://docs.learnbittensor.org/resources/glossary#bittensor-wallet)
defines the wallet as an identity and ownership container, while the
[Coldkey Swap](https://docs.learnbittensor.org/keys/coldkey-swap) documentation describes the swap
as moving the on-chain identity to a new coldkey.

After a coldkey swap, the participant's on-chain identity is controlled by the new coldkey; the
wallet container concept still applies, but the authority moves. Wallet and coldkey swap together
describe different layers of the same identity model: the wallet names the local container, while
coldkey swap names the on-chain operation that can change which coldkey holds the identity.

References:
[Glossary: Bittensor Wallet](https://docs.learnbittensor.org/resources/glossary#bittensor-wallet),
[Coldkey Swap](https://docs.learnbittensor.org/keys/coldkey-swap)

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from local testing to testchain and then mainchain. For a
Bittensor wallet, that sequence changes how readers should interpret evidence about key and identity
management.

In local testing, a Bittensor wallet can be used to manage coldkeys and hotkeys inside an isolated
environment. That is useful for checking wallet creation and key registration mechanics, but it
remains isolated feedback from a local environment.

On testchain, Bittensor wallet behavior can be observed in a shared testing network. This gives
stronger evidence about how coldkeys, hotkeys, and staking operations interact with on-chain
identity than a private local run, while still keeping the result separate from production mainchain
key management.

On mainchain, a Bittensor wallet holds a participant's actual on-chain identity, stake, and key
registrations. The
[Glossary: Bittensor Wallet](https://docs.learnbittensor.org/resources/glossary#bittensor-wallet)
defines the wallet as an identity and ownership container, so production wallet context is tied to
real coldkey and hotkey ownership on the active network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A wallet configuration observed in one environment should
not be read as identical evidence about identity or key state in another environment.

For readers, this keeps Bittensor wallet examples from sounding stronger than their environment
supports. Local, testchain, and mainchain contexts can all describe key and identity management, but
they do not carry the same interpretive weight.
