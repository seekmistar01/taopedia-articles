# External Wallet

An external wallet is wallet software or custody outside the local Bittensor wallet path. The
[External Wallet glossary entry](https://docs.learnbittensor.org/resources/glossary#external-wallet)
places the term in wallet-security vocabulary, where location and custody context matter for how a
wallet is understood.

## Wallet Application Boundary

External wallet and wallet application are related but not interchangeable. The
[external-wallet glossary entry](https://docs.learnbittensor.org/resources/glossary#external-wallet)
points to custody or software outside the local wallet path, while wallet-application vocabulary
describes the software interface used to view wallet state or prepare actions
([Wallets](https://docs.learnbittensor.org/keys/wallets),
[Wallets](https://docs.learnbittensor.org/keys/wallets)).

For readers, an external wallet answers where custody or signing control is managed. A wallet
application answers which interface is being used to interact with wallet information or actions.

## Wallet Address Boundary

External wallet and wallet address are related but separate concepts. A
[wallet address](https://docs.learnbittensor.org/resources/glossary#wallet-address) is a
public-key-derived destination identifier for TAO, while external wallet names custody or software
context. That distinction keeps destination-address vocabulary separate from custody-location
vocabulary.

## Distinction from Local Wallet

Local wallet names files stored through the local Bittensor install path. External wallet names
custody or software outside that path
([Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet),
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)).

The difference is about where wallet control is managed, not about creating a separate Bittensor
asset ([Glossary: Local Wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet),
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)).

## Distinction from Wallet Address

Wallet address names a public TAO destination. External wallet names where custody or signing
control is managed
([Glossary: Wallet Address](https://docs.learnbittensor.org/resources/glossary#wallet-address),
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)).

## Distinction from Ledger Hardware Wallet

Ledger hardware wallet names a device that holds signing material off the host. External wallet
names custody outside the local Bittensor wallet path more generally
([Using Ledger Hardware Wallet](https://docs.learnbittensor.org/staking-and-delegation/using-ledger-hw-wallet),
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)).

## Distinction from Coldkey

An external wallet and a coldkey name different layers of wallet vocabulary. An external wallet
names where custody or signing control is managed relative to the local Bittensor wallet path, while
a coldkey is a wallet component responsible for securely storing funds and performing high-risk
operations such as transfers and staking
([Glossary: Coldkey](https://docs.learnbittensor.org/resources/glossary#coldkey)).

- **External wallet** — names custody, software, or signing control outside the local path.
- **Coldkey** — a specific wallet component for storing funds and high-risk operations.

## Distinction from Hotkey

An external wallet and a hotkey name different layers of wallet vocabulary. An external wallet names
where custody or signing control is managed relative to the local Bittensor wallet path, while a
hotkey is a wallet component used for frequent, lower-risk network operations such as validation and
mining activity ([Glossary: Hotkey](https://docs.learnbittensor.org/resources/glossary#hotkey),
[Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet)).

- **External wallet** — custody or signing-control location outside the local path.
- **Hotkey** — a wallet component used for frequent, lower-risk network operations.

## Distinction from Mnemonic

An external wallet and a mnemonic name different parts of wallet handling. An external wallet names
where custody or signing control is located relative to the local Bittensor wallet path, while a
mnemonic is a sequence of words used to regenerate keys, in case of loss, and restore coldkeys and
hotkeys in the Bittensor wallet
([Glossary: Mnemonic](https://docs.learnbittensor.org/resources/glossary#mnemonic)).

- **External wallet** — names custody or signing-control location outside the local path.
- **Mnemonic** — the word sequence used to regenerate and restore wallet keys.

## Distinction from Custody

External wallet names a custody context, not a different token or subnet role. The
[wallet documentation](https://docs.learnbittensor.org/keys/wallets) gives the broader setting for
Bittensor wallet concepts, while the external-wallet glossary entry narrows the term to wallet
software or custody outside the local path.

## Distinction from Address-Poisoning

External wallets still use destination addresses, so address-poisoning risk remains relevant. The
[address-poisoning guide](https://docs.learnbittensor.org/keys/address-poisoning-scams) focuses on
lookalike destination addresses during wallet use, while the external-wallet term names where wallet
control is managed. The risk and the custody context are connected but not identical.

## Distinction from Yuma Consensus

An external wallet names custody or signing control outside the local Bittensor wallet path. That
custody vocabulary is separate from the tempo pass that converts validator weight submissions into
emission shares
([Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Validators still sign weight submissions with whichever wallet setup they use. Yuma Consensus
settles from those included signals and credits outcomes when the epoch closes
([Emission](https://docs.learnbittensor.org/learn/emissions)).

Readers should keep signing-location vocabulary and settlement vocabulary separate. External wallet
answers where keys are managed; Yuma answers how submitted weights become credited shares on a
netuid ([Wallets, Coldkeys and Hotkeys in Bittensor](https://docs.learnbittensor.org/keys/wallets)).

- **External wallet** — custody or software outside the local Bittensor wallet path.
- **Yuma Consensus** — on-chain weight aggregation that settles emissions.

## Reader Boundary

External wallet should not be read as a subnet role, validator recommendation, wallet provider
endorsement, or balance state. It names custody-location vocabulary for wallet software or signing
control outside the local Bittensor wallet path
([Glossary: External Wallet](https://docs.learnbittensor.org/resources/glossary#external-wallet),
[Wallets](https://docs.learnbittensor.org/keys/wallets)).
