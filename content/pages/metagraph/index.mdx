# Metagraph

The metagraph is Bittensor's structured view of state for a selected subnet. The
[Glossary: Metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph) defines it as a
data structure containing comprehensive information about a subnet's neurons and notes that it helps
calculate emissions. The [Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph)
reference describes it as a snapshot of subnet state at a particular block height rather than a
timeless network summary.

## What It Represents

A metagraph belongs to one netuid and records neuron-level state at a particular block height:
stake-related fields, weights, ranks, trust, consensus, incentives, dividends, emissions, activity
state, and bonds ([The Subnet Metagraph](https://docs.learnbittensor.org/subnets/metagraph),
[Understanding Neurons](https://docs.learnbittensor.org/learn/neurons)).

Each netuid keeps its own neuron list and incentive fields. A metagraph statement should name both
the selected netuid and the block context, because later blocks can change those fields
([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid),
[Glossary: Block](https://docs.learnbittensor.org/resources/glossary#block)).

Examples tied to netuid 1 illustrate one subnet's state snapshot path. They should not be read as
representing production neuron state on another network
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from Neuron

A metagraph is the per-netuid snapshot that lists registered neuron rows and their values at a
block. A neuron is one participant record inside that snapshot
([Glossary: Metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph),
[Glossary: Neuron](https://docs.learnbittensor.org/resources/glossary#neuron)).

- **Metagraph** — subnet-wide participant state at one block height.
- **Neuron** — one registered participant row inside that state.

## Distinction from Deregistration

The metagraph is the snapshot of a subnet's registered neurons and their state at each UID.
Deregistration removes a neuron from a UID slot so that the next metagraph reads the reassigned slot
rather than the removed neuron
([Glossary: Metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph),
[Glossary: Deregistration](https://docs.learnbittensor.org/resources/glossary#deregistration)).

- **Metagraph** — subnet state snapshot at one block height.
- **Deregistration** — removal that vacates a UID slot for reassignment.

## Distinction from UID Slot

Metagraph names the subnet state snapshot for a selected netuid. UID slot names one participant
position inside that snapshot
([Glossary: Metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph),
[Glossary: UID Slot](https://docs.learnbittensor.org/resources/glossary#uid-slot)).

- **Metagraph** — subnet-wide participant state at one block height.
- **UID slot** — one indexed participant position inside that state.

## Distinction from Yuma Consensus

The metagraph records who is registered and what values the chain stores at one point in time. Yuma
Consensus converts validator weight submissions into emission shares each tempo
([Glossary: Metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Metagraph** — subnet neuron state snapshot at one block height.
- **Yuma Consensus** — on-chain settlement that turns validator weights into emission shares.

## Reader Boundary

This page defines the term at a high level. It does not report current subnet membership, current
stake, current emissions, wallet ownership, or live identity fields. Those values change with chain
state and should be checked for the relevant netuid when current data is needed.
