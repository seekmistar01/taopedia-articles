# Transaction Fees

Transaction fees are costs the Bittensor chain can attach to state-changing network actions. The
official [Transaction Fees in Bittensor](https://docs.learnbittensor.org/learn/fees) page separates
base chain transaction fees from swap fees that can apply when stake-related activity interacts with
subnet liquidity. It also notes that reading chain state is free.

## General Transaction Fees

The official fees page describes many state-changing actions as fee-bearing. Their base transaction
fee has two parts: a weight component, tied to computational load, and a length component, tied to
the size of the submitted action. At a Taopedia level, the important point is the distinction
between the base chain fee and other economic effects that may appear around a specific Bittensor
action.

Reference: [Transaction Fees in Bittensor](https://docs.learnbittensor.org/learn/fees)

## Swap Fees

The same source treats swap fees as separate from the base transaction fee. Stake and unstake
activity can also involve an amount-based swap fee when liquidity is transacted through subnet
markets. This means a stake-related action can have both a base chain fee and a liquidity-linked
fee, while ordinary chain reads have neither.

Reference: [Transaction Fees in Bittensor](https://docs.learnbittensor.org/learn/fees)

## Fee-Free Activity

Bittensor documentation identifies chain reads as free and lists some weight-setting and
commit-reveal actions as fee-free. The practical distinction is not whether an action feels small to
a user, but whether the chain classifies that action as fee-bearing. This keeps fee-free activity
separate from actions that change state and consume chain resources.

Reference: [Transaction Fees in Bittensor](https://docs.learnbittensor.org/learn/fees)

## Grouped Actions

The official page also explains that grouped actions inherit fee behavior from their component
actions. If a grouped action contains a fee-bearing component, the group can carry a fee; if every
component is fee-free, the group can remain fee-free. This helps readers avoid assuming that
grouping actions always removes or duplicates fees.

Reference: [Transaction Fees in Bittensor](https://docs.learnbittensor.org/learn/fees)

## Fee Recycling Context

The
[Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)
describes collected transaction fees as recycled tokens deducted from the record of token issuance
for future emission. In Taopedia prose, that links base chain fees to recycling vocabulary rather
than treating every fee as burned or permanently removed.

References: [Transaction Fees in Bittensor](https://docs.learnbittensor.org/learn/fees),
[Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)

## Careful Interpretation

Transaction-fee details can depend on the action being evaluated and on current chain behavior.
Taopedia readers should use this article as conceptual orientation, not as a fee calculator or an
operating guide. For exact current behavior, examples, and estimation paths, use the official
Bittensor documentation.

Reference: [Transaction Fees in Bittensor](https://docs.learnbittensor.org/learn/fees)
