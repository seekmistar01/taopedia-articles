# EdDSA Cryptographic Keypairs

EdDSA cryptographic keypairs refer to the public and private key pairs generated with EdDSA, the
signature algorithm named in Bittensor wallet-key terminology. The official
[Glossary](https://docs.learnbittensor.org/resources/glossary#eddsa-cryptographic-keypairs)
describes EdDSA as the algorithm used to generate public and private key pairs for Bittensor
coldkeys and hotkeys.

## Keypair Context

A keypair has two sides: a public side that can be shared for identification or verification, and a
private side that must remain protected. The
[Public Key glossary](https://docs.learnbittensor.org/resources/glossary#public-key) describes the
public key as shareable material used for verification, encryption, and account reference. The
[Private Key glossary](https://docs.learnbittensor.org/resources/glossary#private-key) describes the
private key as the protected component used to authorize activity.

## Wallet-Key Context

In Bittensor, EdDSA keypairs sit under the wallet-key vocabulary rather than naming a separate
network role or asset. The glossary connects EdDSA key generation with coldkeys and hotkeys, while
separate articles can explain those wallet-key roles in more detail.

## Reading the Term

This article is a concept boundary. It does not describe how to create, store, recover, rotate, or
use keys. For Taopedia readers, the useful point is that EdDSA names the cryptographic method behind
the public/private keypair terminology used in Bittensor.
