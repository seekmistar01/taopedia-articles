# TAO

TAO is the network token used in Bittensor incentive, staking, subnet reserve, and emissions
context. [Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction) frames TAO
as the incentive that connects subnet activity to the Bittensor blockchain, while
[Emission](https://docs.learnbittensor.org/learn/emissions) describes newly created TAO beside
subnet-specific alpha-token emissions.

## Network role

TAO is the token of the Bittensor network as a whole. It connects the blockchain layer to subnet
activity: the blockchain supplies the system-of-record context, and TAO supplies the incentive
context for miners, validators, subnet creators, and stakers.
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction) describes TAO as the
incentive for subnet activity and lists those role families in the platform overview.

## Incentive role

TAO is part of how Bittensor rewards useful work and evaluation. Emission distributes newly created
TAO and subnet-specific alpha tokens to roles that contribute through mining, validation, staking,
and subnet creation. That makes TAO a network-level incentive concept, not just an asset holding
concept. [Emission](https://docs.learnbittensor.org/learn/emissions) covers the TAO and alpha-token
emission framing, and
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) connects
emissions to subnet and role performance.

## Subnet reserve context

TAO also appears inside subnet token context. In Bittensor subnets, the official subnet overview
describes each subnet as having a TAO reserve and a subnet-specific alpha reserve. TAO is the
network token in that relationship, while alpha tokens are specific to individual subnets. This is
why [[dynamic_tao|Dynamic TAO]] and subnet articles often discuss TAO next to subnet alpha tokens
instead of treating every token reference as the same thing.
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) provides the
TAO reserve, alpha reserve, and network-token distinction.

## Staking context

TAO holders can support validators by staking in Bittensor. In subnet staking context, TAO enters a
subnet reserve and the resulting stake is represented in that subnet's alpha units. In root-subnet
context, stake remains expressed in TAO. The important article-level distinction is the asset
context: [[staking_and_delegation|Staking and Delegation]] involves TAO, subnet alpha tokens, and
validator reward context without making those concepts interchangeable.
[Staking/Delegation Overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)
covers the staking relationship, and
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) provides the
subnet reserve context.

## Relation to RAO

TAO and RAO are not separate token systems. RAO is a denomination of TAO, used to express smaller
amounts of the same network token. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#tao) frames TAO as the Bittensor
network token, and [RAO](https://docs.learnbittensor.org/resources/glossary#rao) as a denomination
of it.

For readers, TAO names the token at the unit most contexts use, while RAO names the same token at a
smaller denomination. The two describe the same asset rather than different tokens, so a RAO amount
is a TAO amount expressed at finer granularity.

References: [TAO](https://docs.learnbittensor.org/resources/glossary#tao),
[RAO](https://docs.learnbittensor.org/resources/glossary#rao)

## Relation to alpha tokens

TAO and alpha tokens have different scopes. TAO is the network token, while alpha tokens are
subnet-specific tokens used inside subnet contexts. Emission, staking, and subnet reserve language
can mention both, so a precise explanation should say whether it is discussing TAO, alpha tokens, or
the relationship between them. [Emission](https://docs.learnbittensor.org/learn/emissions) frames
emissions around both TAO and subnet-specific alpha tokens, and
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) contrasts TAO
with subnet alpha tokens.

## Relationship to SS58 Encoded

TAO and SS58 encoded refer to related but different parts of Bittensor wallet vocabulary. The
[Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao) describes TAO as the
Bittensor network token, and the
[Glossary: SS58 Encoded](https://docs.learnbittensor.org/resources/glossary#ss58-encoded) describes
SS58 encoded values as compact representations of the public-key-derived wallet addresses used to
send and receive TAO.

For readers, TAO names the network token itself, while SS58 encoded names the address format that
identifies where TAO is sent. The token and the address encoding that targets it are separate
concepts, so TAO should not be read as an address-format term.

References: [Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao),
[Glossary: SS58 Encoded](https://docs.learnbittensor.org/resources/glossary#ss58-encoded)

## Relationship to Coinbase

TAO and coinbase address related but different parts of Bittensor network vocabulary. TAO is the
Bittensor network token — the asset used for staking, transfers, and forming the TAO-side of subnet
liquidity pool reserves — while coinbase is the per-block protocol mechanism that drives TAO
emission, injects TAO into subnet pool reserves each block, accumulates pending emissions, and
checks epoch boundaries to trigger Yuma Consensus rounds. The
[Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao) describes the network token
itself, and the
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)
documentation describes the per-block operation that orchestrates TAO's distribution through subnet
pools and the emission flow.

TAO is the token that coinbase circulates and injects. Each block, coinbase adds TAO to subnet
reserves through reserve injection and accumulates TAO-denominated pending emissions that are later
distributed to validators, miners, and stakers when a subnet's Yuma Consensus epoch runs. TAO names
the asset; coinbase names the per-block mechanism that moves that asset through the emission and
liquidity system each block.

References: [Glossary: TAO](https://docs.learnbittensor.org/resources/glossary#tao),
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For TAO, that
sequence changes how readers should interpret evidence about token balances, staking amounts, and
emission figures.

In localnet, TAO amounts in examples reflect local chain state. Localnet TAO has no real-world value
and does not represent mainnet supply or participant balances.

On testnet, TAO amounts exist in a shared, non-production network. Testnet TAO is separate from
mainnet supply and carries no production economic value.

On mainnet, TAO is the live production token used to incentivize subnet activity. The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction) describes TAO as the
network token whose emission and staking mechanics apply on the production network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A TAO balance or emission example from one environment
should not be read as representing token state in another environment.
