# TAO Weight

TAO weight is the parameter that describes how TAO stake counts relative to alpha stake when
Bittensor calculates stake weight inside a subnet. The
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight) frames it as
the global parameter for TAO stake's relative influence against alpha stake, while
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) connects
the resulting stake-weight value to validator consensus power and emissions.

## Stake-Weight Role

TAO weight is best read as part of stake-weight vocabulary. It is not a separate token, a staking
role, or a validator score by itself. It explains how one component of stake is treated before the
combined stake-weight measure is interpreted inside subnet consensus.

The [Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) glossary entry
is the companion term: stake weight is the computed value used for validator consensus power, while
TAO weight explains the relative role of TAO stake inside that computation.

## TAO Stake and Alpha Stake

The term exists because validator influence in a subnet can involve both TAO stake and alpha stake.
[TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight) describes how TAO
stake's influence is weighted relative to alpha stake, rather than treating every stake component as
the same kind of input.

That makes the article boundary straightforward. TAO weight is about relative weighting between
stake components. It does not describe total delegated stake, a subnet's task design, or a complete
emission calculation.

## Subnet Consensus Context

TAO weight matters because validator influence is used inside subnet consensus. The
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) documentation
describes validators as evaluating miner work according to subnet rules, and
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) explains how weighted
validator evaluations are resolved into miner and validator emissions.

Within that setting, TAO weight helps explain how TAO stake contributes to a validator's influence.
It is a parameter for interpreting stake weight, not a standalone description of what miners do or
how validators evaluate task quality.

## Root-Proportion Boundary

TAO weight and root proportion both involve TAO stake, but they answer different questions. The
[TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight) glossary entry defines
TAO weight as the relative-influence parameter for TAO stake, while
[Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion) describes the
share of a subnet's dividend context attributed to root-staked TAO.

In plain terms, TAO weight is a weighting parameter used before stake weight is read as validator
influence. Root proportion is a subnet-level share used in a different dividend context. They should
not be treated as interchangeable values.

## Mechanism Boundary

TAO weight should be read inside the consensus path that uses it. The
[Multiple Incentive Mechanisms](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets)
documentation explains that a subnet can run more than one evaluation mechanism, each with its own
weight matrix. That means stake-weight interpretation should stay tied to the relevant mechanism.

The useful reader takeaway is narrow: TAO weight describes how TAO stake is weighted relative to
alpha stake for stake-weight interpretation. It should not be expanded into a universal comparison
across unrelated subnet evaluation systems.

## Reading Boundaries

TAO weight is the right term when the focus is the relative role of TAO stake in stake weight. If
the question is total staked TAO behind a delegate,
[effective stake](https://docs.learnbittensor.org/resources/glossary#effective-stake) is more
precise. If the question is validator consensus influence,
[stake weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) is more precise.

These boundaries keep TAO weight useful as a reference term: it explains a weighting parameter, not
a staking recommendation, a validator ranking, or a full guide to subnet economics.

## Development Stage Context

Bittensor's subnet-development path moves from localnet to testnet and then mainnet
([Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)).
For TAO weight, that sequence changes how readers should interpret examples of TAO stake influence
inside subnet consensus.

In localnet, TAO weight behavior can be tested in an isolated environment. Localnet examples reflect
local subnet settings, local validator stake, and local chain state rather than production subnet
consensus.

On testnet, TAO weight can be observed in a shared, non-production network. Testnet examples can
show realistic stake-weight behavior, but they remain separate from mainnet validator influence and
emission outcomes.

On mainnet, TAO weight is the production weighting context for how TAO stake contributes alongside
alpha stake inside subnet stake weight. The
[Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight) glossary entry
connects stake weight to validator consensus power and emissions.

Mechanism context also matters. In subnets with multiple incentive mechanisms, miner scoring,
weights, bond pools, and emissions should be read in the relevant mechanism context, while validator
stake weight remains the same across mechanisms
([Multiple Incentive Mechanisms](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets)).

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A TAO weight example from one environment should not be
read as representing validator influence, stake weighting, or emission outcomes in another
environment.

That keeps local tests, shared testing observations, and production stake-weight context distinct.
