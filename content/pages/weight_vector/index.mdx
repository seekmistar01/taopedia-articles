# Weight Vector

A weight vector is one validator's structured set of miner evaluations inside a Bittensor subnet.
Each element represents a weight assigned to a subnet miner according to that validator's evaluation
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The scope is narrow. A single vector is one validator's signal before aggregation, not the whole
subnet result. The individual values inside the set are validator weights
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector)).

## What It Represents

A weight vector names the structured payload one validator submits when it sets weights on a subnet.
It lists which miners received scores and what those scores were, before stake-weighted aggregation,
clipping, or bond updates run
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Implementation of the Yuma Consensus Epoch](https://docs.learnbittensor.org/navigating-subtensor/epoch)).

Examples tied to netuid 1 illustrate one subnet's submission path. They should not be read as
representing production behavior on another network
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from Weight Matrix

A weight vector is one validator's miner weight assignments in a single submission. A weight matrix
is the combined validator-to-miner assignment table across the validator set on the selected subnet
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix)).

- **Weight vector** — one validator's miner evaluations in one submission.
- **Weight matrix** — combined validator-to-miner assignments across the set.

## Distinction from Consensus Score

Consensus score is the stake-weighted benchmark used to clip outlier weights. A weight vector is the
per-validator input evaluated before that benchmark is formed
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus: Clipping](https://docs.learnbittensor.org/learn/yuma-consensus#clipping)).

- **Weight vector** — one validator's submitted evaluation signal.
- **Consensus score** — stake-weighted agreement benchmark derived from submissions.

## Distinction from Rank

Rank is the miner-side standing after validator weights are filtered and aggregated. A weight vector
is one validator's raw evaluation input before that aggregation
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector)).

- **Weight vector** — one validator's miner evaluations before aggregation.
- **Rank** — miner-side standing after weights are processed.

## Distinction from Stake Weight

Stake weight names how much validator influence each submitted weight carries in consensus math. A
weight vector names which miners a validator scored, not the stake backing that validator
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector)).

- **Weight vector** — evaluation targets in one submission.
- **Stake weight** — influence scaling applied during aggregation.

## Distinction from Weight Setting

Weight setting is the per-validator action that submits one weight vector. The weight vector is the
structured payload that action records on the selected subnet
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Implementation of the Yuma Consensus Epoch](https://docs.learnbittensor.org/navigating-subtensor/epoch)).

- **Weight setting** — the submission act that records scores on chain.
- **Weight vector** — the evaluation content inside that submission.

## Distinction from Trust

A weight vector is one validator's submitted evaluation signals, while trust is interpreted after
the full validator set is filtered and aggregated into miner-side outcomes
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust)).

- **Weight vector** — one validator's input signal before aggregation.
- **Trust** — downstream miner-side reading after aggregation.

## Distinction from Validator Trust

A weight vector is one validator's submitted evaluation signals, while validator trust summarizes
that validator's clipped influence after the full weight matrix is filtered
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust)).

- **Weight vector** — one validator's input signal before clipping.
- **Validator trust** — validator-side measure after clipping.

## Distinction from Validator-Miner Bonds

A weight vector is one validator's miner assignments in a single round. Validator-miner bonds are
the smoothed relationships built as those assignments repeat across rounds
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)).

- **Weight vector** — one round's miner assignments from one validator.
- **Validator-miner bonds** — smoothed relationships built across rounds.

## Distinction from Weights Version Key

The weight vector is the scored targets a validator submits. The weights version key is the
per-subnet hyperparameter that sets the minimum version a `set_weights` call must carry
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

- **Weight vector** — the evaluation payload in a submission.
- **Weights version key** — compatibility gate on declared submission version.

## Distinction from Subnet Scoring Model

A subnet scoring model is the subnet-defined method a validator uses to judge returned miner work. A
weight vector is the structured set of miner scores that method produces before on-chain submission
([Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

- **Subnet scoring model** — the evaluation rule for miner work.
- **Weight vector** — the per-validator signal that rule becomes before submission.

## Distinction from Yuma Consensus

A weight vector is one validator's structured miner evaluations—the payload produced by a subnet
scoring model before on-chain submission. It names a single validator's input signal, not the
settlement step that allocates emissions across the subnet
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model)).

Yuma Consensus is the on-chain mechanism that runs at the epoch boundary. It reads the weight matrix
assembled from eligible validators' vectors, applies clipping and bonding, and converts the filtered
result into incentive and dividend shares
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

Earlier sections separate weight vectors from weights version keys and from scoring-model rules.
Many vectors combine into the matrix Yuma aggregates; one vector alone does not prove a miner's
final rank or payout.

Readers should treat a weight vector as pre-settlement evaluation data, and Yuma as the mechanism
that settles the combined matrix into emissions.

- **Weight vector** — one validator's miner evaluations before on-chain aggregation.
- **Yuma Consensus** — epoch-boundary settlement from the weight matrix to emission shares.

## Reader Boundary

Weight vector names per-validator consensus-input vocabulary. It describes one validator's
structured miner-evaluation set before subnet-level matrix and consensus processing, not stake
weight, rank, or a complete reward model
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Distinction from UID

A weight vector and a UID relate as a list and the index into it. A UID is the number identifying a
single neuron's position within a subnet, while a weight vector is the full list of weights a
validator assigns across UIDs — each entry keyed by a UID — that Yuma Consensus aggregates
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: UID Slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **UID** — the number identifying one neuron's position in a subnet.
- **Weight vector** — a validator's list of weights indexed by UID.
