# Validator Trust

Validator trust is a per-validator metric in Bittensor's consensus process. The official glossary
describes it as measuring how much weight a validator successfully contributes after weights are
clipped by consensus.

References: [Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)

## What It Represents

Validators submit weight signals after evaluating miner work in a subnet. Consensus filtering can
reduce the influence of outlier assignments. Validator trust describes the remaining consensus
aligned portion of a validator's submitted weights, so it is about validator alignment rather than a
standalone miner score.

References: [Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust),
[Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)

## Relationship to Consensus Score

Consensus score and validator trust describe different sides of the same filtering process. The
consensus score is the agreement threshold around a particular target, while validator trust is a
per-validator view of how much submitted weight remains after that filtering step.

References: [Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust)

## Relationship to Trust

The glossary distinguishes trust from validator trust. Trust is described for the evaluated target,
while validator trust is described for the validator that submitted weights. This distinction helps
avoid reading validator trust as a universal quality label for every participant in a subnet.

References: [Trust](https://docs.learnbittensor.org/resources/glossary#trust),
[Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust)

## Relationship to the Weight Matrix

The weight matrix collects validator weight vectors for a subnet's consensus step. Validator trust
is read after those submitted weights have been compared with the broader consensus signal. The
matrix is the shared input; validator trust is one way of interpreting each validator's retained
influence after filtering.

References: [Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix),
[Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust)

## Careful Interpretation

Validator trust should be read inside its subnet and consensus context. It is not an instruction for
running a validator, not a general reputation score, and not a promise about future outcomes. Exact
calculation details belong in the official references; the Taopedia-level meaning is that validator
trust reflects how much of a validator's submitted signal remains aligned with consensus.

References: [Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)
