# Consensus-Based Weights

Consensus-based weights, also called liquid alpha, are a Bittensor consensus mechanism that makes
validator-miner bond smoothing depend on how a validator's weights align with consensus. In Yuma
Consensus, validators submit weights for miners, and the protocol turns those evaluations into miner
and validator emissions. Consensus-based weights sit inside that bond path by affecting how quickly
a validator's bond to a miner responds after the validator's evaluation is compared with consensus.

References:
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)

## What Consensus-Based Weights Provide

The official Bittensor documentation describes consensus-based weights as a way to better correlate
validator dividends with the performance of the miners that validators weight. Instead of applying
the same smoothing behavior to every validator-miner bond update, liquid alpha varies the response
for each pair. When a validator's weights are closer to consensus, the bond response can be more
direct; weaker alignment keeps the response more conservative.

References:
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)

## Why Consensus Alignment Matters

Yuma Consensus uses validator weights as inputs to miner and validator emission outcomes. It also
penalizes bond values that overstate a miner's consensus evaluation. Consensus-based weights build
on that incentive path by tying bond smoothing to consensus alignment. The point is not to prove
that a single evaluation is correct, but to make validator emissions respond more favorably to
evaluations that fit the subnet's consensus signal.

References: [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)

## Relationship to EMA Bonds

Bittensor uses exponential moving averages to smooth validator-miner bonds, so a bond does not jump
entirely from one evaluation step. Liquid alpha changes the smoothing factor dynamically for
validator-miner pairs. This keeps bonds responsive to fresh consensus-aligned evaluation while still
using EMA smoothing to dampen abrupt swings.

References: [Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)

## Relationship to Weight Copying

Consensus-based weights are related to the weight-copying problem because copied or stale weights
can fall out of line with current consensus. The official consensus-based weights page discusses
liquid alpha alongside delayed weight visibility as tools that help validators identify useful
miners and bond to them quickly. Similar weights do not, by themselves, prove copying; the safer
distinction is between independent evaluation and dependence on public weight information.

References:
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[The Weight Copying Problem](https://docs.learnbittensor.org/concepts/weight-copying-in-bittensor)

## Careful Interpretation

Consensus-based weights are not a separate miner-scoring system, a standalone user feature, or a
statement about subnet-specific alpha tokens. In this context, alpha refers to the smoothing factor
used in EMA bond updates. Exact formulas and current values should be checked in the official
Bittensor references, because the Taopedia-level concept is the role of alignment-sensitive bond
smoothing rather than operational detail.

References: [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)

## Validator Dividend Context

Consensus-based weights sit in the bond path that feeds validator-side emission outcomes. The
official
[Yuma Consensus: Validator emissions](https://docs.learnbittensor.org/learn/yuma-consensus#validator-emissions)
documentation describes validator emissions as bonds to miners, weighted by miner emission results,
while the
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights) page
frames liquid alpha as improving how validator dividends correlate with the miners a validator
actually weights.

For readers, this keeps consensus-based weights inside validator dividend context rather than
treating them as a separate miner-ranking rule.

References:
[Yuma Consensus: Validator emissions](https://docs.learnbittensor.org/learn/yuma-consensus#validator-emissions),
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)

## Outlier Weight Context

Consensus-based weights operate inside Yuma Consensus, where validator weights that diverge too far
from consensus can be clipped. The official
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) documentation describes
clipping as a way to limit outlier weights, while
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights) explains
how liquid alpha changes bond smoothing after weights are compared with consensus.

For readers, clipping and liquid alpha address different parts of the evaluation path: clipping
limits extreme weights, while liquid alpha adjusts how quickly bonds respond.

References: [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)

## Relationship to Multiple Mechanisms

Consensus-based weights are usually discussed for one incentive mechanism at a time, but subnets can
also run multiple mechanisms in parallel. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms) notes
that each mechanism operates with its own bond pool for Yuma Consensus calculations.

For readers, consensus-based weights still adjust bond smoothing inside a subnet, but the mechanism
context matters. Bond smoothing in one mechanism should be read separately when a subnet uses more
than one evaluation system.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)

## Relationship to Yuma Consensus

Consensus-based weights and Yuma Consensus address related but different parts of Bittensor subnet
incentive flow. Consensus-based weights adjust validator-miner bond smoothing by consensus
alignment, while Yuma Consensus aggregates validator weight signals into miner incentives and
validator dividends. The
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)
documentation describes liquid alpha inside the bond path, and
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) describes how validator
evaluations become consensus outcomes.

For readers, consensus-based weights name bond-smoothing behavior inside consensus, while Yuma
Consensus names the aggregation step that turns validator evaluations into subnet-level reward
results.

References:
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)

## Relationship to Validator-Miner Bonds

Consensus-based weights and validator-miner bonds address related but different parts of subnet bond
processing. Validator-miner bonds describe the smoothed relationship between a validator and miners
over time, while consensus-based weights change how quickly those bonds can respond when a
validator's weights align with consensus. The
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)
describes the bond relationship, and the
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)
documentation describes liquid alpha as varying the bond-smoothing factor per validator-miner pair.

For readers, validator-miner bonds name the smoothed relationship, while consensus-based weights
name a consensus-alignment adjustment to how that relationship updates.

References:
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds),
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)

## Relationship to Dividends

Consensus-based weights and dividends address related but different parts of validator-side subnet
outcomes. Consensus-based weights adjust bond smoothing so validator dividends can better correlate
with the miners a validator weights, while dividends name the validator-side emission result
calculated from bonds and miner incentive values. The
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)
documentation frames that dividend correlation goal, and the
[Glossary: Dividends](https://docs.learnbittensor.org/resources/glossary#dividends) describes the
validator-side reward portion within emission distribution.

For readers, consensus-based weights name bond-path smoothing behavior, while dividends name the
validator-side emission outcome that smoothing helps shape.

References:
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Glossary: Dividends](https://docs.learnbittensor.org/resources/glossary#dividends)

## Relationship to Consensus Score

Consensus-based weights and a consensus score refer to related but different parts of Yuma
Consensus. A consensus score is the stake-weighted median used to filter outlier validator weight
submissions, while consensus-based weights adjust bond smoothing after weights are compared with
consensus alignment. The
[Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)
describes the agreement threshold applied to validator inputs, and the
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)
documentation explains how liquid alpha changes bond smoothing based on consensus alignment.

For readers, a consensus score names input filtering inside consensus, while consensus-based weights
name a bond-smoothing adjustment that follows from consensus alignment.

References:
[Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)

## Relationship to Liquid Alpha

Consensus-based weights and liquid alpha address related but different parts of bond smoothing
vocabulary. Consensus-based weights name the broader bond-smoothing adjustment tied to consensus
alignment, while liquid alpha is the specific per validator-miner pair mechanism that varies the
smoothing factor inside that adjustment. The
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)
documentation describes liquid alpha as the per-pair smoothing response, and the
[Glossary: Liquid Alpha](https://docs.learnbittensor.org/resources/glossary#liquid-alpha) describes
the variable alpha factor applied per validator-miner bond.

For readers, consensus-based weights name the bond-smoothing concept, while liquid alpha names the
per-pair smoothing mechanism that implements it.

References:
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Glossary: Liquid Alpha](https://docs.learnbittensor.org/resources/glossary#liquid-alpha)

## Relationship to Commit Reveal

Consensus-based weights and Commit Reveal address related but different parts of validator-weight
processing. Consensus-based weights adjust bond smoothing after validator evaluations are compared
with consensus alignment, while Commit Reveal can conceal those evaluations until a later reveal
time before consensus uses them publicly. The
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)
documentation describes bond-path smoothing behavior, and the
[Commit Reveal](https://docs.learnbittensor.org/concepts/commit-reveal) documentation describes
concealed weight submission and delayed reveal.

For readers, consensus-based weights name bond smoothing inside consensus, while Commit Reveal names
a visibility and timing layer that can protect fresh weight information before aggregation.

References:
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Commit Reveal](https://docs.learnbittensor.org/concepts/commit-reveal)

## Relationship to Drand Time-Lock Encryption

Consensus-based weights and Drand time-lock encryption address related but different parts of
delayed weight visibility. Consensus-based weights adjust validator-miner bond smoothing based on
consensus alignment, while Drand time-lock encryption provides the timing source that prevents
concealed weights from being decrypted before the target reveal round inside Commit Reveal. The
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)
documentation describes liquid alpha inside the bond path, and the
[Commit Reveal](https://docs.learnbittensor.org/concepts/commit-reveal) documentation describes
Drand as the reveal timing mechanism.

For readers, consensus-based weights name bond-path smoothing behavior, while Drand time-lock
encryption names the timing layer behind when concealed weight information becomes available.

References:
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Commit Reveal](https://docs.learnbittensor.org/concepts/commit-reveal)

## Relationship to Validator Weights

Consensus-based weights and validator weights address related but different parts of subnet
evaluation structure. Validator weights are the evaluation signals validators submit for miners,
while consensus-based weights adjust bond smoothing based on how closely those submissions align
with consensus. The
[Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights)
describes the per-validator evaluation inputs, and the
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)
documentation describes liquid alpha as varying bond-smoothing response per validator-miner pair.

For readers, validator weights name the evaluation submissions, while consensus-based weights name a
bond-smoothing adjustment that responds to how those submissions align with consensus.

References:
[Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights),
[Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights)
