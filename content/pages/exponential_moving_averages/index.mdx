# Exponential Moving Averages

An exponential moving average, or EMA, is a smoothing method. It blends a new measurement with a
running value from earlier updates
([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

In Bittensor, EMA vocabulary appears around signals that should respond over time instead of
changing entirely from one observation. Examples include emission flow and validator-miner bond
behavior ([Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds)).

## Smoothing Method

EMA smoothing gives newer measurements influence while keeping part of the earlier running value. A
larger smoothing factor reacts faster, while a smaller factor changes more slowly
([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema),
[Glossary: Exponential Moving Average](https://docs.learnbittensor.org/resources/glossary#exponential-moving-average-ema)).

That behavior is useful when a Bittensor signal should respond to sustained movement rather than
treating one short change as the whole story.

EMA does not choose the source signal. It changes how quickly a running value incorporates new
observations, while the surrounding context identifies the Bittensor value being smoothed
([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema),
[Glossary: Exponential Moving Average](https://docs.learnbittensor.org/resources/glossary#exponential-moving-average-ema)).

The method is therefore about time response. It says how a value changes across repeated updates,
not which Bittensor signal should be measured in the first place.

## Signal Context

EMA is a method, not a complete explanation by itself. A smoothed subnet-flow signal and a smoothed
bond value can both use EMA vocabulary while describing different Bittensor concepts
([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

An EMA claim needs both the smoothing method and the signal being smoothed. Flow, bonds, and
deregistration can all use EMA vocabulary, but the surrounding signal gives the term its local
meaning ([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

This keeps EMA from becoming a generic "stability" label. The meaning changes depending on whether
the smoothed value is flow, bond behavior, alpha price, or another documented signal.

## Emission Flow

Bittensor uses EMA-smoothed net TAO flow when comparing subnet-level emission allocation. Smoothing
helps short movement avoid dominating the flow signal
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Subnet Weights](https://docs.learnbittensor.org/resources/glossary#subnet-weights)).

In this context, EMA smooths the net-flow signal used for subnet comparison. It does not describe
miner or validator outcomes by itself; emission and incentive-mechanism vocabulary explain later
allocation and evaluation ([Emission](https://docs.learnbittensor.org/learn/emissions),
[Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

The useful reading is signal preparation. EMA helps shape the flow input that downstream emission
logic can use, while the emission article explains the reward-allocation context.

## Bonds and Liquid Alpha

EMAs also appear in validator-miner bond smoothing. A bond can evolve from repeated evaluation
signals instead of jumping entirely from one measurement
([Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Liquid alpha changes the EMA smoothing factor for validator-miner bonds according to consensus
alignment. It is a Bittensor rule for varying smoothing inside bond behavior rather than a separate
smoothing method
([Consensus-based Weights](https://docs.learnbittensor.org/concepts/consensus-based-weights),
[Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

Bond smoothing is therefore separate from the raw evaluation signal. EMA explains how the bond
responds over repeated updates; the surrounding consensus and bond sources explain what the bond is
used for
([Glossary: Validator-Miner Bonds](https://docs.learnbittensor.org/resources/glossary#validator-miner-bonds),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

This prevents liquid alpha from being read as a new kind of bond. It is a rule affecting the
smoothing factor used in bond updates, while validator-miner bond remains the surrounding consensus
concept.

## Deregistration Context

EMA vocabulary also appears in subnet deregistration because the removal rule uses a smoothed
alpha-price signal. The point is not that EMA removes a subnet by itself, but that a smoothed signal
participates in the documented selection rule
([Subnet Deregistration](https://docs.learnbittensor.org/subnets/subnet-deregistration),
[Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

That makes the consuming mechanism important. In deregistration context, EMA belongs to the
documented selection rule rather than to a general subnet quality score.

## Consensus Boundary

EMA smoothing can influence values used near consensus, but it is not the consensus rule itself.
Yuma Consensus explains how validator signals are aggregated into subnet outcomes, while EMA
describes smoothing behavior for selected values
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

Keeping that boundary clear prevents the smoothing method from being treated as the full incentive
system. EMA changes how a signal responds over time; the surrounding Bittensor concept defines what
the signal means.

## Reader Boundary

An EMA claim should identify what is being smoothed. In Bittensor, the phrase can refer to subnet
flow, validator-miner bonds, or another reward-related signal, but it is not the same thing as the
latest raw observed value, a reward forecast, or the consensus rule itself
([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

EMA blends new measurements with prior history so Bittensor processes can respond to sustained
changes. The consuming mechanism still needs to be named because EMA is only one part of the
surrounding tokenomics, consensus, or deregistration rule
([Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema)).

## Smoothing Alpha Is Not Alpha Tokens

Official [Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema) documentation
draws a hard line between the smoothing factor used in the EMA formula and the word alpha used
elsewhere in Bittensor tokenomics. The alpha parameter in the moving-average calculation controls
how much weight each new observation carries; it is distinct from and unrelated to
[alpha tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens) as subnet-specific
currencies.

That separation matters because Bittensor prose reuses alpha in several places. Consensus-based bond
smoothing can vary alpha by validator-miner pair, while alpha tokens name subnet assets and pool
reserves. An EMA discussion can still mention alpha as a smoothing factor without implying that the
formula is pricing or moving an alpha-token balance.

The
[Glossary: Exponential Moving Average](https://docs.learnbittensor.org/resources/glossary#exponential-moving-average-ema)
describes the technique as blending current and prior values through a smoothing factor. That factor
belongs to the update rule, not to Dynamic TAO asset vocabulary. When a sentence pairs EMA with
alpha, the useful question is whether the topic is smoothing behavior or subnet tokenomics.

References:
[Exponential Moving Averages](https://docs.learnbittensor.org/learn/ema#mathematical-definition),
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)

## Flow Smoothing Keeps a Long Half-Life

Subnet flow [emission](https://docs.learnbittensor.org/learn/emissions) smoothing uses an EMA with a
thirty-day half-life and an effective window of roughly eighty-seven days, according to the official
[subnet flow emission smoothing](https://docs.learnbittensor.org/learn/ema#subnet-flow-emission-smoothing)
section. Each update keeps almost all of the prior smoothed value and adds only a tiny fraction from
the current block's net [staking](https://docs.learnbittensor.org/staking-and-delegation/delegation)
movement.

The long memory is a deliberate guard against brief manipulation. Official documentation lists
short-term staking spikes, wash-trading-style price games, and flash attacks on emissions as
behaviors the slow EMA is meant to dampen. A
[subnet](https://docs.learnbittensor.org/resources/glossary#subnet) comparison therefore reacts to
sustained net [TAO](https://docs.learnbittensor.org/resources/glossary#tao) flow rather than one
burst of activity.

When smoothed net flow turns negative, official guidance also connects sustained negative flow with
zero emission share for that subnet. The zero outcome follows the smoothed signal, not a single
block of unstaking.

References:
[Exponential Moving Averages: Subnet Flow Emission Smoothing](https://docs.learnbittensor.org/learn/ema#subnet-flow-emission-smoothing),
[Emission: Distribution Across Subnets](https://docs.learnbittensor.org/learn/emissions#distribution-across-subnets)

## Protocol Cost Tracks a Second EMA

Flow-based [subnet weights](https://docs.learnbittensor.org/resources/glossary#subnet-weights) use
more than one smoothed input. Official EMA documentation describes a second parallel EMA that tracks
each subnet's per-block protocol cost, using the same smoothing factor as the user-flow EMA
([Exponential Moving Averages: Protocol Cost EMA](https://docs.learnbittensor.org/learn/ema#protocol-cost-ema)).

Protocol cost in that framing counts TAO injected into the pool plus chain buys minus root staker
claims. When net-flow accounting is active, the allocation input subtracts a scaled share of
positive protocol cost from the smoothed staking flow rather than treating gross user flow alone as
the full signal.

Both EMAs advance every block even when net-flow accounting is toggled, so switching the accounting
mode does not reset smoothed history. EMA vocabulary here therefore names two related smoothing
paths—participant staking movement and protocol-side TAO movement—that combine into one net-flow
comparison input.

References:
[Exponential Moving Averages: Protocol Cost EMA](https://docs.learnbittensor.org/learn/ema#protocol-cost-ema),
[Glossary: Subnet Weights](https://docs.learnbittensor.org/resources/glossary#subnet-weights)

## Environment Scope

Mainnet, testnet, and localnet are separate contexts for observed signal behavior
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)). The EMA method
can be explained across those environments, but an observed EMA example belongs to the network and
mechanism where it was observed.
