# Local Blockchain

A local blockchain is a private Bittensor chain used for development and testing. Official Bittensor
documentation distinguishes local development environments from public network behavior, which makes
the term useful when discussing tools, subnet experiments, and test workflows.

References:
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)

## Development Context

The local blockchain context is isolated from production use. It lets developers exercise Bittensor
components and subnet behavior without treating the result as mainnet state. That separation is the
main point of the term: local behavior is useful for development feedback, but it is not the same as
live network behavior.

References:
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)

## Relationship to Public Subtensor

Local blockchain and Public Subtensor name opposite sides of Bittensor chain access. The
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)
describes a private chain used for development and testing, while the
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)
describes publicly reachable chain access.

For readers, local blockchain names isolated development context, while Public Subtensor names live
network access context. Outputs from a local chain should not be treated as public Subtensor state.

References:
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain),
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)

## Relationship to Bittensor Networks

Local blockchain and Bittensor Networks name related but different parts of network context. The
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference includes
localnet as a user-controlled development environment, while the
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)
describes the private chain used in that setting.

For readers, localnet is the environment label, while local blockchain names the isolated
development chain itself. Mainnet and testnet belong to other network contexts rather than to local
blockchain by default.

References: [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)

## Relationship to Fast Blocks

Fast blocks are described in official documentation as a development-only local-chain timing
configuration. Local blockchain context explains why that distinction matters: faster local block
production can help testing while remaining separate from public Bittensor network behavior.

Reference: [Glossary: Fast blocks](https://docs.learnbittensor.org/resources/glossary#fast-blocks)

## Relationship to Multiple Mechanisms

A local blockchain provides an isolated chain for Bittensor development and testing. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#local-blockchain) describes it as used
for developing and testing subnet incentive mechanisms.

For readers, a local blockchain still names a private development environment. Subnets tested there
may later run more than one incentive mechanism in production, but the local chain itself is not a
mechanism label.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Incentive Mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism)

## Reader Boundary

This page defines the term. It does not describe a live public network, a production subnet, or a
deployment recommendation. Readers should use current official setup documentation when they need
step-by-step local development instructions.
