# Subnet Is Active

Subnet is active is a per-subnet flag, listed in the documentation as `SubnetIsActive`, that
indicates whether a subnet's emissions have started. The
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters) reference
describes it as indicating whether or not the subnet's emissions have started, which separates a
subnet that merely exists on-chain from one that is live and emitting.

## What It Indicates

The flag reports emission status for a subnet. A subnet record can exist at a
[netuid](https://docs.learnbittensor.org/resources/glossary#netuid) before its per-subnet issuance
begins; subnet is active becomes true once that
[emission](https://docs.learnbittensor.org/resources/glossary#emission) path has started
([Emission](https://docs.learnbittensor.org/learn/emissions)).

It is a status indicator rather than a task or scoring control. It does not describe what miners do,
how validators score them, or how much a subnet emits; it only reports whether the emission path for
that subnet has begun
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

## Documented Type and Default

The [Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)
reference lists `SubnetIsActive` as a Boolean with a default of `False`, meaning a newly registered
subnet starts inactive until its emissions begin. Because each subnet carries its own state, the
flag is defined per subnet and reflects only that subnet's emission status.

The value in force is per-subnet chain state, so whether a given subnet is currently active should
be read for that subnet rather than assumed from the default.

## Distinction from Start Call

Start call is the action that begins a subnet's emissions, while subnet is active is the resulting
state that reports whether those emissions have started. One is the trigger; the other is the status
it produces ([Emission](https://docs.learnbittensor.org/learn/emissions),
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

- **Start call** — the operation that switches a subnet's emissions on.
- **Subnet is active** — the flag that reports emissions have started.

## Distinction from Registration Allowed

Registration allowed governs whether new neurons can register on a subnet, while subnet is active
reports whether the subnet is emitting. A subnet can be registered and configured without yet
emitting, so the two answer different questions
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

- **Registration allowed** — whether new neurons may join the subnet.
- **Subnet is active** — whether the subnet's emissions have started.

## BTCLI Reads the Live Value

Any user can read subnet hyperparameters for a selected netuid with
`btcli subnet hyperparameters --netuid 1`. The output reflects the live subnet state, including
whether the subnet's emissions have started, for that subnet at query time
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

Live status is network- and subnet-specific, so an active-state claim should be checked against the
current chain state for the relevant netuid rather than assumed from the documented default.

## Why an Emission-Status Flag Exists

The flag exists because registration and emission are separate milestones. A subnet slot can be
created and configured before its issuance begins, and subnet is active is what tells readers and
tooling that the emission path is now live
([Emission](https://docs.learnbittensor.org/learn/emissions)).

For readers, this means a true subnet-is-active value marks the point where per-subnet emissions
flow, not the moment the subnet was first registered. Claims about a subnet earning emissions should
be tied to this active state rather than to the existence of the netuid alone
([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)).

## Distinction from Subnet Symbol

Subnet is active and subnet symbol are both subnet-level concepts, illustrated here on a subnet such
as netuid 1, but they describe entirely different things. Subnet is active is the boolean flag for
whether a subnet's emissions have started, while subnet symbol is the written character used to
denote that subnet's token in interfaces. One is an emission-status indicator; the other is a
display label for the subnet's currency
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

- **Subnet is active** — whether the subnet's emissions have started.
- **Subnet symbol** — the written character denoting the subnet's token.

## Distinction from Subnet Owner Take

Subnet is active and subnet owner take describe an emission status and an emission share,
illustrated here on a subnet such as netuid 1. Subnet is active reports whether a subnet's emissions
have started, while subnet owner take is the documented portion of those emissions allocated to the
subnet owner. The flag answers whether emission is flowing; the take answers how much of it goes to
the owner once it does ([Emission](https://docs.learnbittensor.org/learn/emissions),
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution)).

- **Subnet is active** — whether the subnet's emissions have started.
- **Subnet owner take** — the owner's share of those emissions once they flow.

## Distinction from Yuma Consensus

Subnet is active names the boolean hyperparameter that reports whether a subnet's emission path has
started. Yuma Consensus is the separate mechanism that converts validator weight submissions into
emission shares each tempo on active subnets
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions),
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

- **Subnet is active** — whether the subnet's emissions have started.
- **Yuma Consensus** — mechanism that turns validator weights into emission shares.

## Reader Boundary

Subnet is active should not be read as a registration gate, an emission amount, or proof that a
subnet is profitable. It names only the boolean flag that reports whether the subnet's emission path
has started on the selected netuid
([Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).
