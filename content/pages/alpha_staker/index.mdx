# Alpha Staker

An alpha staker is a participant who holds subnet alpha through staking in a specific Bittensor
subnet. The official
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution) documentation
describes emissions to stakers holding subnet alpha in a mining subnet separately from emissions to
stakers on the Root Subnet. Alpha staker names the subnet-local staking side of that distinction.

## Subnet Alpha Context

Subnet alpha is a subnet-specific asset in Dynamic TAO tokenomics. The
[Glossary: Stake](https://docs.learnbittensor.org/resources/glossary#stake) describes stake as
currency tokens delegated to a validator UID in a subnet, and alpha staking places that support in
the subnet's alpha context rather than through the Root Subnet TAO path.

For readers, alpha staker should be read as a role tied to one subnet market and its alpha token,
not as a network-wide TAO balance label or a wallet file location.

## Contrast with Root Staker

Bittensor staking prose distinguishes subnet-local alpha staking from Root Subnet TAO staking. The
[Glossary: Root Subnet](https://docs.learnbittensor.org/resources/glossary#root-subnetsubnet-zero)
describes Subnet Zero as the subnet-agnostic TAO staking path, while alpha staker language belongs
to stake held in an ordinary mining subnet's alpha context.

This keeps two ideas separate: root staker names TAO staked through Root Subnet validators, while
alpha staker names participation through subnet alpha in a specific subnet.

## Stake and Emissions Context

Alpha stakers participate in the subnet emission picture described in official distribution
references. The [Emission](https://docs.learnbittensor.org/learn/emissions) documentation treats TAO
and subnet alpha as separate assets in reward flow, and the
[Staking and delegation overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)
frames how TAO holders attach support to validators inside subnet markets.

This article does not quote live balances, slippage outcomes, or staking steps. Those details belong
in official staking documentation and live chain data.

## Relationship to Multiple Mechanisms

An alpha staker supplies stake inside one subnet market. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms) notes
that subnets may run more than one incentive mechanism, each with separate bond pools for Yuma
Consensus calculations.

For readers, alpha staker still names subnet-alpha staking in one subnet. When that subnet runs more
than one incentive mechanism, evaluation and emissions context should be read per mechanism rather
than as one blended alpha-staking label.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)

## Relationship to Root Proportion

Alpha staker and root proportion describe opposite sides of a subnet's dividend split. The
[Glossary: Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion)
defines root proportion as the relative weight of TAO staked through the root subnet compared with
the subnet's alpha-token context. Alpha staker names the participant role on the alpha side of that
comparison.

This distinction helps when reading emission material. Root proportion is a subnet-level measure,
while alpha staker is a participant role. A statement about root proportion describes how dividend
context is divided between root-routed stake and subnet-local alpha stake; it is not itself a label
for any one staker.

References:
[Glossary: Root Proportion](https://docs.learnbittensor.org/resources/glossary#root-proportion),
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution)

## Relationship to Alpha Tokens

Alpha staker depends on alpha tokens but is not the same term. The
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens) describes
alpha tokens as subnet-specific tokens used within individual subnet economies. Alpha staker names a
participant who holds stake through that subnet-alpha context.

The asset-role distinction matters because alpha tokens can be discussed in liquidity, reserve, or
emission contexts without necessarily describing a staker. Alpha staker is narrower: it identifies a
staking participant within a specific subnet market.

References:
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens),
[Staking into Subnets](https://docs.learnbittensor.org/staking-and-delegation/delegation#staking-into-subnets)

## Relationship to Dividends

Alpha stakers are part of the dividend context, but dividends are the reward-side term. The
[Glossary: Dividends](https://docs.learnbittensor.org/resources/glossary#dividends) describes
dividends as the validator-facing reward share from emissions, and the emission documentation
distinguishes root-subnet stakers from stakers holding subnet alpha in a mining subnet.

In reader terms, alpha staker names who is positioned on the subnet-alpha staking side. Dividends
name one reward flow that can be distributed through validator and staker relationships after the
subnet's emission and consensus processes run.

References: [Glossary: Dividends](https://docs.learnbittensor.org/resources/glossary#dividends),
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution)

## Relationship to TAO Weight

Alpha staker and TAO weight are related but distinct parts of Bittensor staking vocabulary. Alpha
staker names a participant who stakes through a subnet's alpha-token context in a mining subnet,
while TAO weight is the global parameter (currently 0.18) that determines how TAO stake contributes
relative to alpha stake in stake-weight and root-proportion calculations. The
[Glossary: Alpha Staker](https://docs.learnbittensor.org/resources/glossary#alpha-staker) describes
alpha stakers as participants who stake into a specific subnet using that subnet's alpha token, and
the [Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight) entry
describes TAO weight as the global tuning parameter that blends TAO stake and alpha stake in
validator-influence arithmetic.

For readers, alpha staker names a participant role, while TAO weight names a global parameter that
shapes how that participant's alpha stake is combined with other TAO stake when consensus and
emission outcomes are computed. The two are not interchangeable: one names who stakes, the other
names how their stake is weighted relative to TAO stake when validator influence is calculated.

References:
[Glossary: Alpha Staker](https://docs.learnbittensor.org/resources/glossary#alpha-staker),
[Glossary: TAO Weight](https://docs.learnbittensor.org/resources/glossary#tao-weight)

## Relationship to Autostaking

Alpha staker and autostaking describe related but different ways emissions reach staked positions in
Bittensor subnets. An alpha staker is a participant who holds subnet alpha through staking in a
specific subnet market, while autostaking is a mechanism that automatically routes a miner's own
emissions into a staking position as rewards arrive—effectively growing an alpha staker position in
the target validator's subnet without manual steps. The
[Auto Staking for Miners](https://docs.learnbittensor.org/miners/autostaking) documentation
describes autostaking as automatically staking a miner's emissions to a chosen validator, removing
the need for repeated manual staking after each reward arrives.

For readers, alpha staker names the staking role—holding subnet alpha in a subnet market—while
autostaking names one automated path that produces and grows that role over time. A miner who
enables autostaking incrementally builds an alpha staker position without separately executing each
staking transaction. The two concepts are additive: autostaking is the mechanism, and the resulting
accumulated stake is the alpha staker relationship to the subnet. The
[Emission](https://docs.learnbittensor.org/learn/emissions) documentation provides the emissions
context in which both alpha staking and autostaking operate.

References: [Auto Staking for Miners](https://docs.learnbittensor.org/miners/autostaking),
[Emission](https://docs.learnbittensor.org/learn/emissions)

## Relationship to Conviction and Locked Stake

An alpha staker and conviction both involve subnet alpha stake, but they describe different things.
An alpha staker is a participant who holds subnet alpha through staking in a specific subnet, as the
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution) documentation
frames the subnet-local staking side. Conviction and locked stake, by the
[Conviction and locked stake](https://docs.learnbittensor.org/staking-and-delegation/conviction-staking)
documentation, describe locking subnet alpha so that it builds a time-based commitment score and the
staked balance cannot be silently reduced below the locked amount.

For readers, alpha staker names who holds staked subnet alpha, while conviction names whether that
alpha has been locked to signal commitment over time. The two are not interchangeable: being an
alpha staker is about holding subnet alpha through staking, whereas conviction is an optional lock
applied to alpha that constrains its reduction and builds a commitment score. An alpha staker can
hold alpha without locking it; conviction is what an alpha staker takes on when they lock a position
for commitment.

References: [Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution),
[Conviction and locked stake](https://docs.learnbittensor.org/staking-and-delegation/conviction-staking)

## Reader Boundary

Alpha staker should not be read as a validator recommendation, a live portfolio report, or a synonym
for nominator in every context. It is a role term for subnet-alpha staking participation. Exact
balances, fees, slippage, and subnet-specific behavior belong in current official staking
references.

## Relationship to Coinbase

An alpha staker and coinbase are related but different parts of Bittensor's staking and emission
flow. An alpha staker is a participant who holds subnet alpha through staking in a specific subnet,
earning subnet-local dividends. The
[Coinbase Implementation](https://docs.learnbittensor.org/navigating-subtensor/emissions-coinbase)
documentation describes coinbase as the per-block protocol mechanism that drives TAO emission,
accumulates pending emissions, and triggers Yuma Consensus rounds at epoch boundaries, while the
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution) documentation
describes the alpha-staking side of distribution.

The connection is that coinbase is part of the upstream emission process the alpha staker's
dividends originate from, while the alpha staker is a participant role on the staking side. Coinbase
advances emissions and fires the epoch calculations each block; the subnet-alpha dividends an alpha
staker receives then arise from that cycle through the subnet's distribution mechanics. An alpha
staker names who holds subnet-local staked alpha; coinbase names the per-block engine whose emission
output those dividends ultimately come from — they are different layers rather than the same
operation.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from local testing to testchain and then mainchain. For alpha
staker, that sequence changes how readers should interpret evidence about subnet-alpha staking
participation.

In local testing, alpha staker behavior can be observed when participants stake subnet alpha in an
isolated subnet environment. That is useful for checking the subnet-local staking path, but it
remains isolated feedback from a local environment and does not reflect production emission
outcomes.

On testchain, alpha staker activity can be observed in a shared testing network. This gives stronger
evidence about how subnet-alpha staking connects to validator rewards and distribution than a
private local run, while still keeping the result separate from production mainchain staking
outcomes.

On mainchain, an alpha staker participates in live subnet alpha staking tied to actual emission and
distribution activity. The
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution) documentation
describes emissions to stakers holding subnet alpha in a mining subnet, so production alpha staker
context is tied to real subnet staking on the active network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. Alpha staker activity observed in one environment should
not be read as identical evidence about subnet-local staking outcomes in another environment.

For readers, this keeps alpha staker examples from sounding stronger than their environment
supports. Local, testchain, and mainchain contexts can all describe subnet-alpha staking
participation, but they do not carry the same interpretive weight.
