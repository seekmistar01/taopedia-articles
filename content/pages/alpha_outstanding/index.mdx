# Alpha Outstanding

Alpha outstanding is subnet alpha outside a subnet's reserve pool. It describes the outside-reserve
side of subnet alpha in Bittensor tokenomics
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The term is a location label. It separates subnet alpha held in the automated-market-maker pool from
subnet alpha outside that pool.

## Outside-Pool Meaning

A subnet liquidity pool has reserve-side alpha and outside-reserve alpha. Alpha outstanding names
the outside-reserve side
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

This does not make alpha outstanding a separate token. It is a way to describe where part of the
same subnet-specific alpha asset sits in the tokenomics model.

## Emission Context

The emission documentation uses alpha outstanding injection for alpha set aside outside the pool
([Emission: Alpha outstanding injection](https://docs.learnbittensor.org/learn/emissions#alpha-outstanding-injection)).

That separates it from alpha reserve injection. Alpha reserve injection supports the pool side,
while alpha outstanding injection belongs to the outside-pool side of subnet emissions.

## Reserve Contrast

Alpha reserve and alpha outstanding are opposite location terms for subnet alpha. Alpha reserve is
pool-held alpha, while alpha outstanding is alpha outside the reserve pool
([Emission](https://docs.learnbittensor.org/learn/emissions)).

The distinction keeps reserve statements from being read as total-alpha statements. A subnet can
have alpha in reserve and alpha outstanding at the same time, but the phrases describe different
locations.

## Stake Context

The subnet overview connects alpha outstanding with total stake in a subnet
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

For a reference page, this context should stay narrow. Alpha outstanding identifies the
outside-reserve side of subnet alpha; it is not a staking walkthrough or return estimate.

## ADR Context

Alpha Distribution Ratio compares alpha emitted outside the pool with alpha injected into the
automated market maker pool
([Glossary: ADR](https://docs.learnbittensor.org/resources/glossary#adr-alpha-distribution-ratio)).

In that comparison, alpha outstanding belongs to the outside-pool side. Alpha reserve belongs to the
pool-injection side.

## Protocol Alpha Boundary

Protocol alpha identifies alpha gathered by the protocol during reserve-injection activity
([Glossary: Protocol Alpha](https://docs.learnbittensor.org/resources/glossary#protocol-alpha)).
Alpha outstanding is broader outside-pool alpha context.

Keeping the terms separate avoids mixing a location label with a protocol-retained alpha category.

## Network Context

Bittensor documentation separates localnet, testnet, and mainnet environments
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

Alpha outstanding examples should keep that environment boundary attached. A localnet or testnet
example can illustrate outside-reserve mechanics, but it is not evidence of a mainnet alpha
outstanding outcome.

## Reader Boundary

Alpha outstanding is a tokenomics term, not a separate token, operating guide, or return estimate.
It should be used when the focus is subnet alpha outside the reserve pool
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

When the focus is pool-held alpha, alpha reserve is more precise. When the focus is the subnet asset
broadly, alpha tokens is more precise. When the focus is the wider tokenomics system, Dynamic TAO is
more precise.
