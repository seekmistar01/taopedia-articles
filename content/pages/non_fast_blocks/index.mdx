# Non-fast Blocks

Non-fast blocks name ordinary block timing in contrast with fast blocks. Official Bittensor
documentation presents the term alongside fast blocks in local-development context.

References:
[Glossary: Non-fast blocks](https://docs.learnbittensor.org/resources/glossary#non-fast-blocks),
[Glossary: Fast blocks](https://docs.learnbittensor.org/resources/glossary#fast-blocks)

## Local Context

The term helps readers distinguish two local timing concepts. Fast blocks refer to accelerated local
block production. Non-fast blocks refer to the non-accelerated side of that same local distinction.

References:
[Glossary: Non-fast blocks](https://docs.learnbittensor.org/resources/glossary#non-fast-blocks),
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)

## Twelve-Second Interval Context

The [Glossary: Non-fast blocks](https://docs.learnbittensor.org/resources/glossary#non-fast-blocks)
describes non-fast blocks as a development-only configuration that adheres to Subtensor's default
12-second block interval, simulating production timing for features like delayed subnet activation.
Readers should treat this as a local timing setting rather than a live mainnet behavior label.

References:
[Glossary: Non-fast blocks](https://docs.learnbittensor.org/resources/glossary#non-fast-blocks),
[Run a Local Bittensor Blockchain Instance](https://docs.learnbittensor.org/local-build/deploy)

## Relationship to Localnet

Non-fast blocks and localnet are related but name different parts of local development context. The
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference includes
localnet as a user-controlled development environment, while the
[Glossary: Non-fast blocks](https://docs.learnbittensor.org/resources/glossary#non-fast-blocks)
describes ordinary local block timing inside that setting.

For readers, localnet is the environment label, while non-fast blocks are a timing configuration
used when simulating production-like block intervals during local development.

References: [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Glossary: Non-fast blocks](https://docs.learnbittensor.org/resources/glossary#non-fast-blocks)

## Relationship to Local Blockchain

Non-fast blocks and local blockchain are related but name different parts of local development
context. The
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)
describes a private chain used for development and testing, while the
[Glossary: Non-fast blocks](https://docs.learnbittensor.org/resources/glossary#non-fast-blocks)
describes ordinary local block timing that follows Subtensor's default 12-second interval.

For readers, local blockchain is the isolated development chain, while non-fast blocks are a timing
setting used within that local environment. Neither term describes live mainnet block behavior on
its own.

References:
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain),
[Glossary: Non-fast blocks](https://docs.learnbittensor.org/resources/glossary#non-fast-blocks)

## Relationship to Fast Blocks

Non-fast blocks and fast blocks name opposite sides of the same local timing distinction. The
[Glossary: Non-fast blocks](https://docs.learnbittensor.org/resources/glossary#non-fast-blocks)
describes ordinary local block timing with Subtensor's default 12-second interval, while the
[Glossary: Fast blocks](https://docs.learnbittensor.org/resources/glossary#fast-blocks) describes
accelerated local production.

For readers, non-fast blocks help simulate production timing in development, while fast blocks trade
that simulation for speed. Neither term describes live mainnet or testnet block behavior.

References:
[Glossary: Non-fast blocks](https://docs.learnbittensor.org/resources/glossary#non-fast-blocks),
[Glossary: Fast blocks](https://docs.learnbittensor.org/resources/glossary#fast-blocks)

## Reader Boundary

This page defines the term. It does not describe live public network behavior or a subnet reward
mechanism.
