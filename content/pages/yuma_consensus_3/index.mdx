# Yuma Consensus 3

Yuma Consensus 3, or YC3, is a refinement of Bittensor's Yuma Consensus mechanism. The official
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)
article describes YC3 as an evolution focused on validator fairness, validator-miner bonds, and
faster recognition of useful miner work.

## What YC3 Addresses

Yuma Consensus turns validator evaluations into subnet incentive allocation. The YC3 article
describes earlier issues around small validator treatment, stale bond allocation, and slow
recognition of miners that only some validators had identified. YC3 narrows those issues by making
bond movement more responsive to individual validator-miner relationships.

Reference:
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Per-Bond EMA Scaling

A central YC3 idea is per-bond EMA scaling. Instead of treating every validator-miner bond with the
same adjustment rate, YC3 lets each pair move according to its own relationship to subnet consensus.
This gives the mechanism a more local view of how a validator's evaluation relates to a miner over
time.

Reference:
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Early Recognition

The official YC3 article emphasizes earlier recognition of promising miners. Under YC3, validators
can begin building bonds with miners before those miners are broadly recognized by the validator
set. This helps distinguish independent evaluation from simply following already established
consensus.

Reference:
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Liquid Alpha Context

YC3 is described alongside Liquid Alpha because both features concern how validator evaluations
respond to changing subnet consensus. The YC3 article says the mechanisms can work together to
support independent evaluation while preserving the broader anti-manipulation goals of Yuma
Consensus. Taopedia readers should treat Liquid Alpha as related context rather than as a separate
operating step inside YC3.

Reference:
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)

## Relationship to Multiple Mechanisms

Yuma Consensus 3 refines validator-miner bonding inside subnet consensus. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms) notes
that each mechanism operates with its own bond pool for Yuma Consensus calculations.

For readers, YC3 still adjusts bonding inside a subnet that may run more than one incentive
mechanism.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)

## Careful Interpretation

YC3 is not a separate subnet and not a guide for changing subnet settings. It is a consensus
refinement within Bittensor's validator-miner incentive machinery. Exact activation status,
settings, and subnet-specific behavior should be checked against current official documentation and
chain state.

Reference:
[How Yuma Consensus 3 Makes Bittensor More Fair](https://docs.learnbittensor.org/learn/yc3-blog)
