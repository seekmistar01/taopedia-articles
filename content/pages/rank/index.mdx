# Rank

Rank is a Bittensor consensus metric for summarizing miner evaluation inside a subnet. It is the
aggregate miner-side result produced by Yuma Consensus from validator weight signals
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

The term belongs to the miner-evaluation side of consensus. It is not one validator's raw view and
not a general reputation label for every Bittensor activity
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank)).

## What It Represents

Rank answers: after all validators have submitted weights and consensus has run, where does each
miner stand in the subnet's evaluation result? Raw validator weights are inputs; rank is the
aggregated output after Yuma Consensus has compared, filtered, and normalized those inputs across
the validator set ([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Examples tied to netuid 1 illustrate one subnet's miner-side consensus result. They should not be
read as representing production evaluation on another network
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from Weight Matrix

The weight matrix collects validator weight vectors as structured input for Yuma Consensus. Rank is
the miner-side standing read after that grouped input is processed, not from one isolated vector
inside the matrix
([Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix),
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Weight matrix** — grouped set of validator weight vectors entering consensus.
- **Rank** — miner-side standing after consensus processes that input.

## Distinction from Trust

Rank is the miner-side aggregate standing after Yuma Consensus filters and combines validator
weights. Trust is the ratio of final rank to pre-rank that shows how much support survived clipping
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Glossary: Trust](https://docs.learnbittensor.org/resources/glossary#trust),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Rank** — miner-side standing after aggregation.
- **Trust** — ratio showing how much pre-rank signal survived filtering.

## Distinction from Consensus Score

Consensus score is the stake-weighted benchmark used to clip outlier validator weights. Rank is read
after that benchmark has shaped miner-side support on the selected subnet
([Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Yuma Consensus: Clipping](https://docs.learnbittensor.org/learn/yuma-consensus#clipping)).

- **Consensus score** — stake-weighted agreement benchmark for clipping.
- **Rank** — miner-side standing after aggregation.

## Distinction from Incentives

Incentives name miner-side emission outcomes credited after rank is normalized. Rank is the standing
input that precedes that credited result on the selected subnet
([Glossary: Incentives](https://docs.learnbittensor.org/resources/glossary#incentives),
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Yuma Consensus: Miner emissions](https://docs.learnbittensor.org/learn/yuma-consensus#miner-emissions)).

- **Rank** — consensus standing before credited miner emissions.
- **Incentives** — miner-side emission outcomes credited after normalization.

## Distinction from Weight Vector

A weight vector is one validator's submitted evaluation signals. Rank is interpreted after the full
validator set is filtered and aggregated into miner-side outcomes
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank)).

- **Weight vector** — one validator's input signal before aggregation.
- **Rank** — miner-side standing after those signals are processed.

## Distinction from Validator Trust

Validator trust measures validator-side clipped influence across neurons. Rank measures miner-side
standing after the same Yuma pass on the selected subnet
([Glossary: Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust),
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank)).

- **Validator trust** — validator-side clipped influence measure.
- **Rank** — miner-side standing from the same Yuma pass.

## Distinction from Dividends

Rank measures miner-side standing after Yuma aggregates clipped weights on the selected subnet.
Dividends are validator-side emission outcomes credited from bonded support and those miner results
in the same pass ([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Glossary: Dividends](https://docs.learnbittensor.org/resources/glossary#dividends),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Rank** — miner-side consensus standing.
- **Dividends** — validator-side credited share from the parallel branch.

## Distinction from Multiple Incentive Mechanisms

Rank and multiple incentive mechanisms connect a consensus standing to a subnet's split evaluation.
Rank is the consensus standing that orders miners, while multiple incentive mechanisms is the case
where one subnet runs more than one evaluation mechanism, each with its own bond pool
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)).

- **Rank** — standing computed separately within each mechanism track.
- **Multiple incentive mechanisms** — split evaluation with separate bond pools per mechanism.

## Distinction from Incentive Compatibility

Rank is the miner-side consensus result after weights merge. Incentive compatibility names design
goals about keeping evaluation signals aligned across validators
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

- **Rank** — miner-side standing after Yuma aggregation.
- **Incentive compatibility** — design goal for aligned validator evaluation.

## Distinction from Yuma Consensus

Rank is the miner-side standing credited after Yuma clips and aggregates submitted validator
weights. Yuma Consensus is the on-chain tempo process that produces that standing from the weight
matrix ([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission: Distribution](https://docs.learnbittensor.org/learn/emissions#distribution)).

- **Rank** — miner-side standing after Yuma clipping and aggregation.
- **Yuma Consensus** — tempo step that turns submitted weights into emission shares.

- **Outcome vs process** — rank names miner-side standing; Yuma names the settlement pass that
  produces it
  ([Yuma Consensus: Miner emissions](https://docs.learnbittensor.org/learn/yuma-consensus#miner-emissions)).
- **Multi-mechanism scope** — on split subnets, rank belongs to one mechanism track rather than one
  combined ordering
  ([Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets)).

## Reader Boundary

Rank is a consensus-result term for miners after Yuma Consensus. It is not a universal quality
label, reward guarantee, or one validator's raw score
([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Distinction from Pruning Score

Rank and pruning score serve different roles for a miner. Rank is the consensus-weighted measure of
a miner's performance derived from validator weights, feeding incentive and emission, while pruning
score is the value the chain uses to decide which UID is dropped when a full subnet must make room
for a new registration ([Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Glossary: Pruning Score](https://docs.learnbittensor.org/resources/glossary#pruning-score),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Rank** — the performance measure that feeds incentive and emission.
- **Pruning score** — the value that decides which UID is deregistered when a subnet is full.
