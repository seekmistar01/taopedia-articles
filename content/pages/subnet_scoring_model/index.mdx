# Subnet Scoring Model

A subnet scoring model is the evaluation method validators use when judging miner responses inside a
Bittensor subnet
([Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The term belongs to incentive-mechanism vocabulary. It names how responses become evaluation scores,
so validators can turn returned work into signals used later in the incentive process.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For subnet scoring
models on a subnet such as netuid 1, that sequence changes how readers should interpret evaluation
methods and observed score outcomes.

In localnet, a subnet scoring model can be exercised in an isolated development environment.
Localnet evaluation examples on netuid 1 reflect local configuration rather than production
validator behavior.

On testnet, scoring models can be observed in a shared non-production network. Testnet evaluation
outcomes on a selected netuid are separate from mainnet metagraph and incentive state
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

On mainnet, subnet scoring model interpretation concerns live validator evaluation on the connected
Bittensor network. Observed scores and their emission effects depend on the active netuid,
mechanism, and tempo context.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A scoring model example from one environment should not be
read as representing production subnet performance in another environment.

## Distinction from Subnet Protocol

Subnet scoring model names how validators judge returned miner work. Subnet protocol names the
exchange rules for how that work is requested and returned
([Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The protocol creates the evaluation surface. The scoring model explains the method applied to
returned work.

- **Subnet scoring model** — how validators judge returned miner work.
- **Subnet protocol** — exchange rules for how that work is requested and returned.

## Distinction from Subnet Task

Subnet task names the work a mechanism asks miners to produce. Subnet scoring model names how
validators evaluate responses against that work target
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The task asks for work. The scoring model judges the responses miners return.

- **Subnet scoring model** — how returned work is judged.
- **Subnet task** — the work miners are asked to produce.

## Distinction from Subnet Miner

Subnet miner names the work-producing role. Subnet scoring model names the evaluation method
validators apply to miner responses
([Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The miner produces responses. The scoring model is the method validators use to judge them.

- **Subnet scoring model** — evaluation method applied to miner responses.
- **Subnet miner** — the work-producing role.

## Distinction from Subnet Validator

Subnet validator names the evaluating role. Subnet scoring model names the method that role uses to
turn returned work into scores before weight submission
([Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The validator evaluates work. The scoring model defines how that evaluation is done.

- **Subnet scoring model** — the method for turning returned work into scores.
- **Subnet validator** — the evaluating role that applies that method.

## Distinction from Weight Vector

Subnet scoring model names the subnet-defined method a validator uses to judge returned miner work.
A weight vector is the structured set of miner scores that method produces before on-chain
submission
([Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Scoring turns returned work into scores. The weight vector carries those scores into consensus.

- **Subnet scoring model** — the evaluation rule applied to returned work.
- **Weight vector** — the structured miner scores a validator submits after evaluation.

## Distinction from Subjective Evaluation

The subnet scoring model and subjective evaluation meet where judgment becomes a defined method. The
subnet scoring model is the subnet-defined method that turns returned work into the scores
validators submit, while subjective evaluation is the challenge that subnet work often has no single
correct answer, so validators may score the same submission differently
([Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

A scoring model can be defined even when the underlying judgment stays subjective and consensus
resolves the differences
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

- **Subnet scoring model** — the subnet-defined method for turning work into scores.
- **Subjective evaluation** — the challenge that subnet work often has no single correct answer.

## Distinction from Consensus Score

A subnet scoring model is the subnet-specific method a validator uses to turn returned miner work
into scores before submitting weights. A consensus score is the stake-weighted agreement benchmark
Yuma Consensus computes from the submitted weights to decide clipping
([Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Subnet scoring model** — the subnet-specific method a validator uses to score miner work.
- **Consensus score** — the stake-weighted benchmark Yuma uses to clip submitted weights.

## Distinction from Yuma Consensus

Subnet scoring model names the subnet-defined method a validator uses to score miner work before
submitting weights. Yuma Consensus is the separate mechanism that aggregates those submitted weights
into emission shares each tempo
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model)).

- **Subnet scoring model** — subnet-specific method for scoring miner work.
- **Yuma Consensus** — mechanism that turns validator weights into emission shares.

## Distinction from Off-Chain Validation Systems

Off-chain validation systems name the subnet-side evaluation layer where validators judge miner work
before chain-facing signals exist. The subnet scoring model names the subnet-defined method that
turns returned work into the scores used in that judgment
([Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Evaluation-layer vocabulary and scoring-method vocabulary answer different questions on the same
path. Off-chain validation describes where subnet-specific judgment runs; the scoring model
describes how that judgment is defined.

## Reader Boundary

Subnet scoring model should not be read as benchmark selection, deployment status, or a full subnet
design. It names concept vocabulary for how miner responses are evaluated inside a subnet incentive
mechanism
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Subnet task names the work target. Validator-miner exchange rules name how work is requested and
returned. Subnet scoring model names how returned work is evaluated before signals move toward
weight vectors and Yuma Consensus
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).
