# External Wallet

An external wallet is wallet software or custody outside the local Bittensor wallet path. The
[External Wallet glossary entry](https://docs.learnbittensor.org/resources/glossary#external-wallet)
places the term in wallet-security vocabulary, where location and custody context matter for how a
wallet is understood.

## Custody Context

External wallet names a custody context, not a different token or subnet role. The
[wallet documentation](https://docs.learnbittensor.org/keys/wallets) gives the broader setting for
Bittensor wallet concepts, while the external-wallet glossary entry narrows the term to wallet
software or custody outside the local path.

## Local Wallet Contrast

The contrast term is local wallet. A
[local wallet](https://docs.learnbittensor.org/resources/glossary#local-wallet) belongs to the local
Bittensor wallet path, while an external wallet places that wallet context outside the local path.
The difference is about where wallet control is managed, not about creating a separate Bittensor
asset.

## Wallet Address Boundary

External wallet and wallet address are related but separate concepts. A
[wallet address](https://docs.learnbittensor.org/resources/glossary#wallet-address) is a
public-key-derived destination identifier for TAO, while external wallet names custody or software
context. That distinction keeps destination-address vocabulary separate from custody-location
vocabulary.

## Address-Poisoning Context

External wallets still use destination addresses, so address-poisoning risk remains relevant. The
[address-poisoning guide](https://docs.learnbittensor.org/keys/address-poisoning-scams) focuses on
lookalike destination addresses during wallet use, while the external-wallet term names where wallet
control is managed. The risk and the custody context are connected but not identical.

## Relationship to Public and Private Keys

External wallet sits near key vocabulary because wallets organize access around key material. A
[public key](https://docs.learnbittensor.org/resources/glossary#public-key) is shareable reference
material, while a [private key](https://docs.learnbittensor.org/resources/glossary#private-key) is
protected authorization material. External wallet does not rename either side of that pair.

## Reading the Term

External wallet should be read as custody-location vocabulary. It does not name a subnet role,
validator recommendation, wallet provider endorsement, or balance state. The
[External Wallet glossary entry](https://docs.learnbittensor.org/resources/glossary#external-wallet)
anchors the concept, with wallet and address references supplying the surrounding context.
