# Hotkeys (Bittensor)

A hotkey is the operational key in the Bittensor wallet model. It is used by participants to perform
routine, on-chain actions needed for roles like miners and validators.

References: [Wallets (learnbittensor.org)](https://docs.learnbittensor.org/keys/wallets),
[Working with Keys](https://docs.learnbittensor.org/keys/working-with-keys),
[BTCLI Reference](https://docs.learnbittensor.org/btcli)

## Responsibilities

- Identify and authorize operational actions by network participants (e.g., register a
  miner/validator, submit evaluation weights).
- Interact with subnet services according to published mechanisms.

## Security model

- Hotkeys are designed to be used on online infrastructure (e.g., validator and miner nodes).
- They should not control funds; if compromised, the coldkey can rotate or revoke authority.

## Operational practices

- Use separate hotkeys for distinct roles or nodes to bound blast radius.
- Monitor activity and rotate keys if suspicious behavior is detected.
- Keep node environments patched and restrict access to hotkey material.

## Tooling

- btcli and the Wallet SDK support generating and managing hotkeys under a wallet.
- Registration and day-to-day actions are performed with the hotkey.

## Relationship to Coldkeys

Hotkeys and coldkeys form a two-tier key model in Bittensor. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#hotkey) describes a hotkey as the
operational key for network roles, while
[Coldkeys](https://docs.learnbittensor.org/resources/glossary#coldkey) hold TAO and stake and sign
financial operations such as staking, unstaking, and transfers.

For readers, a hotkey should not be read as a substitute for a coldkey. A hotkey is safe to run on
internet-connected nodes, but it does not hold funds. The coldkey owns the hotkey and provides the
authority for fund movements; a hotkey compromise does not expose the underlying TAO balance as long
as the coldkey is kept offline.

References: [Hotkey](https://docs.learnbittensor.org/resources/glossary#hotkey),
[Coldkey](https://docs.learnbittensor.org/resources/glossary#coldkey),
[Wallets, Coldkeys and Hotkeys](https://docs.learnbittensor.org/keys/wallets)

## Relationship to Multiple Mechanisms

Hotkeys authorize operational actions such as weight submission for miners and validators. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#weight-matrix) notes that each
incentive mechanism has its own weight matrix for independent consensus calculations.

For readers, a hotkey still names the operational key for subnet actions. When a subnet runs more
than one incentive mechanism, weight-related actions should be read with that mechanism's matrix
context in mind.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix)
