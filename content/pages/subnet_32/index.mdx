It's AI is Bittensor Subnet 32, a subnet built around machine-generated text detection. Its task is
classification: given a piece of text, decide whether it was written by a human or produced by a
language model. The project also exposes a sentence-level "deep scan" that returns a per-sentence
probability of being AI-generated rather than a single label for the whole document. The subnet is
maintained by ITSAI TECHNOLOGIES - FZCO, and the incentive-mechanism code is open source.

## What the Subnet Produces

AI-detection is a moving target: every new language model is a new source of text that a detector
has to recognize. Subnet 32 turns that into a continuous competition. Miners run detection models,
and the network rewards the models that classify human and AI text most accurately, which pushes the
detector to keep improving as new models appear rather than going stale against a fixed test set.

The detector is offered through several consumer-facing surfaces documented in the repository,
including a website, a web app, a browser extension, and an API. For a Taopedia reader the important
point is that the subnet's on-chain incentive is tied to a real classification task with an
objective right answer, not to a subjective output.

## Miner and Validator Roles

Miners are the detector operators. They receive text samples and return a judgment on whether each
sample is human-written or AI-generated, competing to be the most accurate classifier in the
network.

Validators generate the evaluation data and score the miners. According to the repository,
validators continuously query miners with a mix of human-written text and AI-generated text drawn
from more than 30 different open-source language models, and they vary generation parameters and
apply augmentations and adversarial perturbations so that miners cannot overfit to a narrow,
predictable distribution. Each miner's accuracy on this stream determines its score, and those
scores become the weights that feed into [[Yuma Consensus]], which converts them into the incentive
split across miners and validators.

## On-Chain Identity

Live SN32 data, including metagraph state and alpha token pool information, is available on
[TaoStats](https://taostats.io/subnets/32). Readers can reproduce the subnet's registered identity
fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 32 --network finney`.

The subnet's open-source codebase (It-s-AI/llm-detection) is the source of truth for the detection
task and the validation process described above.
