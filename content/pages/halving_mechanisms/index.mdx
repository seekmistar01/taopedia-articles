# Halving Mechanisms

Halving mechanisms reduce token emission rates after supply thresholds are reached. In Bittensor,
the term covers both network-level TAO halvings and subnet-level alpha-token halvings
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

The distinction matters because TAO and subnet alpha tokens have different scopes inside Bittensor
tokenomics ([Emissions](https://docs.learnbittensor.org/learn/emissions)).

## Scope Context

TAO halving is network-scoped, while alpha halving is subnet-scoped. A halving statement should make
clear which asset and scope it is describing
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

That boundary prevents a subnet alpha-token event from being read as a network-wide TAO event, or a
TAO event from being read as a direct statement about every subnet alpha token.

## TAO Halving

TAO halving reduces network-level TAO emission after the relevant TAO supply threshold is reached.
It belongs to the network supply side of Bittensor tokenomics
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emissions](https://docs.learnbittensor.org/learn/emissions)).

For readers, the useful point is scope. TAO halving describes TAO emission, not the full reward
logic inside each subnet.

## Alpha Halving

Alpha halving applies at subnet scope. It reduces a subnet alpha token's emission rate after that
subnet's alpha supply reaches the relevant threshold
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

That makes alpha halving separate from TAO halving even though both use the same broad
supply-threshold idea.

## Emission Relationship

Halving changes the rate at which new tokens enter the emission path. It does not by itself define
miner rankings, validator dividends, or staker reward sharing inside a subnet
([Emissions](https://docs.learnbittensor.org/learn/emissions),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Those later outcomes still depend on subnet evaluation and consensus context. Halving names the
supply-rate change; consensus explains how subnet work is evaluated.

## Dynamic TAO Context

Dynamic TAO gives subnets alpha-token contexts alongside TAO liquidity. That is why halving claims
need to distinguish TAO emission, alpha emission, and alpha-related pool context
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage)).

For readers, a TAO halving, an alpha halving, and a subnet pool change are related tokenomics ideas,
but they should not be collapsed into one claim.

## Reader Boundary

Halving mechanisms should be read as supply-rate rules. They are not date forecasts, staking
instructions, or complete explanations of subnet reward distribution
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)).

The stable article-level point is that Bittensor uses halving vocabulary at both the network TAO
scope and the subnet alpha-token scope.
