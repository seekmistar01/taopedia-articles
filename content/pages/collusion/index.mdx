# Collusion

Collusion is when participants coordinate to bend an outcome in their favor instead of acting
honestly. In a Bittensor subnet, the concern is colluding validators that assign inflated weights to
particular miners so that consensus rewards those targets beyond what honest evaluation would allow
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

## What It Represents

Collusion names coordinated validator inflation toward shared miner targets. Yuma Consensus resists
this through stake-weighted agreement and clipping: weights that outlie the consensus benchmark are
trimmed rather than fully rewarded, and a coalition can move outcomes only in proportion to its
combined stake weight
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight)).

Examples tied to netuid 1 illustrate one subnet's clipping path. They should not be read as
representing production consensus behavior on another network
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Distinction from Sybil Attack

A Sybil actor multiplies identities at registration, while collusion is validators coordinating
inflated weights toward favored miners inside an existing participant set
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Collusion** — multiple validators coordinating inflated weights toward chosen miners.
- **Sybil attack** — one actor registering many identities to seek leverage from numbers.

## Distinction from Weight Copying

Collusion is validators acting together to push inflated ratings toward chosen miners. Weight
copying is validators reusing visible weight signals instead of independently evaluating miners
([Weight Copying Problem](https://docs.learnbittensor.org/concepts/weight-copying-in-bittensor),
[Glossary: Weight Copying](https://docs.learnbittensor.org/resources/glossary#weight-copying)).

- **Collusion** — coordinated inflation of weights toward shared targets.
- **Weight copying** — reusing another validator's visible weights without independent evaluation.

## Distinction from Validator Permit

Collusion is coordinated bias in how permitted validators score miners. The validator permit is the
on-chain flag that records whether a UID may submit non-self weights that count in consensus
([Glossary: Validator Permit](https://docs.learnbittensor.org/resources/glossary#validator-permit),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

- **Collusion** — coordinated validator bias in weight submissions.
- **Validator permit** — the on-chain right to submit non-self weights that count in consensus.

## Distinction from Kappa

Collusion is coordinated inflation of weights toward chosen miners. Kappa is the subnet
hyperparameter that sets the stake-weighted majority ratio for Yuma aggregation
([Subnet Hyperparameters: Kappa](https://docs.learnbittensor.org/subnets/subnet-hyperparameters#kappa),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Collusion** — coordinated inflation toward shared miner targets.
- **Kappa** — protocol threshold for stake-weighted majority in aggregation.

## Distinction from Self-Weighting

Collusion requires multiple validators acting together toward shared targets. Self-weighting is one
validator inflating weights for entities it controls without a coordinating group
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Yuma Consensus: Clipping excess weights](https://docs.learnbittensor.org/learn/yuma-consensus#clipping-excess-weights)).

- **Collusion** — multiple validators coordinating inflated weights toward chosen miners.
- **Self-weighting** — one validator inflating weights for entities it controls.

## Distinction from Stake Weight

Collusion is participants coordinating to inflate weights toward chosen targets. Stake weight is the
stake-derived value that scales how much each colluder's weights count in consensus
([Glossary: Stake Weight](https://docs.learnbittensor.org/resources/glossary#stake-weight),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **Collusion** — coordinated inflation of weights toward shared targets.
- **Stake weight** — stake-derived scale for how strongly weights count in consensus.

## Distinction from Consensus Score

Collusion is validators coordinating to inflate weights toward chosen targets. Consensus score
measures how closely a validator's weights align with the stake-weighted agreement point
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Glossary: Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score)).

- **Collusion** — coordinated behavior a coalition might attempt.
- **Consensus score** — alignment benchmark Yuma uses when clipping outlier weights.

## Distinction from Incentive Compatibility

Incentive compatibility is the design goal that honest contribution is the most profitable choice.
Collusion is validators coordinating to inflate weights so rewards flow beyond honest evaluation
([Anatomy of an Incentive Mechanism](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Glossary: Incentive Mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism)).

- **Collusion** — coordinated deviation clipping must make unprofitable.
- **Incentive compatibility** — property that honest strategy is the rational choice.

## Distinction from Yuma Consensus

Collusion names coordinated validator behavior intended to skew subnet consensus toward favored
miners. Yuma Consensus is the tempo process that reads eligible validator weights, applies clipping
and bonding, and maps the result into emission shares
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

- **Collusion** — coordinated manipulation of validator weight signals.
- **Yuma Consensus** — on-chain settlement that turns validator weights into emission shares.
  Collusion names coordinated weight manipulation; Yuma processes validator weights into rewards
  ([Yuma Consensus: Validator emissions](https://docs.learnbittensor.org/learn/yuma-consensus#validator-emissions)).

## Reader Boundary

This page defines the concept and Bittensor's consensus-level resistance at a high level. It does
not claim collusion is impossible, quantify how much stake would be required, or describe how to
attempt it. The stake distribution and consensus values that determine how strong the resistance is
on any subnet are live chain state and change over time
([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).
