# Subnet Scoring Model

A subnet scoring model is the evaluation method validators use when judging miner responses inside a
Bittensor subnet
([Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The term belongs to incentive-mechanism vocabulary. It names how responses become scores, not the
final consensus result.

## Evaluation Role

The scoring model explains how returned work is judged. The subnet task describes what miners are
asked to produce, while the scoring model describes how validators evaluate the responses they
receive
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model)).

This keeps task vocabulary separate from scoring vocabulary. The task asks for work; the scoring
model evaluates the returned work.

## Numerical Scores From Miner Responses

The glossary defines a subnet scoring model as the part of an incentive mechanism that converts
miner responses into numerical scores. That conversion sits between returned work and the validator
signals that later enter consensus
([Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

For readers, the scoring model is an evaluation method. It is not the miner output itself and not
the final Yuma Consensus rank.

## Relationship to Miners and Validators

Miners produce responses for the subnet, and validators evaluate those responses under subnet
standards. Subnet documentation describes miners as producing a digital commodity and validators as
measuring miner work
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model)).

The scoring model is the method validators apply to that work. It links miner responses to validator
evaluation.

## Evaluation Before Weights

A scoring model belongs earlier in the evaluation path than final weight signals. Weight vectors are
later consensus inputs formed after validator evaluation
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model)).

This distinction keeps scoring formulas and weight vectors separate. The scoring model produces
evaluation scores; weight vocabulary describes the structured signals sent onward.

## Multiple-Mechanism Context

When a subnet uses multiple incentive mechanisms, each mechanism can have its own scoring model for
independent evaluation. Official multiple-mechanism documentation describes separate mechanism
contexts inside one subnet
([Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model)).

For readers, a scoring standard for one mechanism does not automatically describe how miners are
judged in another mechanism.

## Reader Boundary

Subnet scoring model should be read as a concept article term. It is not a universal benchmark, a
deployment claim, or a full implementation description for any specific subnet
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The stable article-level point is that a subnet scoring model converts miner responses into
evaluation scores that validators can use inside a subnet incentive mechanism.
