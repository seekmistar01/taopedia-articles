# UID

A UID is the unique identifier assigned to each participant position in a Bittensor subnet. Each
miner or validator registered to a subnet holds a UID that identifies their position within that
subnet's participant set
([Glossary: UID slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

UID is identifier vocabulary. It names the number, not the position structure itself. UID slot is
the related term for the participant position that the UID identifies.

## Subnet-Specific Scope

UIDs are subnet-specific. The same UID number can appear in different subnets while referring to
different participants. A UID identifies a position within one subnet's participant set, not a
network-wide identity
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

Netuid identifies the subnet itself, while UID identifies a participant within that subnet
([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)). The two terms
answer different questions about the same subnet context.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For UIDs on a
subnet such as netuid 1, that sequence changes which subnet participant set a reader should assume
when matching a UID to a miner or validator.

In localnet, UID assignment can be tested through isolated subnet registration. Localnet UID slots
do not represent production participant sets on live subnets.

On testnet, registration assigns UIDs on a shared non-production network. Testnet metagraph entries
are separate from mainnet subnet state
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

On mainnet, a UID identifies a participant position on the selected
[netuid](https://docs.learnbittensor.org/resources/glossary#netuid) in live metagraph data
([Glossary: Metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph)).

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A UID example from one environment should not be read as
identifying a participant on another network.

## Distinction from Hotkey

A UID names a subnet-local slot number. A hotkey names the account key that registers to and signs
for that subnet ([Glossary: UID slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[Glossary: Hotkey](https://docs.learnbittensor.org/resources/glossary#hotkey)).

Registration assigns a UID to a hotkey, so the hotkey is the account identity and the UID is the
slot that identity occupies on the selected subnet.

## Distinction from Register

Register names the action that places a participant into a subnet. A UID is the slot number assigned
by that registration
([Glossary: Register](https://docs.learnbittensor.org/resources/glossary#register),
[Glossary: UID slot](https://docs.learnbittensor.org/resources/glossary#uid-slot)).

Use register for the registration action; use UID for the numbered position that results.

## Distinction from Metagraph

The metagraph records each subnet's participant set. A UID is the index label for one miner or
validator row in that subnet state
([Glossary: Metagraph](https://docs.learnbittensor.org/resources/glossary#metagraph),
[Glossary: UID slot](https://docs.learnbittensor.org/resources/glossary#uid-slot)).

Metagraph vocabulary names the full participant table; UID vocabulary names one position inside it.

## Distinction from Weight Vector

Validators score miners by UID inside a subnet. A weight vector attaches those scores to the UID
positions listed in the subnet metagraph
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector),
[Glossary: UID slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

Pair UID with netuid when tracing which miner or validator a weight entry refers to
([Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)).

## Distinction from Immunity Period

A UID names a participant position in a subnet. Immunity period is the block window that shields a
newly assigned UID from deregistration
([Glossary: Immunity Period](https://docs.learnbittensor.org/resources/glossary#immunity-period),
[Glossary: UID slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[Subnet Hyperparameters](https://docs.learnbittensor.org/subnets/subnet-hyperparameters)).

- **UID** — the numbered participant position in a subnet.
- **Immunity period** — the shield window for a newly assigned UID.

## Distinction from Netuid

A UID and a netuid are both identifiers but at different levels. A netuid identifies a whole subnet,
while a UID identifies one participant's position inside a subnet. The same UID number can exist on
many subnets, so a UID only pins down a participant when it is read together with the netuid it
belongs to ([Glossary: UID Slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[Glossary: Netuid](https://docs.learnbittensor.org/resources/glossary#netuid)).

- **Netuid** — the identifier of a whole subnet.
- **UID** — the identifier of one participant position within a subnet.

## Distinction from Neuron

UID names the numbered slot identifier inside a subnet. Neuron names the registered participant
record tied to that slot in subnet state
([Glossary: UID slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[Glossary: Neuron](https://docs.learnbittensor.org/resources/glossary#neuron)).

- **UID** — subnet-local slot number for a participant position.
- **Neuron** — registered participant record tied to that slot.

Identifier vocabulary names the slot number; neuron vocabulary names the participant record
([Understanding Neurons](https://docs.learnbittensor.org/learn/neurons)).

## Distinction from Yuma Consensus

UID names the subnet-local slot number for a participant. Yuma Consensus aggregates validator weight
submissions indexed by those UIDs into emission shares each tempo
([Glossary: UID slot](https://docs.learnbittensor.org/resources/glossary#uid-slot),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

- **UID** — subnet-local participant slot identifier.
- **Yuma Consensus** — mechanism that turns validator weights into emission shares.

Identifier vocabulary names participant positions; Yuma vocabulary names settlement from weights
([Glossary: Weight Vector](https://docs.learnbittensor.org/resources/glossary#weight-vector)).

## Reader Boundary

UID is concept vocabulary for subnet participant identification. It should not be read as a network
address, a balance container, or a cross-subnet identity
([Glossary: UID slot](https://docs.learnbittensor.org/resources/glossary#uid-slot)).

UIDs can change through deregistration. A UID that identifies one participant today may identify a
different participant in the same subnet after a registration event
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).
