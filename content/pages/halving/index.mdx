# Halving

Halving is a tokenomics term for a reduction in token emission rate. In Bittensor, it belongs to
emission mechanics because it changes how quickly new tokens enter the reward-distribution path
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

The concept explains a scheduled change in issuance pace. Later reward distribution still depends on
the token, subnet, incentive mechanism, and consensus process involved.

The useful reading starts with three questions: which asset is halving, which threshold triggers the
change, and which environment or source supplies the timing
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)).

## Emission-Rate Role

Halving changes the rate side of emissions. Emissions describe the broader creation and distribution
process, while halving identifies the point where the emission rate is reduced
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

That makes halving different from a reward calculation by itself. A reduced emission rate changes
the input to reward flows; it does not determine every later allocation outcome.

Halving is therefore closest to emissions and issuance vocabulary. It describes the emission-rate
change before downstream rewards are allocated
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

## Threshold Trigger

Bittensor halving is tied to issuance thresholds. The halving mechanism reduces emission rate when
the relevant supply threshold is reached
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

The threshold is part of the definition because it gives the rate change a supply milestone. A
halving claim needs the asset and threshold before it can be read as more than a generic statement
about slower issuance.

That threshold language also separates halving from ordinary supply commentary. Halving is not just
the observation that emissions are lower; it is the named rate reduction associated with a supply
milestone ([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

## TAO and Alpha Scope

Halving language can appear at more than one Bittensor tokenomics scope. TAO halving belongs to the
network-token side, while alpha halving belongs to subnet alpha-token emission
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

The shared word does not erase the asset distinction. TAO and alpha can both be discussed through
emission-rate reduction, but they are not the same token.

This matters in Dynamic TAO discussions because alpha tokens are subnet-specific while TAO is the
network token. A halving reference is clearer when it names whether the reduced emission rate
belongs to TAO or to a subnet's alpha token
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

## Supply Accounting

Halving changes issuance pace; burning and recycling address different movements in token supply.
Burning removes existing tokens from spendable circulation, while Bittensor recycling vocabulary
describes tokens returned to the protocol supply path
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving),
[Glossary: Burning](https://docs.learnbittensor.org/resources/glossary#burning),
[Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)).

These terms can all affect supply interpretation, but they operate at different moments. Halving is
about how quickly new tokens enter the system. Burning and recycling describe what happens to tokens
that already exist.

Issuance vocabulary sits nearby but answers a different question. Issuance describes circulating TAO
supply, while halving describes a rate change in emissions
([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance),
[Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)).

## Reward Flow

Halving sits upstream of reward distribution. The emission documentation describes how emitted value
moves through Bittensor reward paths, and the halving documentation explains the rate-reduction
mechanism that changes the supply entering those paths
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

That relationship matters because a halving event can affect supply entering the system without
fully explaining miner incentives, validator dividends, subnet reserves, or market behavior. Those
later outcomes require their own sources.

The same order applies to subnet alpha. Alpha halving changes an alpha-token emission rate, while
subnet incentive mechanisms and consensus explain how work and validation affect reward outcomes
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## Network Examples

Bittensor separates localnet, testnet, and mainnet environments. Halving examples from one
environment belong to that environment's supply and chain state
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Introduction to Bittensor: Subnet development](https://docs.learnbittensor.org/learn/introduction#subnet-development)).

A localnet example can demonstrate mechanics in isolation. A testnet example belongs to shared
non-production state. A mainnet example belongs to production chain history. The same halving
vocabulary can appear in each environment, but the evidence is not interchangeable.

Specific timing or supply values need the environment and source that produced them. The halving
concept can travel across environments; measured state belongs to one environment.

## Reading Halving

Halving is supply-schedule vocabulary. It is not a date forecast, price forecast, staking strategy,
or complete reward model ([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

The durable concept is emission-rate reduction at a threshold. Exact timing, observed supply values,
and market interpretation belong to the source and environment where they are measured.
