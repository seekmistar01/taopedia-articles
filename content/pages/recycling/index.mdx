# Recycling

Recycling describes token reuse or re-emission inside Bittensor tokenomics. It is supply-accounting
vocabulary for token flows that can return to future emission rather than being permanently removed
from circulation
([Glossary: Recycling](https://docs.learnbittensor.org/resources/glossary#recycling),
[Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)).

The term is easiest to read by contrast with burning. Recycling keeps a token amount available to
the protocol's future emission accounting, while burning emphasizes removal from spendable
circulation.

## Tokenomics Context

Recycling is a supply-flow concept. It describes what happens after tokens enter a protocol-level
accounting process, not an ordinary transfer between users
([Glossary: Recycling](https://docs.learnbittensor.org/resources/glossary#recycling),
[Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)).

For readers, recycling answers a narrow question: whether the deducted amount can return through
future emission accounting.

## Issuance Context

Recycled tokens are deducted from Bittensor's issuance record so the same quantity can be emitted
again later
([Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning),
[Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

This keeps recycling connected to supply accounting rather than to user-facing payment language. The
important point is not who receives a token next, but how the protocol accounts for the amount.

## Relationship to Burning

Recycling and burning are related but different tokenomics terms. Burning removes tokens from
spendable circulation, while recycling allows deducted tokens to be reused or emitted again
([Glossary: Burning](https://docs.learnbittensor.org/resources/glossary#burning),
[Glossary: Recycling](https://docs.learnbittensor.org/resources/glossary#recycling)).

That contrast prevents every token sink from being read as permanent destruction. Recycling is a
reuse concept; burning is a removal concept.

## Relationship to Transaction Fees

Transaction fees can feed recycling context. Bittensor fee documentation treats transaction fees as
costs attached to state-changing actions, while recycling-and-burning vocabulary explains how
collected fees can be accounted for after collection
([Transaction Fees in Bittensor](https://docs.learnbittensor.org/learn/fees),
[Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)).

For readers, transaction fees name the cost side of an action. Recycling names the supply-accounting
side after those amounts enter the protocol flow.

## Relationship to Emission

Emission is the creation and allocation side of Bittensor tokenomics. Recycling matters because
deducted amounts can return to future emission accounting instead of disappearing from the emission
picture ([Emission](https://docs.learnbittensor.org/learn/emissions),
[Glossary: Recycling and burning](https://docs.learnbittensor.org/resources/glossary#recycling-and-burning)).

This makes recycling part of the broader supply-flow vocabulary around issuance, fees, and emission.
It does not by itself define subnet scoring or reward distribution.

## Reading the Term

Recycling should be read as high-level Bittensor tokenomics vocabulary. It is not a valuation claim,
market forecast, or report of exact network totals
([Glossary: Recycling](https://docs.learnbittensor.org/resources/glossary#recycling),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

The stable article-level point is that recycling keeps a deducted token amount available for future
protocol accounting rather than treating it as permanently removed.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. The recycling
concept applies across the Bittensor lifecycle: token reuse and re-emission accounting can be
observed in localnet for isolated testing conditions, on testnet for shared non-production supply
states, and on mainnet for live operation with real emissions.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. Recycling examples or supply-flow outcomes from one
environment should not be read as representing production tokenomics in another environment.

## Supply Cycle Scope

Recycling should be read as supply-cycle accounting, not as an ordinary transfer or a permanent
burn. The glossary places recycling beside burning, while emissions documentation explains how
available token amounts can flow back through protocol distribution
([Glossary: Recycling](https://docs.learnbittensor.org/resources/glossary#recycling),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

That distinction keeps tokenomics language precise. Recycling describes a reusable accounting path,
while burning describes removal from spendable circulation and transfers describe custody changes
between accounts.
