# Alpha Halving

An alpha halving is a subnet-scoped emission reduction for a subnet's alpha token. It applies the
halving idea to alpha-token supply rather than to the network-wide TAO supply
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)).

The concept belongs to Bittensor tokenomics because emissions include both TAO and subnet-specific
alpha-token flows ([Emissions](https://docs.learnbittensor.org/learn/emissions)).

## Subnet Scope

Each subnet has its own alpha token, so an alpha halving is read in relation to one subnet's
alpha-token supply. It should not be treated as a network-wide event affecting every subnet at once
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

That scope is the main difference from the TAO halving. TAO halving is network-level supply
language; alpha halving is subnet-level supply language
([Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

## Emission Reduction

A halving reduces the rate at which new tokens are emitted after a supply threshold is reached. For
alpha halving, the reduced emission is the alpha-token emission for the relevant subnet
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

For readers, the useful point is simple: alpha halving slows future alpha creation for that subnet.
It does not describe a transfer action, a validator evaluation, or a reward-quality score
([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)).

## Alpha Token Context

Alpha tokens are subnet-specific assets used inside subnet tokenomics. An alpha halving matters
because it changes the emission rate of that subnet-specific asset
([Glossary: Alpha Tokens](https://docs.learnbittensor.org/resources/glossary#alpha-tokens)).

This keeps the vocabulary separate: alpha token names the asset, while alpha halving names an
emission-rate reduction for that asset
([Emissions](https://docs.learnbittensor.org/learn/emissions)).

## TAO Halving Context

Alpha halving and TAO halving use the same broad supply-threshold idea, but they apply at different
scopes. One belongs to a subnet's alpha-token supply, and the other belongs to the network's TAO
supply ([Glossary: Halving](https://docs.learnbittensor.org/resources/glossary#halving)).

Keeping that distinction avoids reading a subnet-level alpha event as a network-level TAO event
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

## Reader Boundary

Alpha halving is a concept article about subnet tokenomics. It is not a staking guide, a validator
guide, or a per-subnet schedule
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

The stable article-level point is that alpha-token emission can be reduced at the subnet level while
remaining distinct from the broader TAO halving concept
([Emissions](https://docs.learnbittensor.org/learn/emissions)).
