# Public Subtensor

Public Subtensor refers to publicly reachable Bittensor chain access. Official Bittensor
documentation uses Subtensor to describe the blockchain layer that supports Bittensor network state.

References:
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor),
[Subtensor](https://docs.learnbittensor.org/subtensor-nodes)

## Network Context

The term is useful when distinguishing public chain access from local development context. Public
Subtensor belongs to the live network side of that distinction, while local blockchain context is
private and development-oriented.

References:
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor),
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks)

## Relationship to Subtensor

Public Subtensor and Subtensor are related but name different layers of the same picture. The
[Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor) describes
Bittensor's layer 1 blockchain and the system of record for transactions and rankings, while the
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)
describes publicly reachable access to that chain layer.

For readers, Subtensor is the chain layer itself, while Public Subtensor names live public access to
it. The term does not rename Subtensor and should not be read as a separate blockchain.

References: [Glossary: Subtensor](https://docs.learnbittensor.org/resources/glossary#subtensor),
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)

## Relationship to Local Blockchain

Public Subtensor and local blockchain name opposite sides of Bittensor chain access. The
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)
describes publicly reachable chain access, while the
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)
describes a private chain used for development and testing.

For readers, Public Subtensor belongs to live network access context, while local blockchain belongs
to isolated development context. The two terms should not be read as interchangeable network labels.

References:
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor),
[Glossary: Local Blockchain](https://docs.learnbittensor.org/resources/glossary#local-blockchain)

## Relationship to Bittensor Networks

Public Subtensor and Bittensor Networks name related but different parts of live chain context. The
[Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference groups
mainnet, testnet, and localnet, while the
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)
describes publicly reachable access to the Subtensor chain layer.

For readers, Bittensor Networks is the environment label, while Public Subtensor names public access
to Subtensor inside mainnet and testnet context. It does not describe private local blockchain
development.

References: [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Glossary: Public Subtensor](https://docs.learnbittensor.org/resources/glossary#public-subtensor)

## Reader Boundary

This page defines the term. It does not provide endpoint recommendations, connection steps, or
node-operation guidance. Readers should use current official Bittensor documentation when they need
operational details.
