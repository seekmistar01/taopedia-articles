# Subnet Liquidity Pools

Subnet liquidity pools are the reserve context for subnet tokenomics in Bittensor. A subnet pool
connects network-level TAO with the subnet's own alpha token, so pool context matters when reading
claims about [[dynamic_tao|Dynamic TAO]], staking, unstaking, and emissions.

References: [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage)

## What a subnet pool holds

A subnet pool has a TAO reserve and an alpha reserve. The TAO reserve reflects TAO staked into the
subnet, while the alpha reserve reflects alpha available in the pool. The same subnet economy also
tracks alpha outstanding, which is alpha held outside the reserve context and should not be read as
the same thing as pool liquidity.

References: [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Emission](https://docs.learnbittensor.org/learn/emissions)

## Relationship to alpha tokens

Subnet pools are one reason alpha tokens need separate wording from TAO. TAO is the network asset,
while each subnet uses its own alpha token for subnet-level accounting. A pool claim should
therefore say whether it is describing TAO reserves, alpha reserves, or alpha outside the pool.

References: [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Emission](https://docs.learnbittensor.org/learn/emissions)

## Staking and unstaking context

When TAO is staked into a subnet, the subnet pool context determines the alpha-denominated stake
received. When stake is removed, the pool context is used again to express alpha-denominated stake
back into TAO terms. This is why [[staking_and_delegation|Staking and Delegation]] discussions need
to distinguish TAO, alpha-denominated stake, and pool conversion context.

References:
[Staking/Delegation Overview](https://docs.learnbittensor.org/staking-and-delegation/delegation),
[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage)

## Slippage context

Slippage describes the difference between a simple expected exchange and the result produced by the
pool context at the time of the action. For Taopedia purposes, the important point is conceptual:
staking and unstaking interact with reserves, so pool state can affect the TAO or alpha amount
received. Exact amounts need a direct source for the values being used.

Reference: [Understanding Slippage](https://docs.learnbittensor.org/learn/slippage)

## Emissions context

Bittensor emissions also connect to subnet pools. The official emissions docs describe new liquidity
entering subnet pools and separate TAO reserve, alpha reserve, and alpha-outstanding contexts. This
means pool liquidity, subnet alpha tokens, and reward distribution are related, but they are not
interchangeable claims.

References: [Emission](https://docs.learnbittensor.org/learn/emissions),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)

## How to read pool claims

A subnet-pool claim should identify the asset and scope. A TAO-reserve claim is different from an
alpha-reserve claim, and both are different from a claim about alpha-denominated stake or alpha
outstanding. Claims about exact balances, rates, timing, or subnet examples need the specific source
that reports those values.

References: [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Understanding Slippage](https://docs.learnbittensor.org/learn/slippage),
[Staking/Delegation Overview](https://docs.learnbittensor.org/staking-and-delegation/delegation)
