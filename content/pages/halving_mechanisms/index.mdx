# Halving Mechanisms

A halving reduces a token emission rate by 50%. In Bittensor, halving mechanisms apply to both
network-level TAO emissions and subnet-level alpha-token emissions
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

The important distinction is scope. TAO halvings apply to the network token, while alpha halvings
apply to a specific subnet's alpha token.

That scope distinction keeps halving from becoming a generic reward label. A halving changes an
emission rate; it does not by itself decide miner scores, validator dividends, or the distribution
logic inside a subnet ([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

## What Halving Changes

Halving changes the rate at which new tokens are emitted. It does not by itself define how miner,
validator, or staker outcomes are compared inside a subnet
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

For readers, halving is supply-rate terminology. Reward distribution and consensus describe later
allocation questions.

This makes halving a tokenomics boundary term. It belongs near issuance and emissions because those
articles explain supply and distribution, while consensus articles explain how validator signals
shape allocation after emissions exist
([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)).

## TAO and Alpha Scopes

A TAO halving reduces network-level TAO emission after total TAO issuance reaches a supply-based
threshold ([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance)).

This makes TAO halving an issuance-threshold concept rather than a simple calendar event. The
trigger depends on supply context.

Alpha halving works at subnet scope. It reduces a subnet alpha token's emission rate after that
subnet's alpha issuance reaches the relevant threshold
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

This is different from a TAO halving. TAO halving is network-level; alpha halving is tied to a
specific subnet alpha token.

Readers should therefore ask which asset the halving statement is about. TAO halvings and alpha
halvings both reduce emission rates, but they do so at different scopes and for different assets
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

## Pool Injection Context

TAO halvings also affect subnet pool injection context. When TAO injection into subnet pools is cut
in half, the matching alpha injected alongside TAO is reduced so the pool relationship can be
maintained ([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

That does not mean all alpha rewards halve because of a TAO halving alone. The halving mechanism
separates alpha pool injections from alpha rewards.

For readers, pool-injection language should stay separate from reward-allocation language. A pool
relationship can change when TAO injection changes, while miner and validator outcomes still depend
on the subnet's incentive and consensus context
([Emission](https://docs.learnbittensor.org/learn/emissions),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

## Dynamic TAO Context

Dynamic TAO gives subnets alpha-token reserves and outstanding alpha, so halving statements need to
say which asset and scope they describe
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

For readers, a TAO halving, an alpha halving, and an alpha pool-injection change are related but not
identical tokenomics statements.

## Issuance Threshold Context

Issuance is the supply measure that TAO halving thresholds use. Halving mechanisms are the rules
that change emission rates when the relevant supply thresholds are reached
([Glossary: Issuance](https://docs.learnbittensor.org/resources/glossary#issuance),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

This keeps issuance and halving separate. Issuance names supply context; halving mechanisms name the
rate-change rules.

## Environment Boundary

Halving mechanisms are supply-rule concepts, not environment reports. Bittensor network labels such
as mainnet, testnet, and localnet tell readers where an observation belongs, while the halving term
describes the emission-rate rule
([Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks),
[Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving)).

That boundary matters because a local or testing example can illustrate terminology without
representing every network environment. The durable article point is the distinction between TAO
halvings, subnet alpha halvings, and pool-injection effects.

## Reading the Term

Halving mechanisms should be read as high-level Bittensor tokenomics terminology. They are not
reward tables, exact timing reports, or advice about token movement
([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

The stable article-level point is that Bittensor uses supply-based halving rules to reduce TAO and
subnet alpha emission rates.

## Supply Trigger Scope

Halving mechanisms should be read as supply-based emission triggers, not as block-count triggers or
fixed-date schedules. The halving documentation describes the trigger as a total supply threshold
rather than a block number, and the same source explains that the actual date of each halving is not
fixed ([Halving Mechanisms](https://docs.learnbittensor.org/concepts/halving),
[Emission](https://docs.learnbittensor.org/learn/emissions)).

For readers, the relevant boundary is what the halving measures and when it activates. Bittensor
halvings activate when total token supply crosses a threshold, while the subnet alpha halving uses
its own subnet-level halving index. A precise claim should keep the supply-based trigger separate
from the surrounding block, time, or schedule vocabulary, since the same word can be read with very
different trigger conditions in other systems.
