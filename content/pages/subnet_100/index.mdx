Platform is Bittensor Subnet 100. Its on-chain identity describes the subnet as an auto-research
network where miners compete across multiple challenges to score against a synthetic benchmark,
lists [platform.network](https://platform.network) as the subnet URL, and lists
[PlatformNetwork/platform](https://github.com/PlatformNetwork/platform) as the subnet's GitHub
repository.

## What Platform Provides

The Platform README describes the repository as a multi-challenge Bittensor subnet platform. It lets
independent challenges run under one validator network: it routes miner traffic to the right
challenge, collects raw challenge weights, normalizes emissions, maps each miner to its Bittensor
UID, and publishes a final weight vector for validators to submit on-chain.

According to the README, each challenge lives in its own repository and owns its submissions,
scoring logic, state, and public miner experience, while Platform provides the orchestration layer
that runs those challenges together as a single subnet. A master validator controls the central
registry and orchestration, and each challenge exposes a standard internal weight contract to
Platform.

## Miner and Validator Roles

The README describes miners as choosing a challenge, following that challenge's submission rules,
and monitoring challenge-specific leaderboards through Platform. Each challenge calculates its own
raw per-miner weights from its scoring rules.

Validators run Platform, synchronize the challenge registry, launch the active challenge containers,
collect the raw weights, and submit the final normalized weights to Bittensor at epoch boundaries.
The README notes that if a challenge fails, Platform can isolate that challenge's contribution
without taking down the entire subnet. Operators should treat the repository as the source of truth
for current participant guidance.

## On-Chain Identity

Live SN100 data is available on [TaoStats](https://taostats.io/subnets/100). Readers can also
reproduce the chain identity fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 100 --network finney`.

That Finney identity reports the subnet name as Platform, the subnet URL as platform.network, and
the GitHub repo as PlatformNetwork/platform.
