# Subnet Protocol

A subnet protocol is the rule set that defines how validators and miners interact inside a Bittensor
subnet
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The term belongs to incentive-mechanism vocabulary. It names the interaction pattern around work,
not the work target itself, the scoring method, or the full subnet design.

## Interaction Role

The subnet protocol describes the request-and-response pattern between validators and miners. It
explains how work can be requested, returned, and made available for evaluation inside the subnet
context
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

For readers, protocol is the connective layer between a task and an evaluation. It is the pattern
that lets subnet roles interact around the work being measured.

## Task Boundary

Subnet task and subnet protocol are related but separate ideas. The task names the work target,
while the protocol names the interaction pattern around that work
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol)).

This boundary keeps the question of what miners produce separate from the question of how validators
and miners exchange work and responses.

## Scoring Boundary

The scoring model is another adjacent concept. A subnet protocol can define how responses are
obtained, while a scoring model defines how those responses are judged for the subnet's incentive
mechanism
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

Keeping protocol and scoring separate prevents a request-and-response pattern from being treated as
the full evaluation rule. The protocol can create the evaluation surface; the scoring model explains
how that surface is measured.

## Role Context

Subnet protocols sit between miner and validator roles. Miners produce responses within the subnet's
rules, and validators evaluate returned work against the subnet's standards
([Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets),
[Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol)).

The protocol helps organize those interactions. It does not replace the miner role, the validator
role, or the incentive mechanism that rewards evaluated output.

## Incentive-Mechanism Context

A subnet protocol matters because interaction rules are part of how a subnet turns work into
evaluated output. The protocol works alongside the task and scoring model inside the incentive
mechanism
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

That means protocol claims should stay attached to the subnet's incentive design. A useful protocol
description explains the interaction pattern without turning it into a complete reward formula.

## Multiple-Mechanism Boundary

When a subnet uses more than one incentive mechanism, protocol expectations should be read with the
mechanism being discussed. Different mechanisms can define different work or evaluation contexts
inside the same subnet
([Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)).

This boundary keeps one mechanism's interaction pattern from being treated as the whole subnet's
only protocol context.

## Reader Boundary

Subnet protocol should be read as concept vocabulary for miner-validator interaction. It is not a
claim that every subnet uses the same exchange pattern, not a deployment-status report, and not a
substitute for the subnet's own mechanism documentation
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The stable article-level point is that a subnet protocol defines the request-and-response rules by
which miners and validators interact inside a subnet.
