Verathos is Bittensor Subnet 96. Its on-chain identity describes the subnet as a verified AI
inference and training subnet, lists [verathos.ai](https://verathos.ai) as the subnet URL, and lists
[verathos-ai/verathos](https://github.com/verathos-ai/verathos) as the subnet's GitHub repository.

## What Verathos Provides

The Verathos README describes the subnet as a network for cryptographically verified AI compute.
Rather than trusting that a miner ran the model it claims, Verathos lets each computation be proven:
miners attach a succinct cryptographic proof to their results, and validators check those proofs
before assigning rewards. The README describes the underlying scheme as a zero-knowledge-inspired,
sumcheck-based verification over model weights committed on-chain, and notes that validators can
verify the proofs on ordinary processors in milliseconds — keeping the cost of checking work far
below the cost of doing it.

According to the README, verified inference is live today while verified training is in development.
For inference, a proof accompanies each generation and the network exposes an interface compatible
with common AI clients that routes requests across miners by score. The same approach is being
extended to training, where a job can prove that a stated base model was fine-tuned with the claimed
data and method.

The README lists what the proofs are meant to establish: that a miner used the weights it committed
to, that the underlying computation was carried out correctly, and that the returned output was not
altered afterward. Because only a random subset of a model's layers is challenged on each request,
confidence that a dishonest miner is caught rises toward certainty over repeated queries.

## Miner and Validator Roles

The README describes miners as serving models on GPUs and producing a verification proof alongside
every result. Validators run without a GPU: they periodically probe miners, verify the submitted
proofs, and score miners on speed and proof validity, with a failed proof immediately dropping a
miner's score to zero. Validators then set weights on-chain according to those scores. Operators
should treat the repository as the source of truth for current participant guidance.

## On-Chain Identity

Live SN96 data is available on [TaoStats](https://taostats.io/subnets/96). Readers can also
reproduce the chain identity fields with the Bittensor CLI command documented in the
[btcli reference](https://docs.learnbittensor.org/btcli/#btcli-subnets-get-identity):
`btcli subnets get-identity --netuid 96 --network finney`.

That Finney identity reports the subnet name as Verathos, the subnet URL as verathos.ai, and the
GitHub repo as verathos-ai/verathos.
