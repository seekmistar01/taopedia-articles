# Private Key

A private key is the protected side of a cryptographic key pair. The official
[Private Key glossary](https://docs.learnbittensor.org/resources/glossary#private-key) describes it
as the private component used for securing and authorizing transactions and operations within the
Bittensor network.

## Keypair Context

Private keys are read together with public keys. The
[Public Key glossary](https://docs.learnbittensor.org/resources/glossary#public-key) describes the
public side as publicly available key material used for verifying signatures, encrypting messages,
and identifying accounts. The private side is different: it is the protected authorization material
paired with that public side.

This distinction keeps the two terms from collapsing into a single wallet concept. Public key names
the shareable side of the pair, while private key names the side that must remain protected because
it authorizes activity.

## EdDSA Context

Bittensor glossary terminology connects public and private key pairs to EdDSA. The official
[EdDSA Cryptographic Keypairs glossary](https://docs.learnbittensor.org/resources/glossary#eddsa-cryptographic-keypairs)
describes EdDSA as the cryptographic algorithm used to generate public and private key pairs for
Bittensor coldkeys and hotkeys.

For Taopedia readers, EdDSA names the algorithm context, public key names the shareable side of the
resulting pair, and private key names the protected side of that same pair.

## Authorization Context

The private-key concept is useful because it separates account reference from authority. Public
material can identify or verify, while the private key is the component associated with
authorization. The official Private Key glossary supports this authorization framing, and the Public
Key glossary supports the contrast with public account-reference material.

This article stays at the concept level. It does not describe key files, wallet setup, signing
flows, or any procedure for handling keys.

## Reader Boundary

Private key should not be read as a public key, wallet address, balance, or account profile. It is
protected authorization material in a cryptographic key pair. Claims about a particular account,
key, wallet, or file need a direct source for that specific object.
