# Rank

Rank is a Bittensor consensus metric for summarizing miner evaluation within a subnet. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#rank) frames rank as the final
aggregate judgment produced by Yuma Consensus from validator ratings, while
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) explains how validator
rankings of miner performance are resolved into emissions vectors.

## Consensus Metric

Rank is not just an individual validator's opinion. It sits after validator weights are compared
through the consensus process, so it reflects the part of miner evaluation that remains after
outlier weights are handled. The [Glossary](https://docs.learnbittensor.org/resources/glossary#rank)
describes rank as a final aggregate judgment, and
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) describes clipping as the
process that limits weights outside the consensus benchmark.

## Relationship to Miner Evaluation

Subnets define useful work, miners produce that work, and validators evaluate it. Rank belongs to
that evaluation path: it summarizes how miner work is judged after validator ratings are aggregated.
The [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) page
describes miners as producers of subnet work and validators as evaluators, while
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) connects validator rankings
to miner performance.

## Relationship to Emissions

Rank matters because it feeds the allocation process inside a subnet. The
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) page states that miner
emissions are based on an aggregate ranking weighted by validator stake after clipping. At an
article level, the useful idea is simple: rank is one of the consensus-adjusted signals between
validator evaluation and miner-side emissions.

## Relationship to Trust

Rank and trust are separate consensus measures that describe different parts of the same filtering
path. The [Glossary](https://docs.learnbittensor.org/resources/glossary#rank) frames rank as the
final aggregate judgment on a miner, while
[Trust](https://docs.learnbittensor.org/resources/glossary#trust) describes how much the path from a
miner's pre-rank evaluation to its final rank changed under consensus clipping.

For readers, rank says where a miner landed after aggregation, while trust says how much consensus
clipping shaped that result. They are related but not the same signal, so the two are best read as
complementary views of the consensus process rather than interchangeable values.

References: [Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Trust](https://docs.learnbittensor.org/resources/glossary#trust)

## Relationship to Consensus Score

Rank and consensus score are separate Yuma Consensus measures that describe different stages of the
same evaluation path. The [Glossary](https://docs.learnbittensor.org/resources/glossary#rank) frames
rank as the final aggregate judgment Yuma Consensus produces for a miner, while
[Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score) describes the
stake-weighted benchmark that validator weights are compared against before that judgment is
reached.

For readers, the consensus score belongs to the filtering step that decides which validator weights
survive clipping, while rank is the aggregated result that remains afterward. They are produced by
the same [Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) process but answer
different questions, so a rank value should not be read as the consensus score itself.

References: [Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Consensus Score](https://docs.learnbittensor.org/resources/glossary#consensus-score),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)

## Relationship to Multiple Mechanisms

Rank is usually discussed for one incentive mechanism at a time, but subnets can also run multiple
mechanisms in parallel. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms) notes
that miner performance in one mechanism does not affect their rating in another.

For readers, rank still summarizes consensus-adjusted miner evaluation inside a subnet, but the
mechanism context matters. A rank value for one mechanism should not be read as describing miner
performance in another when a subnet uses more than one evaluation system.

References:
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets),
[Glossary: Multiple Incentive Mechanisms](https://docs.learnbittensor.org/resources/glossary#multiple-incentive-mechanisms)

## Interpretation Boundaries

Rank should be read as a subnet-specific consensus result, not as a standalone guarantee about a
miner, a universal quality score, or a replacement for the subnet's own incentive design. The
[Glossary](https://docs.learnbittensor.org/resources/glossary#rank) links rank to Yuma Consensus,
and [Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets) explains
that each subnet defines its own work and evaluation standards.

## Relationship to Yuma Consensus

Rank and Yuma Consensus address related but different parts of Bittensor subnet incentive flow. Yuma
Consensus aggregates validator weight signals into miner incentives and validator dividends, while
rank names the final aggregate judgment produced for a miner inside that process. The
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank) describes rank as the
final aggregate judgment from validator ratings, and
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus) describes clipping, bonding,
and emission calculation.

For readers, Yuma Consensus names the aggregation process, while rank names one consensus-adjusted
outcome metric for an evaluated miner inside that process.

References: [Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus)

## Relationship to Validator Weights

Rank and validator weights address related but different parts of subnet evaluation structure.
Validator weights are the evaluation signals validators submit for miners, while rank summarizes how
those signals combine into a final aggregate judgment after consensus clipping. The
[Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights)
describes the per-validator evaluation inputs, and the
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank) describes the final
aggregate judgment produced from those inputs.

For readers, validator weights name the evaluation submissions, while rank names the aggregated
consensus result built from those submissions.

References:
[Glossary: Validator Weights](https://docs.learnbittensor.org/resources/glossary#validator-weights),
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank)

## Relationship to Weight Matrix

Rank and the weight matrix address related but different parts of Yuma Consensus vocabulary. The
weight matrix collects validator weight vectors as structured input to Yuma Consensus, while rank
names the final aggregate judgment for a miner after that input is filtered and aggregated. The
[Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix)
describes the matrix built from validator submissions, and the
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank) describes the final
aggregate judgment produced from those combined signals.

For readers, the weight matrix names the organized validator input collection, while rank names the
miner-side consensus outcome built from that collection.

References:
[Glossary: Weight Matrix](https://docs.learnbittensor.org/resources/glossary#weight-matrix),
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank)

## Relationship to Validator Trust

Rank and validator trust address related but different parts of Yuma Consensus clipping vocabulary.
Rank names the final aggregate judgment for an evaluated miner, while validator trust describes how
much of a validator's submitted weight signal remains after consensus filtering. The
[Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank) describes the final
aggregate judgment, and the
[Glossary: Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust)
describes the validator-side clipping outcome.

For readers, rank names the miner-side consensus result, while validator trust names how much of a
validator's submitted signal survived filtering inside the same process.

References: [Glossary: Rank](https://docs.learnbittensor.org/resources/glossary#rank),
[Glossary: Validator Trust](https://docs.learnbittensor.org/resources/glossary#validator-trust)
