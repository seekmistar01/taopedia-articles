# Subnet Protocol

A subnet protocol is the rule set that defines how validators and miners interact inside a Bittensor
subnet
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The term belongs to incentive-mechanism vocabulary. It names the interaction pattern around work. A
task says what work the subnet wants, the protocol says how miner-validator exchange is organized,
and the scoring model judges returned work.

## Development Stage Context

The
[Introduction to Bittensor](https://docs.learnbittensor.org/learn/introduction#subnet-development)
describes subnet development as moving from localnet to testnet and then mainnet. For subnet
protocol, that sequence changes how readers should interpret miner-validator interaction examples.

In localnet, protocol patterns can be tested in an isolated environment. Localnet exchange examples
do not represent production subnet behavior.

On testnet, protocols can be exercised in a shared non-production network. Testnet interaction
outcomes are separate from mainnet subnet state.

On mainnet, subnet protocol concerns live miner-validator exchange on the production Bittensor
network.

The [Bittensor Networks](https://docs.learnbittensor.org/concepts/bittensor-networks) reference
separates mainnet, testnet, and localnet. A protocol example from one environment should not be read
as representing production subnet performance in another environment.

## Distinction from Synapse

Subnet protocol names the rule set for how validators and miners exchange work inside a subnet.
Synapse names the structured message object that carries that exchange in Bittensor client code
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Glossary: Synapse](https://docs.learnbittensor.org/resources/glossary#synapse),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The protocol defines exchange rules. Synapse is the structured object that carries those rules in
client code.

- **Subnet protocol** — rules for how validators query miners and miners respond.
- **Synapse** — the structured message object carrying that exchange.

## Distinction from Axon

Subnet protocol names the exchange rules between validators and miners. Axon is the server-side
component subnet miners deploy to receive incoming work
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Glossary: Axon](https://docs.learnbittensor.org/resources/glossary#axon),
[Understanding Neurons](https://docs.learnbittensor.org/learn/neurons#neuron-to-neuron-communication)).

The protocol defines what belongs in the exchange. Axon is how miners receive that exchange on the
server side.

- **Subnet protocol** — rules for validator-miner exchange.
- **Axon** — server-side endpoint miners use to receive work.

## Distinction from Dendrite

Subnet protocol names how validators request work from miners. Dendrite is the client-side component
validators use to transmit those requests
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Glossary: Dendrite](https://docs.learnbittensor.org/resources/glossary#dendrite),
[Understanding Neurons](https://docs.learnbittensor.org/learn/neurons#neuron-to-neuron-communication)).

Validators initiate through dendrite clients. The protocol defines what those requests contain.

- **Subnet protocol** — rules for requesting and returning work.
- **Dendrite** — client-side component validators use to send requests.

## Distinction from Subnet Task

Subnet task names the work a mechanism asks miners to produce. Subnet protocol names the interaction
pattern for how validators request that work and miners return responses
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The task asks for work. The protocol defines how that work is requested and returned.

- **Subnet protocol** — the interaction pattern for requesting and returning work.
- **Subnet task** — the work miners are asked to produce.

## Distinction from Subnet Scoring Model

Subnet protocol names how responses are obtained through miner-validator exchange. Subnet scoring
model names how validators judge those responses once they arrive
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Glossary: Subnet Scoring Model](https://docs.learnbittensor.org/resources/glossary#subnet-scoring-model),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The protocol creates the evaluation surface. The scoring model explains how returned work is judged.

- **Subnet protocol** — how responses are obtained through exchange.
- **Subnet scoring model** — how returned work is judged.

## Distinction from Subnet Miner

Subnet protocol names the exchange rules between roles. Subnet miner names the work-producing role
that returns responses within those rules
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Glossary: Subnet Miner](https://docs.learnbittensor.org/resources/glossary#subnet-miner),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

The protocol organizes exchange between roles. The miner is the work-producing role within those
rules.

- **Subnet protocol** — exchange rules between roles.
- **Subnet miner** — the work-producing role returning responses.

## Distinction from Subnet Validator

Subnet protocol names the exchange rules for how validators request work and miners return
responses. Subnet validator names the evaluating role that operates within those rules
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Glossary: Subnet Validator](https://docs.learnbittensor.org/resources/glossary#subnet-validator),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The protocol defines how work is exchanged. The validator is the evaluating role within those rules.

- **Subnet protocol** — exchange rules for requesting and receiving work.
- **Subnet validator** — the evaluating role within those rules.

## Distinction from Incentive Mechanism

A subnet protocol is one required piece of an incentive mechanism. The mechanism also needs a task
definition and a scoring method validators can apply to returned work
([Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism),
[Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Glossary: Incentive Mechanism](https://docs.learnbittensor.org/resources/glossary#incentive-mechanism)).

The protocol defines exchange rules. The incentive mechanism is the broader rule layer that turns
evaluated work into rewards.

- **Subnet protocol** — miner-validator exchange rules for a subnet.
- **Incentive mechanism** — the broader rule layer that evaluates work and assigns rewards.

## Distinction from Yuma Consensus

Subnet protocol names the interaction rules between miners and validators inside a subnet. Yuma
Consensus is the separate mechanism that converts validator weight submissions into emission shares
each tempo ([Yuma Consensus](https://docs.learnbittensor.org/learn/yuma-consensus),
[Emission](https://docs.learnbittensor.org/learn/emissions),
[Understanding Subnets](https://docs.learnbittensor.org/subnets/understanding-subnets)).

- **Subnet protocol** — miner-validator interaction rules for a subnet.
- **Yuma Consensus** — mechanism that turns validator weights into emission shares.

## Reader Boundary

Subnet protocol is concept vocabulary for miner-validator interaction. It explains exchange rules
rather than deployment status, payout expectations, or a subnet's full mechanism design
([Glossary: Subnet Protocol](https://docs.learnbittensor.org/resources/glossary#subnet-protocol),
[Understanding Incentive Mechanisms](https://docs.learnbittensor.org/learn/anatomy-of-incentive-mechanism)).

The stable point is that a subnet protocol defines the request-and-response rules by which miners
and validators interact inside a subnet. Subnet task, scoring model, incentive mechanism, and
network-environment language describe adjacent parts of the same system
([Glossary: Subnet Task](https://docs.learnbittensor.org/resources/glossary#subnet-task),
[Multiple Incentive Mechanisms Within Subnets](https://docs.learnbittensor.org/subnets/understanding-multiple-mech-subnets)).
